init

Initializes embedded mode for a given activity for Data Entry. This method assumes the specified activity has already been created. A new session will be created as a part of the initialization process. This is supported by eApp and Illustration.

NOTE: If resources for multiple modules are loaded loadModule must be called prior to init in order to load the correct module.

Parameters

  • token: string

    The string returned from the API Token Service that uniquely identifies the current call to the client API (for tracking purposes). The string that uniquely identifies the current call to the client API (for tracking purposes).

  • activityId: string

    The GUID of the activity to be loaded.

  • isNewActivity: Boolean

    A Boolean (true/false) value to indicate if the current init operation is for a new activity or an existing activity.

  • enableErrorHandling: Boolean

    A Boolean (true/false) value to indicate whether FireLight should show its own error dialogs when system errors occur. If you set this to false, you should handle errors in your embedding code.

  • enableAuthProxy: Boolean – defaults to true

    A Boolean (true/false) value which indicates whether or not the Auth Proxy should be used. The Auth Proxy is used to store cookies for a session so the browser does not have to, letting users load embedded while using browsers that don’t support third party cookies (including Safari and Firefox).

Returns void
