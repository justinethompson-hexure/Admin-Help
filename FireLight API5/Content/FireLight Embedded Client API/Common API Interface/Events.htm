Events

Events
Event Defined in / Description

fireLightLoaded

Subscribe to this event to be notified when the FireLight® eApp script has completed loading. Once this event has fired, initialization of the module should be done. See init, loadModule and loadEmbeddedESign for details on initialization.

errorEvent

Subscribe to this event to be notified when FireLight has encountered an error. Scenarios where this event could be raised include script errors, issues communicating with the server, or exceptions thrown by the server. This event returns an ErrorEvent model.

navigationEvent

Subscribe to this event to be notified when an action that may affect the navigation graph has completed. This event returns a NavigationEvent model.

statusEvent

Subscribe to this event to be notified when the status of the activity has changed. This event also gets fired after the init method has been called and before the application has completed loading. The status is reflected as a percentage of fields that do not have validation error messages associated with them. This event returns a StatusEvent model.

ruleMessagesEvent

Subscribe to this event to be notified when the activity has received validation error messages. The response will be a RuleMessagesEvent model, keyed by data item ID. Use getRuleMessages to obtain the actual messages.

notificationsEvent

Subscribe to this event to be notified when a notification is raised within FireLight. This event returns a NotificationsEvent model.
