Getting Started

IMPORTANT! It is highly recommended to link to the FireLight® embedded bundle that is hosted on our web site as described below. If your organization is hosting the FireLight bundle content, and you do not update the version hosted on your website with every FireLight release and patch, it will be stale, missing new functionality and bug fixes, and it also will be incompatible with the backend FireLight code hosted on our servers.

To get started with the embedded API, there are a few things that need to occur. The steps listed below allow you to establish a session with FireLight and embed a FireLight wizard within your application.

  1. Obtain a token by connecting to our token service described here.

  2. Create an activity using the CreateActivity method.

  3. Link to the FlexBoxGrid CDN for layout styling.

    • https://cdnjs.cloudflare.com/ajax/libs/flexboxgrid/6.3.1/flexboxgrid.min.css

    • Example: <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/flexboxgrid/6.3.1/flexboxgrid.min.css" type="text/css">

  4. Link to the following JavaScript resources (Adjust BASE URL to match target environment):

    1. Embedded Illustration

      1. Format: {BASE URL}/Resource/Scripts/illustration

      2. Example: https://firelight.insurancetechnologies.com/EGApp/Resource/Scripts/illustration

    2. Embedded Application

      1. Format: {BASE URL}/Resource/Scripts/eApp

      2. Example: https://firelight.insurancetechnologies.com/EGApp/Resource/Scripts/eApp

        Optionally, to specify the name of the div where embedded FireLight will be rendered, use the “destination” query parameter. Example: https://firelight.insurancetechnologies.com/EGApp/Resource/Scripts/eApp?destination=myCustomDiv

    3. E-Signature

      1. Format: {BASE URL}/Resource/Scripts/eSign

      2. Example: https://firelight.insurancetechnologies.com/EGApp/Resource/Scripts/eSign

        Optionally, to specify the name of the div where embedded FireLight will be rendered, use the “destination” query parameter.

        Example: https://firelight.insurancetechnologies.com/EGApp/Resource/Scripts/eSign?destination=myCustomDiv

    4. Multiple Resources (i.e. Application and E-Signature)

      1. Format: {BASE URL}/Resource/Scripts/eApp~eSign

      2. Example: https://firelight.insurancetechnologies.com/EGApp/Resource/Scripts/eApp~eSign

        NOTE: When multiple module resources are loaded loadModule must be used to determine which module to load and display.

        Optionally, to specify the name of the div where embedded FireLight will be rendered, use the “destination” query parameter.

        Example: https://firelight.insurancetechnologies.com/EGApp/Resource/Scripts/eApp~eSign?destination=myCustomDiv

  5. Link to the following CSS resources (Adjust BASE URL to match target environment):

    1. Embedded Illustration

      1. Format: {BASE URL}/Resource/CSS/illustration

      2. Example: https://firelight.insurancetechnologies.com/EGApp/Resource/CSS/illustration

    2. Embedded Application

      1. Format: {BASE URL}/Resource/CSS/eApp

      2. Example: https://firelight.insurancetechnologies.com/EGApp/Resource/CSS/eApp

        By default, all available styles, including fonts and colors, will be included in the CSS returned by the calls above. To return only the root styles that provide the framework for rendering the embedded UI, add “rootOnly=true” to the query string of the request. This will reduce the payload of the CSS response by ~30KB and will make setting your own fonts and colors easier.

    3. Embedded E-Signature

      1. Format: {BASE URL}/Resource/CSS/eSign

      2. Example: https://firelight.insurancetechnologies.com/EGApp/Resource/CSS/eSign

  6. Create a div element in your web page called fireLightMainDiv.

  7. Once the fireLightLoaded event has fired, an instance of a FireLight wizard can be loaded using the init method of the Client API along with the activityId that was returned from the CreateActivity method called above.
