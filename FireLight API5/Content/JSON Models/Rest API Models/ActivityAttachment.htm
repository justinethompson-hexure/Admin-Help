ActivityAttachment

Type: object

Properties:

  • AttachmentName string

    The name of the attachment

  • AttachmentType int (OLI_LU_ATTACHMENTTYPE): optional

    The type of the attachment

  • DocumentType int (ActivityAttachmentDocumentType)

    • For template forms, the DocumentType will always be 6, which corresponds to Additional Doc With Signatures.

  • DocumentFormat int (ActivityAttachmentDocumentFormat): default 3

    The file format of the attachment

  • DocumentTag string

    Document tag values are set per the organization's UploadDocumentTypes list profile and correspond to the entry's <Description/> value.

     

    For non-template forms, the DocumentType and the <Description/> tag must match for the request to be accepted. For example, if you are uploading a non-template form with a DocumentType of 4, the DocumentTag must match the <Description/> value that is set for that <UploadDocumentType/> entry within the list profile.

     

    For template forms (these must have a DocumentType value of 6), the DocumentTag must match the <Description/> value of any configured <UploadDocumentType/> entry that has an <AdditionalDocWithSignatures/> DocType value. The Document Tag field in the Form Library corresponds to the value passed for template documents.

  • AttachmentData64Binary string

    Attachment data in Base64 binary format
