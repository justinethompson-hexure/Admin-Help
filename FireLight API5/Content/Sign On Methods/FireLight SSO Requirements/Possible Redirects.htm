Possible Redirects

After authentication the user will be redirected to one of the following pages within the FireLight® system:

  1. If the SSO contained a TX103 transaction, the user will be redirected to the Edit Application View.

  2. If the URL contains the query string “appId” and the referenced application does not exist within FireLight, the user will be redirected to a page indicating the application is not on file.

  3. If the URL contains the query string “appId” and references a valid FireLight application identifier owned by the authenticated user, the user will be redirected to the Edit Application View, unless the RelayState is set to eSign.

  4. If the RelayState is Set to eSign and the URL contains the query string “appId” and the appId references a valid FireLight application identifier owned by the authenticated user and the application is Locked or PendingSignature Status, then the User will be redirected to the eSign View.

  5. If the RelayState is Set to eSign and the URL contains the query string “appId” and the appId references a valid FireLight application identifier owned by the authenticated user but the Application is not in a valid status for Signatures, the user will be redirected to an Invalid Request Page.

  6. If no TX103 exists and the user is authenticated, the user will be redirected to the FireLight Home page.

  7. If the request fails authentication or validation for any reason, the user will be redirected to a FireLight error page.

  8. If the form post RelayState field contains one of the following values, the user will redirected to the corresponding page in FireLight:

    Home => Home page

    NewApplication => New Application page

    Applications => All Applications page

    OSJ => Review Queue page
