Redirect Options within FireLight

By default, a user redirected through this route into FireLight® will land on the home page. However, two query variable options are available for the purpose of indicating different redirects within FireLight (once authentication and session has been established). These are:

  • appid

    This will be accompanied with a specified app/activity’s “AppId”, a guid that identifies it within the FireLight system. This will redirect the user directly to that app/activity’s page, so long as the user has access to that app/activity.

  • relayState

    This variable functions the same way as in the SAML redirect mechanism (also called relayState). The possible options are:

    • Home

      Simply redirect the user to the home page of FireLight

    • NewApplication

      Redirect the user to the New Application page

    • Applications

      Redirect the user to the Applications page

    • OSJ

      Redirect the user to the OSJ page

    • eSign

      Redirect the user to the E-Signature page – when this option is sent, an ‘appid’ query variable must also included – the appid will correspond to an application/activity that the token user has access to, and is in a Locked or PendingSignature Status

  • embed

    This will set a flag within the FireLight session indicating the UI is Embedded in a client's environment. The key functionality this provides is to hide unwanted UI elements when displayed in Embedded. Primarily the toolbar will no longer be visible. The embed flag is only available for relayState eSign or OSJ.

Examples:

?appid=84604672-acaa-4f5e-a906-70f6bbda2f9b

?relayState=NewApplication

?appid=84604672-acaa-4f5e-a906-70f6bbda2f9b&relayState=eSign&embed=true
