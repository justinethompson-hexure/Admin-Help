Submitting Compliant Illustration

You can add to the transaction a compliant illustration as a base64 encoded string under the <Attachment> element within the <Holding> aggregate. The original format of the document must be PDF. The document will be included within the final package submitted to the carrier for processing.

The table below lists the required elements that must be part of the <Attachment> aggregate.

Required Elements for the <Attachment> Aggregate
Element Description
<Description> A short 255 characters or less description of the document.
<AttachmentData64Binary> The base64 encoded string value of the PDF document.
<AttachmentType> “tc” value of “7” for OLI_ATTACH_ASORIGSOLDILLUS.
<ImageType> “tc” value of “4” for OLI_IMAGE_PDF.
<AttachmentLocation> “tc” value of “1” for OLI_INLINE

The following is a sample <Attachment> aggregate:

<Attachment> Aggregate

 <Attachment> <Description>Compliant Illustration</Description> <AttchmentData64Binary>JVBERi0xLjQKJeLjz9MKMiAwIG9iaiA8PC9...</AttchmentData64Binary> <AttchmentType tc="7">OLI_ATTACH_ASORIGSOLDILLUS</AttachmentType> <ImageType tc="4">OLI_IMAGE_PDF</ImageType> <AttachmentLocation tc="1">OLI_INLINE</AttachmentLocation> </Attachment>

Submitting a Third Party Needs Determination

You can attach a needs determination transaction completed outside FireLight using attachments. A needs determination attachment differs from other attachments in that it cannot be deleted or modified in the newly created application. It can be used to replace a needs determination template in the package, allowing it to be included in the edit application view and in the signing ceremony. An Upload Document type of Needs Determination must be added to the organization’s list profile to support this workflow.

The following is a sample needs determination <Attachment> aggregate:

Needs Determination <Attachment> Aggregate

 <Attachment> <Description>NeedsDetermination</Description> <AttachmentData64Binary>JVBERi0xLjQKJeLjz9KMiAwIG9Ia8PC9…</AttachmentData64Binary> <ImageType tc="4" /> <OLifEExtension VendorCode="25"> <DocumentTag>NeedsDetermination</DocumentTag> </OLifEExtension> </Attachment>
