Launch into FireLight Dashboard – Delayed Use of 103

Posting over into FireLight® with a 103 that does not contain the product ID and jurisdiction will launch into the FireLight dashboard. Client data from the 103 can be retained and utilized for the duration of the session. The New Business service does additionally require the refid to be included in the URL in order to make use of the delayed 103 functionality. The 103 should also include one of the following parties: insured, annuitant, or owner.

The following is an explanation of each step:

  1. The user is authenticated and authorized by the external system.

  2. The user clicks on a link to launch into the FireLight dashboard.

  3. The external system sends back to the user a specially crafted response to force the browser to POST an ACORD Tx1228 transaction within a hidden form field to the FireLight passive endpoint.

  4. FireLight authenticates with the STS using a predefined shared user login and password set up specifically for the implementation. If the credentials are valid, FireLight will impersonate the user identified within the 1228 for the remainder of the session. If the credentials are not valid, the user will be redirected to a generic error page within FireLight.

  5. FireLight will send back a redirect to the FireLight dashboard.

  6. The user will select the product and jurisdiction for the new application.

  7. The system will allow the user to choose whether to use the data from the 103 for the application being created. The checkbox for ‘Create the Application for [Insured Name], [Annuitant Name]’, or [Owner Name] will default to checked.

  8. If the user does not uncheck the box, the associated fields on the forms are pre-filled with the data from the current session’s 103. The 103 client information can continue to be used for duration of the session for any applications created.
