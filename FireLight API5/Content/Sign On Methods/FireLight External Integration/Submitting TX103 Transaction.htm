Submitting TX103 Transaction

Typically, you will want to create a proxy from the FireLight® web service wsdl. The proxy is then used to communicate with the web service. At a minimum, the sequence is to call the web service, and then check the response to determine if the request succeeded. The following example shows what a typical transaction will look like on the wire.

The “tc” value within the <TransResult> aggregate is used to determine if the transaction succeeded or failed. Possible values returned from FireLight are shown in the following table.

"tc" Values and Descriptions
Value Description
1 - RESULT_SUCCESS The transaction was processed successfully with no additional information.
5 - RESULT_FAILURE The transaction was not processed. The reason for the failure can be identified in the <ResultInfo> aggregate.
