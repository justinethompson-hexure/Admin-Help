Elements within the Tx1228

The following table describes each element within the Tx1228.

Elements within the Tx1228
Element Description
<UserAuthRequest> Required aggregate when the Tx1228 is a POST to the passive FireLight endpoint. When used in SSO this should not be included. It contains the predefined login information used by FireLight to authenticate the external system with FireLight®.
<UserLoginName> The predefined login name set up for this subscription to FireLight. All requests will use the same login name and password.
<UserPwd> Required aggregate containing the predefined password for authenticating the external system with FireLight.
<CryptType> Required with a value of “AES”.
<CryptPswd> Required. This element contains a SHA1 hash of the shared password appended to a SHA1 hash of the <TXLifeRequest> aggregate, which is then signed using AES and the entire value base64 encoded.
<UserDomain> Required. Used by FireLight to identify the organization subscription to FireLight.
<VendorApp> Required aggregate used to identify the name of the external system.
<AppName> External system name.
<TXLifeRequest> Required aggregate containing information about the actual user of the system.
<TransRefGUID> Required and must be unique for each transaction.
<TransType> Required and “tc” value must be “1228” (OLI_TRANS_TRNPRODINQ).
<TransSubType> Optional, but must have a “tc” value of “22800” (OLI_TRANSSUB_PRODDETALL).
<OLifE> Required aggregate.
<SourceInfo> Optional aggregate used by FireLight logging to indicate the actual referring system identifier. For instance, if you have multiple ways into FireLight, you can use the <SourceInfoName> to identify which launch mechanism (e.g. portal) was used.
<Party> Required aggregate containing information used by FireLight to identify the actual user of the system. Optionally, used to identify Firms the user is associated with.
<FullName> Optional if <Person><FirstName> and <LastName> are supplied. Required if <Person> aggregate is not used.
<Person> Required if <FullName> not included within the <Party> aggregate. Note: When the <NickName> attribute is used with the <Person> aggregate, it allows a user to log in to FireLight under another user’s account and still maintain accurate case ownership information. The FireLight audit records record the nickname for the logged in user. The owner of the case retains the agent information and it is displayed in various parts of the system.
<Producer> Required aggregate used to uniquely identify the user within the FireLight system.
<CarrierAppointment> Required aggregate containing an identifier used by FireLight to identity the user and/or firms distribution agreement.
<CompanyProducerID> Unique identifier from the external system that will be used by FireLight to uniquely identify the user.
<CarrierCode> Used by FireLight to identify the organization subscription to FireLight.
<Relation> Required aggregate used to identify the role of the <Party> aggregate.
<DistributionAgreementInfo> Optional aggregate to identify the distribution agreement aggregate for that organization.
<Registration> Optional aggregate to identify the organization firm information.
<FirmName> Optional aggregate to identify the organization firm name.
<FirmRegistrationNum> Optional aggregate to identify the organization Firm ID.
<Relation> Required aggregate used to identify the role of the <Party> aggregates and the relation between Person and Organization <Party>.
Aggregates when specifying distribution agreements and associated firms.
<DistributionAgreement> Optional aggregate to identify product availability.
<PolicyProductInfo> Optional aggregate to identify a product defined in FireLight.
<ProductCode> External code to help identify product within FireLight.
<OLifEExtension> Aggregate to extend the ACORD (v2.27) model.
<CusipNum> External code to help identify a product defined in FireLight.
<DistributionAgreement> Optional aggregate to identify product availability.
