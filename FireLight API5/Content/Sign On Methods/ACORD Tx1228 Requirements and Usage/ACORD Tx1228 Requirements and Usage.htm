ACORD Tx1228 Requirements and Usage

Overview

FireLight® provides support for an ACORD Tx1228 during the sign on process. This ACORD request can be used to provide login credentials and agent information as well as specify firm information and product availability. When specifying firm information, applications can be tied to a firm so managers who need to review their firm’s applications can easily do so.

During SSO login, any parties identified with a relation role code of 120 (OLI_REL_AGENTOF) in the 1228 that have an email address (identified by PrefEmailAddr, primary email, or the first email address if not specified) and a producer registration FirmRegistrationNum value will be maintained and identified within FireLight by the FirmRegistrationNum. When an application lands in the review queue, if a match for the application’s Firm identifier is found in the firm contact information maintained in FireLight, the notification will be sent to that firm’s email address. If not found, the queue email address will be used as before.

The topics in this section detail the usage and requirements FireLight has for the Tx1228 used in the sign on process.
