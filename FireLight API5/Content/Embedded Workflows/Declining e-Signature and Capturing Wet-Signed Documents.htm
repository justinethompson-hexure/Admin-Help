Declining e-Signature and Capturing Wet-Signed Documents

In the regular FireLight® user interface, the user is given an option to decline electronic signature. When a user declines electronic signature, the system provides instructions on how to upload wet signed documents. This same workflow does not exist within the embedded, but the following steps can be followed to recreate the experience in your own application.

  1. Start by configuring the group option to Disable E-Sign Decline. This will instruct the signing ceremony to not ask the user if they consent to electronic signature. This assumes that the question will be asked outside of FireLight.
  2. Create a page in your application that asks the user if they would like to do electronic signature. This is a consent form that allows the user to accept or decline the use of electronic signature. Based on the answer, set the FLI_DECLINED_ESIGN value to true if they decline the use of electronic signature or false if they accept the use of electronic signature.
  3. If FLI_DECLINED_ESIGN is set to false, and the group option has been enabled, redirect the user into the FireLight signing ceremony. The user will not be asked again for consent to electronic signature. Refer to proceedToSignatures to learn how to redirect the user to the signing ceremony.

  4. If FLI_DECLINED_ESIGN is set to true, implement the UI functions to allow users to upload documents. Use the following APIs to implement this function.

    • GetActivityPDF to get the PDF to sign.
    • UploadDocument to upload documents and associate with the activity.
    • showDocumentsDialog to show our documents dialog that would allow the user to upload the wet signed documents. This will require the embedded session and client libraries to be loaded.
