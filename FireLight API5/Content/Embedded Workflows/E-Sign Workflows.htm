E-Sign Workflows

Steps for Using FireLight E-Sign within an Embedded Application

  1. Enable the proxy server from within your embedded application.

    1. FireLightAPI.init(token, activityId, isNew, enableErrorHandling, useAuthProxy)

  2. (User fills out the embedded application.)

  3. Implement an application completion process. For example, check that the application is complete by using FireLight.statusEvent.subscribe(), and checking status.percentComplete in the callback.

  4. Call API.proceedToSignatures(). This verifies the application is ready for signatures and raises a signatureStatusEvent. Check status.succeeded to continue.

  5. Navigate to FireLight E-Sign using SSO. See the SSO information on how this may be done. Using the Token SSO approach is recommended because a token has already been retrieved. This approach would require setting the following query string parameters: appId & relayState=eSign.

Steps for Using Embedded E-Sign within an Embedded Application

  1. Load the Embedded E-App application:

    1. Call FireLightAPI.loadModule('EApp')

    2. Call FireLightAPI.init(token, activityId, isNew, enableErrorHandling, useAuthProxy)

  2. (User fills out the embedded application.)

  3. Implement an application completion process. For example, check that the application is complete by calling the FireLight API's IsComplete method.

  4. Call API.proceedToSignatures(). This verifies the application is ready for signatures and raises a signatureStatusEvent. Check status.succeeded to continue.

  5. Load the Embedded E-Sign application:

    1. Call FireLightAPI.loadModule('eSign')

    2. Call FireLightAPI.loadEmbeddedESign(activityId)

  6. When signatures are complete the user should be navigated away from eSign. For example navigate back to the eApp module.

    1. Call FireLightAPI.loadModule(‘EApp’)

    2. Call FireLightAPI.loadEmbeddedEApp(activityId)

Features currently missing from Embedded E-Sign:

  • Canceling of an Active Signer

    • Re-Signing is still available. The signer must complete the signature process and then use Re-Sign.
