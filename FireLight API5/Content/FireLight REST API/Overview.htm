Overview

The REST-based API services utilize JSON web tokens for security and access. Accessing a FireLight® API service endpoint requires a JWT token first received from the FireLight token endpoint. Getting access to a service endpoint is a two-step process:

  1. A call to the FireLight Token endpoint to get a token. Note this requires credentials pre-defined for your organization. The token is returned in an encoded string format.

  2. Embedding the received encoded token string into the subsequent service endpoint call’s Authorization header.

    • FireLight API tokens can be used multiple times within their time limit window. A FireLight API token expires after 15 minutes, after which you will need to get a new one.

Please read Getting the Token for a detailed description on how to retrieve a token.
