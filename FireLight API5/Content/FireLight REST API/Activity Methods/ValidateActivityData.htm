/Activity/ValidateActivityData

HTTP post that validates the given activity data, sending back a list of errors if necessary.

Requires API Token, but does not need an API Token with an associated user.

Request body ActivityValidationRequest Model *required

A list of data items used for the activity along with a unique ID to identify the request:

  • Id

  • CUSIP

  • Jurisdiction

  • CarrierCode

  • AcordType

  • DataItems

  • ReturnUpdatedDataItems

Response

The response contains:

  • Id

  • ResultCode

  • Messages

  • ExceptionError

  • ValidationMessages

  • DataItems (if ReturnUpdatedDateItems is true)
