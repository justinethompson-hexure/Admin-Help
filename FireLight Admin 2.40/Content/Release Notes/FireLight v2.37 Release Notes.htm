FireLight v2.37 Release Notes

New Functionality

SuranceBay Integration Updates: CR-3520, CR-2674, CR-2603

Login widget replaced with Username/Password:

  • AgencyID validated as part of Surancebay credentials also displayed in Preferences,

  • Compliance widget in Recent Application List,

  • Application started call to Surancebay for AcordType 103

Updates to the Distributor Profile Logic: CR-2669

The Distributor Profile (Previously Commissions) under the Production Configuration section of the FireLight administration tool has been enhanced.

  • The Distributor Profile table includes Filename, Upload Date, Carrier, Distributor List, Start Date, and Status.

  • The user can now view the DPFA XML by clicking on the File Name.

  • The user can now view the DPFA Details Dialog when clicking on Status.

  • When the user selects Promote in the DPFA Details Dialog a confirmation message will be presented.

  • The full list of distributors and subfirms present in the file will be viewable when the user hovers over the field on the distributor profile.

  • One to many subfirms can now be added to the distributor profile.

Enhance Options to Add List Values to a Drop Down Field in Wizard Designer: DIST-2244

To support ease of use, more options will be provided to populate list values for a drop-down field. Currently, a common tag can be used to populate values to a drop-down field, or the values can be entered manually. With this enhancement, the option to populate values for a drop-down from a custom list or an external file will be introduced.

Impact

  • FL Admin > Wizard Designer: the following changes will be available on the properties dialog of a drop-down field.

    • Drop-down to select custom list – to link or attach values to the drop-down field,

    • Import button to select values from an external file.

  • FL Admin > Form Designer: below changes will be available to the properties dialog of a combo box field.

    • Drop-down to select custom list – to link or attach values to the drop-down field,

    • Import button to select values from an external file.

  • FL Admin > Wizard Library > Wizard History (tab): record and display changes made to the drop-down field to fill values from custom list or import file method.

  • FL Admin > Form Library > Form History (tab): record and display changes made to the drop-down field to fill values from custom list or import file method.

New Field Type on Wizard Designer with Capability to Auto-Validate Inputs: Validation Rule Field Type Improvements: DIST-2244

To support ease of use and facilitate quicker turnaround to create wizards and rules, a new field type will be added to auto-generate validation rules at run time for input types such as Number, SSN, TIN, Amount, Zip, etc. Using this field type will not require to write validation rules to format generic values such as email addresses, percent value, numeric value, 10-digit phone number, etc.

  • Allows inputs such as Number, AlphaNumeric, SSN, Zip, etc. and ability to set the input type and choose validation property entered value,

  • Ability to validate date values to apply system generated rules for past and future date validations.

Admin Report Enhancements: DIST-3330

To provide more usability to a few of the admin reports, below changes will be made.

  • Remove Option to Generate Admin Reports in PDF Format

  • Add CUSIP and Product GUID to Sales Agreement Admin Report

  • Add Label, Attachment and Description to Package Forms Admin Report

  • New Admin Report to List Deployment Info

Impact

• FL Admin > Utilities > Reports: Drop-down to select report format on Admin reports will be removed.

• FL Admin > Utilities > Reports > Admin History Reports > Sales Agreement Report: Two new columns “CUSIP” and "GUID" will be added to the sales agreement admin report.

• FL Admin > Utilities > Reports > Package Forms admin report: Add three columns to the report – Label, Attachment and Description.

• FL Admin > Utilities > Reports > Deployment History Report: Add new report as “Deployment History Report” on this page

Add New Data Tag to Identify an External Data Entry Session: DIST-3827

With this enhancement, a new data tag will be added to distinguish between an agent’s data entry session and an external client session (client fill or fill & sign) during rules processing. A new data item “FLI_DATA_ENTRY_CONTEXT” is added to return context values such as "Agent," "Client," or "Reviewer" based on the user logged in for data entry.

Telesign Signing Option Enabled for Single Signer on Multiple Signer Contract: EAES-4567

The original implementation of Telesign was configured to work with only one signer with one signer type (who is not the agent). This update allows Telesign as a signing option for a single signer on an application that has other non-agent signers.

Support for Multiple Type Code Values on a Custom Action: EAES-4046, EAES-4947

These two stories allow users to specify multiple Type Codes to be printed when creating a Print Custom Action Button. In addition, if multiple Type Codes are passed in and one of them is "73", we've changed it to disregard the special logic tied to that specific Type Code for ACORD 951 eForm. But you will be able to print forms with TC Type "73"

Error Message Now Displays Multiple Errors: PC-2779

We updated error messages to display multiple errors. Allows the system to show multiple errors in an application when errors occur.

Impact

When a person is completing an application, if there are multiple errors on a page, a popup window will appear with all errors being stated.

Mask that Allows for Percentages %: PC-2818

Create a Percentage Mask that can be used or update the current mask, so that it can go as high as XXXXXXXX.XXXX% to the thousandths digit percentage (.00X%)

Impact

The user will be able to pick a new mask the numerial mask, to show large numbers without have a comma in the number. The person will have to go to the Admin program > Organization Link > Organization > Activity > Select "ADD" link next to "Edit Masks." > for Mask Types select "Percentage: 9999999.999%."

Numeric Mask that Can Be Used or Update the Current Mask to Not Force a 1000 Separator: PC-2509

We implemented a Numeric Mask that updates the current mask to not force a 1000 separator.

Impact

Uses can use the new mask to display large numbers without commas in the number (Admin program > Organization Link > Organization > Activity > Select "ADD" link next to "Edit Masks." > for Mask Types select "Percentage: 9999999.999%").

Resolved Issues

Product Config List Is Showing "Commissions" Option When Opened from Page Other than Home Page: CR-3220

We resolved an issue where the Product Configuration list was displaying the Commissions option when opened from a page other than the Home page. The Product Configuration list now displays the Distributor Profile option as designed.

Clicking on "Distributor Profile" Button on Admin Home Page Is Giving Null Exception Error: CR-3391

We resolved an issue in the Admin app where clicking the Distributor Profile button resulted in a null exception error. The button is now working as intended.

Issues Uploading a DPFA File: CR-3497

We resolved an issue where DPFA files failed to be uploaded properly. DPFA files are now uploaded as expected.

Third Party Service Configuration Should Prefer Carrier Organization Settings: CR-3578

We streamlined GetThirdPartyServiceConfig calls for greater efficiency.

MIB Related Audit Message Is Not Appearing Partywise: CR-3623

We resolved an issue where party-specific information was not included in MIB-related audit messages. MIB-related audit messages are now complete.

Additional Insured MIB Related Response File Not Showing on Support Tool: CR-3624

We resolved an issue where responses were not appearing correctly in the support tool. Responses are now appearing as designed.

Additional Insured - "AURA_EF_MIB_STATUS_AdditionalInsured" Response Field Is Not Mapping with any Response: CR-3625

We resolved an issue where response fields were not mapping as intended. Response fields are now mapping as designed.

Changing Organization Name Logs Other Changes that Did Not Occur: DIST-3904

We resolved an issue where following an organization name change the History tab also erroneously listed changes that had not been made. Now the History tab lists only actual changes.

Office Use Only Field Triggered by a Rule Does Not Immediately Render as Office Use Only: DIST-4082

We resolved an issue where Office Use Only fields were not clearly rendered as such, in green, on initial view. Office Use Only fields now appear as green on initial screen view.

Activity Audits Are Not Being Purged: DIST-4115

We resolved an issue where Activity page data was not being purged as designed. Activity page data is now also purged with admin information.

Deploy Now for a Single Organization Was Specified but Instead All Updates Were Deployed: EAES-3805

We resolved an issue where clicking Deploy Now for a single organization resulted in all updates being deployed. The Deploy Now button is now working as designed.

Email Address with Leading or Trailing Spaces Is Not Allowed, Preventing Continue: EAES-4101

We resolved an issue where the system failed to trim leading and trailing spaces from email addresses. The system is now trimming leading and trailing spaces, allowing applications to continue.

E-Delivery Agent Directed Change Requests Impacted by Delete Refactor: EAES-4692

We resolved an issue where clients received daily reissue requests rather than just one. Clients are now receiving a single reissue request.

Deleted Firms Are Not Being Purged for Firm Management: EAES-4731

We resolved an issue where deleted firms were not also being purged. Deleted firms are now also purged.

Upload Illustration Button Not Working: EAES-4741

We resolved an issue where clicking the Upload Illustration button resulted in no action. Clicking the button now launches the Illustration document type as designed.

Drop-Down Text Color Is Different in Wizard UI: EAES-4742

We resolved an issue where drop-down text in a screen displayed differently than designed. The drop-down text now appears as intended.

Error When Signing Application If Activity Name Contains Brace { } Characters: EAES-4974

We resolved an issue where application names containing curly brace characters caused an error to occur when generating the email template, preventing applications from being signed. Applications can again be signed as designed.

Always Use Agent Name in AGENT_NAME Email Template Variable: EAES-4986

We resolved an issue where the AGENT_NAME variable in the email template was not populating properly. The AGENT_NAME variable is now working as designed.

Request Expiration Reminder Email Coming from Organization Instead of Agent: EAES-4991

We resolved an issue where the request expiration reminder email From information was the organization’s rather than the agent’s. Request expiration reminder email From information is now the agent’s.

Session User Mismatch When Reopening FireLight App or Admin: EAES-4893

We resolved an issue where a mismatch in session user information caused an error when logging in to FireLight. Login now works as designed.

If Users Have Curly Braces in App Name then Agent Not Able to Start Signing: EAES-5046

We resolved an issue where if curly braces were included in an application name agent signing could not be completed. Agent signing now works as designed.

Unlock Activity - Option Always Available to Select Even in Data Entry Phase: PC-1450

We resolved an issue where Unlock Activity was available from the Actions menu during data entry. Now, Unlock Activity is greyed out and unavailable during data entry.

Illustration Activity - "Recover Unsaved Data" Popup Not Showing on Direct Closing of Browser/Tab without Save: PC-1481

We resolved an issue where "Recover Unsaved Data" pop up is not appearing, due to which, if user unknowingly close the browser without save, then, there is no option left to recover the entered data.

React Remote e-Signing Doesn't Populate Checkmarks Once They Review/Approve the Terms: PC-2230

We resolved an issue where checkmarks failed to be selected after review/approval of terms during e-Signing. The checkmarks are now selected upon review/approval of terms as designed.

Timestamp Showing in DOB: PC-2663

We resolved an issue where the Request Clients to Complete Filing and Signing in the Application dialog erroneously displayed a timestamp in the Date of Birth field. The field now displays only the date of birth.

While Testing French Cases, the Mettre a jour Is Cut Off: PC-2766

We resolved an issue where in the French version of the application the Add Activities screen was partially cut off. The screen now appears as designed.

FLI_FIRM_NAME dataitem ID Is Not Getting Pre-Filled on Application Transferred to Other User from FL Access Account: PC-2944

We resolved an issue where after transferring a case to another user the FLI_FIRM_NAME dataitem appears as blank for the transferred case. Now after transferring a case the FLI_FIRM_NAME dataitem is populated.

Need French Translation for Text on Documents Dialog: PC-2957

We resolved an issue where pop-up text was not translated into French and Spanish. The pop-up text is now translated and appears in French and Spanish when in those environments.
