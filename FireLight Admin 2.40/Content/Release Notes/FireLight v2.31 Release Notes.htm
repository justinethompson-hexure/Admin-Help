FireLight v2.31 Release Notes

August 18, 2023, Production

FireLight August 2023 Release – 08/14/2023

Updates

Active Signer Types: Update to allow Active Signers to be used in rules

A new well known data item is created, FLI_ACTIVE_SIGNER_TYPES. It is a comma delimited string that contains a list of all of the Active Signer Types, and if there are no active signer types, the DataItem is an empty string. The list of possible signers is all of the signer types in the Signer Type dropdown, but the data item will only show those that will complete a signature. The Active Signer Type Names will always be in English, and this DataItem will only work on activities that are Auto Saved.

Add a Data Item to Identify Review Queue Name in Emails

The existing email replacement tag <QUEUE_NAME> for review queue name is made available for Post Submit RQ email templates. This replacement tag is used in the email templates under the category “Review Queue” to identify the review queue referred to when approval/decline emails are sent.

Add Auto Deploy Column to Sales Agreement Admin Report

A new column “Auto Deploy” has been added to Sales Agreement admin report. This column will have value “True” or “False” for each sales agreement in the report.

Address Errors when Running Agent Activity reports.

Update report architecture to prevent “deadlock” issues when running activity reports. This includes updates that impact the following reports:

  • AgentFirmPipelineReportProvider

  • AgentSubmissionFirmReportProvider

  • ApplicationReportProvider

  • ANXPendingAppReportProvider

  • GroupEnrollmentReportProvider

  • HMLPipelineReportProvider

  • IllustrationUserDetailReportProvider

  • SAGApplicationSubmissionReportProvider

Activity Completion Admin Report

A new Activity Report has been added “Activity Completion Report” that will display the completion duration of an activity sent for remote signature. This report will track the duration from log in to signature completion

Non FireLight User Login time stamp is retained when they login using a link.

When an External users logins into Firelight via a link for Client Fill and Sign, Client Fill, Remote eSignature or PreSubmit Review Queue the login timestamp is retained.

Update the Lexis Nexis integration to include Emailage via a custom Action parameter

We are adding "LexisNexis Emailage" as a Custom Action Parameter. The custom action type will still display as "LexisNexis," but the Action Parameter will allow the user to select "LexisNexisEmailage" in the textbox.

Update the Lexis Nexis integration to include InstantID via a custom Action parameter

We are adding functionality to support mapping the dobVerified response.
