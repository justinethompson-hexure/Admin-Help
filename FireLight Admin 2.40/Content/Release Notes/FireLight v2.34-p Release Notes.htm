FireLight v2.34-p Release Notes

April 2024 Release—4/18/2024 Production

New Functionality

Add SuranceBay Key to Third Party Settings: CR-2290

This enhancement allows for the input of the SuranceBay APIKey under Third Party Configuration Settings (Organization Settings) as part of the Agency Onboard Process.

SuranceBay My Preference Page - SuranceBay Login Section - Showing Two Different Validation Messages in Specific Scenario with Test and Save Button: CR-2043

When user enters invalid SuranceBay credentials and click on “Save” button, Red validation message appears as expected. When the user enters valid credentials and clicks on “Test” a Green message appears a expected. User should then click save to ensure the red validation message is cleared.

SuranceBay - APIKey Is Not Valid or Null Message Provided to the User: CR-2497

If an administrator provides an invalid or null API Key and a user initiates an application activity, the system displays, “Your System Administrator enabled SuranceBay Integration; however, API Key is having issues. Please contact your System Administrator immediately to fix the issue. Please be advised by proceeding with the application, the integration is not enabled at this moment,” and during the application process no widgets or messages from SuranceBay appear.

Update 1035YellowPages Integration to Version 2.11: CR-1393

We updated the 1035 YellowPages integration to version 2.11 to align with Cooperative Technologies most current production version.

Passcode Validation to Download Documents for Enrollment: DIST-2888

This enhancement is related to user authentication method for Enrollment apps. Once the individual (child) enrollment app is signed and submitted by the client, they receive an email to download the application documents. There was an issue with authentication method when the link to download documents was clicked, only allowing to enter SSN/DOB info. With this enhancement, clients will be able to access the link using the passcode validation also.

Impact

Passcode validation will be used as an authentication method to download documents for an individual app created for Enrollment activity.

Resolved Issues

File Upload – Note on Documents Dialog to Mention about File Being Compressed AfterUpload: DIST-2876

We resolved an issue where the Documents dialog did not inform users that files may be compressed during upload. This message is now displayed.

Review Queue – Data Entry Percentage Is Not Updating as Expected When Data Entry Issues Are Resolved: DIST-2880

When reviewer documents contain mandatory fields, the percentage is now updating as expected when the mandatory fields have been completed.

Report Scheduler – FTP Information Should Not Be Displayed on the Report Scheduler as This Information Applies Only to Production: DIST-3017

This information has been removed from the Report Scheduler screen. Text has been added to remind the user to correctly set the FTP information via the Organization Settings when the requested delivery method is SFTP.

Activity Reports – Client Signed Reports Is No Longer Generating as Expected: DIST-3195

The data items reference for this report have been updated to correct the error. Report is now generating as expected.

Support Tool Unable to Display Signature Info Due to Missing Agent Name Column: DIST-3196

We resolved an issue where signature information failed to be displayed in the support tool. Signature information is now appearing as designed.

FireLight Direct Login - No Longer Able to Log In to Accounts in Organizations That Have 2-Factor Authentication Enabled UI-898

Implementation of the new Login screen has been rolled back to correct an issue that was reported on Multi Factor Authentication. Users logging directly into Firelight will see the old Login screen.

FireLight Administration Tool - CORS Origins Settings Are Removed When Updating SFTP Settings: DIST-3107

We updated the logic for saving and maintaining the CORS Origins and SFTP settings in the Administration tool to ensure that both can be updated independently.

FireLight Administration Tool – “Link to Package” Link Is Not Displayed as Expected in the Sales Agreement: DIST-2866

The location and alignment of this link have been updated so that it displays as expected.

Firelight Administration Tool – The Form Field Report Is Not Displaying as Expected and an Error Is Generated: DIST-2743

The form field report has been optimized; however, in some cases generation of the PDF report may still time out. When this is the case, we recommend generating the form field report as a CSV file.

Form Properties – Passcode Only Validation Behaves Differently between Post Issue and Enrollment: DIST-2769

We resolved an issue where passcode-only validation was not working as designed, resulting in a discrepancy in how it operated between post-issue and enrollment. It is not working as intended.

eSignature - Status Notification for Agent Being Sent to Client: DIST-3224

An error that incorrectly changed the recipient of this notification has been corrected. Agent is not receiving the status notification as expected.

Some Email Notifications Are Incorrectly Determining When and Where to Send the Message: DIST-3237

We resolved an issue where status update email was being sent to agents even when the group option Enable Use Agent Email and the user preference Use Agent Email for All Communications were not selected. Now only agents designated to receive status update email via this option and preference are receiving such notifications.

FireLight – Error Occurred When Opening an Activity After Adding or Removing Optional Forms: EAES-3342

The logic to add and remove options forms has been corrected to resolve this issue.

Inefficient Regular Expression Causes Endless Rules Execution which Can Affect System Stability: EAES-3387

We resolved an issue where rules were executed in a loop, delaying opening activities and impacting system stability.

Error Sending Email on e-Delivery Case Due to Invalid applicationId: EAES-3429

We resolved an issue where e-Delivery case email was not distributed. The email is now being sent as designed.

Email Variable Replacements Should Replace Variable with Blank Value If Data Item Value Has Whitespace: EAES-3536

We resolved an issue where automated email was not rendered as intended. The email is now rendered as designed.

eApplication – Create New Application Page Occasionally Displays an Incorrect Error Message Briefly, at the Bottom of the Page: EAES-3549

The code has been updated to ensure that an incorrect error message no longer appears.

Case Transfer – When Transferring a Case via Using a Custom Action the Permissions for the User the Case Is Transferred to Are Not Working as Expected: EAES-3817

The logic for custom action transfers has been updated so that group permissions are correctly initialized based on the associated role code. Permissions are now being set correctly.

API – CreateActivity/UploadDocuments Functionality Is Not Validating the Document Type as Expected Leading Failed Uploads: PC-1310

Validation on the document type has been temporarily disabled to allow document upload via API to complete without errors.

***Please note that the JIRA ticket numbers are provided for reference. JIRA tickets are not accessible to users outside Hexure.***
