FireLight v2.30 Release Notes

June 09, 2023, Production

Enhancements

Foundation

Enhance Pre-Sale Activity to Prevent Multiple App Launches (IC 273402)

This enhancement allows FireLight clients to configure a Pre-Sale activity to generate one or multiple eApplications or other “Next Activities” as part of a workflow. This is similar to the existing functionality that automatically links an FL Illustration to the corresponding generated eApplication. This functionality has been replicated for a Pre-Sale activity and any “Next Activities” that may be generated from that Pre-Sale.

This is configurable at the Sales Agreement level to allow customers to define when the Pre-Sale and eApplication/Next Activity linkage requires a 1:1 relationship. When enabled, it prevents users from generating more than one eApplication/subsequent activity from a single Pre-Sale activity. This also allows for the use case where a customer has more than one pre-sale activity configured and will allow flexibility as to how those pre-sale activities interact with available “Next Activities.”

Impact:

• FL Admin: New checkbox “Create single activity when Next Activity is selected” added on the Options tab on the Sales Agreement page on the admin tool.

• FL App: Continue/Apply button on the activity navigation.

Foundation

Enhancement for Share Activity Permission (IC 280428)

This enhancement allows organizations to share activities with users with limited or partial permissions. Previously, when an agent/user shared an activity with another user, they could provide read-only access or full control over that activity. This enhancement provides more control while sharing an activity if another user can complete the activity before data-entry or the signing ceremony.

When the new option Share Limited Control is enabled, four control options are available. The user can select to allow the party with whom the activity is shared to complete data entry, complete signing, complete signing without primary agent, or complete pre-signature review. Only one of these options can be selected. When the email is generated for the shared activity, the type of shared functionality is provided.

Impact:

• FL Admin > Organization Settings > Groups > App (tab) > Share/Transfer (section)

• Two new checkboxes are added:

• Disable Share Full Control

• Allow Share Limited Control

• FL App > Other Actions > Share

• When a user is sharing an activity with another user with limited control, they is able to choose the level of control.

Add Product Type Filter for All Activities and Review Queue (IC 276970)

With this enhancement, a new "Product Type" filter is added to the advanced search option for Review Queue and All Activities pages so that users can filter their pre-review queue apps by product type (Annuity, Life, etc.).

Impact:

• FL App: Advanced Search dialog on Review Queue page

• FL App: Advanced Search dialog on All Activities page

Option to search in "Replace Form/Wizard/Rule Set in Package" (IC 278324)

With this enhancement, users can search for items within the dropdown on the Replace Form/Wizard/Rule Set in Package dialog on the Packages page in the admin tool. It is more user-friendly and similar to the dropdown used on the Copy Fields dialog on the form/wizard designer.

Impact:

• FL Admin: Dropdowns on Replace Form/Wizard/Rule Set in Package dialog on the Packages page.

Foundation

Masking Passcode Phone Number

With this enhancement, organizations can mask the phone number used as part of the one-time passcode functionality (i.e., ###-###-1234) when the email is sent. This is controlled via a new option on the Organization Settings page.

Impact:

• FL App -> SMS Authentication

• FL Admin -> Organization Settings -> Signature

Foundation

Adding Provider Hook for Pre-Submit Queue

Previously, the Provider had access to a hook to pull a Name Value Pair File (NVPs) for activities in the Post-Submit Review Queue. With this enhancement, the Provider also has access to hooks to pull NVPs for activities in the Pre-Submit Review Queue. Each Queue (Pre- and Post-) will have hooks for the Provider: CallBeforeOSJ and CallAfterOSJ.

• A provider hook for CallBeforeOSJ is added before the case transitions to pre- or post-signature review.

• A provider hook for CallAfterOSJ is added after the case is approved and before submission for the post-signature review process. The existing CallBeforeOSJ using a workflow instance is used to preserve the pre-existing functionality.

• A provider hook for CallAfterOSJ is added for the pre-signature review after the approval process and before the case is moved to another review queue or ready for signing state.

Impact:

Provider: Access to data during the Review Queue process.

NOTE: Impacted clients need to change the CallBeforeOSJ to a CallAfterOSJ before the move to production.

Foundation

Disable Continue after Pre Signature Review Queue: When no signatures are present (IC 279795)

iConnect 267762 previously implemented a new Groups Option feature (Disable Continue after Pre Signature Review Queue). When implemented, it did not account for all potential workflows. With this enhancement, the feature is extended to include activities that have:

• A review queue present

• The review queue bypassed

• No signatures

NOTE: This feature works in Wizards and Wizards & Forms, but Forms-only activities may behave differently.

Impacts:

FL App: Wizard and Wizard & Forms UI Activities

DTCC Updates

As part of the DTCC App Sub Initiative, the following changes were made to the DTCCEntityAutoGen Class to bring it to current standards:

• Added New Properties and marked replaced properties as obsolete.

• Added new enumerations and marked replaced enumerations as obsolete.

• Implemented corrections for properties and enumerations.

• Removed obsolete enumerations values.

Provider updates are required to access the updates.
