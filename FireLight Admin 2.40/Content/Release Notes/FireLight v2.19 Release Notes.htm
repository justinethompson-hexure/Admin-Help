  • Added more details to the "Allow Submit on E-Sign Decline" option in Organization Settings on the Signature tab as follows:

    Allow Submit on E-Sign Decline – Select the checkbox to allow activities to be submitted when electronic signature is declined. This setting uses the carrier setting to override the distributor setting. FireLight looks at the setting of the two organizations. If the distributor has it cleared (unchecked) so that they do not receive submissions when e-signature is declined, but the carrier has the option selected (checked), the carrier still receives the transmission, but the distributor does not.

    For distributors that want to review applications that were declined from e-signature (FLI_DECLINED_ESIGN), you can enable Allow Submit on E-Sign Decline in Organization Settings on the Signature tab. Or, if you do not want apps that were declined e-signature to be transmitted to carriers that do have Allow Submit on E-Sign Decline enabled, the distributor will need to work with each carrier to have the carrier provider modified to not transmit the application.

    NOTE: FLI_DECLINED_ESIGN is what is set when the agent clicks decline e-sign. It is also a common tag value that can be set via rules if the client wants to decline e-signature based on something in the app. And, if the firm does not support e-signature at all, they set that with the organization setting for the logged in org. Since you can use rules to set that value, the carrier could set it via rules, but if the carrier sets that value globally, all orders will skip the e-sign selection screen and decline e-sign. “Allow Submit on E-Sign Decline” is a different setting. That means, send me the data even if they do not e-sign and the FLI_DECLINED_ESIGN is true.

E-Delivery

  • CMR 235019 (Carriers: E-Delivery Clients) E-Delivery is enhanced to support new workflows for Signing Only and Deliver Only. This release focuses on Deliver Only - Client Directed; Deliver Only - Agent Directed; and Sign Only - Client Directed.

    Both of these workflows will enter FireLight as locked cases and will not allow edits or Request Change. For these workflows, the agent or client will have a different option called Request Print that will allow the client to request a printed version. Selecting this option will generate an 1122 to the carrier indicating the status of RequestPrint.

    - Deliver Only child cases is marked complete as soon as the child authenticates and is presented the option to review and download. The parent case is marked complete when all of the child cases are marked complete.

    - Sign Only cases are intended to support documents that only need a signature. These child cases is marked complete as soon as each of the child cases complete their signing. Each signer will have the option to download and receive an email with the link to login and download.

    E-Delivery Impacts:

    • These new workflows is supported in the 121 with subtypes 12122 for Sign Only and 12123 for Deliver Only. Fill and Sign will remain 12100.

    • The Admin Activity setting will have checkbox options added for these new workflows.

    • The Admin Email will have email templates added for these new workflows.

    • New Activity Type Acord Type values is seen in the Package and activity types so that firms can support different wizards and packages for the different workflows.

    • A new eDelivery Complete email was created specifically for eDelivery workflows and will eliminate the usage of the standard template in Activities called Signature Complete.

    • Sign Only and Fill and Sign workflows will both now support the Signer Complete email and the Package Complete email that will apply when there is an agent signature.

    • All workflows will now have a Download 1122 substatus and display status. If the client download, the display status is updated and the 1122 is sent to the carrier with the substatus for DownloadComplete.

    • All workflows will appear in the eDelivery Dashboard.

    • 121 Cancels and Updates are not supported for Deliver Only and Sign Only workflows.

    NOTE: Updates to the FireLight E-Delivery Guide are in progress and not yet now available in Admin Help 12/16/2020.
