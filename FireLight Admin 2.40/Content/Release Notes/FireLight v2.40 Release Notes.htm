FireLight v2.40 Release Notes

New Features

Updated User Interface is Now Available in UAT for All Clients

The New User Interface is now available in UAT. New User Interface can be enabled/ disabled by users in the UAT environment by checking or unchecking the check box that is found in user preferences. For Clients who wish to have the new User Interface available in Staging or Products please submit a JIRA ticket with that request. More information on the new User interface can be found in the FAQ Document and the PDF guide which can be found in Admin Help.

DTCC Producer Authorization. Error Message Enhancements: CR-4088

We updated all “hard stop” error messages to indicate the specific reason that the producer is not authorized to sell.

DTCC Producer Authorization. Messaging for Producer When Its Neither Soft Stop Nor Hard Stop: CR-4178

We added the following DTCC Producer Authorization message: “All requirements are satisfied. You can proceed with selling this product.””

DTCC Producer Authorization. Update Verbiage in the Soft Stop Message: CR-4181

We updated the FireLight soft stop message to read: “DTCC Producer Authorization: You are authorized to sell this product, though certain requirements are not yet fully met”

DocuSign Integration. Gracefully Handle Unaccounted DocuSign Events: CR-4175

We modified payload handling to more gracefully handle unaccounted DocuSign events and reduce error logging.

Prompt to update missing email address in Preferences when user logs in via SSO: DIST-5026

Added logic so that user will be prompted to add Agent email address when it is not present in the SSO.

Update Next Activity Default Messages with User-Friendly Wording: EAES-3905

We updated error messages to be more clear and user-friendly, including the following:

  • "This product has not been configured for your organization. Press OK to continue."

  • "The product you are attempting to access is not available. Press OK to continue."

Report Enhancements and Additions: DIST-4763, DIST-4764, DIST-4788

The BrandID Column was added to the agent Submission Activity Report.

The following updates have been made to various Admin reports:

  • We added the name of FireLight instance—such as Prod, UAT—to the title of FireLight Audit Report

  • We moved the Sales Agreement report to the report list from Admin History reports drop down

  • We added the column "Product Type" Sales Agreement admin report.

Updated Title of FireLight Audit Report: DIST-4860

We updated the title of the FireLight Audit Report to include the name of the environment (QA, UAT, Staging or Production).

Update to Support “Distributor Only” Forms for Back Office Transmissions: EAES-5027, EAES-6452

In the Admin Tool, we added an option to the form Detail “Send to Distributor Only” to support the handling of forms that should only be sent to the distributor. In Firelight we added the functionality required to support this new option.

Update to the Audits in Firelight Admin Tool to Deletion of a Form or Wizard:EAES-5166

We added an audit line for form removal in the Package History and in the Deployment > Package Changes to be displayed when a form or Wizard has been deleted.

Reporting: Added a Transaction Detail Report: EAES-5321, EAES-6449

If enabled for an organization, the completion of a transaction will generate a Transaction Detail Report. The report captures a comprehensive record of information related to a transaction, including wizards, forms, and other key points of data. The feature can be enabled at the organization level and permissions can be adjusted at the group level for more fine-tuned access. We also added a new option on dropdown/combo box controls which, when enabled, will limit displayed values in the Transaction Detail Report for that specific field.

Updates to the DPFA Upload to Accommodate Additional Scenarios for Carrier Appointments and Commission Options: CR-4308

We addressed a DPFA issue where duplicate carrier appointments are generated due to improper handling of CarrierCode and DistributionChannelCode elements in the file. To resolve the issue and prevent future duplicates, the NAICGroupCode is introduced in the system. This will uniquely identify and associate each carrier appointment entry to prevent duplication when dealing with carriers that share the same CarrierCode and DistributionChannelCode. The "Commission Option" dropdown in FL UI is now a text field. This is done to account for the different versions observed in the DPFA.

Error Message Now Displays Multiple Errors: PC-2779

The system now displays multiple errors in an application when errors occur.

Impact

When a person is completing an application, if there are multiple errors on a page, a popup window will appear with all errors being stated.

NOTE: This update has been enabled in QA and UAT but will not be generally enabled in production at this time.
