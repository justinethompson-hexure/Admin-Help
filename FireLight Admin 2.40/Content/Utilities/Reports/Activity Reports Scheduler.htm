Activity Reports Scheduler

The Reports Scheduler Tool allows clients to schedule any available Detailed or Summary activity reports from the Admin App. Clients can configure the sending of required reports at a specific

  • Frequency (Daily/Weekly/Monthly/etc.)

  • Format (CSV/PDF)

  • Send mode (Email or SFTP)

Reports Scheduler can be accessed by users with the proper permissions—granted through the Admin App—to configure activity reports.

Note: The Report Scheduler UI will be available in the organization's UAT environment, though the report generation will happen in their production environment. Organizations using SFTP as their delivery method will manage their FTP info on their production environment.

Add New Group Setting to Allow Access to Report Scheduler

A new group setting, Enable View Report Scheduler, is available from FL Admin (UAT) Organization Settings > Groups page > Admin tab > Enable View > Report Scheduler to provide user groups with access to Report Scheduler.

When this setting is enabled for a user group, users from that group can log in to the Admin App to access the Report Scheduler link from the Utilities menu, available from both the homepage and the top menu.

Note: Managing this setting will be available in the organization’s UAT environment.

Users with the Vendor role do not require the group setting to be enabled for their user group. Users with the Vendor role for an organization can log directly in to the Admin App to access the Report Scheduler and view/schedule activity reports for their organization or the organization that they select on the homepage.

Manage Activity Report Schedule

The updated interface allows users with the correct permissions to select existing detailed and summary activity reports and schedule them for their organization.

Users can add a new schedule, edit an existing schedule, or delete a schedule for their organization. Using the report scheduler screen, admin users can select available detailed and summary activity reports and schedule them at a frequency as per their need to be sent via email or at an FTP location.

Following is the screen for the report scheduler page.

Users can select the following from this page:

• Schedule Name: Name to identify the schedule for a set of reports. This can be duplicated within an organization for multiple report configurations.

• Report Category: Detailed / Summary / Custom

• Report Name: Dropdown listing existing activity reports, based on the report category selected.

• Schedule: Dropdown listing options Daily, Weekly, Monthly, Quarterly, Semi Annual and Yearly.

• Schedule Day:

o When schedule “Weekly” is selected, user can choose day of the week when the report is sent. Default value is “Friday.”

o When schedule is selected as “Monthly,” user can select the day of the month from 1st till last day of the month. Default selected day of the month for "Monthly" schedule is "Last Day of the Month."

• Delivery Method: Email / SFTP

• Recipient Email: When delivery method “Email” is selected, recipient email is mandatory to be entered.

• Report Format: PDF/CSV/Both/Email

• Orientation: Landscape/Portrait

• Report Parameters: When delivery method “SFTP” is selected, this field will show the FTP Host info from the Org Settings page that was saved for activity reports. This field remains in-editable and only for display purpose.

• Description: Section will display the short desc about the selected report.

• Key – Value: This section displays the optional parameter keys for the selected report and value to supplied when generating the report for delivery. The entered value for the key is saved when its checkbox is selected when scheduling the report.

FTP information will be managed from the organization's production environment.

A note will appear on the bottom of the page to let the user know that they must add FTP info for the activity reports to be delivered to that location. Also, this info needs to be verified by the FL support team to create an FTP connection pool to be utilized by the activity reports job scheduler.

The note is “In case delivery method "SFTP" is selected, ensure that correct FTP credentials to send the Activity Reports are entered on Organization Settings > Admin (tab) > Activity Reports FTP Info (link). Once the FTP credentials are entered request FL Support to open the SFTP firewall to start delivering the report.”

Deleting a Schedule

You can delete a schedule by clicking “X” next to the schedule on the grid. An alert message, “Are you sure you want to delete '<Report Name>'?, appears. Confirming this action removes schedule parameters for the report, prevents future delivery to recipients, and cannot be undone.

Confirming Yes deletes the schedule.

View Report Schedule History

Audit history of all schedule changes can be viewed in the Schedule History tab on the Report Scheduler page.

View Report Delivery History

The Delivery History tab on the Report Scheduler page displays the delivery status of the selected schedule. When the reports are delivered successfully at the given delivery mode and schedule, it displays “Success” for the delivery date and time. If there was an issue with the delivery, it displays “Failure” along with a link to view the exception detail. Report delivery history can be viewed by both admin users and users with the vendor role.

Duplicate Schedule Alert

When a new report schedule is added or an existing one is editing, there is a check to not allow creating duplicate schedules for a report. An alert is raised when duplicate schedule with below values is selected at the time of creation or editing:

  • Report Name

  • Report Frequency

  • Report Format

  • Delivery Method

The alert message displayed is "Selected report schedule already exists. Set a different schedule for this report."

Manage FTP Credentials to Deliver Activity Reports

On the Organization Settings page, on the Admin tab, the “Activity Reports FTP Info” links allows entering FTP info for the activity reports.

Clicking this link opens a dialog with text boxes to enter the following FTP info.

  • FTP Credential

  • FTP Host

  • FTP Port

  • FTP Directory

  • FTP Username

  • FTP Password

Note: FTP credentials will be managed from the organization’s production environment.

The note below appears to let users know that after entering the FTP information, FireLight support must be contacted to create an FTP connection pool.

“All fields are required. Ensure that entered FTP info is correct. Once the FTP info is saved, request FL Support to open the SFTP firewall to start delivering the activity reports.”

After FTP details are entered, they cannot be removed/deleted for that organization.

Scheduling Custom and Internal Reports

Custom and internal (for all orgs) reports can be scheduled by the vendor role only.

Scheduling Custom Report for an Org

When a user with the Vendor role logs in to FireLight and selects an organization, they can schedule custom reports for that specific organization. Because custom reports use data items specific to an organization, only Vendor users can schedule custom reports. Once a custom report is scheduled for an organization, the schedule is visible to the organization admin, who can edit or add another schedule for their organization using the same custom report.

Scheduling Internal Use Report

Users with vendor roles can schedule internal reports that generate data used for billing and accounting purposes for all or individual organizations. The internal reports, identified in the Report dropdown by a “**”, are visible to the user with Vendor role only when they land in the Report Scheduler page.

When the vendor user loads the Report Scheduler page without selecting any organization from homepage, they can schedule the internal report for all organizations. On the Report Scheduler page, a Scheduled for All Organizations checkbox appears to schedule reports for all organizations. Reports scheduled for all organizations checking this flag generate report data for all organizations.

If the vendor user loads the Report Scheduler page after selecting an organization from the homepage, they can schedule the internal report for the selected organization. These report schedules will not be available to the admin user for that organization when they log in to their instance (org) of the Report Scheduler page.
