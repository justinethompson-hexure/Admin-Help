Admin Reports

The Reports page contains available Admin reports for your organization.

NOTE: Submit a JIRA ticket to request that we enable the applicable Admin reports for your organization. Any reports with Personally Identifiable Information (PII) need to be Admin Reports that are accessed and printed securely via FireLight Admin on the Reports page in Utilities.
Spotlight Video: See the Reports Spotlight Feature Video for more information.

Generate, View, and Save Admin Reports

  1. On the navigation bar, click Reports to go to the Reports page.
  2. Under Available Reports on the left, select a report type.

  3. Under Report Inputs on the right, enter the applicable report options.

    NOTE: Where applicable, select an activity in the Activity list to filter the report by activity type or select the All option to include all activities in the report.

    NOTE: Sales Agreement Report inputs now include the As of Today checkbox. If selected, generated Sales Agreement reports include only Sales Agreements as of the current date.

    4. Note: CSV is the format for all reports other than the Application Audit and Transaction Detail reports, which are always generated as PDFs.

  4. Click Generate to create the report.

  5. Right-click in the report and click Print to print the report.

Sample Admin Reports

Following are sample Admin reports. Click a link to view a sample report in PDF format.

  • Abandoned Activity Report Abandoned Activity Report lists the activities that are not submitted where the last updated date is X days. Report includes the columns: Application Name, Policy Number, App Create Date, Carrier, Product, Product Type, Agent ID, Agent Role Code, Current Status, Last Updated Date and Days Since Last Updated.

  • Admin History Reports include reports for the following: Activity History Audit Report, Forms History Audit Report, Group History Audit Report, Package History Audit Report, Product History Audit Report, Rule Set History Audit Report, Sales Agreement History Audit Report, Sales Agreement Report, and Wizard History Audit Report.

    • Activity History Audit Report
    • Forms History Audit Report
    • Group History Audit Report
    • Package History Audit Report
    • Product History Audit Report
    • Rule Set History Audit Report
    • Sales Agreement History Audit Report
    • Sales Agreement Report
    • Wizard History Audit Report

    The data in the Admin History reports depend on the selections made in the Report, Select Option, Start Date, and End Date lists. History report columns include the Action, Old Value, New Value, Create Date (with timestamp), and the User Making Changes. The report values coincide with the Action, Old Value, New Value, Action Date, and Login columns in the Activity History, Product History, Package History, Rule Set History, and Sales Agreements History tabs in Activities, Products, Packages, Rule Sets, and Sales Agreements pages in Product Configuration, on the Form History and Wizard History tabs in Form Library and Wizard Library pages in Designer, and on the Group History tab on the Groups page in Organization Configuration. The Sales Agreement Report columns include the Distributor Org, Product Name, Carrier Agreement (if the Carrier Agreement is TRUE or FALSE), Distributor Agreement (if the Distributor Agreement is TRUE or FALSE), State Approvals, Assigned Packages, Last Updated, Firm Org, CUSIP ID, and Product GUID.

    Spotlight Videos: See the Deployments - Additional Details for Admin Deployments and Add Sales Agreement Report to Admin Tool Spotlight Feature Videos for more information.

  • Agent Activity ReportAgent Activity Report lists activity by producer ID. The agent activity report is a pipeline report. It shows the activities that are currently in the works. It shows only activities that have not been set to the complete status, so anything transmitted or less. The Elapsed date is the time since the activity was first created. The report columns include Status, Elapsed, Product Name, Create Date, Application Name, User Name, External Map Code, External Role Code, Carrier, and Transaction Type.
  • Agent Activity Firm Report Agent Activity Firm Report lists Status, Elapsed Days, Product Name, Create Date, Application Name (activity name), Username, External MapCode, Firm, and Channel.
  • Agent Firm Report Agent Firm Report lists Submit Date, Policy Number, Product Name, Date Created, Application Name, Username, External MapCode, Firm, and Channel.
  • Agent ReportAgent Report lists activity submission history by producer ID. The agent report is a submission report. It shows only activities that have been completed. Columns include Submit Date, Policy Number, Product Name, Create Date, Application Name, User Name, External Map Code, Carrier, and Transaction Type.
  • Agent Share ReportAgent Share Report lists the agent names, dates, shared users, and share rights. Columns include Agent Name, Date, Shared User, and Shared Rights.
  • Application Audit ReportApplication Audit Report lists the application packet information and activity audit history.
  • Business Detail ReportBusiness Detail Report lists all submitted transactions in the system; allows filtering based on activity type, firm, and user; and provides an option to select a time frame for the report based on when the transaction was updated.
  • Client Signed Report Client Signed Report lists how the client is signing the application (Remote, Sign Now) and indicates whether Client Fill or Client Fill & Sign was used. Report includes Carrier, Distributor, Agent ID, Agent Code, Application ID, Number of Signers, Client Role Type, Signature Identifier, Signer Signed Multiple Signer Roles, Signing Type, Client Fill (Y/N), and Client Fill and Sign (Y/N).
  • Form Field ReportForm Field Report lists the form fields information for the selected form. Report includes Form Number, Form Name, Pg, Ctl Type, Global, Export, E-Label Name, and Signer Type. The CSV file is an extended version of the report with additional field properties: Edition, Jurisdiction, Input Type, Current Data Group, Current Custom Action, Current Custom List, Protection, Text, Hyperlink, Alignment, Tool Tip, Soft Max, Max Len, Error Placement, Mask, and Font.

  • LexisNexis Instant ID ReportLexisNexis InstantID Report lists the LexisNexis Ref ID, LexID/Order Number, CVI Score, NAS Score, NAP Score, IID Risk Indicator(s), and Pass/Fail Score.

  • LexisNexis Risk Classifier ReportLexisNexis Risk Classifier Report lists the LexisNexis Ref ID, Quoteback ID, Result Codes(s), Risk Score, and Reason Code(s).

  • Package Forms ReportPackage Forms Report lists the information for each form in the selected packages. Report includes Form Name, Form Number, Edition, Jurisdictions, Label, Attachment, and Description. Also included are the:
    • Duty type selected for the form/wizard in the package

    • Number of copies selected for the form in the package

    • Effective start and end dates for the form

    Along with above information, the order of the form listed in the report will be kept the same as the order set into the package.

  • Package Rules ReportPackage Rules Report lists the data item rules for the Form/Data Item ID. Report includes Form/Data Item ID, Package Name, Rule Set, Global, References, Affects, Effective Start Date, and Effective End Date.
  • Pending Agent Signature Report Pending Agent Signature Report lists the activities that are pending agent signatures. Report includes Policy Number, Insured Name, Plan, Risk Amount, Presence of Cash with Application, Date Signed by Insured, Producer Name, Producer Number, and Producer District.
  • Unused Data Report List of Forms/Wizards not found in any packageUnused Data Report List of Forms/Wizards not found in any package lists the forms and wizards that are not found in any package. Report includes Form or Wizard Name, Form or Wizard Number, Form or Wizard, Edition, and Last Update.
  • Unused Data Report List of Packages not found in any Sales AgreementsUnused Data Report List of Packages not found in any Sales Agreements lists the packages that are not found in any sales agreements. Report includes Package Name and Last Update.
  • Unused Data Report List of Rule Sets not found in any packageUnused Data Report List of Rule Sets not found in any package lists the rule sets that are not found in any package. Report includes Rule Set Name, Description, and Last Update.
  • User Detail ReportUser Detail Report lists system users by User Name, External Map Code, External Role Code, and Group Name.
  • User Share ReportUser Share Report lists the User Name, User Share Name, Date Share Initiated, Share Type (Full Control or Read Only), Share Control (Single Activity or All Activities), Activity Name, and User Email.
  • Wizard Field ReportWizard Field Report lists the wizard fields information for the selected wizard. Report includes Wizard (Form) Number, Wizard (Form) Name, Page, Control Type, Name, and Text. The CSV file is an extended version of the report with additional field properties including Read Only, Office Use Only, Input Type, Is Multi Line, Max Length, Text Align, and Visibility.
  • Workflow ReportWorkflow Report shows the worklow stops of all submitted transactions. Report includes Timestamp, Status, User, Role Code, Wealthscape ID, Carrier Name, Product Name, CUSIP, Product Type, Transaction Status, Confirmation Number, Advisor Name, Advisor ID, Account Number, Plan Type, Money Source, Transaction Premium Amount, and Commission Amount.

Custom Report

The following custom client-specific report can be added. Submit a JIRA ticket to have the report developed.

  • Pre-Application Report Pre-Application Report is a report that has dataitems specific to the client. The following elements are added in CSV format: Owner Type, Joint Owner, Owner gender, Owner city, Owner state, Owner zip, Joint address same, Joint city, Joint state, Joint zip, S&P, Nasdaq, Russell, Bloomberg, Fixed, Non-qualified, Traditional IRA, Roth IRA, Rider, ACH 401k, ACH IRA, ACH qual plan, Transfer Company, Add Premium Amount, Bene type, and Contingent bene type. This report can be automated to run on a scheduler. Because this is an Admin report, it requires a release build to implement. It also requires a JIRA ticket and a work order.

Report Column Descriptions

  • Action = type of action performed — asppears in various audit history reports

  • Activity Name = Name of a single activity that was shared

  • Affects = the action triggered by a rule (e.g., Trigger 1035 replacement Forms)

  • Agent ID = agent identification number (e.g., CRD Number)

  • Agent Code, Agent Role Code = agent group Role Code set up in FireLight Admin

  • App Create Date = date when the activity was created

  • Application ID = the activity identifier in FireLight

  • Application Name = activity name (e.g., Jane C Needs Determination, Jane C Application, Client Illustration, etc.)

  • Background Color = background color (Transparent or White) of the control on a form

  • Barcode Format = format (e.g., hyperlink, etc.) of a Barcode control on a form

  • Barcode Type = type (QRCode or Datamatrix) of a Barcode control on a form

  • Captured Name = name of signer

  • Captured Signature = signer's signature

  • Carrier = carrier's name in FireLight

  • Channel = channel name if channel names are set up in FireLight

  • Client Fill = indicates whether a Client Fill request was sent to the client for the activity

  • Client Fill & Sign = indicates whether a Client Fill and Sign request was sent to the client for the activity

  • Client Role Type = the signer type in FireLight (Owner, Annuitant, Insured, Joint Owner, Agent, Beneficiary, etc.)

  • Control Type, Ctrl Type = type of control (Text Box, Radio Button, etc.) on a form or wizard in FireLight

  • Current Custom Action = custom action button control on a form or wizard

  • Current Custom List = Custom List that the custom action button control opens on a form or wizard

  • Current Data Group = a data group for controls associated with a custom action button on a form or wizard

  • Create Date = activity creation date or the date an action occurred in an audit report

  • Current Status = current activity status (Pending Approval, Active Application, Locked, etc.)

  • Date Share Initiated = date when the user share was initiated

  • Date Signed by Insured = date when the Insured signed the activity

  • Days Since Last Updated = number of days since the activity was last updated

  • Description = rule description (e.g., Validates address fields)

  • Distributor = distributor's name in FireLight

  • Edition = edition of the form in FireLight Admin

  • E-Label Name, Name = the name of the control (e.g., Owner_DOB) on the form or wizard

  • Elapsed, Elapsed Days = days since the activity was updated

  • Email Address = email ID of signer

  • Enforce Match State/State Match Warning = warns about solicitation state/signing state mismatch in a Signature control on a form

  • Error Placement = location of the validation message (Default, Bottom, Top, Left, Right) for a control on a form

  • Exclude From PDF = excludes the control when the form is printed to PDF

  • Export, Export Value = the export value for the control (True, False) on the form

  • External Map Code = agent first name, last name, and identification number (e.g., CRD Number)

  • Firm = firm name if firm names are set up in FireLight

  • Font = font (Arial, Calibri, etc.) for a control on a form

  • Font Color = color of the font for a control on a form

  • Font Size = size of the font for a control on a form

  • Form Name = form name in FireLight Admin

  • Form Number = form number in FireLight Admin

  • Global, Is Global = indicates whether the control on the form is set for global use of a control that is added to the form more than once

  • Group Name = name for a group of signatures for a Signature control on a form

  • Hyperlink = hyperlink in the Url field property in a Label control on a form

  • Input Type = HTML5 text input “Type” attribute (empty, Email, Number, Color, Tel, Url) on a form or wizard

  • Insured Name = name of the Insured in an activity

  • Is Multi Line = whether a control can have multiple lines of text or text wrapping on a form or wizard in FireLight

  • Is Password = whether a control is a hidden entry text box on a form

  • Jurisdiction, Jurisdictions = approved jurisdictions for the form in FireLight

  • Last Update = date when the rule set was last updated

  • Last Updated Date = date when the activity was last updated

  • Mask, Edit Mask = specified format (Date, Date Long, SSN/TIN, Phone, Phone w Extension, Zipcode, Numeric, Routing) for a control on a form or wizard

  • Max Length, Max Len = maximum number of characters that can be entered into a control on a form or wizard

  • Name = name of the control (e.g., Owner_SSN) on a wizard in FireLight

  • New Value = the new value after the action occurred — appears in various audit history reports

  • Number of Signer = number of signers for the activity

  • Office Use Only = whether a control is for Office Use Only on a form or wizard

  • Old Value = the existing value before the action occurred — appears in various audit history reports

  • Organization = organization name for the carrier, distributor, etc.

  • Package = the name of the package in FireLight Admin

  • Page = name of the wizard page in FireLight

  • Pg = page number in the form in FireLight

  • Plan = name of the Plan in an activity

  • Policy Number = policy number if applicable for the activity

  • Presence of Cash with Application = indicates whether cash is presented with the application

  • Producer District = producer district

  • Product Name = product name shown in FireLight

  • Producer Number = producer identification number

  • Product Type = type of product (Term Life, Variable Annuity, etc.)

  • Read Only = whether a control is Read Only on a form or wizard

  • References = rule action references (e.g., Add Additional Agents, etc.)

  • Request Type = only for client signers, Client Fill & Sign; for client and agent signers, E-Sign Email Request

  • Risk Amount = amount of Risk

  • Rule Set Name = name of the rule set in FireLight Admin

  • Sent On = data application sent for review and signing

  • Share Control = type of share control (Single Activity, All Activities)

  • Share Rights, Share Type = type of user share (Full Control, Read Only)

  • Shared User = user to whom the activity was shared

  • Signature Identifier = signature identifier in FireLight

  • Signed On = date application signed

  • Signer Signed Multiple Signer Role = indicates (Y or N) whether a single signer signed for multiple signer roles (e.g., Owner and Annuitant = the same person who signed for both roles)

  • Signer Type = type of signer (Owner, Agent, Annuitant, etc.) on a form or wizard

  • Signing Type = type of signature capture (Sign Now, Sign Later)

  • Soft Max = Soft Limit (maximum character limit) field property for a control on a form

  • Source Item = the Item Source list in a Combo control on a form

  • Status = current activity status (Data Entry, Review, Submit, etc.)

  • Submit Date = date of activity submission to back office

  • Tab Index = tabbing order of the control on a form if different from the default tab order

  • Text = display text for a control name on a form or wizard in FireLight

  • Text Alignment, Text Align = text alignment of Left, Center, or Right for a control on a form or wizard

  • Tool Tip = Help Text property in a control on a form or wizard

  • Transaction Type = activity type (Account Opening, PreSale, Application, e-Delivery, Illustration, Quote, etc.)

  • Use Lexis Nexis = use LexisNexis third-party client authentication for a signer role, set in the Signature control on a form

  • Use SMS Text = use SMS authentication for the signer role, set in the Signature control on a form

  • User Email = user email address

  • User Making Changes = user name of the user who made the change — appears in various audit history reports

  • Username, User Name = name of the user in FireLight

  • User Share Name = the user share name in FireLight

  • Using IP Address = IP address used by signer

  • View On = data application viewed

  • Visibility = the Protection setting (None, Public, Private, Client, Agent, Signer, Reviewer) for visibility of the form or wizard

  • Wizard Name = name of the wizard in FireLight

  • Wizard Number = number of the wizard in FireLight
