Custom Reports

The Reports page contains available Custom reports for your organization. Contact Hexure for assistance with developing Custom reports for your organization.

NOTE: Submit a JIRA ticket to request that we enable the applicable Custom reports for your organization. Any reports with Personally Identifiable Information (PII) need to be Custom Reports that are accessed and printed securely via FireLight Admin on the Reports page in Utilities.
Spotlight Video: See the Reports Spotlight Feature Video for more information.

Generate, View, and Save Custom Reports

  1. On the navigation bar, click Reports to go to the Reports page.
  2. Under Available Reports on the left, select a report type.

  3. Under Report Inputs on the right, enter the applicable report options.

    NOTE: Where applicable, select an activity in the Activity list to filter the report by activity type or select the All option to include all activities in the report.

    NOTE: Sales Agreement Report inputs now include the As of Today checkbox. If selected, generated Sales Agreement reports include only Sales Agreements as of the current date.

    4. Note: CSV is the default format for all reports other than the Application Audit report, which is always generated as a PDF.

  4. If desired, on the lower right select PDF to generate the report in a format other than CSV.

  5. Click Generate to create the report.

  6. Right-click in the report and click Print to print the report.

Report Column Descriptions

  • Action = type of action performed — appears in various audit history reports

  • Activity Name = Name of a single activity that was shared

  • Affects = the action triggered by a rule (e.g., Trigger 1035 replacement Forms)

  • Agent ID = agent identification number (e.g., CRD Number)

  • Agent Code, Agent Role Code = agent group Role Code set up in FireLight Admin

  • App Create Date = date when the activity was created

  • Application ID = the activity identifier in FireLight

  • Application Name = activity name (e.g., Jane C Needs Determination, Jane C Application, Client Illustration, etc.)

  • Background Color = background color (Transparent or White) of the control on a form

  • Barcode Format = format (e.g., hyperlink, etc.) of a Barcode control on a form

  • Barcode Type = type (QRCode or Datamatrix) of a Barcode control on a form

  • Captured Name = name of signer

  • Captured Signature = signer's signature

  • Carrier = carrier's name in FireLight

  • Channel = channel name if channel names are set up in FireLight

  • Client Fill = indicates whether a Client Fill request was sent to the client for the activity

  • Client Fill & Sign = indicates whether a Client Fill and Sign request was sent to the client for the activity

  • Client Role Type = the signer type in FireLight (Owner, Annuitant, Insured, Joint Owner, Agent, Beneficiary, etc.)

  • Control Type, Ctrl Type = type of control (Text Box, Radio Button, etc.) on a form or wizard in FireLight

  • Current Custom Action = custom action button control on a form or wizard

  • Current Custom List = Custom List that the custom action button control opens on a form or wizard

  • Current Data Group = a data group for controls associated with a custom action button on a form or wizard

  • Create Date = activity creation date or the date an action occurred in an audit report

  • Current Status = current activity status (Pending Approval, Active Application, Locked, etc.)

  • Date Share Initiated = date when the user share was initiated

  • Date Signed by Insured = date when the Insured signed the activity

  • Days Since Last Updated = number of days since the activity was last updated

  • Description = rule description (e.g., Validates address fields)

  • Distributor = distributor's name in FireLight

  • Edition = edition of the form in FireLight Admin

  • E-Label Name, Name = the name of the control (e.g., Owner_DOB) on the form or wizard

  • Elapsed, Elapsed Days = days since the activity was updated

  • Email Address = email ID of signer

  • Enforce Match State/State Match Warning = warns about solicitation state/signing state mismatch in a Signature control on a form

  • Error Placement = location of the validation message (Default, Bottom, Top, Left, Right) for a control on a form

  • Exclude From PDF = excludes the control when the form is printed to PDF

  • Export, Export Value = the export value for the control (True, False) on the form

  • External Map Code = agent first name, last name, and identification number (e.g., CRD Number)

  • Firm = firm name if firm names are set up in FireLight

  • Font = font (Arial, Calibri, etc.) for a control on a form

  • Font Color = color of the font for a control on a form

  • Font Size = size of the font for a control on a form

  • Form Name = form name in FireLight Admin

  • Form Number = form number in FireLight Admin

  • Global, Is Global = indicates whether the control on the form is set for global use of a control that is added to the form more than once

  • Group Name = name for a group of signatures for a Signature control on a form

  • Hyperlink = hyperlink in the Url field property in a Label control on a form

  • Input Type = HTML5 text input “Type” attribute (empty, Email, Number, Color, Tel, Url) on a form or wizard

  • Insured Name = name of the Insured in an activity

  • Is Multi Line = whether a control can have multiple lines of text or text wrapping on a form or wizard in FireLight

  • Is Password = whether a control is a hidden entry text box on a form

  • Jurisdiction, Jurisdictions = approved jurisdictions for the form in FireLight

  • Last Update = date when the rule set was last updated

  • Last Updated Date = date when the activity was last updated

  • Mask, Edit Mask = specified format (Date, Date Long, SSN/TIN, Phone, Phone w Extension, Zipcode, Numeric, Routing) for a control on a form or wizard

  • Max Length, Max Len = maximum number of characters that can be entered into a control on a form or wizard

  • Name = name of the control (e.g., Owner_SSN) on a wizard in FireLight

  • New Value = the new value after the action occurred — appears in various audit history reports

  • Number of Signer = number of signers for the activity

  • Office Use Only = whether a control is for Office Use Only on a form or wizard

  • Old Value = the existing value before the action occurred — appears in various audit history reports

  • Organization = organization name for the carrier, distributor, etc.

  • Package = the name of the package in FireLight Admin

  • Page = name of the wizard page in FireLight

  • Pg = page number in the form in FireLight

  • Plan = name of the Plan in an activity

  • Policy Number = policy number if applicable for the activity

  • Presence of Cash with Application = indicates whether cash is presented with the application

  • Producer District = producer district

  • Product Name = product name shown in FireLight

  • Producer Number = producer identification number

  • Product Type = type of product (Term Life, Variable Annuity, etc.)

  • Read Only = whether a control is Read Only on a form or wizard

  • References = rule action references (e.g., Add Additional Agents, etc.)

  • Request Type = only for client signers, Client Fill & Sign; for client and agent signers, E-Sign Email Request

  • Risk Amount = amount of Risk

  • Rule Set Name = name of the rule set in FireLight Admin

  • Sent On = data application sent for review and signing

  • Share Control = type of share control (Single Activity, All Activities)

  • Share Rights, Share Type = type of user share (Full Control, Read Only)

  • Shared User = user to whom the activity was shared

  • Signature Identifier = signature identifier in FireLight

  • Signed On = date application signed

  • Signer Signed Multiple Signer Role = indicates (Y or N) whether a single signer signed for multiple signer roles (e.g., Owner and Annuitant = the same person who signed for both roles)

  • Signer Type = type of signer (Owner, Agent, Annuitant, etc.) on a form or wizard

  • Signing Type = type of signature capture (Sign Now, Sign Later)

  • Soft Max = Soft Limit (maximum character limit) field property for a control on a form

  • Source Item = the Item Source list in a Combo control on a form

  • Status = current activity status (Data Entry, Review, Submit, etc.)

  • Submit Date = date of activity submission to back office

  • Tab Index = tabbing order of the control on a form if different from the default tab order

  • Text = display text for a control name on a form or wizard in FireLight

  • Text Alignment, Text Align = text alignment of Left, Center, or Right for a control on a form or wizard

  • Tool Tip = Help Text property in a control on a form or wizard

  • Transaction Type = activity type (Account Opening, PreSale, Application, e-Delivery, Illustration, Quote, etc.)

  • Use Lexis Nexis = use LexisNexis third-party client authentication for a signer role, set in the Signature control on a form

  • Use SMS Text = use SMS authentication for the signer role, set in the Signature control on a form

  • User Email = user email address

  • User Making Changes = user name of the user who made the change — appears in various audit history reports

  • Username, User Name = name of the user in FireLight

  • User Share Name = the user share name in FireLight

  • Using IP Address = IP address used by signer

  • View On = data application viewed

  • Visibility = the Protection setting (None, Public, Private, Client, Agent, Signer, Reviewer) for visibility of the form or wizard

  • Wizard Name = name of the wizard in FireLight

  • Wizard Number = number of the wizard in FireLight
