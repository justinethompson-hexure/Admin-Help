Package Conversion

NOTE: To enable the Package Conversion utility, go to the Organization Configuration/Groups page Admin tab and select the Enable View Utilities checkbox. This enables all utilities for your organization.

Use the Package Conversion utility to create new forms, wizards and rule sets within a package that contain the new dataitem tags from the mapping spreadsheet and then select an expiration date to stop using the existing forms, wizards and rule sets with the old tags and start using the new forms, wizards and rule sets with the new tags.

This utility is especially useful in converting existing tags to common tags (see Common Tags for more information).

Before Converting a Package

Use the Download Tag List utility to download a spreadsheet of existing dataitem tags within a package and add the mapping "to" dataitem tags to the spreadsheet. See Download Tag List for instructions. This is especially useful if most of the dataitem tags are going to be converted to common tags.

NOTE: The client and the Project Manager and developers need to coordinate all updates to prevent issues with data.

Convert a Package

  1. On the navigation bar, click Utilities/Package Conversion.
  2. Under Upload Mapping Spreadsheet, drop the mapping spreadsheet or click in the box and select the mapping spreadsheet.
  3. In the Package to Convert list, select the applicable package to create new forms, wizards and rule sets with new tags from the mapping spreadsheet.
  4. In the Package Items to Convert grid, select or clear the checkboxes to include only the applicable items to convert to the new tags.
  5. (Optional) Click in the Select Expiration Date box and select a date to expire the current forms, wizards and rule sets within the package and start using the converted forms, wizards and rule sets with the new tags within the package. NOTE: If you do not set an expiration date, you can update the package later after verifying the converted forms and rule sets converted correctly to common tags (see step 10).
  6. (Optional) In the TimeZone list, select the time zone for the expiration date.

  7. (Optional) Click APPLY EXPIRATION DATE. NOTE: This step can be delayed until after you verify the converted forms, wizards and rule sets in the package. You can return to the Package Conversion utility and then set the expiration date.

    The Expiration Date (Switchover Date) column displays the selected expiration date.

  8. Select the Add converted items to the package checkbox to add the new converted forms, wizards and rule sets to the package.

  9. Click CONVERT PACKAGE and wait for processing to complete.

    When processing completes, the Package Items to Convert grid displays "Converted" in the Status column.

  10. Verify the converted forms, wizards and rule sets. Converted items appear with "-Converted [DATE]" next to the form, wizard or rule set name in the grids at the top of the pages.

    • Go to Designer/Form Designer page to view the dataitem names in the converted forms. Go to Designer/Form Library page and ensure "This Form Begins On" date is correct on the converted forms after you apply an expiration date.
    • Go to Designer/Wizard Designer page to view the dataitem names in the converted wizards. Go to Designer/Wizard Library and ensure "This Wizard Begins On" date is correct on the converted wizards after you apply an expiration date.
    • Go to the Product Configuration/Rule Sets page to view the converted rule sets and ensure "This Rule Set Begins On" date is correct after you apply an expiration date.
  11. Go to the Product Configuration/Packages page. Verify the converted forms, wizards and rule sets are in the product Package.
  12. When the begin date for the package conversion arrives, go to the Utilities/Deployments page. Deploy the product Package Configuration or All Configuration for the applicable carrier or distributor environment.
  13. Next Step: Contact your Strategic Account Manager for developer assistance with converting old tags to new tags in the Provider. Work with the FireLight integration team to schedule the converted Provider to be deployed when the new forms, wizards and rules begin.
