Testing Process

Guidelines

  • To avoid accidentally overwriting work in Staging, it is imperative that you always deploy from left (UAT) to right (Staging and then Production).

    This ensures that the product changes you test and approve are not accidentally overwritten when a product is deployed from UAT to Staging.

  • Deploy and test all products and development changes in UAT.
  • Promote from UAT to Staging.
  • Promote from Staging to Production.

NOTE: When logged in to the UAT or Staging environment in FireLight® Admin, the environment name appears at the top of each page to the left of the navigation bar.

Distributor Testing Options

Test Carrier Products in UAT

To test the carrier products, you can clear the Auto Deploy checkbox in UAT. Doing this stops carrier changes from being auto deployed (promoted) from UAT to Staging and allows the distributor to test carrier changes in UAT. When the distributor has completed their testing in UAT, the distributor must manually promote to Staging. If you want to force that change to Production at the same time, select the Force Deploy checkbox during the deployment. Doing this allows the product changes to move to Staging and to Production in one step.

No Testing for Carrier Products

Select the Auto Deploy checkbox in UAT and Staging. If enabled in both UAT and Staging, a carrier deployment to that distributor’s UAT deploys to the distributor’s UAT, Staging, and Production environments with one step and requires no manual promotion.

Carrier Testing Options

There are three configurations for carriers. To support the various configuration options, we have deployment scopes that are used to support configuration 1 (carrier-to-carrier) deployments and separate scopes to support deployment to isolated distributor environments, configuration 2 (carrier-to-distributor). If a carrier falls under configuration 2, they have to deploy once using a carrier-to-distributor scope and deploy a second time using a carrier-to-carrier scope.

Support Carrier Instance of FireLight Only

If your organization does not promote products to distributor instances of , follow the options in this section.

  • Option 1: Test your products in UAT Only. When you are done testing your products in UAT, you can deploy the changes to your Staging and Production environments by using a carrier-to-carrier scope and selecting the Force Deploy checkbox. This promotes your changes from your UAT environment to both your Staging and your Production environments in one step.
  • Option 2: Test your products in UAT and Staging. You have the option to test your products first in UAT and then in Staging. We do ask that you always start in UAT so that your products do not accidentally get overwritten. Use a carrier-to-carrier scope and clear the Force Deploy checkbox.

Support Distributor Instance of Only

If your organization promotes products to distributor instances of only, follow the options in this section.

  • Option 1: Test carrier products in UAT. Test all of your changes within your UAT environment. When testing is complete, you promote your changes to the distributor’s instance of using a carrier-to-distributor scope. This promotes all of your changes to either the distributor’s UAT environment, the distributor's UAT and Staging environments, or the distributor’s UAT, Staging and Production environments. The environments updated depend on how the distributor sets the Auto Deploy checkbox within the environment. If the Auto Deploy checkbox is selected, the deployment updates the current environment and automatically updates the next environment (e.g., Staging, Production).
  • Option 2: Test carrier products in the distributor environment. To test your changes in a distributor environment requires you to obtain a login to the distributor’s instance of .

Support Carrier Instance and Distributor Instances of FireLight

If your organization promotes products to both your instance of and distributor instances of , follow the options in this section.

  • Option 1: Test your products in UAT. Test all of your changes within your UAT environment. When testing is complete, you promote your changes to the distributor’s instance of using a carrier-to-distributor scope. This promotes all of your changes to the distributor’s UAT environment, the distributor's UAT and Staging environments, or the distributor’s UAT, Staging and Production environments. The environments updated dependent on the distributor sets the Auto Deploy checkbox within the environment. If the Auto Deploy checkbox is selected, the deployment updates the current environment and automatically updates the next environment (e.g., Staging, Production).

    You also need to use a carrier-to-carrier deployment to promote your changes to your instance of Staging, or Staging and Production. Check the “Force Deploy” option to promote the changes to both Staging and Production at the same time.

  • Option 2: Test your products in the distributor environment. To test your changes in a distributor environment requires you to obtain a login to the distributor’s instance of . Follow option 1, then test using the distributor login within their environment.

  • Option 3: Test your products for distribution in UAT, and test your products for your instance in UAT and Staging. Promote your changes to the distributor’s instance of using a carrier-to-distributor scope. This promotes all of your changes to the distributor’s UAT environment, the distributor's UAT and Staging environments, or the distributor’s UAT, Staging, and Production environments. The environments updated dependent on the distributor sets the Auto Deploy checkbox within the environment. If the Auto Deploy checkbox is selected, the deployment updates the current environment and automatically updates the next environment (e.g., Staging, Production).

    Promote the changes to your Staging environment by using a carrier-to-carrier scope and clearing the Force Deploy checkbox.

Hot Fixes

Hot fix testing is permitted in Staging. Because all normal testing is occurring in UAT, Staging is a clean environment that mirrors production. Please note that any product changes that are done directly in Staging need to be redone in UAT.

If the distributor does not want to test the hot fix changes, they should set Auto Deploy in Staging. When a carrier deploys a new product into the distributor's Staging environment, it deploys to Staging and also deploy to Production. This scenario should occur only for hot fix testing.

NOTE: Any hot fix made to Staging that is deployed to Production requires the change to be done manually to the UAT environment. This ensures that when UAT is deployed to Staging, any hot fix in Staging and Production is not overwritten.
