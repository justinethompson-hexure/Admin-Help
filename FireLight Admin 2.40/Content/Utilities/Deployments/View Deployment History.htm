View Deployment History

On the navigation bar, click Utilities/Deployments to go to the Deployments page. Click the History tab.

Use the History tab on the Deployments page to view deployment history.

NOTE: When logged in to the UAT or Staging environment in ® Admin, the environment name appears at the top of each page to the left of the navigation bar.

  1. On the navigation bar, click Utilities/Deployments to go to the Deployments page.

  2. Click the History tab on the left.

  3. In the Synchronization Scope list, select a configuration.

    All Configuration includes additions and changes to the top level of data (Activities, Custom Lists, Products, Quote data, Forms/Wizards, Packages, Sales Agreements, and Rule Sets). Carriers use the All Configuration to promote changes, including Sales Agreements, to their next environments. Distributors use the All Configuration to deploy distributor information to their next distributor environments and to deploy carrier information to their next distributor environments, including Sales Agreements, when not using Auto Deploy.

    Commission Netting Configuration (for carriers only) includes each DPfA that can be deployed. Multiple DPfAs can be selected for deployment, but each one selected appears as a single, individual change. See in Product Configuration.

    Custom List Configuration includes additions and changes to custom lists modified through the Custom Lists page in Organization Configuration.

    Data Item Sync Configuration includes synched Data Items. See Mapping in Organization Configuration.

    Fees and Expenses Configuration is used by carriers to deploy manually uploaded fee files from Staging into Production.

    Firm Configuration includes the Firm additions, changes, and deletions on the Firm Management page in Organization Configuration.

    Form Configuration includes the form and wizard level of data and nothing else. See Designer.

    Group Configuration includes all changes to the role codes modified through Groups in Organization Configuration.

    Localization/Branding Configuration includes changes to localization or branding modified through the optional Localization or Branding features.

    Mapping Configuration includes changes to mappings modified on the Mapping page, Mapping tab in Organization Configuration.

    Organization Configuration includes all data that is specific to the organization such as email addresses, email messages, corporate information, etc. By design, the IDP Identifier, IDP Certificates, and Purge settings are not included in the Organization scope. See Organization Settings in Organization Configuration.

    Package Configuration includes the product, package, rule set, and form/wizard data. See Packages in Product Configuration.

    Product Configuration includes additions and changes to Products and nothing else.

    Repository Product Configuration includes additions and changes to products in the .

    Review Queue Configuration includes changes to the review queue script and to review queues modified through Organization Configuration in Review Queue.

    Rule Set Configuration  includes changes to rule sets modified through the Form Designer Rules Window, Wizard Designer Rules Window, and through Rule Sets in Product Configuration.

    Tele-Sign Script Configuration includes additions and changes to the Tele-Sign Scripts in Organization Configuration.

    Carrier Only All Configuration includes the top level of data (Activities, Custom Lists, Products, Quote data, Packages, Rule Sets, and Forms/Wizards). Distributors use this scope to deploy carrier top-level configurations, excluding Sales Agreements, to distributor environments when not using Auto Deploy.

    Carrier Only Product Configuration includes additions and changes to Products and nothing else. Distributors use this scope to deploy carrier product configurations to distributor environments if not using Auto Deploy.

    Carrier Only Form Configuration includes only the form and wizard level of data. Distributors use this scope to deploy carrier forms and wizards to distributor environments if not using Auto Deploy.

    Carrier Only Package Configuration includes the product, package, rule set, and form/wizard data. Distributors use this scope to deploy carrier packages to distributor environments if not using Auto Deploy.

    Carrier Only Rule Set Configuration  includes changes to rule sets modified through Form Designer Rules Window, Wizard Designer Rules Window, and through Rule Sets in Product Configuration.

    Carrier Only Custom List Configuration includes only the custom list level of data. Distributors use this scope to deploy carrier custom lists to distributor environments if not using Auto Deploy.

    Distributor All Configuration includes the top level of data (Activities, Custom Lists, Products, Quote data, Packages, Sales Agreements, Rule Sets, and Forms/Wizards) that are specific to the selected distributors. Carriers use this scope to deploy carrier top-level configurations to distributor environments.

    Distributor Product Configuration additions and changes to Products and that are specific to the selected distributors. Carriers use this scope to deploy carrier product configurations to distributor environments.

    Distributor Form Configuration includes only the form and wizard level of data. Carriers use this scope to deploy carrier forms and wizards to distributor environments. Select one or more forms and wizards for deployment.

    Distributor Package Configuration includes the product, package, rule set, and form/wizard data. Carriers use this scope to deploy carrier packages to distributor environments.

    Distributor Rule Set Configuration  includes changes to rule sets modified through Form Designer Rules Window, Wizard Designer Rules Window, and through Rule Sets in Product Configuration.

    Distributor Custom List Configuration includes changes to custom lists modified through the Designer page. Carriers use this scope to deploy carrier custom lists to distributor environments.

  4. Optionally select a server environment (UAT, Staging, Production) in To Server, select an organization in To Organization, or enter a date range in calendars in Start and End to filter the history.

  5. Click Load History to load deployment history. Click the Cancel button on the right to cancel loading the history.

  6. On the left, click a row in the grid to select a deployment history.

  7. On the right, view deployment statistics and deployment status history for the selected deployment process data changes. Click a link to view detailed Prosduct, Package, Form, Sales Agreement, Rule Set, Custom List, and Activity changes in a dialog. The Package Changes, Form Changes, and Rule Set Changes dialogs display the changes in a panel format with search filtering available at the top of the dialog, individual forms, packages, or rule sets listed in the left pane, and Date/Time, Action, Old Value, and New Value columns in a grid format on the right.

  8. In Deployment Statistics, click the Open link to view a list of items updated in the Deployment Detail dialog.

NOTE: The system purges history for all deployment scopes and audits that are more than one year old.
