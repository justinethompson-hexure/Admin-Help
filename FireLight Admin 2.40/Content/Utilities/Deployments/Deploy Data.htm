Deploy Data

Guidelines

  • To receive deployment notification emails, enter one or more email addresses in the Deployment Notification Email field on the Organization tab of the Organizations Settings page in Organization Configuration.
  • When viewing or uploading Fee data in UAT, it is actually stored in Staging. This is because there is only one DTCC test region, which is another potential source of the fee data for carriers that utilize it, and regardless of the source (manual upload or DTCC), the fee data is stored in the FireLight® DTCC subsystem. It is accessible from UAT and/or Staging to allow testing in either environment, but it resides only in Staging. Fee data that comes from DTCC cannot be deployed – instead, it must be passed through the DTCC production region.
  • Carriers use the non-Distributor configurations in Synchronization Scope to deploy their data to the next carrier environment, and they use the Distributor configurations to deploy their products/packages/forms to distributors.
  • Distributors use the non-Distributor configurations in Synchronization Scope to deploy the distributor products/packages/forms, etc., to the next distributor environment, and they use the Carrier Only configurations to deploy the carrier products/packages/forms, etc., to the next distributor environment.
  • When Auto Deploy is enabled in UAT on the Sales Agreements page in Product Configuration, auto deploy will be in effect for all carriers to that distributor from carrier UAT to distributor Staging. Otherwise, if the distributor enables Auto Deploy in Sales Agreement settings for a given Carrier Org, auto deploy will be in effect for that carrier to the distributor from carrier UAT to distributor Staging. In Staging, if the distributor organization enables Auto Deploy in Organization settings > Admin, auto deploy will be in effect for all carriers to that distributor from carrier UAT to distributor Production (or from carrier Staging to distributor Production). Otherwise, if the distributor enables Auto Deploy in Sales Agreement settings for a given Carrier Org, auto deploy will be in effect for that carrier to the distributor from carrier UAT to distributor Production (or from carrier Staging to distributor Production).
  • If a carrier has the Deploy Email Group added on the Sales Agreements page for a distributor, the distributor emails recipients are notified of carrier deployments that may affect the users' experience for that carrier's deployment configurations in the distributor environment, regardless of the distributor Auto Deploy setting.
  • The system does not allow deployments from Production to prevent overwriting data in Staging.

  • When an admin user for a Firm Org is logged in, they have access to Deployments. However, that access is greatly restricted. Admin users of the Firm Orgs have the Synchronization Scope default to "Group Configuration," and that is the only item available to them from the dropdown list. The list of Groups they see is only Groups that are associated with their organization.

Process

Use the Deployment tab on the Deployments page to move new or updated data from the UAT environment (where all of your development and your testing occurs) to the Staging environment and then to the Production environment.

  1. Ensure you are logged in to the correct environment (UAT or Staging) for the deployment.

    The environment name appears to the left of the navigation bar.

  2. On the navigation bar, click Utilities/Deployments to go to the Deployment tab on the Deployments page.

  3. In the Synchronization Scope list, select a configuration. The default is Select Scope, which speeds the loading of the Deployments page.

    All Configuration includes additions and changes to the top level of data (Activities, Custom Lists, Products, Quote data, Forms/Wizards, Packages, Sales Agreements, and Rule Sets). Carriers use the All Configuration to promote changes, including Sales Agreements, to their next environments. Distributors use the All Configuration to deploy distributor information to their next distributor environments and to deploy carrier information to their next distributor environments, including Sales Agreements, when not using Auto Deploy. NOTE: Activity data is part of the All Configuration, but some of the Activity data is part of Branding (Activity Name, Apply Label & Messages). If the user changed the name, label, or messages in Activity they should also deploy the Localization/Branding Configuration along with the All Configuration.

    Commission Netting Configuration (for carriers only) includes each DPfA that can be deployed. Multiple DPfAs can be selected for deployment, but each one selected shows as a single, individual change. See in Product Configuration.

    Custom List Configuration includes additions and changes to custom lists modified through the Custom Lists page in Organization Configuration.

    Data Item Sync Configuration includes synched Data Items. See Mapping in Organization Configuration.

    Fees and Expenses Configuration is used by carriers to deploy manually uploaded fee files from Staging into Production.

    Firm Configuration includes the Firm additions, changes, and deletions on the Firm Management page in Organization Configuration.

    Form Configuration includes the form and wizard level of data and nothing else. See Designer.

    Group Configuration includes all changes to the role codes modified through Groups in Organization Configuration.

    Localization/Branding Configuration includes changes to localization or branding modified through the optional Localization or Branding features. NOTE: Activity data is part of the All Configuration, but some of the Activity data is part of Branding (Activity Name, Apply Label & Messages). If the user changed the name, label, or messages in Activity, they should also deploy the Localization/Branding Configuration along with the All Configuration.

    Mapping Configuration includes changes to mappings modified on the Mapping page, Mapping tab in Organization Configuration.

    Organization Configuration includes all data that is specific to the organization, such as email addresses, email messages, corporate information, etc. By design, the IDP Identifier and IDP Certificates are not included in the Organization scope. See Organization Settings in Organization Configuration.

    Package Configuration includes the product, package, rule set, and form/wizard data. See Packages in Product Configuration.

    Product Configuration includes additions and changes to Products and product notifications and nothing else.

    Repository Product Configuration includes additions and changes to products in the .

    Review Queue Configuration includes changes to the review queue script and to review queues modified through Organization Configuration in Review Queue.

    Rule Set Configuration  includes changes to rule sets modified through the Form Designer Rules Window, Wizard Designer Rules Window, and through Rule Sets in Product Configuration.

    Tele-Sign Script Configuration includes additions and changes to the Tele-Sign Scripts in Organization Configuration.

    Carrier Only All Configuration includes the top level of data (Activities, Custom Lists, Products, Packages, Rule Sets, Quoting data, and Forms/Wizards). Distributors use this scope to deploy carrier top-level configurations, excluding Sales Agreements, to distributor environments when not using Auto Deploy. Deployment by a distributor does not impact carrier custom list changes.

    Carrier Only Product Configuration includes additions and changes to Products and nothing else. Distributors use this scope to deploy carrier product configurations to distributor environments if not using Auto Deploy.

    Carrier Only Form Configuration includes only the form and wizard level of data. Distributors use this scope to deploy carrier forms and wizards to distributor environments if not using Auto Deploy.

    Carrier Only Package Configuration includes the product, package, rule set, and form/wizard data. Distributors use this scope to deploy carrier packages to distributor environments if not using Auto Deploy.

    Carrier Only Rule Set Configuration  includes changes to rule sets modified through Form Designer Rules Window, Wizard Designer Rules Window, and through Rule Sets in Product Configuration.

    Carrier Only Custom List Configuration includes only the custom list level of data. Distributors use this scope to deploy carrier custom lists to distributor environments if not using Auto Deploy.

    Distributor All Configuration includes the top level of data (Activities, Custom Lists, Products, Packages, Sales Agreements, Rule Sets, Quoting data, and Forms/Wizards) that are specific to the selected distributors. Carriers use this scope to deploy carrier top-level configurations to distributor environments. Deployment by a carrier does not impact distributor custom list changes.

    Distributor Product Configuration additions and changes to Products and product notifications that are specific to the selected distributors. Carriers use this scope to deploy carrier product configurations to distributor environments.

    Distributor Form Configuration includes only the form and wizard level of data. Carriers use this scope to deploy carrier forms and wizards to distributor environments. Select one or more forms and wizards for deployment.

    Distributor Package Configuration includes the product, package, rule set, and form/wizard data. Carriers use this scope to deploy carrier packages to distributor environments.

    Distributor Rule Set Configuration  includes changes to rule sets modified through Form Designer Rules Window, Wizard Designer Rules Window, and through Rule Sets in Product Configuration.

    Distributor Custom List Configuration includes changes to custom lists modified through the Designer page. Carriers use this scope to deploy carrier custom lists to distributor environments.

  4. Select an Organization from the list.

  5. (Optional) Click Load Deployment Information to enable the Description, Organization, and Deploy options of the Deployment details or click the Cancel button to cancel loading the deployment information. A deployment of all changes can be scheduled without completing this step.

  6. (Optional) Type in the text box above the grid to filter deployment information by description. The system filters the deployment items relevant to that particular organization and description and displays only the items available for deployment for the organization.

    Example: When Synchronization Scope is set to Product Configuration and the grid shows all product configurations for the selected organization, type Annuity in the text box to filter and show only the organization's Annuity products in the grid.

  7. When deploying Distributor Configurations, the Correct Out-of-Sync Environments checkbox becomes available.

    • Leave the checkbox cleared to have the deployment comparison from the carrier UAT to the farthest distributor environment (Production if Auto Deploy is enabled in UAT and Staging). Deployment to Production includes all necessary updates and succeed, but if any intermediate environment is out of sync, the deployment to that environment could fail.

    • Select the checkbox to have the deployment first compare with the next immediate environment, and if any differences are detected, those changes are used for the deployment to all destination environments. This enables deployment to out-of-sync intermediate environments to succeed and be brought back into sync. It is possible that a deployment is successful but not complete to all higher environments, requiring deploying again (one or two times until the items no longer show for deployment) to get all environments updated with the latest changes. Once all environments are back in sync, this option can be cleared, and a single deployment successfully updates all environments.

  8. If using Force Deploy, select the Force Deploy checkbox before selecting the checkboxes for the updates to deploy. Selecting the Force Deploy checkbox re-loads the list of items for the Production environment. NOTE: To enable the Force Deploy feature, go to Organization Configuration/Organization Settings/Admin tab and select the Enable Force Deploy checkbox.

    Force Deploy is not available for carriers when deploying to a distributor. The Force Deploy option is available only for carrier-to-carrier deployments and distributor-to-distributor deployments in the UAT environment.

    Select the Force Deploy checkbox: Carriers select Force Deploy to deploy from the carrier's UAT environment to the carrier's Staging and Production environments in one step. Distributors select Force Deploy to deploy from the distributor's UAT environment to the distributor's Staging and Production environments in one step.

    Clear the Force Deploy checkbox: Carriers clear Force Deploy to deploy the configuration from the carrier's UAT environment to the carrier's Staging environment. Distributors clear Force Deploy to deploy the configuration from the distributor's UAT environment to the distributor's Staging environment.

    Example: A carrier would select this checkbox when retiring a product so it is no longer available for selling.

  9. On the Deployment tab, select the checkboxes for items in the list to deploy to Staging or Production. This list can include items to add, change, or remove in the Production environment. Select or clear the Check/Uncheck All checkbox above the grid to select or clear all of the checkboxes in the Deploy column.

  10. On the right, click the applicable Product Changes, Form Changes, Package Changes, Sales Agreement Changes, Rule Set Changes, Custom List Changes, and Activity Changes to open a dialog and view the history for the selected items in the grid for the Synchronization Scope. Clicking one of these links shows all changes since the last deployment that is deployed for the current item or for all selected items, depending on which link you click. The Package Changes, Form Changes, and Rule Set Changes dialogs display the changes in a panel format with search filtering available at the top of the dialog, individual forms, packages, or rule sets listed in the left pane, and Date/Time, Action, Old Value, and New Value columns in a grid format on the right.

    EXAMPLE

    Information about the selected item appears on the right. For example, All Configuration is selected in Synchronization Scope. The pane on the right displays Product Changes, Form Changes, Package Changes, Sales Agreement Changes, Custom List Changes, and Activity Changes for the product selected in the grid on the left. Under Total for Selected Items, view the number of all product, form, package, sales agreement, and rule set changes for all selected items on the left since the last deployment for those items.

    On the right, click the Form Changes link to open a Form Changes dialog and view all form changes since the last deployment for the item(s) selected in the grid on the Deploy page.

    NOTE: The Form Changes, Package Changes, and Rule Set Changes dialog display the changes in a panel format with search filtering available at the top of the dialog, individual forms, packages, or rule sets listed in the left pane, and Date/Time, Action, Old Value, and New Value columns in a grid format on the right.


  11. On the upper right of the Deployments page, type a Reason for the deployment.

  12. displays deployment process status on the upper right:

    Locked indicates that the request is locked because a deployment process is currently running.

    Not Locked indicates that the request is not locked because a deployment process is not running.

  13. (Optional) On the right, click the View All Scheduled Deployments link to open the Scheduled Deployment List dialog and view Synchronization Scope, Time of Deployment, and Items that are scheduled to deploy. In the Items column, pause the pointer over View Items to display the items that are scheduled to be deployed.

  14. Choose a deployment option.

    Deploy Now

    Click the Deploy Now button to deploy the selected Synchronization Scope now. On the Confirm Deploy dialog, ensure the target deployment environment is correct and click the OK button to deploy the selected data items.

    Schedule Deployment

    NOTE: Deployments are scheduled for the Mountain Time zone.

    1. Click in the Schedule Deployment at (Mountain Time) box and click the deployment date on the calendar.

    2. In the Schedule Deployment at (Mountain Time) box, change the time of day for the deployment and click the Done button.

      NOTE: The Deploy Now button becomes gray and disabled after selecting a schedule deployment time.

    3. Verify the date and time in the Schedule Deployment at (Mountain Time) box and click the Schedule Deployment button.

    Clear a Scheduled Deployment

    In the Schedule Deployment at (Mountain Time) box, delete the date and time. The Deploy Now button becomes enabled.
