Common Tag Conversion

IMPORTANT! Be sure to enable the common tag conversion utilities. Go to the Organization Configuration/Groups/Admin tab and select the Enable View Utilities checkbox. This enables all utilities for the organization.

Additional Information about Common Tag Conversion

Initial QE/UAT Development and Testing

  1. Use the Download Tag List utility to download a spreadsheet of tags for the existing forms, wizards and rules within a package.

    TIP: Optionally, add all forms, wizards and rule sets to a "tag conversion" package to convert all the forms, wizards and rule sets in one run. Then move the new converted forms, wizards and rule sets to the applicable packages after verifying the converted tags. This would produce a very large spreadsheet for mapping dataitems, but would ensure that all form, wizard and rule set tags were converted at the same time.

  2. Create a mapping spreadsheet by updating the downloaded spreadsheet to add new tag names for dataitems and export list values in the "to" columns of the spreadsheet.
  3. Use the Generate Mapping Rule Set utility to create a mapping rule set that converts old tags to new tags. Add the mapping rule set to the applicable package. This can be useful for mapping one set of data item tags to another set of data item tags (e.g., from carrier to distributor tags) or it can be used during a holdover period while preparing for a package conversion.

  4. Use the Package Conversion utility to create the new forms, wizards and rule sets that uses the new tags and replace the old forms, wizards and rule sets. Schedule the old forms, wizards and rule sets expiration date and schedule the new converted forms, wizards and rule sets to begin on the following day.

    NOTE: If you find a mistake in the new tag names, go back to the mapping spreadsheet, make the correction and repeat the package conversion to the updated tag names.

  5. On the Product Configuration/Packages page, verify the new converted forms, wizards and rule sets are in the applicable product package.
  6. Contact your FireLight® PM to arrange for developer assistance with converting old tags to new tags in the Provider. Work with the FireLight integration team to schedule the converted Provider to be deployed when the new forms, wizards and rules begin.

  7. Test all scenarios to ensure activities function correctly through submission.

    • New activities
    • Activities created prior to conversion
    • Submission of activities locked prior to conversion
    • Submission of activities created prior to conversion
    • Submission of new activities

Staging and Production Deployment and Testing

  1. On the Utilities/Deployments page, schedule the deployment of the Products with updated forms, wizards and rules. (See Deploy Data for deployment instructions.)
  2. Schedule the deployment of the updated Provider on the same date and time.

  3. Test all scenarios to ensure activities function correctly through submission.

    • New activities
    • Activities created prior to conversion
    • Submission of activities locked prior to conversion
    • Submission of activities created prior to conversion
    • Submission of new activities

After Deployment

When all of the changes have been deployed, the following scenarios are handled by the new forms, wizards and rule sets.

Create New Activity

  • New activities only contain the new tags. There is no need for the rules to map to the new tags.
  • The forms, wizards and rules use the new tags.
  • The Provider uses the new tags to generate outbound calls.
  • The Provider does not need to map to the new tags in the DataItem collection.

Existing Active Activity

  • The activity initially contains only old tags.
  • When the activity is opened, the conversion rule maps the old tags to the new tags. The activity now has both old tags and new tags.
  • The forms, wizards and rules use the new tags.
  • The Provider uses the new tags to generate outbound calls.
  • The Provider does not need to map new tags in the DataItem collection.

Existing Locked Activity (Activity Data Cannot Change)

  • The activity contains only old tags.
  • The forms and wizards display the stamped page images.
  • The rules do not run on locked activities.
  • The Provider needs to map new tags in the DataItem collection.
  • The Provider uses new tags to generate outbound calls.
