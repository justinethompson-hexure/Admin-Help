Step 2. Lock and Begin an Order Review [Reviewer Instructions]

NOTE: When an activity is locked by the reviewer (status = "In Pre-Submit Review" or "In Review") and the application is opened by the agent/user in FireLight, a message appears with text, "Application is locked for review." The agent/user cannot unlock or cancel the application. The Unlock option on the Other Actions menu and the Unlock button on the Rename/Summary dialog on the Other Actions menu are disabled while the activity is in review. The Pending Requests dialog that opens from the Requests link on the Other Actions menu does not show the Cancel Request link,

Lock and Review an Order in a PreSubmit or Post-Submission Review Queue

  1. The reviewer clicks the Review button to open an order pending review.

  2. The reviewer clicks the Lock and Review Order button to lock the order so other reviewers cannot access the order while it is in review. The reviewer clicks the Cancel button to leave the order open for review by other reviewers who have access to the queue.

    OPTIONAL (Post-Submission Review): To automatically lock orders when clicking the Review button without the Lock and Review Order warning dialog appearing, a post-submission reviewer can select the Don’t show me again checkbox or click My Preferences on the navigation bar and set Warn before lock to No. The change takes effect the next time the reviewer logs in to FireLight®.

    When the reviewer selects the Don’t show me again checkbox, the following message appears on the dialog. The reviewer clicks the Lock and Review Order button to continue with the review or clicks Cancel to return to the queue without locking the order for review.

Lock and Review an Order in an Automated Review Queue

Locking an order (case) in an automated review queue causes the Provider to be put into a holding pattern for that case. This locking causes the Provider to bypass the case in the automated review queue. It is the responsibility of the reviewer to reject or move the case. The Provider retry is cancelled. The Provider will then have a time interval to check back on the case, but the lock is honored. If a reviewer unlocks the case, the Provider can re-lock and pick back up where it left off.

  1. After successful log on, FireLight opens to the Home page. On the right under Manage, reviewer clicks Review Queue.

  2. On the left under Automated Queues, the reviewer clicks the name of an automated queue.

  3. In the automated queue, the reviewer finds the pending order to review and clicks the Review button to lock the order for review.

  4. The reviewer follows the standard post-submission review process for approving, rejecting or moving the case.

  5. To release the order back to the automated queue without review, the reviewer clicks the Unlock button on the order so the Provider can pick back up where it left off.
