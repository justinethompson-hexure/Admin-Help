History [Reviewer Instructions]

The History page displays orders with the reviews completed. Reviewers use the History section to view the approval status, activity history, queue history, and Audit Report of individual orders. Date range, queue, and action lists along with an advanced search feature provide filters for limiting the orders in the viewing section.

  • A green check mark on the right indicates the order was approved.

  • A red X Rejected by [Reviewer] indicates the order was rejected by the reviewer.

  • A red X DeadlinePassed indicates the order was rejected because the maximum review period elapsed because the review queue expiration deadline expired.

  • A red X AutoRemoved by [Reviewer] indicates the order was removed from the queue when the reviewer moved it to another review queue.

NOTE: FireLight® purges all of the Personally Identifiable Information (PII) data and documents from the system on the purge date. FireLight retains a skeleton version of the application containing basic information about the application, the audit history for the application, and the audit history for the review queue so the user can still see it in All Activities and review History but no application data or documents are retained.
