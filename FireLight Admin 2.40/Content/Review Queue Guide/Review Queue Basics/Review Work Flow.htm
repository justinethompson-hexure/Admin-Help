Review workflow [Reviewer Instructions]

The review process begins in a queue.

Following is a typical post-submission workflow for reviewing orders within a queue for submission to the back office for processing.

  1. The reviewer selects an order from one of the queues.

  2. The reviewer clicks the Review button and sets the order status to “Locked” so other reviewers cannot access the order while it is in review.

    NOTE: If the initial reviewer cannot complete the review process, the reviewer clicks the Unlock button to make the order available for other reviewers to continue the review process.

  3. If the organization has a review Deadline Length, the order shows an expiration date in the queue. If the order is not reviewed before the deadline expiration date and time, FireLight® approves or rejects the order from the review queue depending on the organization’s review deadline settings. If the order is rejected, the agent needs to go into FireLight to review and determine if the order is still suitable, then copy the rejected activity, make necessary changes, and resubmit the order.

    The order displays an Expires date when a deadline is set for a review queue. This text turns red when two or fewer business days remain for the review deadline.

  4. If the organization has the Enable Manual Queue Move option enabled in FireLight Admin, the reviewer can move an order to another review queue. The reviewer selects another queue in the list of queues and clicks the Move button.

  5. The reviewer opens and reviews the order documents and history by clicking the Application PDF, Application History, Queue History, Needs Determination (if available), supplemental application documents (Illustration, etc.), and Audit Report links. The reviewer can also click Attach Documents to add documents to the order.

  6. If needed, the reviewer requests more information from the agent for the order by clicking the More Info button and then proceeding with the written request. The order status changes from “Locked” to “Request Sent. Pending Reply”.

  7. The organization determines whether to include reviewer documents in the order package and whether reviewer documents are optional or required. If the organization requires reviewer documents, FireLight disables the Approve button until after the reviewer completes the required reviewer documents. When reviewer documents are optional, the Approve button remains enabled.

    If the order package includes reviewer documents, the reviewer clicks Reviewer Documents, fills in the applicable fields on each reviewer document, and clicks a button to Save or Complete (lock) the reviewer document. The reviewer clicks Back to Review Queue to return to the order review.

    NOTE: When using Sequential Review Queues, FireLight retains the reviewer documents and reviewer comments in the original queue. However, since each queue has a brand new document set, FireLight does not retain these comments across sequential review queues. Despite this, these comments can be copied through a client's provider. If interested in this option, the organization can reach out to the Strategic Account Manager.

  8. After reviewing all of the information available for the order, the reviewer approves or rejects the order by clicking the applicable Approve or Reject button.

  9. Go to Step 1. Access the Review Queue [Reviewer Instructions] in Queues for detailed reviewer instructions.
