Advanced Search [Reviewer Instructions]

  1. On the side navigation bar, the reviewer clicks a section name (Queues, History, or Reviewers).

  2. If searching the Queues section, the reviewer selects a queue on the left. If searching the Reviewers section, the reviewer selects a reviewer on the left. The History section shows all orders.

  3. At the top of the page, the reviewer clicks the Advanced Search button.

  4. On the Advanced Search dialog:

    The reviewer selects the applicable date range: Week, Month, Quarter, YTD, or All.

    NOTE: The available advanced search options vary by organization. For example, if the Tele-Sign feature is enabled in FireLight® for an organization, the Advanced Search dialog includes the Tele-Sign App ID.

    The search options available in the Queues section are different from the search options in the History and Reviewers sections.

    Example – Queues section: searching by user

    The reviewer selects the applicable search options:

    User list includes the Review Queue users that currently have a login active and have access to that review queue. "All Users" is the default user in the list. Select a user in the list to search for cases reviewed by that user. The Advanced Search filters by the reviewer, showing all cases the reviewer has locked but may not be currently locked by the reviewer. If the History shows the user has been a reviewer for the case, the case will display when filtered by that user.

    Activity Type list includes activity types for which the selected user has access (e.g., All, Application, Illustration, Multiple Activities, etc.).

    Product Type list includes filters for activities based on the type of product for which the app/activity was created.

    Status list includes the various statuses for the activities (Any, Data Entry, Signatures, PreSubmit Review, Review Queue, Submit Requested).

    Sort list includes options to sort the search results by Date, Activity Name or Product Name.

    Signature Type list includes options: All, e-Signature, or Wet Signature. This list provides the ability to search for cases with the FLI_DECLINED_ESIGN tag, or by accepting or declining e-Sign in FireLight. Declining e-signature in the FireLight UI causes the application to show up as "Wet Signature" in the search results.

    Example – History or Reviewers section: searching by queue and action

    The reviewer selects the applicable search options:

    Queue select a Queue to search for cases within a specific queue.

    Action select an action to search for cases by reviewer action: Approved, Rejected, Auto Removed, Deadline Passes.

    Signature Type list includes options: All, e-Signature, or Wet Signature. This list provides the ability to search for cases with the FLI_DECLINED_ESIGN tag.

  5. The reviewer enters search parameters in the applicable fields: Activity Name, Issue State, Policy Number, Product Name, etc.

    NOTE: Search parameters are specific to the organization and may differ from the ones listed here. Any of the search items that are set within the List Profile in the Provider will display in the Advanced Search.

  6. Optionally, the reviewer enters a search date range in the Begin Date and End Date fields.

  7. The reviewer clicks one of the following buttons:

    • Save – opens a Save Search dialog. The reviewers enters a Name and clicks Save to save the search options for future use. The reviewer needs to save the search options before initiating the search. The reviewer clicks Cancel to cancel saving the search.

    • Search – initiates the search process and displays the search results on the right.

    • Cancel – cancels the search and closes the Advanced Search dialog.

  8. When in a Queue, the reviewer can click the Export button to download a .csv file that contains the search results. See Export Data.
