FireLight Admin Information Checklist

Use the following checklist to gather information for setting up product configurations in ® Admin.

Click the Print button on the upper right to print this page.

FireLight Admin Information Checklist
For each Product: For each Form or Wizard when added:
  • Carrier (manufacturer product name)
  • Sales Agreement product name
  • Product Type
  • Rank Order
  • CUSIP ID
  • Carrier Code
  • State Approvals

For each Package:

  • Package name
  • Package Type
  • List of forms and wizards to include
  • List of role codes to include

  • Form inclusion type:

    • Required
    • Conditional
    • Optional
    • Print Only
    • Submit Only
    • Input Only: Required
    • Input Only: Conditional
    • Signing Ceremony Only
  • Rule sets to include

For each Rule Set:

  • Data item IDs and rules

For each Sales Agreement:

  • Originating carrier (manufacturer) name
  • Selling organization
  • Products to include
  • Alternate product names, if applicable
  • Agreement start and end dates, optional
  • Time zone for agreement, optional
  • Back Office script data for workflow
  • State approvals for agreement
  • Which packages for each product in the agreement
  • Whether to auto deploy (distributors) or force deploy (carriers), optional
  • The Form PDF file (file name and location on your PC or network)
  • Form Number
  • Form or Wizard Name
  • Description
  • Edition/Version of the form or wizard, optional
  • Date to activate the form or wizard, optional
  • Date to expire the form or wizard, optional
  • Time zone for dates, optional
  • Form or wizard that will replace this form or wizard on expiration, optional
  • Is this form part of Back Office?
  • Is this form a Report?
  • Is this form a Template?
  • Document tag type
  • State approvals for this form or wizard

For each Form or Wizard edited in the Designer:

  • Data item ID of any field to be copied in from another form or wizard
  • List of fields and data item IDs to add
  • List of fields and data item IDs to edit
  • List of fields and data item IDs to delete
  • Rule set for the form or wizard
  • Locations for placement of fields
  • Field properties for each fields
  • Tabbing order for fields

For Form or Wizard to FireLight Mapping:

  • Data item ID for each field to be mapped

For Email Templates:

  • New email wording
  • Variable field names for data fill
  • Localized email wording, optional
