Typical FireLight Admin Workflow for Applications

  1. Log in to ® Admin.
  2. Upload Forms and Create Wizards

    Add or create forms and wizards for the applicable activities in FireLight.

    • Upload existing forms (in PDF or XPS format), add form detail, and view form history using the Designer/Form Library page and Upload Forms dialog.

    Forms with existing named tag fields as added through Adobe Acrobat is present on the uploaded forms.

    • Create Wizards on the Designer/Wizard Library page.
  3. Design Forms and Wizards and Add Rules
    • Design forms using the Designer/Form Designer page by selecting master settings and adding input controls, field properties, and rule sets or rules.
    • Design wizards using the Designer/Wizard Designer page by adding pages, containers, fields, and rule sets or rules to each wizard.
  4. Set Up Product Configurations

    Access the Product Configuration pages. Add or edit activities and products, add packages with the applicable forms and wizards, add and edit rule sets, add and edit sales agreements, and optionally add and edit fees, expenses, and commission repository information and quote for Annuity and Life products.

  5. Set Up Organization Configurations

    Access the Organization Configuration pages. Add and edit organization settings and custom lists, edit email templates, edit mappings from source data to , add pre-submission and post submission review queues, add and edit groups for access to activities, cases, products, features, and training videos. Some Admin changes may take 20–30 minutes to take effect in the FireLight application.
  6. Log On to FireLight Web Application

    Log on to the application for end users.

  7. Test the Forms and Wizards

    Test the forms and wizards using the application for end users. You can edit in Admin and validate your changes in the FireLight application by refreshing or navigating back to Home and re-entering the order.

  8. Complete the Setup in Admin

    Log on to Admin to complete the setup.
  9. Optional: Export Forms, Wizards, and Rule Sets

    Export selected forms or wizards with metadata (fields) using the applicable Form Library or Wizard Library page. Use the Export feature to back up forms and wizards (in case forms are deleted in the system), if planning to use the forms and wizards in multiple environments, or to use the same form or wizards multiple times; for example, when requiring a proof of citizenship form or wizard for joint or additional insured. Export rule sets using the Product Configuration/Rule Sets page.
  10. Deploy

    Use the Utilities/Deployment page to deploy the products, forms/wizards, packages, rule sets, organization, groups, etc., configurations to staging and production. The All Configuration deploys the top level of data: Activities, Products, Forms/Wizards, Packages, Sales Agreements, and Rule Sets.
  11. Generate and View Reports

    Use the Utilities/Reports page to generate and view admin reports for your organization. You can add activity reports that run on a scheduler. Activity reports can be scheduled using the Activity Reports Scheduler or can be requested by submitting a JIRA ticket.
  12. Utilities

    Use the Utilities pages to access maintenance utilities (for example, to convert data item IDs to common tags or to transfer cases from one User ID to another User ID).
  13. Support

    Use the Support page to access the Rule Tracing debugging tool for creating rule traces, to access the API Insights analysis tool that shows the API data at each step in the workflow for a given trace ID, and to access the Application Data tool to view information on each case that is being sent to the DTCC; their app/sub status.

  14. Log off from Admin.

When available, click on the upper right to access help for the current field.

See FLI_APP_ORIGIN Data Item for information about capturing details of how an application was created or copied.
