What's New in FireLight

The following enhancements are now available for all clients unless noted otherwise. See the "FireLight v2.40" Release History" topic in Admin Help for fixed defects and links to design documents and Admin Help topics.

New Features

Hard Stop Error Message: CR-4088

We updated the hard stop error message to the following: "Unable to continue. You are not authorized to sell:

<Message>

<Message>

You may create this application, but cannot submit it until all the necessary requirements to sell the product are met.”

Gracefully Handle Unaccounted DocuSign Events: CR-4175

We modified payload handling to more gracefully handle unaccounted DocuSign events and reduce error logging.

Messaging for Producer When Its Neither Soft Stop Nor Hard Stop: CR-4178

We added the following DTCC Producer Authorization message: “All requirements are satisfied. You can proceed with selling this product.”

Update Verbiage in the Soft Stop Message: CR-4181

We updated the FireLight soft stop message to read: “DTCC Producer Authorization: You are authorized to sell this product, though certain requirements are not yet fully met”

Compliance Reports: DIST-4595, DIST-4596, DIST-4988, DIST-5133

A few compliance reports are added as part of admin reports in the admin tool.

1. Age Limit

2. Net Worth

3. Income

4. Tax Bracket

5. Limited Investment Experience

6. VA/Net Worth %

7. Elderly Modest Means & Limited Experience

8. Non-Natural Person Accounts

9. Tax Qualified Sales

10. Large Surrender Replacements

11. Pre-threshold Surrender Sales

12. High 1035 & Exchange Reps

13. High Switching Reps

Impact:

FL Admin: New report "Compliance Report" will be available on the Reports page in the Admin section. User will be able to select any of the above compliance reports to generate them in CSV format.

Master Repository for Wizard Templates: DIST-4589, DIST-5033, DIST-4511, DIST-4512, DIST-5134

As part of increasing wizard efficiency, a master repository of wizard templates will be made available in the admin tool. While adding a new wizard, the admin user will be able to navigate through the available wizard templates to view if they meet their requirement and then import it into the wizard library. Any rule set attached with the wizard template will also be imported. They can then go ahead and customize the wizard as per their need, without building the wizard from scratch.

Impact:

FLAdmin: Under the Designer menu, a new menu option will be available to view the existing wizard templates in the master repository and then select and import them into the wizard library.

Update Next Activity Default Messages with User-Friendly Wording: EAES-3095

We updated error messages to be more clear and user-friendly, including the following:

  • "This product has not been configured for your organization. Press OK to continue."

  • "The product you are attempting to access is not available. Press OK to continue."

  • "The product you are attempting to access is not available. Press OK to continue."

Implement Logic for New Checkbox on Dropdown Controls: EAES-6450

We have added to the Forms and Wizards controls a new checkbox, TDR-Selected Value Only, to pare down combo box and dropdown lists and reduce the length of reports.

Impact:

Forms and Wizards/TDR – Selected Value Only

Resolved Issues

Search Fields in Wizard Packages-DIST-4369

We resolved an issue where clicking the Search button on the Search Fields in Wizard Packages tab in the Wizard Library resulted in an error. Clicking the Search button now works as designed.

ReviewQueueReport Activity Report Error: DIST-4864

We resolved an issue where an error resulted when generating the ReviewQueueReport. The report can now be generated without issue.

EDeliveryPendingExpirationReport Activity report Error: DIST-4865

We resolved an issue where an error resulted when generating the EDeliveryPendingExpiratoinReport. The report can now be generated without issue.

Getting Server Error When User Approved or Reject the App from Review Queue: DIST-4932

We resolved an issue where a server error was received when users approved or rejected an app in the review queue. In this scenario now no error is received.

Error Occurring in Activity Completion Report Due to Missing Login Date Value: DIST-5020

We resolved an issue where an error occurred when generating the Activity Completion Report. The report can now be generated without issue.

Timeout Error When Generating Business Detail Report: DIST-5132

We resolved an issue where generating the Business Detail Report for a date range longer than two days resulted in an error. The report can now be generated without issue.

Admin Reports Are Not Showing in the Report Section, Except for the Application Audit and Transaction Detail Report: DIST-5194

We resolved an issue where not all Admin Reports appeared in the Report section. Now all Admin Reports display, as designed.

Spurious Application Messages for Passcodes Being Created but Not Sent: EAES-3264

We resolved an issue where an application message for Passcode Application would be created and stored but not sent. Passcode Application messages are now sent as designed.

User with Apostrophe in the Name, Not Able to Be Assigned as (Producer/Admin/Member) in Any of New or Old Workgroups: EAES-3919

We resolved an issue where a user with an apostrophe in the name could not be assigned to a workgroup. Now such users can be assigned to workgroups.

In the Resend Passcode Email Functionality, Entering an Email Address with Leading or Trailing Spaces Results in a Validation Error: EAES-4745

We resolved an issue where when attempting to resend a passcode email, entering an email address with leading or trailing spaces resulted in a validation error. Now such addresses can be entered without issue.

In the Send Reminder Email Functionality, Entering an Email Address with Leading Spaces Results in a Validation Error: EAES-4746

We resolved an issue where when attempting to send a reminder email, entering an email address with leading or trailing spaces resulted in a validation error. Now such addresses can be entered without issue.

In the Send New Email Functionality in Message Center, Entering an Email Address with Leading or trailing Spaces Results in a Validation Error: EAES-4747

We resolved an issue where when attempting to send a new email in the Message Center, entering an email address with leading or trailing spaces resulted in a validation error. Now such addresses can be entered without issue.

Cancel Tele-Sign Link Color Is Different than Normal Signers Revoke Request Link: EAES-5158

We resolved an issue where in the signers list the Cancel Tele-Sign link color was incorrect and did not match the link color of other signers. Now the Cancel Tele-Sign link appears as designed.

When "Allow E-Sign" Is Disabled, No Audit Report Is Generated When It Skips the Submission Process to Go to Complete: EAES-5413

We resolved an issue where when Allow E-Sign is disabled no audit report was generated. Now in this scenario an audit report is generated as designed.

When "Allow E-Sign" Is Disabled, a Record Is Not Inserted in ApplicationTrans Table Used for Billing: EAES-5418

We resolved an issue where when "Allow E-Sign" was disabled, the app went to complete without a record being inserted in the ApplicationTrans table used for billing. Now in this scenario a record is inserted in the ApplicationTrans table.

App Status Showing 'Client Signatures Complete' After Re-sign Process Initiated: EAES-5882

We resolved an issue where after the re-sign process was initiated the app status erroneously showed “Client Signatures Complete.” Now in this scenario the app status shows as In Signatures with the request canceled.

Activity Created but Unable to Open Due to Invalid Product Organization ID: EAES-5891

We resolved an issue where an erroneous ProductOrganizationID of 0 caused issues when attempting to open an activity. Activities can now be opened as designed.

Tele Sign MVC Signing: Signer Name and Signer Email Not Autofill in Forms Access Page for Send Via Email Page: EAES-5893

We resolved an issue where on the Tele Sign access page the Signer Name and Signer Email fields failed to autofill. Now these fields autofill as designed.

'Locked for review' Audit Not Showing in TDR for Review Queue: EAES-5999

We resolved an issue where when the pre-submit review was approved or rejected, the TDR report field was blank. Now in this scenario the report field is populated.

After the App Is Submitted and Completed, the Audit Details of the Completed App Are Not Appearing in the TDR: EAES-6054

We resolved an issue where when an app was submitted and complete audit details of the completed app failed to appear in the TDR. Now in this scenario audit details appear in the report.

Scheduled Assemblies Configured to Run Every 1 or 2 Minutes Are Instead Running Every 5 Minutes: EAES-6108

We resolved an issue where scheduled assemblies ran every five minutes instead of the every one or two minutes they were configured to run. Scheduled assemblies are now running every one or two minutes, as configured.

Overlapped Checkboxes Are Displaying for Optional Forms: EAES-6262

We resolved an issue where on optional forms checkboxes overlapped. Checkboxes on these forms now display correctly.

“(OFFICE USE ONLY)” Is Appearing at the End of Certain Error Messages: EAES-6305

We resolved an issue where “(OFFICE USE ONLY)” erroneously appeared at the end of some error messages. Error messages now appear correctly, as designed.

Error Messages Are Displaying "Office Use Only". Provider Colors Are Not Always Displayed: EAES-6306

We resolved an issue error messages displayed “Office Use Only” and did not include provider colors. Error messages now appear as designed.

Documents Uploaded via the REST API Have the Wrong Visibility: EAES-6394

We resolved an issue where documents uploaded the REST API had the incorrect visibility of CarrierOnly. Now such documents have the correct visibility.

The Value 'Unknown' Is Not Displayed for the Newly Created Data Item 'FLI_APP_ORIGIN' When an App Is Created Using the QETestPage.aspx Sample App: EAES-6397

We resolved an issue where the value Unknown failed to be displayed for the newly created data item FLI_APP_ORIGIN when an app was created using the QETestPage.aspx sample app. Now in this scenario the value Unknown is displayed.

Unable to Continue after Signatures Due to Pending Review Request: EAES-6428

We resolved an issue where applications could not be completed in certain circumstances. Now applications can be completed without issue.

Deployment of Firm with All-Numeric ID Not Working: EAES-6436

We resolved an issue where deploying firm data failed when firm records contained numeric-only values. Now in this scenario firm data deploys without issue.

Log an Exception When There Are No Available Continue Actions: EAES-6443

We now log an exception when there are no available continue actions and the user cannot complete nor submit the activity.

Exclude From Back Office Checkbox Missing: EAES-6525

We resolved an issue where the Exclude from Back Office checkbox was missing from the Form Library page. The checkbox is now available on the page, as designed.

Value of Data Item FLI_APP_ORIGIN Displays Unknown Instead of “Firelight UI” When Application Is Created by Clicking the New Application Button: EAES-6566

We resolved an issue where the data item “FLI_APP_ORIGIN” displayed “Unknown” when creating an application using the native UI. Now in this scenario “Firelight UI” appears.

We resolved an issue where distributors were able to modify state approvals of a carrier sales agreement. Now, distributors can restrict (remove) jurisdiction approvals from a carrier sales agreement but cannot add approvals that the carrier did not specify.

We resolved an issue where on the Sales Agreement page any company using the Firm Management feature could view all other Firm Organizations. Now accounts using Firm Management see only their own Firm Organization view.

Open Source License Info Page Is 404: PC-2704

We resolved an issue where the open source license info page failed to display. This page now appears as designed.

Wizard Does Not Show Rule Processing Exceptions: PC-3271

We resolved an issue where the form/wizard failed to display rule processing exceptions. These exceptions now appear on the first visible form or wizard.

Form/Wizard History Tab Are Not Showing the Edition’s Old and New Values: PC-3424

We resolved an issue where after adding or editing the edition on the Form/Wizard details page the old and new values did not display on the History tab. Now both the old and new values appear on the tab in this scenario.

Template Rules for SSN Are Disallowing Some Valid Numbers: PC-3473

We resolved an issue where template rules disallowed some valid SSNs. Now all valid SSNs are allowed.

Organization Spelling Incorrect: PC-3637

We corrected a spelling error on the Manage Sales Agreement page.

Exceptions Occurring Rendering Layout Due to Missing EnableMixPanel View Property: PC-3676

We resolved an issue where layout rendered incorrectly.
