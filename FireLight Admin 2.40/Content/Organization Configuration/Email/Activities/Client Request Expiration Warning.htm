Client Request Expiration Warning

Client Request Expiration Warning
Action, To Name/Email, From Name/Email, CC Address, Available Tags, and Notes

Action

Notifies the Client that a pending request (for signature, client fill & sign, etc.) to the client is nearing expiration1

Template Name: Client Request Expiration Warning

Email Subject Line: Client Request about to Expire

Message:

Dear <CLIENT_NAME>,

Your pending request in application <APPLICATION_NAME> is about to expire. Please take all necessary actions that this requires. To complete this request, click on <URL_LINK> link. If a new window does not automatically appear, you may have to copy the link and paste it into the address bar of a new browser window.

The Expiration templates are available for sending when the Expiration Warning Threshold and Requests Timeout options are set in Organization Configuration/Organization Settings on the Activity tab.

To Name / Email

Client Name/Email used in original request

From Name / Email

Default: Session User’s Preference Name/ Email2

Use Agent: Owning Agent’s Preference Name/Email

Use DMARC1: Organization's Support Email

CC Address

If Enable Cc Sender In All Requests is set in Organization Configuration/Groups on the App tab, cc email sent to ‘From Email’ address.
