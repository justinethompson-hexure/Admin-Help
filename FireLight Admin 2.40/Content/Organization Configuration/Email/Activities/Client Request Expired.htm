Client Request Expired
Action, To Name/Email, From Name/Email, CC Address, Available Tags, and Notes

Action

Notifies the Client that a pending request (for signature, client fill & sign, etc.) to the client has expired1

Template Name: Client Request Expired

Email Subject Line: Client Request Expired

Message:

Dear <CLIENT_NAME>,

Your pending request in application <APPLICATION_NAME> is expired.

The Expiration templates are available for sending when the Expiration Warning Threshold and Requests Timeout options are set in Organization Configuration/Organization Settings on the Activity tab.

To Name / Email

Client Name/Email used in original request

From Name / Email

Default: Session User’s Preference Name/ Email2

Use Agent: Owning Agent’s Preference Name/Email

Use DMARC1: Organization's Support Email

CC Address

If Enable Cc Sender In All Requests is set in Organization Configuration/Groups on the App tab, cc email sent to ‘From Email’ address.
