Client E-Delivery Expiration Warning

Client E-Delivery Expiration Warning
Action, To Name/Email, From Name/Email, CC Address, Available Tags, and Notes

Action

Notifies the client when an E-Delivery is nearing expiration1
Template Name: Client E-Delivery Expiration Warning

Email Subject Line: Client E-Delivery about to Expire

Message:

Dear <CLIENT_NAME>,

Your pending E-Delivery in <ACTIVITY_NAME> <APPLICATION_NAME> is about to expire in <EXPIRY_DAYS> days. Please take all necessary actions that this requires. To complete this request, click on <URL_LINK> link. If a new window does not automatically appear, you may have to copy the link and paste it into the address bar of a new browser window.

This template is available for sending when the E-Delivery checkbox is selected in Organization Configuration/Groups page, Activity Access tab and the E-Delivery Expiration Warning Threshold and Requests Timeout options are set in Organization Configuration/Organization Settings on the Activity tab.

To Name / Email

Client Name/Email used in original request

From Name / Email

Default: Session User’s Preference Name/ Email2

Use Agent: Owning Agent’s Preference Name/Email

Use DMARC1: Organization's Support Email

CC Address

If Enable Cc Sender In All Requests is set in Organization Configuration/Groups on the App tab, cc email sent to ‘From Email’ address.

Available Tags3

<AGENT_NAME>
<APPLICATION_NAME>
<APPLICATION_ID>
<CLIENT_NAME>
<COMPANY_NAME>
<DATE_TIME> (current datetime in UTC)
<EXPIRY_DAYS>5
<HYPER_LINK>
<PRODUCT_NAME>
<PRODUCT_TYPE>
<STATUS>
<TRANSACTION_TYPE>
<URL_LINK>
ApplicationSearchItems4 (e.g., add <CLIENT_NAME>)

Notes:

5Add the <EXPIRY_DAYS> [count] variable in the expiration warning email template to convey the number of days until expiration. See the E-Delivery Reminder Frequency option on the Organization Configuration/Organizations Settings page/Activity tab, which allows organizations to set the frequency (Never, Daily, Weekly) of E-Delivery warning emails.
