Revoke E-Delivery Request (Undeliverable Email)
Action, To Name/Email, From Name/Email, CC Address, Available Tags, and Notes

Action

This email goes to clients to notify them that the transaction was canceled due to an undeliverable email.

Template Name: Revoke E-Delivery Request

Email Subject Line: <ACTIVITY_NAME> request for <APPLICATION_NAME> Canceled

Message:

Transaction has been canceled due to an undeliverable email. Client cases have now been revoked.

This template is available for sending when the E-Delivery option is selected in Organization Configuration/Groups page, Activity Access tab.

To Name / Email

Client Name/Email typed in email fields or pre-filled from mapped dataitems

From Name / Email

Default: Session User’s Preference Name/ Email2

Use Agent: Owning Agent’s Preference Name/Email

Use DMARC1: Organization's Support Email

CC Address

If Enable Cc Sender In All Requests is set in Organization Configuration/Groups on the App tab, cc email sent to ‘From Email’ address.

Notes:

2You can disable preference fields and send with the incoming FireLight® request. If template fields are not disabled, users can also type the From name and Email. There are two ways that the preference fields can be disabled. One is using the Organization Configuration/Groups, App tab setting 'Disable Edit Preferences'. The other is done automatically, similar to the DMARC option. If the user comes in through an external FireLight request, the preference fields are disabled automatically. If the Groups, App tab setting ‘Enable Override SSO’ is enabled, the name and email fields are never editable in the send email screen even with the enable override SSO for e-Delivery transactions. The name and email fields can be mapped to allow them to be edited and passed into signing through data items. If the organization does not want them to be editable, they can implement a comparison rule that would require the case to be reissued if the field is changed.

3Available replacement variables are the names of the data items that are saved in the activity. For example, AGENT_NAME is the name the user entered for the Agent_FullName data item on the activity form. If replacement tags are being used directly in the app portal template view, pipe enclosure must be used in the tag as browsers will not allow direct replacement of brackets. For example, use |AGENT_NAME| rather than <AGENT_NAME>. Additionally, use of unavailable tags may result in email and other browser view errors. You do not need a field named Agent_FullName on the form to pre-fill these. Like the client names, it is coming from the 121 that is posted over. It may be able to edit through the use of data items that are mapped using the Admin Mapping tool, but it is not what is filling the agent name for e-Delivery.
