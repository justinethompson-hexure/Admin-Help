Transfer Activity Request Expiration Warning

Transfer Activity Request Expiration Warning
Action, To Name/Email, From Name/Email, CC Address, Available Tags, and Notes

Action

Notifies the Agent that a pending transfer activity request to the agent is nearing expiration1
Template Name: Transfer Activity Request Expiration Warning

Email Subject Line: Transfer Activity Request about to Expire

Message:

Dear <CLIENT_NAME>3,

Your pending transfer activity request for <ACTIVITY_NAME> <APPLICATION_NAME> is about to expire in <EXPIRY_DAYS> days. Please take all necessary actions that this requires.

The Expiration templates are available for sending when the "Expiration Warning Threshold" and "Requests Timeout" options are set in Organization Configuration/Organization Settings on the Activity tab.

To Name / Email

Agent Name/Email used in original request

From Name / Email

Default: Session User’s Preference Name/ Email

Use DMARC1: Organization's Support Email

CC Address

No cc Sender

Available Tags3

<ACTIVITY_NAME>
<AGENT_NAME>
<APPLICATION_NAME>
<CLIENT_NAME> - In this email template, CLIENT_NAME is the Agent's name.
<DATE_TIME> (current datetime in UTC)
<EXPIRY_DAYS>5
<HYPER_LINK>
<STATUS>
<URL_LINK>

Notes:

5Add the <EXPIRY_DAYS> [count] variable in the expiration warning email template to convey the number of days until expiration. See the Expiration Reminder Frequency option on the Organization Configuration/Organizations Settings page/Activity tab, which allows organizations to set the frequency (Never, Daily, Weekly) of expiration warning emails.
