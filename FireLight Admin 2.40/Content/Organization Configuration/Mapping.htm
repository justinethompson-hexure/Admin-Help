Mapping

On the navigation bar, click Organization Config and click Mapping to go to the Mapping page.

Mapping Tab

Use the Mapping tab to set the mapping from the FireLight® global (FLI_*) data items (System Data Id column) to the form data items (Form Data Id column) in . This is how maps identification data within the forms.

The Mapping tab is not used to create any FireLight global data items (FLI_*). It is used only in the signature process to map a well-known identifier (FLI_*) to an organization-specified data item for pre-filling the identification fields.

Edit Form or Wizard Data Item IDs

You can map the data from the FireLight global data items (FLI_*) to the identification form data items for your organization.

FireLight global data items (FLI_*) in the System Data Item ID column cannot be changed. Values are case sensitive.

  1. On the navigation bar, click Organization Configuration and then click Mapping.

  2. On the Mapping tab, select a Product Type in the list.

    The list of available data items is automatically populated from the signer types defined within FireLight.

  3. Make your changes in the Form Data Id column to specify the mapping of the Form Data Item ID.
  4. Click Save Mapping to save your updates.

  5. To undo all of your changes and set the mapping back to the original system defaults, click Remove Mapping.

    CAUTION: Clicking this button will remove all changes you made to the Form Data IDs, not just the changes you made in this session.

DataItem Sync Tab

NOTE: This tab is available only for Distributor organizations.

Distributors use the DataItem Sync tab to map one data item to another data item so that when one data item changes, the linked data item also changes.

Link Data Items

  1. On the navigation bar, click Organization Configuration and then click Mapping.

  2. In the grid on the DataItem Sync tab:

    1. Type the primary data item ID in the Primary Data Id column. This is the data item to link from.
    2. Type the secondary data item ID in the Secondary Data Id column. This is the data item to link to.
    3. Type a description for the data item linking in the Description column.
  3. Click the Add New Row button to add another pair of linked data items.

  4. Click the Save Mapping button to save the entries.

EXAMPLE

EXAMPLE

The distributor form uses D_Owner_DOB data item for the owner's birth date field and the carrier's form uses Owner_DOB data item for owner's birth date.

Enter D_Owner_DOB in Primary Data Id and enter Owner_DOB in Secondary Data Id and save the mapping.

When a producer enters the owner's birth date with the D_Owner_DOB data item on the distributor's form, FireLight will automatically pre-fill the owner's birth date with the Owner_DOB data item on the carrier's form.

Delete Linked Data Items

In the grid on the DataItem Sync tab, locate the row of linked data items to delete and click the red X in the Delete column of that row.
