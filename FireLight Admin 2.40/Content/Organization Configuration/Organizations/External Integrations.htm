External Integrations

On the navigation bar, click Organization Config > Organization Settings and click External Integrations to go to the External Integrations page.

Use the External Integrations page to set up, edit, update, or delete illustration services external integrations.

This integration exposes the FireLight Illustration SDK to support connection to FireLight Illustrations so that anyone providing an illustration functionality can integrate with the Hexure platform. This is done by creating REST APIs to help carriers wrap their engines, provide self-service tools exposing a variety of ways to hook their engines into the platform, or include capabilities to leverage existing engines as-is.

Guidelines

NOTE: For more information, see the Illustration Integrations APIs (SDK) design document.

API Authentication

  • The desired IllustServices API endpoint version has been determined based on the product's configuration data.

  • The version 0 and version 1 API model classes are defined separately. (Day 1 is set for version 1).

  • The correct model classes are used for the IllustServices API that is getting called.

  • Current illustrations run as expected.

  • Version 0 implementations do not have a standardized Health endpoint. All existing IllustServices API configurations is configured as version 0.

  • Version 1 implementations do have the Health endpoint defined. If the desired IllustServices API that is getting called implements version 1 of the API, then the Health endpoint is used to determine if the IllustServices API endpoint is operational and available for illustrations. The health of the IllustServices API endpoint can be determined by calling the Health endpoint and getting a "healthy" response. Applications that implement the Health endpoint but respond with a non-successful HTTP status code, or return an "unhealthy" response, are excluded from illustration. The {Base URL}/Health GET operation returns a 200 status code and relevant health data in a response. The {Base URL}/Health GET operation returns a 500 status code if an exception is generated, and a relevant error response is generated.

Add a New Illustration Service

NOTE: Information in the Illustration Services table is not included in deployments for Organization Sync Scope.

Different environments can have different Illustration Service configuration data for the same Illustration Service.

Admin users need to configure the Illustration Service applications in each of the various environments separately. The Illustration Service configuration data is not transferred from one environment to another during a release.

The name of the Illustration Service must be the same in all environments, but the URL, Authentication type, and Key Value properties can be different environments.

  1. Below the Illustration Services table, click the Add New Illustration Service link to open the Service Name Configuration dialog.

  2. In the text boxes, enter the Service Name, Base URL, and Key Value.

  3. In the lists, select the Version and type of Authentication (None, API Key, HMAC).

  4. Click the Save Endpoint button.

  5. Click the Test Endpoint button to test the endpoint. This returns the JSON results to let the user know if the set-up is "Healthy" or “Unhealthy” which would receive a response such as : HTTP: 401 or HTTP: 500.

Delete or Update an Existing Service Name Configuration

Delete:

In the Illustration Services table, click the red “X” on the right of the row for the corresponding service Name to delete the service.

Update:

  1. In the Illustration Services table, click the service Name link.

  2. In the Service Name Configuration dialog, make changes to the service and click the Save Endpoint button to save the changes.

  3. (Optional) Click the Test Endpoint button to test the endpoint and verify whether the endpoint is Healthy or Unhealthy.

  4. Click the X on the upper right of the Service Name Configuration dialog to close the dialog.
