Mobile Configuration

Step through the following to find instructions for entering settings for your organization.

  1. On the Organization Settings page, click the Mobile Configuration link to open the Mobile Configuration dialog. This is where you configure access parameters for ® Console (disconnected) users.

  2. Add and edit mobile configuration information.

    Max Check-Out Applications Maximum number of applications that can be offline (checked out) at one time per device. The default is 10 applications. If no maximum, type 0.

    Max Check-Out Days Maximum number of days the application can be kept offline (checked out). When an application expires all data associated with the application is removed from the device; 24 hours prior to expiration the application is highlighted to notify the user it is about to expire. The default is 14 days. If no maximum, type 0.

    Max Devices per Organization Maximum number of mobile devices that can be added per Organization for each user account. The default number of devices is 2.

    Heartbeat Check (Seconds) Number of seconds between each heartbeat check. The heartbeat check executes only when the Console has access to the web server. During the heartbeat, mobile configuration settings are updated and Password Reset as well as Device Deactivation commands are handled. To prevent too much unnecessary network traffic, the minimum is 300 seconds. The default is 600 seconds.

    Access PIN Duration (Days) Number of days until the Console Access PIN expires. The minimum is 1 day. The default is 7 days.

    Offline Enabled Select the checkbox to enable offline console support for the organization. The default is cleared (unchecked).

    Disable Check for Topaz Pad Select the checkbox to disable the Console from checking for the presence of a Topaz signature pad.

    Type of Username Select the type of Username to be required along with the Password when logging into the Console:

    None (Default) Only the password is required.

    External ID The Username must match the External ID provided in the SSO NameIdentifier assertion.

    Custom The user provides a Username during the activation process.

    Min Password Length Length of the password. The minimum is 1.

    Password Expires in (Days) Maximum number of days until a new Password is required. The minimum is 1 day.

    Min Numbers in Password Minimum numbers required in the Password. (The default is the same as Minimum Password Length for backwards compatibility.)

    Min Symbols in Password Minimum symbols required in the Password. (The default is 0 for backwards compatibility.)

    Login Attempts Window (Seconds) The number of seconds allowed for authentication attempts. If the Max Login Attempts is exceeded within this time frame, the Wipe On Failed Login setting is checked to determine if the data should be deleted from the mobile device. If the Wipe On Failed Login is not set, the console is locked from entering a Password for the Authentication Window duration. The default is 180 seconds. The minimum is 60 seconds. If no maximum, type 0.

    Max Login Attempts Maximum number of Password entry attempts before the application is locked or wiped clean. The default is 3 attempts. The minimum is 1 attempt. If no maximum, type 0.

    Wipe On Failed Login Select the checkbox to wipe the data from the device if the number of failed login attempts is exceeded. The default is cleared (unchecked).

    Idle Timeout (Seconds) Number of seconds of idle time before the Console is locked and the user is required to type a Password to re-log in to the Console. The default is 300 seconds. If no maximum, type 0.

  3. Click the Save button to save your entries.
  4. Click the X on the upper right to close the dialog.
  5. On the Organization Settings page, click the Save Organization button.
