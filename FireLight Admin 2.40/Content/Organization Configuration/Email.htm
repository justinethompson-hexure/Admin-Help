Email

On the navigation bar, click Organization Config and click Email to go to the Email Template page.

The Email Template page contains a collection of default email templates the system uses to automatically generate emails when certain events occur in FireLight®.

Select an Email Template

  1. In the Email Category list, select a category to filter the email templates by the selected category.
  2. Select an email template to modify as applicable for your organization using the individual email topics under the selected category in the Admin Help as a guide.
  3. If applicable, select a Locale.
  4. If applicable, select a Branding Firm.
Email Guidelines

Email Guidelines

  • The "To Address" cannot contain more than one address in an email. Multiple "To" addresses are not supported.

  • If the email address field can be edited, DMARC logic is automatically applied if the Use DMARC checkbox is selected in Organization Settings on the Activity tab. The From Address is replaced by the organization's email address to avoid email server rejection due to domain mismatch. The Reply To address will still be the agent's email. If you do a direct login or you do not send in an email address, we enable the Use DMARC feature automatically and do the replacement. If you send in the email address but have the "Use DMARC" setting enabled, we also do the replacement. The only time we do not replace the From address with the organization's email address is when you send in the email address and do not have the "Use DMARC" setting enabled on the Organization Settings, Activity tab.
  • You can disable preference fields and send with the incoming FireLight request. If template fields are not disabled, users can also type the From name and Email. There are two ways that the preference fields can be disabled. One is using the Organization Configuration/Groups, App tab setting "Disable Edit Preferences." The other is done automatically, similar to the DMARC option. If the user comes in through an external FireLight request, the preference fields are disabled automatically. However, if the Groups, App tab setting "Enable Override SSO" is enabled, it enables the group to still edit the preferences full name and email address even if they come in through a request.

  • A new replacement variable using an activity’s dataitems can be added to an email template by requesting a database change to add the new value to the activity search items list. This activity search item can be used in the Advanced Search in the All Activities page in FireLight or can simply be saved off for use in the email templates. Contact your Strategic Account Manager to add the new replacement variable.

  • The CLIENT_NAME tag requires a dataitem of the same name to be added to the activity and set to the name of the signer wanted for the replacement value.

    For example, to set it to the owner name, a rule would be added to the form:

    Set Client_Name to Owner_FullName

     <if> <condition> <not> <isnullorwhitespace> <diget dataitemid="Owner_FullName" /> </isnullorwhitespace> </not> </condition> <diset dataitemid="Client_Name"> <diget dataitemid="Owner_FullName" /> </diset> </if>

NOTE: When possible, the From name & email address are based on the owning agent's login information, but several factors can result in that not being used. If any of the following are not the case, the From name and AGENT_NAME email variable will default to the organization name (rather than organization email address):

  • The group option Enable Use Agent Email must be selected.

  • The user Preferences option Use Agent Email for All Communications must be selected.

  • The agent must have a name passed in their SSO or configured in Preferences.

 Add Logo and Color Branding to an Email Template

Add Logo and Color Branding to an Email Template

NOTE: Firms can have a separate HTML template for each Sub Firm.

  1. Use HTML to add a logo and colors to an email template.

    IMPORTANT! The CSS needs to be inline and any images that are in addition to the logo need to be inline base 64 encoded.

  2. Provide the logo in .png format (it does not need to be base 64 encoded).

  3. Please submit a JIRA ticket with separate HTML templates, logos, and branding ID for each Sub Firm. This ticket will be assigned to a developer for review, testing, and loading into the firm's database. You cannot edit the HTML in Admin. All changes must be submitted via a JIRA ticket.

Following is a sample HTML template for an email:

<html><body><div>

<table style="background-color:blue;color:white;">

<tr><td colspan="2"><img src="cid:$$OrgLogo$$" /><br /><br /></td></tr><tr><td></td>

<td><b><span style="color:white;font-size:10pt;font-family:Arial;">$$Body$$</span></b></td></tr></table></div></body></html>

NOTE: $$OrgLogo$$ in the sample string is replaced with the logo for your firm in FireLight. If you create an html string for a Sub Firm, you need to replace the $$OrgLogo$$ with the URL of the logo image that you want to use for the Sub Firm. For example, the Sub Firm would use “https://www.firelighteapp.com/EGApp/Content/Images/[Sub-firmLogo].png” for the URL in place of the $$OrgLogo$$.

Edit Email Subject Lines, Locale, and Template Text

Edit Email Subject Lines, Locale, and Template Text

NOTE: Do not alter the Message Center Notification email template, which is a system generated template for multiple system messages.

Change the email text for your organization.

  1. On the navigation bar, click Organization Config and then click Email.
  2. In Email Category, select a category of emails, for example, select Review to filter the list of email templates to show only Review related email templates. Select All to view a non-filtered list of all emails.
  3. In Email Template Type, select a template to edit.

  4. If enabled for your organization, select a Sub Firm in the Branding Firm list to change an email template for a Sub Firm.

    NOTE: If you need branding for Sub Firm emails, contact your Strategic Account Manager to have the FireLight team create the list of Branding Firms for you. Once that list is created, you can manage the Sub Firm emails on the Email page.

  5. If localization is enabled for your organization, select a language for the email template in Locale.

  6. Add, change or delete text in the Email Subject Line and the Email Template sections as appropriate for your organization and the selected Locale.

    Important: Do not change the text in angle brackets < >. This identifies a data field name variable that is populated with data for the message. For example, the variable <CLIENT_NAME> will populate the email message with the client’s name from the application.

  7. When finished editing a template, click the Save Email Template button at the end of the page. The change takes place immediately.

 Ensure Review Queue Emails Are Sent to the Agent

Ensure Review Queue Emails Are Sent to the Agent

The following settings must be in place in FireLight Admin and FireLight to ensure the agent receives Review Queue emails.

In FireLight Admin:

  • In Organization Configuration/Groups on the Review Queue tab, ensure the Disable Action Email Notifications checkbox is cleared (unchecked) for the reviewer role code.
  • In the Post-Submit Review Queue Approve and Post-Submit Review Queue Reject email templates, ensure the |MESSAGE| or <MESSAGE> replacement marker is somewhere within the email body.

In FireLight:

  • On the My Preferences page, ensure the agent's Send Message Center Emails checkbox is selected (checked).
 Settings That Can Affect All Emails

Settings That Can Affect All Emails

The following settings in Organization Configuration on the Organization Settings and Groups pages can affect all emails.

Organization Settings/Activity tab:

  • Prevent Email Editing Select the option to prevent system generated emails from being edited by agents or clients. For example, FireLight presents an email template in the request process for the client fill and client fill and sign requests, but the user cannot edit the email template.
  • Use DMARC Select the option to use the email address that is entered in the organization’s Support Email field instead of the Agent’s email address for all outgoing email communications. This option helps avoid email rejection due to DMARC authentication. The agent email is used as the "Reply to" email address.

Groups/App tab:

  • Disable Edit Preferences Select this option to disable access to the My Preferences page in FireLight. The My Preferences button on the Home page and on the top toolbar will not be visible. If disabled, FireLight will use the default preference options or the information sent in through the incoming Single Sign-On (SSO) request.
  • Enable Cc Sender in All Requests Select this option to enable automatic CC of the sender for any agent generated email (for example, Client App Fill Request, Sign Later Signature Request, Reviewer email).
  • Enable Use Agent Email Select his option to enable and show the Use Agent Email for All Communication checkbox on the My Preferences page in FireLight. This allows the agent to determine if all assistant actions should use the agent’s email for outgoing communications. The Use Agent Email option is based on the activity owner’s preference, not the current session user’s preference.
  • Enable Override SSO Select this option to enable the agent to override the entry in the Your Name field on the Send Email To Client To Request Signatures page (this overrides the User Full Name field entry on the Preferences page). This allows agents to use their preferred names on client signature requests. Without this option selected, the default behavior is to lock the User Full Name and the User Email Address fields within My Preferences to the values sent over in the Single Sign On (SSO) Request that was used to establish the FireLight user session.
