Create and Edit Groups

Use the Groups page to define FireLight® user access groups and role codes, and applicable firms, for your organization.

General Guidelines:

  • The role code is sent with the SSO information if your organization uses Single Sign On.
  • The group Name and TC Role Code are based on the type of user and role.

  • Tabs at the bottom of the Groups page define the access permissions for the selected group in the grid at the top of the Groups page. For example, users in an Admin group require different access permissions than users in an Agent group or Test group.

    Following is a simple sample of user Groups and TC Role Codes for the sample Peak Life organization.

Firm Management Guidelines:

  • The grid at the top of the page has a Firm Org column. When the Admin user does not have firms set up in the Firm Management page under Organization Configuration, the Firm Org column in the grid and the Firm Org dropdown list below the grid are blank.
  • If an Admin user does NOT have a Firm associated with their user group, the Admin user sees the Main Org view.

  • If an Admin user has a Firm associated with their user group, the Firm Org dropdown list below the grid includes all firms that are set up on the Firm Management page under Organization Configuration for their organization and a blank entry that displays all firms in the grid. The list is arranged with the blank entry being first (this is the default selection) followed by the entries for the firms, sorted alphanumerically by the firm's name in the grid.

    The Groups view displays the Firm Org in the grid. The Firm Org dropdown list below the grid does not allow for the user to make changes. No other firms are available for selection to the firm Admin user. The dropdown selection can be edited only by the IT Vendor role or Main Organization/Firm level Admin.

Add groups and associated role codes for each type of user.

  1. On the navigation bar, click Organization Configuration/Groups.
  2. On the upper left of the Groups page, click Create New.
  3. Below the grid in Name, type a unique name for the user group. For example, type Agent for a group Name.

  4. In TC Role Codes, type the role code(s) for the FireLight user group within your organization. NOTE: If a user is a member of a Sub Firm, the TC will begin with a group identifier, and be separated from the TC by an underscore (e.g., Dell_TEST8).

  5. In Illustration Services Role Code (if the Illustration activity is included for your organization), type the Illustration Services Role Code for the user group.

    The Illustration Services Rode Code is used when calling the Illustration Services for calculation results for Quick View and Illustration PDF files. Illustration Services uses the role code to determine which results to return or which reports to return. This is something that you need to know as an Admin or request it from the Strategic Account Manager.

    Specify this role code in two places:

    • Groups – in the Illustration Services Role Code. This is used only when the selling organization is the same as the products organization (i.e., the Carrier). In this case it essentially works as an override to the Sales Agreement role code. This allows the carrier to have Admins or Quality Assurance users that can get more calculations or more PDF files for development and testing purposes.

    • Sales Agreement – in the Illust Services Role. This defines the role code to use when a carrier has a distributor selling their product.

  6. FireLight adds the Groups to the Case Access tab. When you select a group in the grid at the top of the Groups page, go to the Case Access tab to select the checkbox for each group that can have access to the selected group's cases. For example, select an Agent group in the grid at the top of the page, click the Case Access tab, and select the user groups that can have access to the Agent group's cases. You may want to grant case access to Agent group cases to the Sales Desk group. However, you would not grant access to the Agent group to prevent agents from seeing cases from other agents. See Case Access and User Share Comparison for descriptions and a comparison chart based on case access by group Role Code, Firm ID, and User Share.

  7. Select checkboxes on the remaining tabs at the bottom of the Groups page, as applicable, to grant access to FireLight features and functions for each user group. Refer to the individual Help topics for each tab.
  8. When finished with selections, click the Save button at the bottom of the Groups page.

Create External Role Code

For external requests, an EXTERNAL role code defines the permissions of users entering FireLight through external requests and leaves the existing DEFAULT role code behavior for the agent SSO scenario.

  1. Log in to the Admin app.

  2. Under the Groups settings, click the App tab, then select the required role code.

  3. Under Signing general settings, deselect Agent Not Required to View All Pages and then select Disable Signing Until All Pages Viewed.

  4. Save the Groups setting.

  5. Log in to User app.

  6. Create a new application (Multiple Agent Signing Product).

  7. Click Continue.

  8. Click Use E-Signature for React E-Sign.

  9. Select Client (Owner/Agent) as the signer and then select Proceed for Send Email Request.

  10. Enter data on request window the Send Email Request to the external Signer.

  11. Log in with the Pickup link received in mail or displaying on the generated link.

  12. Click Sign Activity on External Signer.
