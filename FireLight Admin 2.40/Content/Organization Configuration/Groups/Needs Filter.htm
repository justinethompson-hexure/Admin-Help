Needs Filter

Add a Filter Category

Filter category names are the columns across the grid. Category names are unrestricted strings and each category name must be unique.

  1. On the Groups page, select a TC Role Code in the upper list.

  2. On the App tab, select Enable Create Needs Determination.

    Additionally, to require the User Group to always do a Needs Determination before an application, also select the Disable Create Application checkbox.

  3. Click the Needs Filter tab.
  4. Select an Organization in the list. This is the organization whose products is filtered.
  5. Click the New Category button.
  6. On the right, type a name for the filter category.

  7. In the list, select a filter type:

    String if the Needs Determination transaction contains the given data item for the filter rule, and the data item’s value matches what is defined for the filter rule, then the corresponding product is allowed to be shown (unless other filter rules prevent it from being shown). Inputs are unrestricted. Use semi-colons to delimit the string, allowing multiple options for the given product.

    StringOverrideFilter if the Needs Determination transaction contains the given data item for the filter rule, and the data item’s value matches what is defined for the filter rule, then the product must be shown, no matter how the other filter definitions match up. Inputs are unrestricted. Use semi-colons to delimit the string, allowing multiple options for the given product.

    ValueGreaterThanX if the Needs Determination transaction contains the given data item, and the data item’s value is greater than what is defined for the filter rule, then the corresponding product is allowed to be shown (unless other filter rules prevent it from being shown). Enter numbers only in a decimal format (for example: 10,000.34).

    ValueLessThanX if the Needs Determination transaction contains the given data item, and the data item’s value is less than what is defined for the filter rule, then the corresponding product is allowed to be shown (unless other filter rules prevent it from being shown). Enter numbers only in a decimal format (for example: 10,000.34).

    NumericRange if the Needs Determination transaction contains the given data item, and the data item’s value is between or equal to the numbers that are defined for the filter rule, then the corresponding product is allowed to be shown (unless other filter rules prevent it from being shown). Enter only two numbers separated by a dash (for example: 10,000.34–800,000.67).

    ValueContainsString if the Needs Determination transaction contains the given data item, and the data item’s value contains the value that is defined at any point within the data item’s value, the corresponding product is allowed to be shown (unless other filter rules prevent it from being shown). Input is unrestricted.

    NotEqualString if the Needs Determination transaction contains the given data item, and the data item’s value is not equal to what is defined, then the corresponding product is allowed to be shown (unless other filter rules prevent it from being shown). Inputs are unrestricted. Use semi-colons to delimit the string, allowing multiple options for the given product.

  8. Type the Data Item ID from the Needs Questionnaire form.
  9. On the left, select the Products in the list to filters will apply.
  10. Enter the applicable filter value for each product. If the filter category does not apply to a product, leave it blank for that product.
  11. Click the Save button.
  12. Enter the filter categories on this tab for each product to which the selected role code has access.
  13. Repeat the process for each applicable User Group.

Upload a Group Filter Definition CSV

Use the Upload feature to upload multiple filter category definitions in a CSV or XLSX file.

  1. On the Groups page, select a TC Role Code in the upper list.

  2. On the App tab, select Enable Create Needs Determination.

    Additionally, to require the User Group to always do a Needs Determination before an application, also select the Disable Create Application checkbox.

  3. Click the Needs Filter tab.
  4. Select an Organization in the list. This is the organization whose products is filtered.

  5. On the upper left of the tab, click the Upload button, browse for the group filter definitions CSV or XLSX file, and click the Upload button.

    If an uploaded data set is too large, the group filter definition table will not render on the Needs Filter tab and a "Data set too large - unable to render table" message appears under the Upload button. The data is saved, but you are unable to view it on the Needs Filter tab.
