NOTES:

The following data items apply for Paramed Ordering.

  • FLI_UNDERWRITING_PARAMED_REQUIRED allows the paramed ordering to be either enabled or disabled based on carrier preference. FLI_UNDERWRITING_PARAMED_REQUIRED= true will enable paramed ordering, while FLI_UNDERWRITING_PARAMED_REQUIRED=false will disable the functionality.

  • FLI_UNDERWRITING_PARAMED_SELECTED gives the user the choice to either choose between the APPS ordering platform or the ExamOne ordering platform. "APPS" or "ExamOne" is used as list values. For example: FLI_UNDERWRITING_PARAMED_SELECTED const value=APPS would allow the APPS ordering platform to be called, while FLI_UNDERWRITING_PARAMED_SELECTED const value=ExamOne would allow the ExamOne ordering platform to be called.

  • FLI_UNDERWRITING_PARAMED_APPS is specific to the APPS Ordering platform. This dataitem can be used to enable/disable this vendor.

  • FLI_UNDERWRITING_PARAMED_EXAMONE is specific to the ExamOne Ordering platform. This dataitem can be used to enable/disable this vendor.

Allow Share Limited Control

When an administrator selects Allow Share Limited Control link for a selected user, two control options are provided, as described below.

  • Complete Data Entry: The user/agent with whom the case/app is being shared will only be able to get the data entry to 100% complete for the shared app. The user will not be able to process the app further from the data entry step.

  • Complete Signing without Primary Agent: The user with whom the case/app is shared will be able to complete the data entry and also proceed for signing ceremony for the shared app, though the user will not be able to sign as the primary agent (Signer Type = “Agent”) on the application. Based on the agent sign order, they will be able to complete signatures for all other signers, except primary agent. If pre-signature review is enabled, they will be able to complete that step before the signing ceremony.

  • Complete Pre-Signature Review: The user/agent will be able to complete data entry and the pre-signature review process for the shared app. They will not be able to proceed for Signatures. Activities/firms that do not have pre-signature review enabled (or manually disabled via rule or data items) but have this option is selected, they will complete data entry on the shared app and not be able to proceed to the signing ceremony.

  • Complete Signing: The user with whom the case/app is shared will be able to complete the data entry and also proceed to the signing ceremony to complete all the signatures on the shared app. They will not be able to proceed to the step after signing. If pre-signature review is enabled, they will be able to complete that step before signing.
