Admin Basics for Setting Up Pre-Sale

Pre-Sale is creating a new activity that will precede an activity that follows. Several places within the Admin allow configurability to the pre-sale. This section details each one.

Activity Detail

Activity Name

The Activity Name can be configured to a business need. This allows for the user to name their activity specific to what they’re trying to accomplish. When adding a new Activity, this will need to be altered.

ACORD Type

ACORD Type drives the activity and associates it with Products, Packages, and Sales Agreements. This has replaced the Transaction Type from previous releases and makes an activity and package accessible to both a carrier and distributor. The type chosen will reflect the ACORD value associated with it.

Activity Type

This feature is attached to the ACORD Type and is locked down as it cannot be altered.

Branding Icon

The icon reflected in FireLight® can be configured by the client. The icons are featured from FontAwesome, and a link is provided to view each professional icon. The textbox can be altered, and the name of the icon needs to be entered to display the chosen icon.

Continue/Apply Button

A value can be entered to customize the label of the Continue button.

Allow Pre/Post Signature Review

These checkboxes are mutually exclusive and allow for a pre-signature or a post-signature pre- submission review. This option can be chosen along with a post-submission review.

 

Bypass New Activity Prompts

This feature allows you to skip choosing the jurisdiction and product if only one product and one carrier is set to this activity. This feature applies only to the initial activity. For example, if there was only one Needs Determination listed for the Activity, then once selected in FireLight®, it will immediately launch the forms listed:

Edit Activity Pop-up messages

Select pop-up and toast messages can be customized based on the client’s business need. If a user wants to return to the default message, a selection can be made to change the message. These messages are cached, so it will take 20 minutes for changes to be reflected.

 

 

Next Activity

Add in the Next activities that will be associated with the Pre-sale. Multiple next activities can be associated with the Pre-sale and will be displayed once the Pre-sale activity has been submitted.

If only one Activity is selected for the Next Activity and only one product is associated with that Activity, it will bypass the Jurisdiction and Product type menu and open up the next Activity

 

without displaying the menu. If multiple products are associated with the next activity, the normal FireLight® prompts appear.

Groups

The Pre-sale activity needs to be selected for each role code in the Activity tab to be visible.

Using Rules and Next Activity Options

A new dataitem has been created to reflect the Next Activity through ACORD type. This is FLI_NEXT_ACTIVITY_ACORDTYPE, and this dataitem can be utilized to specify the next Activity.

For example, to have an illustration show for the next activity when multiple activities are listed, add the following rule:

<diset dataitemid="FLI_NEXT_ACTIVITY_ACORDTYPE">

<const value="111" type="String" />

</diset>

Even though multiple activities are listed within the field properties, the rule will designate the activity reflected.

 

 

Utilizing the Needs Filter through Pre-Sale

The Needs filter can be used to create a greater filter for the pre-sale activities. Within the groups tab, the Needs filter can filter certain next activity products based on dataitems and export values.

An example would be if a product needed to be filtered based on the Owner’s age. The age would be added along with the products needing to be filtered down:

Once the value is entered into the pre-sale and the user is advanced to the next activity, the needs filter shows the products they are allowed to select based on the query.

 

Please NOTE: The Needs Filter is based on the selected role code, so multiple filters can be created if there are multiple role codes.

For more information about the Needs Filter, please see the Admin Help.

The Deployment Process

When deploying a Pre-Sale either into the higher environments like Staging and Production or deploying into a distributor environment, there is not much of a change with the current deployment process. The difference is the configuration for activities. For both carriers and distributors, the activity name will have to be created.

If a carrier is deploying to a distributor, the current process of pairing the package to the product for that selling organization is the same within the Sales Agreement; however, the defining item is that the package is set by the ACORD Type that is the same as the Activity.

 

 

 

While the activity is not deployed to distributors, the product and package are and will be tied to the distributor’s activity through the ACORD Type. This gives the opportunity to the distributor to name the Activity according to their business needs but still tie the carrier’s product and package to it.
