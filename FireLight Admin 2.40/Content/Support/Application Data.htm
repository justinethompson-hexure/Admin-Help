Application Data

Use the Application Data page to view information on each case that is being sent to the DTCC and their App/Sub and attachment status.

NOTE: The information contained in this Application Data tool shows only what is transmitted from FireLight® to the DTCC. This does not reflect any information from the DTCC to the client organization.

Enable Application Data in Admin

  1. In Organization Configuration on the Groups page, select a group Name in the grid.

  2. In Groups on the Debugging Tools tab, select the Enable Viewing of Application Data checkbox.

View and Download Application Data

  1. On the Home page or navigation bar, go to Support and click Application Data to open the Application Data page.

  2. On the left, enter an Application ID or Application Control Number and click LOAD. NOTE: After clicking the LOAD button, FireLight displays the following message for entry of an incorrect Application ID or Application Control Number: "Application ID or Application Control Number does not exist."

    The window on the right shows any information listed about the application. This includes Application Name, Application Control Name (DTI Number), Application ID (GUID), Distributor Code, Carrier Code, Request Type, Submission Status of the application submission, and View AppSub File button, which shows after the request to the DTCC has been received. Click the View AppSub File button to display the AppSub below the table.

    NOTE: The View AppSub File, View and Download buttons are available only in the QE, UAT, and Staging environments. The buttons are not available in Production to protect Personally Identifiable (PI) information.

  3. View specific details on the application, specifically for DTCC transmissions. On the right of the page, the table includes the Type, Instant (date and timestamp), Status of the DTCC Transmission, and a view column. The Status of the DTCC Transmission can include the following responses: Success, Pending, Confirmed, Transmitted, Error, AppSub, and N/A. Click the View button next to the "Transmitted" status to view the header of the transmission to the DTCC.

    • The order of a successful transmission is: Pending, Transmitted, Confirmed, and Success.

    • The order of a rejected transmission is: Pending, Transmitted, Transmission Accepted, Transmission Rejected, transmitted, Transmission Accepted, and Success. The transmitted, Transmission Accepted, etc., can occur multiple times until a Success happens.

    • The Confirmation response field will show either Confirmed or Rejected or is blank until the response is received by the DTCC. Click the View button next to the Response status of "Accepted" or "Rejected" to view the DTCC response. Note that the "Accepted" status is the same as the "Success" status.

    • The confirm header is viewable and downloadable as a txt file upon viewing. Only the latest confirmation is viewable and downloadable. Any previous confirmations will show type, timestamp, and status, but not a viewable file.

    • Confirmation is visible in all environments: QE, UAT, Staging, and Production.

    • A rejected response will have a View button to help the user troubleshoot the issue.

    • AppSub will always be viewable, either when a successful or rejected transmission occurs.

  4. In the right column of the table, click the View button to view the Request, Transmission, and Response to display the information in a separate section below the application details. Clicking the View button creates an audit that a support person viewed this transmission in the Audit History.

    NOTE: The View button is available only in the QE, UAT, and Staging environments. It is not available in Production to protect PI information.

  5. In the Request, Transmission, or Response details section on the lower right, click the Download button to download a TXT file with the detailed information.

    NOTE: The Download button is available only in the QE, UAT, and Staging environments. To protect PI information, it is not available in Production.
