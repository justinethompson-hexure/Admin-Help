FireLight v2.39 Release History

What's New in FireLight

The following enhancements are now available for all clients unless noted otherwise. See the "FireLight v2.39" Release History" topic in Admin Help for fixed defects and links to design documents and Admin Help topics.

New Functionality

Integration with DTCC Producer Authorization: CR-1807

We have added an integration to our Can Sell functionality to utilize the DTCC Producer Authorization. This integration can be selected via settings in the administration tool. Please note that this functionality is available only to carriers who are participating in the DTCC Producer Authorization and is mutually exclusive to the use of the SuranceBay Producer Eligibility functionality.

Impact

FL Admin > Org Settings > App > Submit - either SuranceBay Producer Eligibility OR DTCC Producer Authorization can be selected under Can Sell Check.

FL eApplication (only)

When enabled, a call is made to the DTCC to verify the producer eligibility to sell the selected product.

Moved App Settings from appsettings.config to ExternalIntegration Config Files: CR-705

We enhanced security by moving the app settings for the DTCC PA, LexisNext, Magnum, Aura, DocuSign, and SuranceBay apps from appsettings.config to ExternalIntegration files.

Remove DTCC Producer Authorization from Sales Agreement: CR-4041

We removed DTCC PA as a selectable option in FireLight.

Impact

FL Admin > Sales Agreement > Options tab > Can Sell Check > “DTCC PA” no longer available

Enhancements Related to Smart Text Box and Smart DatePicker: DIST-2244

To provide more options to validate numbers and date inputs, new properties are added to the Smart Text Box and Smart Datepicker fields.

To validate whether numbers entered on the app side are between a specific range or greater than or less than a specific number, admin users can now select properties on the field without writing rules.

Date validation options are enhanced to determine whether the entered date should be within a date range in the past or if the date should be within a date in the past or beyond that date.

Impact

  • New Property “Validation Range” added for smart text box: FL Admin > Wizard Designer > Smart Text Box (field type) > Validation Range (new property)

  • More options added to the Date Validation property for the smart datepicker: FL Admin > Wizard Designer > Smart Datepicker (field type) > Date Validation (property)

Admin Reports Modifications: DIST-3813

Two new reports are available from the Admin tool:

  • Business Detail Report

  • Workflow Report

Impact

  • FL Admin > Utilities > Reports: new reports can be generated in CSV format.

    • Business Detail Report

    • Workflow Report

  • Provide Firm Org filter for the reports below and add a column listing firm org to these reports:

    • Sales Agreement Audit History Report

    • Package Audit History Report

  • The Firm Filter drop-down lists firms based on access to the logged-in admin user.

  • The Form Fields report has new columns for field Height and Width.

Allow Importing First Four Columns from the Custom List or Import File to the Drop-Down Field: DIST-4390, DIST-4389, DIST-4388, DIST-4362

As a further enhancement to importing values to a drop down from a custom list or an external file, users can now import up to four columns while importing values.

Impact

  • FL Admin: On the wizard designer, the changes below will be available to the properties dialog of a drop-down field - Wizard Designer > Dropdown field> Field Properties > Edit List Name Values link.

    • Drop-down to select custom list – to link or attach values to the drop-down field,

    • Import button to select values from an external file.

  • FL Admin: On the form designer, the changes below will be available to the properties dialog of a combo box field - Form Designer > Combo box field> Field Properties > Edit List Name Values link.

    • Drop-down to select custom list – to link or attach values to the combo box field.

    • Import button to select values from an external file.

Activity Report Modifications: DIST-4575, DIST-4779, DIST-4780

New activity reports are available to be scheduled from Reports Scheduler:

  1. Detail Report to list Illustration activity data: Illustration Activity Detail Report

  2. Custom Report to list commission netting data for distributors: Commission Detail Report

  3. Custom Report to list money settlement report for distributors: Money Settlement Report

Impact

FL Admin > Utilities > Reports Scheduler: New reports are available to be scheduled.

  • Under the detailed reports - Illustration Activity Detail Report

  • Under the custom reports - Commission Detail Report

  • Under the custom reports - Money Settlement Report

Notify Reviewer When Agent Uploads Documents in Post Submit Review: DIST-4710

Currently, when an agent adds or removes a document from an activity while it is being reviewed in the pre submit review, a notification is being sent to the reviewer to let them know of the action. The same functionality is now enhanced for the activity when reviewed by post submit review. An email notification will be sent to the reviewer in post submit review that a document is added or removed by the agent.

Impact

  • FL Admin > Organization Configuration > Email > Review Queue (Email Category) > Review Queue Attachment Update (Template Type)

  • FL App > Activity > Other Actions > History: Display document upload update and email notification sent update

  • FL App > Review Queue > Application History: Display document upload update and email notification sent update

Update APIs to V2: EAES-2088

We updated the following APIs to V2 so that they return a specific error code and reason when an error is encountered while using them: GetActivityDetails, GetActivityPDF, GetAvailableActions, GetDocumentList, and GetProductAvailability, Submit, Upload Document, UnlockActivity, and ValidateActivityData.

Prevent External Signer Types from Printing PDFs (Implement for React): EAES-4596

We added the ability to prevent specific external signers from printing forms while reviewing them.

Update Data Item with Role Code of Last User Who Accessed an App: EAES-5646

We added the ability to have the FLI_CURRENT_ROLE_CODE data item updated with the role code of the user who most recently accessed an application.

SetToIXN Updated to Current Specifications: EAES-5883

The setTOIXN property now states its current function, such as toggling between the two quote tools, and allows for more accurate comments.

Edition Attribute Added to FireLight Admin Modules: PC-1941

We added the ability for users to add the edition attribute to the Wizard Library, Rulesets, and Package modules for more efficient version tracking of wizards, rulesets, and packages.

Error Message Now Displays Multiple Errors: PC-2779

The system now displays multiple errors in an application when errors occur. Note: This update has been enabled in QA and UAT but will not be generally enabled in production at this time.

Impact

When a person is completing an application, if there are multiple errors on a page, a popup window will appear with all errors being stated.

Add Ability to Add More than One Product at a Time When Managing Sales Agreements: PC-2838

We added the ability to add multiple products to multiple selling organizations when managing sales agreements.

Impact

In Product Configuration > Sales Agreements, a new menu will be available for the customer to use to select multiple products and assign them to multiple selling organizations.

Ability to Redirect to a Dynamic URL on Submit: PC-2842

We added the ability to redirect to a dynamic URL on submit. This new function allows that at submit (regardless of the group option), retrieve the app data and if data item FLI_SUBMIT_REDIRECT_URL exists and has a value, it will redirect to that URL (regardless of the URL configured in group settings).

Impact

User will be able to submit (regardless of the group option), retrieve the app data, and if data item FLI_SUBMIT_REDIRECT_URL exists and has a value, it will redirect to that URL (regardless of the URL configured in group settings)

Add New Product Type - Contingent Deferred Annuity (CDA): PC-3381

We expanded user options by adding a new product type, Contingent Deferred Annuity.

Resolved Issues

We resolved an issue where when creating an application with a smart text box and date picker wizard and entering data the data entry percentage remained at 0%. In this scenario the data entry percentage display is working as designed.

We resolved an issue where clicking the View Sample CSV link on the Import Participants pop-up resulted in an error. When clicking this link the sample CSV is now downloaded as designed.

Unable to Filter by Queue When Viewing Reviewer's Activities: DIST-4797

Unable to Filter by Date and Keyword When Viewing Reviewer's Activities: DIST-4871

We resolved an issue where filtering by date and keyword on the Reviewers tab of the Review Queue page was not working as designed. These elements are now working as expected.

We resolved an issue where the confirmation dialog text was other than expected when submitting an application.

We resolved an issue where having an application with curly braces in the name caused sending an email signing request and generating a link to fail. Sending an email signing request and generating a link now work as designed, even when an application name includes curly braces.
