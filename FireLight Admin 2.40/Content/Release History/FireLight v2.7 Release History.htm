Client Designated Enhancements

Admin

  • [126598] Sort the grid by Name on the Groups page in Profiles – In FireLight Admin on the Groups page, made the list of groups in the grid alphabetical first by the “Name” column and then by the “TC Rode Code” column.
  • [128213] Added email notification after application is submitted when e-Sign is declined – Added a new email template in Admin for when electronic signature is declined during the application signature process.

Audit History

  • [127287] Added the user logged in during the agent signing to the audit trail – In FireLight, added the logged in username to each individual audit message during the signing ceremony. Displays in the FireLight Packet Audit Report and in the History link in All Applications.

Forms Visibility

  • [125552] Added Distributor only visibility – Added the ability to restrict forms from visibility of both distributors and carriers.   • [127624] Added the ability to disable the application submit button based on criteria – Added the ability to disable the Submit button and force applications to go to the manual review queue (step 3) prior to submission in FireLight.
  • [128021] Added the agent code to the review queue email – Added the ability in Admin to add the <AGENT_CODE> data element to the manual reviewer requests for application review email template.

Signature Types

  • [129406] Added new Authorized Signer signature types – Added new signer types for Authorized Signer, Authorized Signer 2, Authorized Signer 3, Authorized Signer 4, and Authorized Signer 5 in the Signature Box control in the Admin form designer.
  • [131311] New sub-brand logos were not being saved in the database – Now sub-brand logs are being saved in the database.

October 30, 2015 Production Release

NOTE: If a design document is associated with an enhancement, click the link in the iConnect number to open the design document.

Client Designated Enhancements

Branding

  • [106534 ]Added the ability to brand UI elements – Added the ability for organizations to custom brand FireLight UI elements (UI Labels, System Strings, Audit Messages, List Profile, Email Templates, and Validation Error Messages) and add or update sub/firms using the Branding page in FireLight Admin. The vendor must first enable the organization to allow UI branding and then the Change UI Branding permission under Admin needs to be selected on the Groups page in Profiles to add the Branding page and a Branding button on the Home page in FireLight Admin. We highly recommend contacting your Strategic Account Manager or business analyst before using the Add or Update Sub/Firm Brand feature.

Console

  • [119418] Enhanced FireLight Console security – Enhanced the Console security by adding more control over password content, limiting the number of active devices a user can have and limiting the number of times a password can be changed. On the Organizations tab in FireLight Admin/Profiles, the Mobile Configuration dialog now includes the ability to set the minimum numbers and symbols in the Console login Password (formerly called the PIN) along with the ability to include a required Username field on the Console login dialog. The Mobile Configuration dialog contains new fields for Max Devices per Organization, Min Numbers in Password, Min Symbols in Password, and Type of Username (None, External ID, or Custom). It also renamed some of the old fields (PIN Length to Min Password Length, PIN Duration to Password Expires in, Max Invalid PIN Attempts to Max Login Attempts, and Authentication Window to Max Login Attempts window). In FireLight web on the Preferences page under Mobile Access, the Reset PIN button name is renamed to Reset Password. In the FireLight Console, the Reset PIN button is renamed to Reset Password. The Console login dialog can be custom branded with colors, text, images, and footer text.

Display/Print PDF

  • [103510] Added ability to designate placement of the validation report – Added properties to allow the provider to designate the placement of the validation report within the Display/Print PDF option. The validation report can now be placed first or last in either the application forms section or within the optional forms section of the print order list.

Form Designer

  • [110098] Documented the ability to copy/move fields from one page to another in a form – In the FireLight Admin User Guide on the form Design page, added additional instructions for using the Toolbox to copy and move fields from one page to another within a form.
  • [111825] Enhanced “exclude from back office” functionality so that it can be conditional via rules – In FireLight Admin on the form Design page, added two new nodes in advanced rules (getbackofficeformstate and setbackofficeformstate) so “exclude from back office” functionality can be conditional via rules.
  • [118329] New Save prompt in FireLight Admin on the form Design page – In FireLight Admin, when a user attempts to leave a form without first saving changes, a new Form Changes dialog asks the user to save form changes. Clicking “Yes” saves the changes and clicking “No” discards the changes.

Forms

  • [118048 form / 118048 product] New replace/update form and update product features in FireLight Admin – In FireLight Admin on the Forms page, added a new Replace Form button that opens the upload form page where the admin user can upload a new replacement form for the selected form on the Forms page. Also added a new Replace Product field on the Sales Agreement page. The admin user can enter start and end dates and a time zone for a product to expire and then select a similar type replacement product from the Replace Product list. Active applications are switched over to the new product on the old product expiration date.

Groups

  • [112151] New group setting for Agents not required to view all pages – In FireLight Admin under Profiles on the Groups tab, the new “Agent Not Required to View All Pages” checkbox on the App tab allows Agents to sign without viewing all pages within a document set. If this feature is selected and Disable Signing Until All Pages Viewed feature is also selected in FireLight Admin, the “Confirm all documents without reviewing individually” checkbox appears in FireLight during the Agent signing process. The Agent can select the checkbox and click OK on the confirmation dialog to proceed with signing without viewing all of the form pages.

Integration

  • [118292] New ability to limit Resident Jurisdiction – All carriers can now restrict residence to jurisdictions they pass in the TX 1228 if the 1228 contains jurisdiction restrictions. When creating an application in FireLight, this limits the options in the Resident Jurisdiction list to only those states in which the agent is licensed to sell the selected product.

Localization

  • [99197 ]Added support for multi-lingual capabilities – Added a Localization page under Profiles in FireLight Admin that is available for organizations that are located in a country other than the U.S. The FireLight UI elements is translated into the selected language using Google Translate. Then the organization has the ability to modify the translation of the FireLight UI elements (UI Labels, System Strings, Audit Messages, List Profile, Email Templates, and Validation Error Messages) using the Localization page. The English language version is always available to users. FireLight users will see the localized UI when they first log on, but will have the choice to select English or the localized language on the upper left of the Home page for subsequent logons.

Organizations

  • [118252 ]Added a Validate Identity Verification option in FireLight Admin – Under Profiles on the Organizations page in FireLight Admin, added a Validate Identity Verification checkbox that applies to the Client Identification Verification entry during the e-signature capture process in FireLight. Select the checkbox in FireLight Admin to: (1) not auto-populate the SSN and DOB fields in the Client Identification Verification section for e-Signature, and (2) require the SSN and DOB fields to match the mapped application values for the signer type. If the values do not match, a validation message indicates that the entered value does not match the application entry value and prevents signing in FireLight.

Other Enhancements

Branding

  • [125999] Added ability to set a default time zone and limit the list of time zones – In FireLight Admin, added Time Zone to the Branding page under the List Profile. Admin users with permissions to change branding can limit the number of time zones displayed for the organization on the My Preferences page in FireLight. In FireLight, users can select their time zone from the Default Time Zone list on the My Preferences page.

Console

  • [125723] Implemented conditional template forms in the disconnected Console – Implemented the ability for template forms to be conditional and set off from rules in the Console. This was already implemented in the FireLight web application in version 2.6.

Form Designer

  • [121762] Made the focused field on form entry more prominent – Added styles in the Site.Css to make the focused field on form entry more prominent, keeping consistency through all browsers.
  • [125988/126934] Added a Custom Upload action to the Button control in FireLight Admin and the ability to upload documents and non-pdf images to an application in FireLight – In FireLight Admin on the form Designer page, added a new Custom File Upload action and related field properties to the Button input control. FireLight users can click the button control to open the Application Documents dialog, select a Document Type (or Other), browse to the location of the document or image file and click the Upload button to upload the document or image to the application Documents list on the Other Actions menu. FireLight accepts image files in .jpg, .tiff, .png, .gif, and .bmp formats. Image files in non-pdf format can only be uploaded via a custom button.

Forms List

  • [120205] Get Form Definitions from Provider/Integration – In FireLight Admin, added a Label field to the Form List page. Label is a free-form string field that can be used to determine which forms to get definitions for and obtain item IDs from the form definition. It can also be used for specifying additional version/edition information for a form.

Global Data Items

  • [126685] Added a new FLI_ORGANIZATION_NAME to Default Data on application creation – Added a new FLI_ORGANIZATION_NAME to the default data on application creation. We already set FLI_ORGANIZATION_ID and just added the name as well.

Integration

  • [124555] Added the ability to call out to update the user when an application is created on behalf of another user – When an application is created on behalf of another user, the back office is called and asked to request a new 1228 for that user.

Other Actions Menu

  • [121773] Added new document type options – In FireLight on the Other Actions menu in Documents, added two new Document Type options (Select Document Type and Other). The user must now select a Document Type in the list.
  • [126438] Tied the Illustration document type in the upload document dialog to Additional Doc With Signature – In FireLight, “Illustration With Signature” document type is now labeled “Illustration” in the document upload dialog.

Quoting

  • [118204] New Quoting capability for Term Life products – Added a Quoting page to FireLight and a Quoting tab to FireLight Admin under Profiles. Admins use the Quoting tab in FireLight Admin to prepare and upload a rate file and add product and rate information for existing Term Life products. FireLight users use the Quoting page to see comparison quotes for Term Life products and they can begin a new application from the Quoting page in FireLight.

Signer Types

  • [119843] Added a new Additional Payor signer type – Added a new Additional Payor signer type in the Signature Box control, Signer Type list, in FireLight Admin. This allows separate signature controls payors for Applicant A and Applicant B in FireLight.
  • [126341] Added a new Additional Applicant role/signer type – Added a new Additional Applicant role, signer type. In FireLight Admin, the new role is added to the List Profile section in custom Branding.

Fixed Defects

  • [121462] There is no auto-formatting in the disconnected Console application – Added field mask implementation to the Console. This has been corrected.
  • [122105] Client Fill email request: Removed the radio buttons Client and Admin and replaced them with a checkbox, “Complete annotated areas only” on the Request Another Person to Complete Filling the Forms dialog in FireLight.
  • [126123] Corrected width, height and position of some controls – Some controls appeared to be either slightly smaller or nudged down in FireLight EApp. The proper offsets needed were determined and adjusted accordingly. No changes were made to existing clients.
  • [126348] Update FireLight user database from inbound 103 request – Updated the logic to update the user’s role code during the processing of the 103 data. This will update the user whenever the user information is sent.
  • [126565] Generic error appears if you ask a client to fill out an application – The error was corrected and no longer occurs when asking a client to fill out an application.
  • [126639] Review Queue deployment is not including email group and additional info fields – This has been corrected. The Review Queue deployment now includes the email group and additional information fields.
  • [126768] Submit Action is available if a separate session changes and unlocks an application – An agent in the field was able to submit an application with no signatures that had been unlocked in a separate session on a different machine. Validation was added to the Submit action to ensure the application is in a state to be submitted. If validation fails, an exception is thrown.
  • [126879] OverrideDataItems is only showing a subset of the copied items – This has been corrected. Data is now prepopulated for override dataitems.
  • [127061] Problem Opening apps in the same session once an app has been copied using FireLight Console – An error was being thrown on copied applications. One of the files for the application was not copied and it failed when accessing the app. This has been corrected. The file is now being copied and not causing errors.
  • [127453] Long Combo Box control text in FireLight Admin wraps into another option item in FireLight – This has been corrected. In the Site.css, made the line not wrap to keep the integrity of the combo box text in FireLight.
  • [127466] Save popup dialog on the form Designer in FireLight Admin needs formatting – In FireLight Admin, the Save popup dialog has been formatted so the “f” in the word “form” is capitalized and the “Save form changes?” text is centered over the “Yes/No” buttons.
  • [127628, 127691] On the Create a New Application page, the long product names are not wrapping – This has been corrected. The line now wraps on long product names on the Create a New Application page in FireLight.
