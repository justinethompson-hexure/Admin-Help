FireLight v2.39-p Release History

What's New in FireLight

The following enhancements are now available for all clients unless noted otherwise. See the "FireLight v2.39-p" Release History" topic in Admin Help for fixed defects and links to design documents and Admin Help topics.

New Functionality

Support Filtering of Report Data to Include Only Embedded Applications: DIST-5115

We added the ability to filter report data so that only embedded applications are included.

Capture Application Origin in New Data Item: EAES-5673

We created a new data item, “FLI_APP_ORIGIN,” to capture details of the environment in which an application was created or copied, whether FireLight UI, REST API, or Unknown. See FLI_APP_ORIGIN Data Item for more information.

Resolved Issues

Create SuranceBay Config File with Production Settings: CR-4168

We resolved an issue where no config file with SuranceBay Production settings existed in the prod config directories. This config file is now available.

Comm Netting Issue: CR-4182

We resolved an issue where an error was encountered when attempting to calculate commission netting. Commission netting can now be calculated as designed, without error.

Update Docusign Production appSettings: CR-4210

We resolved an issue where production settings for Docusign environments were incorrect in the appSettings.config file. These settings are now correct.

Once the App Status Is Marked as Complete, the User Can Remove the Uploaded Documents: EAES-6348

We resolved an issue where users were erroneously able to remove documents from completed applications. Now documents cannot be removed from completed applications.

Unable to Create Activity When User's Group Has Many Role Codes: EAES-6380

We resolved an issue where multiple role codes resulted in a database error when attempting to write the application audit. Application audits are now written without error.

Form That Is Excluded from Back Office Has the Wrong Visibility, Causing It to Be Included: EAES-6385

We resolved an issue where forms were mistakenly set as Carrier Only instead of Client Only, causing them to be included with applications. In this scenario, such forms are not included with applications.

System Not Honoring Signature System Selection: EAES-6458

We resolved an issue where selecting either Pre-Signature Review or Post-Signature Review did not work as expected. Selecting these checkboxes now works as designed.

BeforeOSJ Workflow for Pre-submit Review Queue Is Not Working: EAES-6511

We resolved an issue where the BeforeOSJ workflow did not work as expected. The workflow now works as designed.

Distributor Should Not Be Able to Modify Firm Organizations on Carrier Sales Agreement: PC-3641

We resolved an issue where distributors were erroneously able to modify firm organizations on carrier sales agreements. Distributors are no longer able to modify firm organizations.

Validation Messages Are Not Getting Cleared Correctly: PC-3666

We resolved an issue where validation messages continued to display after error correction. Validation messages now clear after errors are corrected.

Integration Provider Unable to Load Rebex Assembly Due to Version Update: PC-3672

We resolved an issue where some integration providers encountered errors after loading the Rebex component due to a version change. The Rebex component can now be loaded without error.

Error Loading Application Due to Duplicate traceFailedRequests Config File Entry: PC-3675

We resolved an issue where loading an application resulted in an error due to duplicated config file entries. Applications now load without errors.

Excessive setrowvisibility Rule Exceptions: PC-3681

We resolved an issue where an excessive number of setrowvisibility errors appeared in exception logs. The number of setrowvisibility errors is now within the anticipated threshold.

Loading of Pages Is Very Slow: UI-964

We resolved an issue in the Admin App where pages loaded slower than expected. Pages now load within acceptable limits.

Password Reset and Change Password Not Working Correctly: UI-2300

We resolved an issue where password reset and change did not work correctly. Password reset and change now work as designed.

 

***Please note that the JIRA ticket numbers are provided for reference. JIRA tickets are not accessible to users outside Hexure.***
