2.8.0.322 (July 22, 2016 UAT Release)

Enhancements

Review Queue

  • DR 142679 Functional Status for Apps in Review Queue – In FireLight Admin on the Review Queue page, added the ability to set priority status levels in the review queue script via rules. Font color has been updated to reflect priority status level (Normal = Green, Urgent = Red, Low = Blue). If priority status level is not set in the review queue script, the priority status in the Review Queue defaults to "Normal" in the green font color.

2.8.0.111 (May 6, 2016 Production Release)

Client Designated Enhancements

Birth Date Format

  • [116962 / 118569 / 117947] Birth Date format during signing ceremony – In a birth date input mask using the format 99/99/9999 was added to the signing ceremony for client verification, providing the format automatically for users. The input mask formatting was also added to the external client fill request and external signature request.

Form Designer

  • [84044 / 127721] Set the hide/show visibility of field controls based on business rules – New read only and visibility nodes (setvisibility, getvisibility, setvisibilityrange, setreadonly, getreadonly, setreadonlyrange) were added to the Rules Window in the Admin form designer. Fields that are set to read only is read only regardless of the rules ran on them. For rules to control read only, the field must be set to not read only. Fields appear and disappear based on the visibility flag.
  • [88157] Add support for Hyperlink label functionality for use within step 1 of the workflow – Added a Url field property to the Label control field properties in the Admin form designer. Enter the display text for the hyperlink in the Text field property and add the URL for the hyperlink in the new Url field property. In , the active hyperlink will display and be clickable during the edit application process, form acknowledgment, the eSignature process, and for any pdf generation of the application.
  • [117395] Keep a history of changes on Form Design in the Admin Tool – Added a Form History dialog on the Admin form design page. Use the dialog to view a history of form changes. Form history is updated when form changes are made and the form is saved.
  • [127480 / 127634 / 106035 / 125053 / 111891] Pop-ups – In Admin on the form Designer page, added the ability to add HTML in rules for pop-ups. In Admin on the Sales Agreements page, added support for HTML in the More Info field to display in the product info pop-up. HTML encoding is automatically generated behind the scenes upon saving the page.

Groups

  • [122279 / 127681] Building on User Sharing/Transfer – In Admin, added a new Allow Transfer Application group permission that allows a user to transfer an application to a new user by clicking Transfer Application on the Other Actions menu in . Also provides the ability to transfer an application to a user managed by the Integration Provider. Application transfer is only available before an application is completed. Basically, if an application can be unlocked, it can be transferred to another user.

Review Queue

  • [125234] Allow Reviewer to fill forms during review – Added the ability to add reviewer dashboard forms to the Review Queue process. In Admin on the Design page in Form Properties, set Protection to "Public" and in the Field Properties, set each form field to "Is read only" for each field where data is imported from the application (e.g., FLI_OWNER_NAME field). On the Packages page in Profiles, create a new "Reviewer Package" type, add "reviewer dashboard" forms into this package and set the forms to Required. Then add the package to applicable product sales agreements. When Review Queue users have required reviewer dashboard forms to be filled for an application and they click the Review button for the application, a Fill Reviewer Forms dialog appears. The reviewer clicks the Fill Reviewer Forms button on the dialog to open the required reviewer form and proceed with filling in the form and the application review.

Signing Ceremony / Organizations

  • [125434] Build functionality to notify all e-signing parties of opportunity timeout or opt-out – In Admin on the Organizations page in Profiles, added a new Expiration Warning Threshold field. Enter the number of days prior to the Requests Timeout field days to send warning emails to the client and agent that the electronic signature request is near expiration. In Admin on the Email page in Profiles, added four new email templates for client and agent expiration warnings and expiration notices. The new email templates are also added to the Branding and Localization pages in Admin.
  • [127589] Add Driver’s License Verification during the Sign Now Ceremony – Upon an organization's request to vendor, added the ability to allow agents to upload their client's driver license images during the Sign Now identity verification and attach the front and back driver license images as part of the application PDF. To implement this feature, contact your FireLight PM for additional work in the organization CSS that is required to enable this feature.
  • [127714] Signature pad font – Added the ability to use script signature fonts instead of the standard block font for the typed signature in the signing ceremony. In Admin on the Organizations page in Profiles, click the Signature Fonts "Click to Edit" link next to open the Signature Font Styles dialog, select the checkbox for each script Font Style to include for your Organization. In , the signer selects a script font in the Select Font list on the Capture Electronic Signature page.

Other Enhancements

Admin Help System

  • [127727] Create an integrated Admin help system – In Admin, added an About FireLight Admin panel on the Admin home page that includes a What's New section with featured updates, a FireLight Release Notes section with the release notes for the current release and a link to the online Admin Help, which contains the Admin user guide contents along with added content and features.

Branding

  • [137513] Update to branding – In the Completed Signatures list in Firelight, the font color for the signer types was changed from red to black.

Forms Designer

  • [127482] FireLight Admin: Tabbing Order enhancement – In Admin, added the ability to set tab order vertically.

  • [127509 / 121469] FireLight Admin: Moving multiple fields – In Admin, added the ability to move multiple fields at the same time using the mouse.

  • [127510] FireLight Admin: Display all tag names – In Admin, added the ability to display all tag names on a page.

  • [127594] Firelight Admin: Copy Fields and Rules Update – In the Admin form designer Copy Fields or Rules feature, added the ability to search for forms by Name, Number, or Edition in the Copy From list.
  • [128373] FireLight Admin: Select page option in Copy Fields and Rules window – In the Admin form designer Copy Fields or Rules feature, added the ability to filter fields by name or page number in the Search field.

Forms Lists

  • [127581] Enhance usability and create uniformity on all Form lists in Admin – In Admin, updated all form lists on the Forms, Designer, Packages, Reports and Deploy pages to include the same amount of form data (Form Number, Form Name, Edition).

Notifications

  • [127011] Provide a carrier controlled message that a form has been updated - both before and after lock down – In FireLight, if any changes are made in a package and an application is in FORM ENTRY status, the system displays a message to the user regarding the changes. If the application is in LOCKED status, the system displays a message to the user regarding the changes and the necessity to unlock the application to get latest changes.

Purge Policy

  • [131123 / 137556] Changes to the purge policy and reports – Added to the purge process the deletion of purged applications. For applications that have a purge date and are not in Complete status, permanently delete the application when the last update is two times the Purge Uncompleted Days policy. Modified the policy to treat submitted applications the same as completed applications by including them in the Purge Completed Days policy rather than the Purge Uncompleted Days policy. Also modified the Pending Purge Apps report and added a new Pending Deletion report. The new Pending Deletion report lists completed/transmitted applications that is reaching the delete from database date (two times the Purge Completed Days policy). To get the new Pending Deletion Report, you need to submit a request to your Strategic Account Manager.

UAT Environment

  • [127726] FireLight Admin: Additional Dev development environment that can deployed to staging as an enhancement to version control. (Merged with 127642 FireLight Admin enhancement: Multiple versions / branching feature for product deployments) – Added a new UAT environment that is used for client testing before deployment to staging and production.
