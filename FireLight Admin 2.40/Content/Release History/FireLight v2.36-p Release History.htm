FireLight v2.36-p Release History

New Functionality

Disallow Copying an Activity with Expired Sales Agreement: DIST-3730

Currently, a user cannot proceed with data entry of an activity with an expired/invalid sales agreement. As a validation, it will be restricted to creating a copy of an activity with invalid sales agreement.

Impact

FireLight App: Copy option will be unavailable on below areas for the activities whose sales agreement in invalid.

• Other Action menu

• All Activities page

Make CSV the Default Format When Generating Admin Reports: DIST-3679

With recent enhancements to admin reports, changes have been made to the CSV format due to width/column limitation in the PDF format of reports. For this reason, to generate the entire report, user must select CSV from the dropdown because the default option is currently PDF.

With these changes, the default selected format will be for CSV to generate admin reports.

The exception to this change will be Application Audit and Transaction Summary reports.

Impact

FL Admin: Display CSV as the default format on admin reports page under the Utilities menu.

New Detailed Activity Reports are Available with the Report Scheduler Functionality: DIST-3807 and DIST- 2341

The following activity reports have been to the Report Scheduler. They will be listed under detailed reports:

  • Application Bypass RQ: List activities that have bypassed both pre and post submit reviews

  • Apps Submitted to Suitability Review: List activities that were submitted to post submit review

  • Apps Suitability Review Received: Review status received for activities submitted to post submit review

Prevent System from Scrolling All the Way to the Top When User Launches SireLC Can Sell: CR-2355

When a user browses the Product Selection Page in FireLight and selects a product with SureLC Parity, system now returns to the same row, instead of scrolling to the very top of the Product Selection page. We have improved the user experience by maintaining the cursor on the selected product.

Include MIB Response as Part of the Audit Report: CR-3254

When user calls Evidence Framework, FireLight is now able to get the final MIB Response and record it in the Audit Report for the primary insured that is associated to Case ID. Work is in progress to add the ability to include the MIB report for additional insureds.

Add New DataItem to Determine When MIB Is Complete: CR-3435

FireLight now indicates when an MIB is complete.

Disable Print PDF for External Signers: EAES-4825

A new checkbox has been added to signature controls. The checkbox is called "Disable Print PDF" When true, it will prevent the signer type assigned to that specific control from viewing the application as a PDF as well as printing during the signing ceremony. This only applies for external users. This only applies to Sign Only.

Implement Ability to Cancel Paramedic Appointment: CR-2853

We have implemented the ability for a user to cancel either an APPS or ExamOne appointment after it has been scheduled. The ability to cancel the appointment is set up using a custom action with the parameter “ExamOneCancelAppointment” or “APPSCancelAppointment.” A cancellation message will be presented.

Include the Paramed Transaction in the Audit History: CR-2923

When cancellation of an appointment is successfully executed the Audit history will display the following text: "Paramed [ExamOne/Apps] has been successfully cancelled" For failing cancellations the audit report will display the message that is returned from the ParaMed service.

Resolved Issues

LexisNexis Emailage in Signatures - Service Call in Signature Workflow Is Not Consistent for REACT and Legacy Signing System: CR-2302

We resolved an issue where the use of the field properties “Use LexisNexis” and “Use LexisNexis Emailage” for the Signature Control Type were inconsistent between the Legacy and REACT signing systems. The signing systems now work as designed.

Redtail Mapping Issue with "Trust Name" in Redtail: CR-3062

We resolved an issue where "Trust Name" in Redtail mapped to "Owner Last Name." "Trust Name" now maps to "Trust Name" as designed.

Paramed ExamOne Cancellation Response Is Not Mapping: CR-3201

We resolved an issue where the ExamOne Paramed Cancellation Response was not invoked after the Cancel Appointment button was clicked. The Cancel Appointment button is now working as designed.

When "Required" Property Is Checked for the Radio Button and Checkbox Control, the Application Is Not Displaying the Required Message: DIST-3707

We resolved an issue where when the Required property was checked, the required message did not appear. The Required property now works as expected.

Copy Form Field Functionality in Wizard Designer Stuck on Loading: DIST-3720

We resolved an issue where the Copy Form field in the Wizard Designer was taking an excessively long time to load. The Copy Form field now works as designed.

Hide Allow ApplicationAudit Report Flag from Group Page: DIST-3727

We resolved an issue where the group setting Allow Application Audit Report was errantly available when selecting Org Settings > Groups page in the Admin app. This groups setting is now no longer available when make this selection.

Unable to Submit Application When External Login Username Contains a Vertical Bar | Character: DIST-3814

We resolved an issue where audit reports failed to be generated when the external login user name contained a vertical bar character (|). Audit reports are now generated without issue regardless of whether a vertical bar character is included in the external login user name.

If Group Setting "Copy with Changes" Is Enabled for the User, They Try to Copy the Expired Product from All Activities Getting Server Error: DIST-3816

We resolved an issue where an error resulted from clicking the Copy button for an expired product on the All Activities page. The Copy button on the All Activities page now works as designed.

Clear Audit and Delivery History When Adding a New Schedule: DIST-3909

We resolved an issue where when adding a new schedule the Schedule History and Delivery History tabs displayed history information about the previously selected schedule. The tabs are now working as designed.

Illustration Signature Came in on Wrong Page: EAES-3269

We resolved an issue where the Illustration signature appeared on the wrong page. The signature now appears on the correct page.

Data Item Values and Form Include State Are Not Consistent with Rules That Set Form State (Sporadic): EAES-4126

We resolved an issue where data item values and rules were inconsistent.

Applications Are Not Getting Submitted after Data Entry and Signatures: EAES-4733

We resolved an issue where applications were not being submitted after data entry and signatures. Application submission is now working as expected.

When a Document Upload Using a Template Is Required to Complete Data Entry the “Data Entry Met” Popup Does Not Always Appear as Expected: EAES-4975

We resolved an issue where the “Data Entry Met” popup was not presented until the user refreshed the screen after a required upload. This is now working as expected and the popup is presented without refreshing the screen.

 

***Please note that the JIRA ticket numbers are provided for reference. JIRA tickets are not accessible to users outside Hexure.***
