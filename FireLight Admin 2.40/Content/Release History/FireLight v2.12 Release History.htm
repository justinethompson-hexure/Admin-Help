2.12.0.394 September 14, 2018 Minor Production Release

Enhancements

1035YellowPages

  • CMR 185303 Updated FireLight to pull back the business street address from 1035YellowPages.com as default instead of pulling back the mailing address as default. If the street address is not found, FireLight pulls back the first address provided.

Integration

  • CMR 185353 Enhanced FireLight base to pass dataitems to integration on integration initialization. Providers that Clear Mapped Data will have to remove it to prevent losing passed in dataitems.

Product Types

  • CMR 181617 Added a new Pre Sale Suitability Review product type to FireLight.

Fixed Defects

Admin

  • DR 182381 Delete Rule tooltip did not disappear until navigating away from the page. Also, the message asking if the user wanted to overwrite changes was appearing when only one user was in the rule set. User had to cancel Save to close the message. This was fixed.
  • DR 182565 Saving of rule sets was taking too long. This was fixed by making a change in the application firewall and by fixing the 'user has changed rule set' error by disabling the ability to click Save while the system is in process of saving a rule set.
  • DR 184056 Rules were not being reset for the search option in packages until either the package rules were saved or the search cleared. This was fixed.
  • DR 184958 Deployments page did not clearly state the timezone that is used for scheduled deployments. This was fixed by changing the name of the "Schedule Deployment at" box to Schedule Deployment at (Mountain Time).
  • DR 185197 If a custom CRM was configured, it was not possible for the organization to also use a Redtail CRM. This was fixed. Enter a CRM Parameter of Redtail to specify Redtail CRM. Patched UAT/Staging/Production 7/17/2018.
  • DR 186328 Unable to access rule sets to set approved states. Audit records were being written for all rules in a rule set regardless of whether they were changed, which was causing a timeout while loading the rule set. This was fixed by adding a check to ensure audit records are only written for rules that have changed.
  • DR 186715 A Standard email template was being sent when a Branded email template should have been sent. This was fixed.

Applications

  • DR 181970 New applications incorrectly had an override date defined when they should not. This was fixed. Patched Production 8/7/18.
  • DR 184640 Input-only pdfs were showing up in the external signature process, and should not have been. This was fixed by adding a check to ensure document visibility was taken into account for external signature pdf viewing.
  • DR 184835 An error occurred when entering the signing ceremony if the application had e-sign declined. This was fixed.
  • DR 185343 Product Type drop-down list on the new application view should be sorted alphabetically. This was fixed.

Check Boxes

  • DR 184204 Check boxes were not allowing users to select multiple checkboxes and were functioning like radio buttons. This was fixed by updating the system to check for an empty group name in a checkbox. If the group name is empty, the system only updates that checkbox. Otherwise, the system updates all items in the group.

Client Sign Request

  • DR 184640 Input-only PDFs were showing up in external signature process, and should not have been. This was fixed by adding in a check to make sure document visibility was taken into account for external signature PDF viewing.

  • DR 185368 Requests dialog was showing the signature request was complete when it was not complete. This was fixed. Now the dialog will not show the signature request is complete until all document sets have been signed.

Combo Boxes

  • DR 183346 Rolled back the fix for this as it introduced a new bug. Initial default values of combo boxes are being set to an empty value when the dropdown options do not contain an empty option. The previous resolution was to set the dropdown value to the default option (1st option) if the value is empty. Because the combo control is updated with a value, but the data item is still blank, the combo displays a value, but a required rule on the field generates a validation message, since the data item is empty. This is working as expected and is not a bug.

CRM and Commission Lookups

  • DR 187121 CRM lookups and commission options selection were not populating the information in applications. A bug was introduced in the UAT release on 8/17/2018. This was fixed. Patched UAT 9/6/2018.

Document Upload

  • DR 185267 When uploading a document template, the selected document type in the combo box did not always match the internal document type of the document being uploaded. This was fixed. Patched UAT/Staging/Production 7/17/2018.

DTCC Codes

  • DR 185200 Updated to DTCC September 2018 spreadsheet and added new DTCC Codes (RI - RIA and AP - Authorized Party) to FireLight base.

FireLight Console

  • DR 182763 For a checked out application on the Console, client information such as SS# and name was not pre-filled on the applications. This was fixed by adding new rule nodes to the Console and having the Console update the dataitem of current form location on page update.
  • DR 184442 Console was repeatedly bringing up e-signature forms to sign. The document set was repeating because it did not know it was complete. This was fixed.

Needs Determination

  • DR 176881 After completing a Needs Determination, Firelight was preventing users from selecting another product after navigating to a Product Type that had no products in it. This was fixed.
  • DR 186387 In the application workflow, the Needs Determination PDF file was not being included in the application Documents list. This was fixed.

Signatures

  • DR 184427 On the Client Identification Verification page, the validation message for an invalid email address appeared as "The field ToEmail is invalid" instead of the expected "Email entered is invalid" message. This was fixed.

Wizards

  • DR 185111 In wizard fields that had diset rules, fields would not repopulate with the default value when the user entered blank in the field. The user had to go to another page and return to the field for the field to repopulate with the default value. This was fixed.
  • DR 186850 Conditional forms with enabled signature controls were not added to the signing ceremony for wizard products. This was fixed.
  • DR 186875 Validation was not getting triggered on required fields when an activity was opened and when a new page appeared. This was fixed. Required fields now display with a red background.

2.12.0.223 July 13, 2018 Production Release

Enhancements

Applications

  • CMR 147808 Added the ability to automatically submit after the signatures are complete for an application, and optionally notify the agent by email, when the Pre-Submit (Manual) review step is either not available, skipped or declined. In FireLight Admin in Profiles/Groups/App tab, added the "Enable Auto Submit" and "Enable Auto Submit Email Notification" role code options. In Profiles/Email, added a new Auto Submit Email template. Added a new FLI_AUTO_SUBMIT data item that can be used in a rule to override functionality of the Enable Auto Submit role code option.
  • CMR 182071 FireLight now prevents users from deleting applications that have been submitted or complete.

Comm100 Chat Service

  • CMR 179243 Added the ability to integrate with the Comm100 chat service across FireLight for customer support. The Comm100 chat service portal allows for real time communication and email between the User and Agent when help is needed. FireLight clients must contract directly with Comm100 to use their chat service. See Comm100 Chat Service Integration in the Admin Help for setup information.

FireLight Access

  • CMR 181602 Added config settings and signing options to MDE Provider and config, which allow MDE providers to encrypt and sign the XML Payload for FireLight Access.

Groups

  • CMR 150199 In FireLight Admin, added a Disable View Review Queue option on the Admin tab of the Groups page in Profiles. When the option is selected, the Review Queue option under Profiles on the Admin home page and the Review Queue option under Profiles on the navigation bar will not be visible. The user will not be able to edit the review queues or the review queue script in FireLight.

Integration - Remote Signature Request

  • CMR 184096 Created a new web service endpoint that allows an organization to initiate a remote signature request when the 103 comes in with no interaction from the agent. Patched UAT.

Product Type

  • CMR 182819 Added product type Hospital Indemnity to the Health product category/line of business in FireLight.

Reports

  • Added the following ten Activity reports to FireLight:

    • Carrier Transactions by Product Type / Distributor Transactions by Product Type
    • Carrier Transactions by Distributor / Distributor Transactions by Carrier
    • Carrier Transactions by Distributor by Product Type / Distributor Transactions by Carrier by Product Type

    • Number of Registered Users by Organization ID (for Carriers and Distributors)
    • Number of Users with Submissions by Organization (for Carriers and Distributors)
    • Number of users with Only 1 Submission by Organization (for Carriers and Distributors)

Review Queue

  • CMR 181385 Added sort order for review queues. Review queues now display in sequential order.

Salesforce

  • Added detailed setup instructions for adding a FireLight Integration connected app in Salesforce. See the Salesforce Integration topic in the Admin Help and the "Salesforce Integration Setup" topic in the user guide.

Fixed Defects

Admin

  • DR 181472 In wizards, rules were not running on an invalid or unformatted date. This was fixed. Modified the date picker control to send all values when the user navigates off the control, even if the value is not a valid date.
  • DR 182176 In the wizard designer, setcomboitems rules did not work (they would not replace or append). This was fixed.
  • DR 182382 Saving of Mappings was not working. This was fixed. Applied database change to QE, UAT, Staging, and Production 5/15/2018.
  • DR 182397 Mappings were not showing up for deployment after saving. This was fixed by updating the timestamp when saving or deleting mappings. Also updated button captions to be plural to indicate that all mappings is saved/deleted.
  • DR 182487 In the wizard designer, wizard pages were getting deleted when adding new items to previous pages. This was fixed by modifying the page selection dropdown change handler. Patched UAT 5/17/18.
  • DR 182612 The setpackagestatebyname rule caused an exception if a form had the "Will Be Replaced By" property set. The real problem is the logic that loads package names for the application forms does not take into account forms that are replaced by another form, leading to the exception due to a null package name. A workaround is to use "This Form Expires On" property for the form to be replaced and use "This Form Begins On" property for the new form instead of using the "Will Be Replaced By" property.
  • DR 182769 Include source organization and environment in deployment notification emails. This was completed.
  • DR 182809 In the wizard designer, if a date picker or dropdown was placed next to a text box, the fields did not align properly. This was fixed by changing the CSS class. Patched UAT 5/25/18.
  • DR 183169 In Profiles/Email, the Branding Firm option was not working correctly when there was no locale. This was fixed.
  • DR 183771 In the Rules window for the form designer and the Rules window for the wizard designer, the Yes and No buttons were in reverse order in the save rule set confirmation dialog. This was fixed. Now the Yes button is on the left and the No button is on the right for both save rule set confirmation dialoges.
  • DR 184746 Distributor organizations could not save sales agreements that had a Start and/or End date because the Time Zone, Start, End and Replaced Product fields were not being posted to the server. This was fixed.

Activities

  • DR 181599 When users with Role Codes that had the Disable Save Activities option selected in Groups tried to save an activity when data entry reached 100%, the standard Error page did not redirect to the Error view. This was fixed by disabling all actions that result in saving the application (continue, print, custom action, transfer, etc.).

All Activities

  • DR 180209 All Activities page did not have the ability to filter by Locale. A new Locale dropdown list control was added above the grid on the All Activities page. The Locale list is available only when activities exist for more than one locale.
  • DR 182962 The rule editor dialog would randomly close without providing any error information. Fixed server-side error handling within the wizard designer and rule editor so the user is provided with the proper information and dialogs are not closed improperly.
  • DR 183969 Filter by Activity type, Status and Sort were missing in UAT. This was fixed in the code to prevent this and added a migration script to prevent this from happening in Staging and Production.
  • DR 184591 Search results in All Activities were not retained past the first page of results. When the user went to the next page, the search results were lost. This was fixed.

Applications

  • DR 181580 After declining signatures and clicking Continue to submit the application, the user would be returned to the edit application view where the Continue button would still be enabled. The application was being rolled back to locked status due to the conflicting settings of the distributor's environment and the carrier's for the Allow Submit on E-Sign Decline option. This was fixed. The app will now be set to submitted rather than rolled back to locked in the case where the application is transmitted through the carrier when their Allow Submit on E-Sign Decline option is turned on and the distributor's setting is off.
  • DR 182243 Client data for delayed 103 was being saved off, but was not populating the form fields. This was fixed by adding the missing session ID at application creation to allow the delayed 103 data to populate the form fields. Patched Production 5/16/2018.
  • DR 182460 When Enable Residence State option was enabled in the Sales Agreement and the user chose to cancel or change Jurisdiction or Product Type, FireLight did not display any products.
  • DR 183382 An error locked an application if a form contained a Calendar control with a Date Format and the date value was invalid. This was fixed by modifying the code to ignore errors parsing the date.
  • DR 183765 Conditional packages not always shown on initial creation of application; after adding a package containing a required template document, upload of the document within the same session was not required. This was fixed.

Check Boxes

  • DR 184204 Check boxes were not allowing users to select multiple checkboxes and were functioning like radio buttons. This was fixed by updating the system to check for an empty group name in a checkbox. If the group name is empty, the system only updates that checkbox. Otherwise, the system updates all items in the group.

  • DR 182247 Client Fill and Sign parties were not populating if the application had not been saved. This was fixed. Patched UAT, Staging, and Production 5/12/18.

  • DR 183346 Initial default values of drop-downs were being set to an empty value when the drop-down options did not contain an empty option. Set the drop-down value to the default option (1st option) if the value is empty. Patched UAT 7/11/2018.

  • DR 184412 Read only combo fields did not work correctly in Internet Explorer and Edge web browsers. This was fixed.

  • DR 184872 Disabled combo box and calendar control not re-enabled if disabled and then made read-only. This was fixed. Patched UAT 7/11/2018.

2.12.0.162 May 11, 2018 Production Release

Enhancements

Account Opening

  • CMR 170065 Added a new Account Opening pre-sale transaction type. Account Opening is a submitted transaction separate from other transaction types for reporting and downstream processing. Account Opening has its own Post Over transaction type to send users directly into the Account Opening transaction or users can click Account Opening on the FireLight Home page. The initial transaction screen includes the Jurisdiction list and a Custodian list if more than one Custodian exists.

All Activities

  • CMR 163505 Added the ability for users to retain the last search criteria used during the session. When searching for a particular activity in All Activities, the user can go into an activity to view the details and then come back to the search results. FireLight retains the search criteria in All Activities during the session until the user clears the search criteria.
  • CMR 169972 Added a search feature for the User List (formerly Owner List) on the All Activities page. This allows a user who has visibility into multiple agents’ applications, through either User Share or Group code access, to search for a specific user name.

Allow Submit on E-Sign Decline

  • CMR 174621 Added the ability to allow the carrier to receive submission even if the distributor has Allow Submit on E-Sign Decline feature disabled in Organizations. Implemented as another workflow type, SubmitCarrierOnly, which will call the carrier provider only when the user completes an application if: 1) in a distributor environment, 2) the distributor has the Allow Submit on E-Sign Decline checkbox cleared (disabled) in Organizations, and 3) the carrier has Submit on E-Sign Decline selected (enabled) in Organizations.

API Layer to Call FireLight to Run Illustrations

  • CMR 172089 Added a flexible API layer to FireLight. The API returns calculation results based on the provided request values. This API forwards the request to the ForeSight Illustration Services web service to provide the calculation results. Part of what FireLight adds is a call to the Rules engine to validate the inputs before ForeSight Illustration Services is called.

Auto Save

  • CMR 170040 Added auto save functionality so when an activity reaches 100%, it is automatically saved. This enables other actions (e.g., print options) immediately after an activity reaches 100%, without the user having to first click the Save button. In FireLight Admin> Profiles/Groups/App tab, select Enable Auto Save to enable the Auto Save feature for each applicable role code.

Benchmark Tool

  • CMR 164112 Added a "Data Item Properties (P)" test condition to compare data item properties (value, visible, enabled, read-only) for pass/fail of benchmarks. See the FireLight Platform User Guide for Benchmark tool instructions.
  • CMR 170466 Added a "Form Name/Number (N)" test condition to compare form name and number that are in the benchmark so users can quickly identify incorrect forms.
  • CMR 170491 Added the ability to update multiple benchmarks using a new Accept button. For example, after a new release, you can select multiple failed benchmarks and click the Accept button to accept the benchmark run results as the new benchmarks.

Client Fill and Client Fill & Sign Requests

  • CMR 170556 Added the ability for clients to attach documents during the Request Client to Fill App and Request Client to Fill & Sign requests. Clients can click the “Documents” link on the top navigation bar in “Client Fill” and “Client Fill and Sign” pages, which uses the existing FireLight functionality for uploading outside documentation, including the upload dialog buttons to quick-launch an upload, template functionality, notification that a template is required, document type association with a template, signature tags on the template, inclusion in the percent complete if a template is required, and to remove attached documents.

CSS Enhancements

  • CMR 177593 Added the ability to hide e-signature tags during form entry.
  • CMR 177601 Added the ability to hide e-signature tags during the signing ceremony.
  • CMR 177611 Added the ability to add branding and customize the text on the session expired page.

Deploy Mappings

  • CMR 170107 Added the ability to deploy mappings to other environments via a new Mapping Configuration synchronization scope on the Deploy page/Deployment tab in FireLight Admin.

Designer (Forms and Wizards)

  • CMR 150069 In FireLight Admin, added the ability to add a new rule for multiple fields in the forms Designer. Because users need to select multiple fields in a form, this feature is available only in the Designer/Form Designer page and its associated Rules Window. It is not available in the Rule Sets page in Profiles. This feature is for new rules only and cannot be used for updating rules.
  • CMR 170106 In FireLight Admin, enhanced checkbox controls to support export values just like a radio button.

Direct Login Clients

  • CMR 157909 Added the ability for direct login client users to reset their own password for the FireLight application and FireLight Admin.
  • CMR 170214 In FireLight Admin, added a setting on the Organizations page in Profiles that requires Two-Factor authentication before allowing direct login access to FireLight. See the "Two-Factor Authentication (Direct Login Users Only)" topic in the FireLight Platform User Guide and 2-Factor Authentication on the Organizations page.

DTCC

  • CMR 178903 Extended the FireLight base class to allow the providers to set the new options reflected in the latest DTCC record layout from March, 2018.

Edit Masks

  • CMR 165125 In FireLight Admin, added a numeric currency Edit Mask for currency or dollar amount on the Organizations page in Profiles. You can apply the mask to Text Box fields in the forms Designer.

Email – Sub Firm Branding

  • CMR 180598 Added the ability to brand the email templates for sub firms on the Email page. To add branding for sub firm emails, contact your Strategic Account Manager to have the FireLight team create the list of Branding Firms for you. Once that list is created, you can manage the sub firm emails on your own by selecting an email template and then selecting a sub firm in the Branding Firm list on the Email page. NOTE: Organizations that currently have Branding Firms have had their Branding Firm list created in their environments.

FireLight Access

  • CMR 180032 FireLight Access distributors were generating duplicate policy numbers as their respective carriers. This was fixed.
  • CMR 181149 Modified the CSV import to update existing users.

FireLight Illustration

  • Added the optional FireLight Illustration feature. FireLight Illustration adds basic illustration components (user interface wizards, rule sets, illustration engine, and output) that provide the Guided Illustration and Illustration activities in FireLight. FireLight Illustration currently uses the ForeSight® illustration engine to provide the calculations and output for illustrations. (Hosting of third-party engines in FireLight Illustration is available in a future release.) The FireLight Illustration feature supports only wizards (no forms) for data entry. Contact your Strategic Account Manager for detailed information. See FireLight Illustration in Features for an overview.

FireLight Statuses

  • CMR 169970 Updated application statuses to reflect the true status of the application order from a client and agent perspective in Recent Activity, All Activities, History, and the Audit trail. See FireLight Display Status Workflows.

Global Data Item: FLI_REQUIREMANUALREVIEW

  • CMR 173296 When the organization setting for Allow E-Approval (manual PreSubmit review) was selected and the FLI_REQUIREMANUALREVIEW global data item was set to "false" and the user clicked the Continue button, the user was still seeing the sub menu with the "Request Review" and "Submit" options. This was updated. Now when the organization setting for Allow E-Approval is selected and the FLI_REQUIREMANUALREVIEW data item is set to "false," the system will not show the Review step or the sub menu with the "Request Review" and "Submit" options. When the organization setting for Allow E-Approval is selected and FLI_REQUIREMANUALREVIEW is set to "true," FireLight takes the user directly to the PreSubmit review process and does not show the sub menu with the "Request Review" and "Submit" options.

Groups

  • In FireLight Admin on the Profiles/Groups page, several options were renamed to match the name change from "application" to "activity" in FireLight.
  • CMR 177603 In Profiles/Groups on the App tab, added a new Disable Log Off option. Select the Disable Log Off checkbox to hide/remove the Log Off option on the top navigation bar in FireLight.
  • CMR 177604 In Profiles/Groups on the App tab, added a new Disable Show Annotations option. Select the Disable Show Annotations checkbox to disable the Show Annotations feature on the Other Actions menu in FireLight.
  • CMR 177609 In Profiles/Groups on the App tab, added a new Enable Custom Submit Page option that enables a custom submit page by role code. When enabled, the client is directed to a custom page after submission that can be branded by organization.

Home Page Styling

  • Beginning with FireLight v2.12, we updated the styling for the Home page because we needed to adjust the layout of the Home page to support all of the new activities. Based on feedback from clients, we made some additional changes to the Home page so that if you enable 3 activities or less the Home buttons will stack vertically like they did in v2.11. Once you exceed 3 activities, the Home buttons will shift to a two-buttons-per-row layout that you see today in v2.12.

    You do also have the ability to customize the icons on the home page. We now support the usage of Font Awesome but can also include custom images if you had custom images defined in 2.11. More details about font awesome can be found on their website, https://fontawesome.com/v4.7.0/icons/

Localization

Added the ability for an agent to select from multiple languages/locales in FireLight in which to select products and view lists, labels and messages in the localized language for an activity. Contact your Strategic Account Manager to initiate the localization process for your organization, including translation updates, database updates, and possible Provider changes. See Localization in the Admin Help or Admin Guide for setup instructions.

Notifications

  • CMR 157022 In FireLight Admin, added the ability to send an additional email notification to each organization receiving the deployment. In addition, added support to enter multiple email addresses for each organization separated by a semicolon in the Deployment Notification Email field on the Organizations page in Profiles. If distributors add one or more deployment notification email addresses in this field, notifications of Distributor configurations that any of their carriers deploy will also be sent to the distributor.
  • CMR 163150 In FireLight application, added an 'X' to validation messages so users can close a validation message to view the field beneath the message.
  • CMR 170425 In FireLight Admin, added an automatic notification if an unlocked Rule Set is in use by another user and has been saved by that user since they began their work. The notification asks the second user if they want to overwrite the first user's saved changes, discard their own changes, or cancel their own changes.
  • CMR 170426 In FireLight Admin, added an automatic notification if an unlocked Form is in use by another user and has been saved by that user since they began their work. The notification asks the second user if they want to overwrite the first user's saved changes, discard their own changes, or cancel their own changes.

Product Type

  • CMR 180645 Added the Indexed Variable Annuity product type to FireLight base.

Required Templates

  • CMR 170776 FireLight now includes required templates in the application % complete indicator and displays an error if the required template form is missing by highlighting the template form in red font in the Open form list. Clicking the missing template form in the form list opens the template warning message with links to complete the requirement (upload the missing form).

Reports

  • CMR 170244 Added a new Admin report for abandoned activity. Abandoned Activity Report lists the applications that are not submitted where the last updated date is X days. Report includes Application name, Policy Number, App Created Date, Carrier, Product, Product Type, Agent ID, Agent Role Code, Current Status of App, Last Updated Date and Time passed from today to last updated (days).
  • CMR 170245 Added a new Admin report for how the client signed (Remote, Sign Now) and indicates whether Client Fill or Client Fill & Sign was used. Report includes Carrier, Distributor, Agent ID, Agent Role code, App ID, Number of signer roles on the application, Client role type, Signing session ID, Signer, Signer multiple roles, Signing Type, Client Fill (Y/N), and Client Fill and Sign (Y/N).

Restrict Size of Attachments

  • CMR 170075 To reduce transmission errors caused by users uploading extremely large attachment documents, added a "Maximum Upload File Size" setting in FireLight Admin on the Organizations page that allows organizations to set a maximum file size limit on documents uploaded by their users. Users can upload files from the Other Actions menu/Documents, from a Button control with Action Type set to Custom File Upload, or from attaching documents in the Review Queue or from Client Fill and Client Fill and Sign requests. If a carrier and distributor both have a maximum upload file size set, FireLight will use the most restrictive setting.

Review Queue

  • CMR 170326 Enhanced Reviewer forms functionality in the post-submission review queue workflows to allow the reviewer to manage the reviewer only forms and access them when they are needed. In FireLight Admin, you can now set reviewer forms to optional or required. In FireLight Admin/Profiles/Packages page, added Reviewer Optional in the Package Type list for the optional reviewer forms package. Only one type of reviewer-only package is available per Sales Agreement—a Sales Agreement cannot have both a ‘Reviewer’ (required) type package and a ‘Reviewer Optional’ type package. If a Sales Agreement contains this combination, an error message will display when the Save button is clicked and the Sales Agreement will not be saved. In FireLight Admin on the Profiles/Sales Agreement page, the Package Duty list is hidden on all Reviewer and Reviewer Optional packages and those packages default to Required. Additionally, the Review button can now be branded through the CSS to be named in a manner that the firm will understand (e.g., "Lock" instead of "Review"). Contact your Strategic Account Manager if you would like to rename the review button.

Salesforce Integration

  • CMR 180269 Added integration with Salesforce CRM that allows data to be pulled from Salesforce into FireLight using a custom action button on a form. For setup instructions, see Salesforce Integration in Features.

Signer Types

  • CMR 178970 Made adding signer types configurable without a code change. Now, only a SignerType list profile change is needed to add a signer type. There are no changes from a user interface perspective for existing signer types.

Wizard User Interface

  • CMR 172087 Enhanced FireLight Admin to include a Designer tool for Wizards. Carriers and distributors can build their own user interface with reflexive questions in a wizard presentation layer, create rule sets for the wizards, and assign them to a product, package, and selling agreement.
