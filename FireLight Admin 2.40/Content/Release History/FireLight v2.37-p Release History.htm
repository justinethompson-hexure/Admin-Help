FireLight v2.37-p Release History

New Functionality

Removal of Milliman Language from the Pop-Up Message: CR-3742

We updated a pop-up message to remove a reference to Milliman. The message now reads "Underwriting Decision Received."

Support for Multi-Level Nodes in DPFA File Needs to Be Accounted for in Comm Netting: CR-3746

We updated DPFA file support to account for multi-level nodes/axes, reflecting different rates.

Up to Ten DPFA Files Can Be Uploaded at a Time: CR-3786

We updated DPFA file uploading to allow up to but no more than 10 DPFA files to be uploaded at one time.

Verify Customization Buttons in Magnum Interview Display Component: CR-3803

We updated Lifestyle Questionnaire buttons to be labeled “Back,” “Save,” and “Continue.”

Enhancements to Pop-Up Messages and Button Labels in Milliman and Magnum Integrations: CR-3838

We provided the client with the ability to modify the default pop-up message in Milliman integration, and default button names now appear.

Create a New Report Category as "FTP Only" for Reports That Have PII Data: DIST-3826

We have added a new indication with single asterisk "*" in the report dropdown and at the bottom of the page for reports containing PII data and that can be scheduled/delivered via the SFTP method.

Modify User Detail Admin Report: DIST-4283

The current admin report "User Detail" has been modified to display the columns below to make the report usable to requests for lists of users with active accounts who have access to FireLight:

  • User Creation Date

  • Last Log On Date

  • Disabled

Impact

  • FL Admin > Utilities > Reports > User Detail (report)

Ability to Identify SFTP Use Only Reports on Report Scheduler: DIST-4477

Recently a few activity reports have been added that can be delivered only via SFTP method, since these might contain PII data. To identify such reports when scheduling and limit them to be only delivered via SFTP method, these reports will be suffixed with an asterisk "*" and with a note displayed mentioning that these reports are SFTP Use Only Reports.

Impact

  • FL Admin > Utilities > Report Scheduler > Schedule Detail (tab) > Report (drop-down)

Finish Legacy Annuity Quoter Code Cleanup: EAES-5125

We have fully removed legacy Annuity Quoter code from FireLight.

Make the ALQ Quoter Endpoint Configurable by Firelight Environment: EAES-5162

All Firelight environments can now be configured to point to IXN/ALQ endpoints independently of other environments.

Make the AAQ Quoter Endpoint Configurable by FireLight Environment: EAES-5202

All Firelight environments can now be configured to point to IXN/AAQ endpoints independently of other environments.

Reduce Clicks Required to Create a New Quote in FireLight: EAES-5215

We streamlined the process for users to navigate from the FireLight App home page directly to the ALQ/AAQ quoter page.

During SSO, Fetch FirmRegistrationNum from 1228 Data and Prefix on Role Code: PC-3222

We now allow for dynamic creation of the User Role code when SSOing into the system by combining the Firm ID and the passed-in User Role code.

Resolved Issues

DPfA File Name Not Updated When Replacing or Reuploading a DPfA File:CR-3745

We resolved an issue where the file name was not being correctly updated when the user indicated that the file was a replacement of an existing file.

Forms and Wizard Are Selected by Default When a User Searches in All Forms/Wizards or Forms/Wizards in Package: DIST-3956

We resolved an issue where, on the Forms/Wizards in Package tab, forms and wizards were erroneously highlighted and selected by default. On this tab forms and wizards are no longer highlighted and selected by default.

Pasted Values Are Not Displaying in Wizard and Form History: DIST-4184

We resolved an issue where attempting to paste values into Wizard and Form designers was unsuccessful. Pasting values into Wizard and Form designers now works as designed.

Value Added Form Custom List Are Not Displaying Correctly in Form History: DIST-4284

We resolved an issue where values appeared incorrectly in Form History. Form History now displays values correctly.

Values Are Not Updated If User Changes the Value: DIST-4288

We resolved an issue where values updated in the Organization tab were not being reflected in History. Updated values are now reflected in History as designed.

Smart Text Box and Smart Date Picker Page Not Displaying Required in Forms Expand Menu: DIST-4548

We resolved an issue where when creating applications on pages that include the smart text box and smart date picker, the page error icon failed to appear. The page error icon now appears as designed.

All Activities View Is Not Populating Correctly in Conjunction with Rep Code Visibility Logic: EAES-5163

We resolved an issue where the All Activities view displayed a count of activities but not the activities themselves. The All Activities view now displays both the count of activities and the activities in that count.

Quote Cannot Be Added to New User Rolecode Group: EAES-5279

We resolved an issue where creating a quote failed when performed with a new user rolecode group. Now creating a quote works as designed.

Certificate Validation Failure on Outbound Connections During Workflow Retry: EAES-5300

We resolved an issue where certificate validation failed on outbound connections during workflow retry. Certificate validation now works as designed.

Token Service Is Generating Valid Tokens Regardless of Date Header Value: EAES-5378

We resolved an issue where the token service failed to validate date header values, resulting in continuous generation of tokens. The token service is now validating data header values as designed, disallowing timestamps more than 30 seconds outside of the current UTC date and time.

Unable to Complete Reviewer Forms Activity Due to Use of Obsolete Browser Events: PC-3016

We resolved an issue where document reviewers using Chrome were unable to complete review due to the Complete button failing to appear. The Complete button now appears as designed.

 

***Please note that the JIRA ticket numbers are provided for reference. JIRA tickets are not accessible to users outside Hexure.***
