FireLight v2.34 Release History

March 2024 Release—3/28/2024 Production

Enhancements

SuranceBay Integration to Support Can Sell Check

This enhancement adds a base integration with SuranceBay. Clients who utilize SuranceBay for Can Sell check can now perform those CanSell checks within the Firelight system.

Impact

The agency sets up the SuranceBay credentials in the Organization Third-Party Settings.

Admin Changes

  • Adding Group Options - FLAdmin > Groups> Admin(tab) > Select "Enable SuranceBay" Box > Message shown "Checking this service will automatically call SuranceBay Can-Sell integration.” > Save

User Preference Changes

  • Adding SuranceBay Login - FL home screen > Preferences > SuranceBay Login (Only shows when SuranceBay has been approved in Admin) > Input username and password > Save

  • New Application Selection Changes:

  • New Application Showing SureLC Information widget from each product - FL home screen > Applications > (Make sure "Information from SureLC" is on top of screen, and producer profile from SureLC is showing) > Product Selection > Pick State > Any insurance product that can be sold will have a CanSell widget on it.

New Application Selection Changes

  • New Application Showing SureLC Information widget from each product - FL home screen > Applications > (Make sure "Information from SureLC" is on top of screen, and producer profile from SureLC is showing) > Product Selection > Pick State > Any insurance product that can be sold will have a CanSell widget on it.

Data Entry Changes

  • Prior to Signature Confirmation Box - Upon selecting Continue the SureLC CanSell Widget is displayed, providing the options to continue, stop, or launch into the SureLC Portal through the "Fix Issues" button.

Increase the File Size Limit of Single File Upload Up to 50 MB

With this enhancement, the file size for attachments to activities has been increased. The maximum available file size is now 50 MB. The maximum file size for your organization can be set in the FireLight administration tool. The feature can enable users to upload large files (up to 50 MB) without splitting them into multiple files.

Impact

  • FL Admin > Organization Config > Organization Setting > Activity (tab) > Maximum Upload File Size - Single (dropdown)

  • FL App > Activity > Other Actions (menu) > Documents (upload dialog)

Impact of Changes to Sales Agreement/Product Availability

With this enhancement, an agent will not be able to submit an application (unlocked) for which the sales agreement is no longer valid. Currently, an agent cannot create an application for an invalid/expired sales agreement, but if the sales agreement became invalid while the application was not yet locked (in data entry), the agent is still able to continue to signature and submit the application. With this enhancement, when the agent tries to continue/submit an application in data entry, with an invalid sales agreement, FL will provide an alert and make the application read-only. They would still be allowed to submit a locked application.

Impact

  • App: When an agent loads an activity in “Data Entry” with an invalid sales agreement, an alert will appear, disallowing the agent to proceed.

Enhancement for Custom Activity Pop-up Messages

This enhancement allows suppressing pop-up message for an activity action. The FireLight Administration Tool User can save a blank message for an activity action so that no alert appears for that action within the FireLight workflow. The default system message can be restored to enable the system alerts.

With this enhancement, admin users can also place links in the custom alert messages for activity action.

Impact

  • FL Admin > Product Config > Activities > Activity Details (tab) > Edit Activity Pop-up Messages (link)

  • Allow saving blank message for an activity action.

  • Allow using workflow links to activity custom pop-up messages.

Manage Form Protection for Sign-Only Duty Types

With this enhancement, Agent and Client protection on forms are applied to the duty types related to signing. Both the protections will apply when printing and signing these forms.

A client protected form will not be available to an agent for printing and signing unless agent signatures are required. Similarly, an agent protected will not be available to a client for print and signing unless client signatures are required.

Both of these protections are applicable to the following duty types:

  • Signing Ceremony Only

  • Signing Ceremony Only: Conditional

  • Signing Ceremony Only: All

  • Signing Ceremony Only: All Conditional

  • Print Only

Impact

  • Agent and Client UI for App: Other Actions > Display/Print PDF

  • Agent and Client UI for App: Activity > Signatures

Impact

  • Expand Field Name dropdown on field properties for long values - FLAdmin > Designer > Form Designer > Field Properties

  • Enhance the forms/wizard search dropdown in Admin:

  • FLAdmin > Product Config > Packages > Package Detail (tab) > Replace Form/Wizard/Rule Set in Package (link) > Form/Wizard in Package (tab) > Form/Wizard to Replace (dropdown)

  • FLAdmin > Product Config > Packages > Package Detail (tab) > Replace Form/Wizard/Rule Set in Package (link) > Form/Wizard in Package (tab) > Replace with Form/Wizard (dropdown)

  • FLAdmin > Product Config > Packages > Package Detail (tab) > Replace Form/Wizard/Rule Set in Package (link) > Rule Set in Package (tab) > Rule Set to Replace (dropdown)

  • FLAdmin > Product Config > Packages > Package Detail (tab) > Replace Form/Wizard/Rule Set in Package (link) > Rule Set in Package (tab) > Replace with Rule Set (dropdown)

  • FLAdmin > Designer > Form Designer > Copy Fields (link) > Copy From (dropdown)

Add Common Tag for Product Code

This enhancement adds a new common tag to the FireLight system that maps to the product code

Impact

A product code will appear as a new common tag in Admin, mapping tool, the forms, wizard tools, and other locations that common tags will appear. The customer is not required to take any action for the new common tag to appear in FireLight. If the customer wants to use the new common tag, they will set the data item or rule to use the new common tag.

Change Behavior of Case Firm ID When Copying Cases

This enhancement allows the client’s first request for the Firm ID values to be derived from the firm ID of the user when copying activities, most of which are currently copied from the original case in FireLight today.

Impact

  • The customer will not have to take any actions for the new functionality to work in FireLight.

  • When the customer copies or transfers an activity, the new functionality will automatically occur.

LexisNexis Enhancements

This enhancement adds additional information to the LexisNexis section of the Application Audit report.

Impact

  • The customer will not have to do any actions for the new functionality to work in FireLight.
