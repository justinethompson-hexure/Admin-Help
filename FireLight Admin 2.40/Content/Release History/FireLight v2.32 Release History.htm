FireLight v2.32 Release History

NOTE: Documentation for this release is currently incomplete and in a "DRAFT" status.

October 2023 Release—10/13/2023 Production

Enhancements

Ability to Reject an Activity in Data Entry Review: DIST-1266

We added the ability to reject applications in the data entry review queue. The ability to reject applications is now available on all the review queues, including the data entry review queue. This option is also available to the API call to allow rejecting an activity from Data Entry Review.

Impact

  • New display status when activity is rejected from data entry review.

  • Update Review Queue API (ReviewQueue.UpdateReviewQueueState) to allow rejecting an activity for data entry review.

  • Update Review Queue page (FL App) to allow rejecting an activity for data entry review.

  • Lock / set read-only an activity when it is rejected from data entry review.

  • Update audit history to manage rejection of data entry review.

  • Update admin reports to include new display status for required conditions.

  • Update activity reports to include new display status for required conditions.

LexisNexis—One-Time Passcode: CR-574

We added the ability to add the LexisNexis - One-Time Passcode service to be used within either the Date Entry or eSignature workflow.

Impact

  • New Custom Action Parameter for LexisNexis - One-Time Passcode.

  • Update Audit History to include new display of One-Time Phone Finder service calls made and responses received.

  • Update Admin reports to include the new display section of One-Time Passcode service being utilized and responses returned.

  • Updated Activity reports to include the new display section of One-Time Passcode service being utilized and responses returned.

LexisNexis—Phone Finder: CR-539

We added the ability to add the LexisNexis - Phone Finder service to be utilized within either the Date Entry or eSignature workflow.

Impact:

  • New Custom Action Parameter for LexisNexis - Phone Finder.

  • Update Audit History to include new display of Phone Finder service calls made and responses received.

  • Update Admin and Activity reports to include the new display section of Phone Finder service being utilized and responses returned.

LexisNexis - Emailage - Additional Data Inputs for Service Call: CR-969

This enhancement adds a section to the data entry section of the Emailageaudit reports to verify information that was entered in that section of the application.

Impact:

• Emailage integration

• Emailage audit reports

New Inclusion Option

We added the ability to create two new options for a form in a package: "Signing Ceremony Only: All" and "Signing Ceremony Only: All Conditional." These new options differ from "Signing Ceremony Only" versions in that the form is displayed to all users during the signing ceremony if a signature control is on it, as opposed to only the user whose signature needs to be collected.

Impact

  • Admin Packages: Form duty

  • Signing Ceremony: Forms displayed

Signature Option: CS-12926

We added the ability to add two new group options in Admin, both on the App tab. The first option, Lock Signature Email Addresses, is under the Emails category. When that option is selected, when an agent sends a signature request, they will not be able to update/change the email address to which the request is sent. The second option, under the Signing: General category, is Hide Recipient Link for Agent. When this option is selected, the link the user uses to log in is displayed to the agent on the Signature Request screen.

Impact

  • Admin: Group Options

  • Signature Requests

Replace ID Issue Jurisdiction Selection with Country Selection: iConnect 278642

We have extended the ListProfile for ID verification to include an info node. This node is used to specify whether the issuing jurisdiction should be a Country or a Jurisdiction/State dropdown.

Impact

• Client ID Verification

Work Groups Participation/Coordination (Intra Org)

We enabled intra organization workflows, allowing users to have shared visibility into an activity and receive notifications for the activity life cycle. This also allows multiple individuals to access an activity without being required to take a specific action, such as Transfer, Share, etc. Status and notification functionality is enhanced to add a list of individuals to receive notifications/have visibility of a single activity from start to finish, enabling a "Team Like" activity visibility.

Impact

• Admin: Organization, Admin tab: New group option

• Admin: Group Options, App tab: New Work Groups section

• Admin: Audits

• App: Communications

• App: Preferences

Updates to AURA Next Integration

We completed various updates to the integration with RGA AURA Next to complete the XML passed to AURA Next and update navigation data for button logic passed into the AURA Next User interface.

Impact

  • Update XML Content for Holding/Insured relationship.

  • Update Tracking ID to reflect application ID.

  • Update JSON passed to AURA to correct navigation buttons.
