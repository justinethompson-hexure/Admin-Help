Differences between Combined Wizard+Forms UI and Legacy Forms UI

Spotlight Video: See the Wizards and Forms in the Same UI Spotlight Feature Video for information about the combined Wizards and Forms UI.

Some differences between the legacy forms UI and the combined wizard+forms UI follow.

[Differences Between Legacy Forms UI and Combined Wizard + Forms UI]
Legacy Forms UI Wizard+Forms Combined UI
Uses chevrons for previous/next page navigation Uses buttons for previous/next page navigation
The entire form/viewable area scrolls The content between the header and footer scrolls
Differences in styling of fields, buttons, and validation messages Differences in styling of fields, buttons, and validation messages
Text box fields do not display the number of characters in the field as the user types in the field. Text box fields display the character countdown on the lower right of the field with the number of characters typed along with maximum length of the field. For example, "42/100" shows the text box contains 42 characters of a possible 100 characters per the maximum length setting in the field properties.
An activity that contains only forms and doesn’t have any wizards in any of its packages uses the legacy forms UI. If a wizard is added to any package in the activity, the combined UI is used. If the distributor has any wizards in their packages, this can take effect in distributor environments even if the carrier doesn’t use any wizards.
Legacy forms operates on the current page regarding radio buttons and updating a radio button affects only fields with the same Group on the current page. Wizard (React UI) intrinsically operates on the entire wizard and forms collection, and updating a radio button affects all fields with the same Group for the application as a whole.
By default, all Combo Box dropdown lists display the first item in the list when a user opens a form. When the user changes any other field on the form, all the Combo Box dropdown lists become blank. A blank item can be added to a combo box List Name Values field property for the combo box to be blank when the user opens the form, which is the same behavior as the form and wizard UI. By default, all Dropdown List fields are blank when a user opens a wizard.
