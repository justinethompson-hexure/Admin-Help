Add Template Forms

NOTE: FireLight templates are currently displayed only in the portrait format. If a document is uploaded in a landscape format, the upload appears partially within FireLight®. Once viewing the upload within the Print Forms section, the upload appears completely. This can be more of an issue when signatures are collected on the uploaded page. Please submit a JIRA ticket to request support in setting up or troubleshooting template forms.

A template form is used as a placeholder to add an additional document to the order package. For example, use a template form to upload a picture of a client's driver's license or check in a PDF file or use a template form to determine the placement of signatures on a supplemental form (e.g., illustration, etc.) in the signing ceremony.

FireLight validates whether the template is uploaded before the user data entry can reach 100%. The template appears in the Open list to provide instructions. The template form can include the Custom Action Upload and Custom Action Print in Button controls to provide clear instructions with easy links to complete the template requirement.

A signature template form needs to be one page. Once the supplemental form is uploaded in FireLight, the template form is replaced with the supplemental form. The form properties of a signature template form include the signature locations on the page for the associated document type. If relative, the actual page stamped is based on its relative position in the document. If static, the system stamps the captured signature based on the coordinates of the template form signature controls.

NOTE: Template forms can also be conditional and set off from rules.

Adding a signature template form requires uploading the template form on the Form Library page, adding signature controls on the Designer page, and adding the form to a Package and making it a required form on the Packages page in Product Configuration.

Upload a template form and make it a required form in the product package.

  1. Designer/Form Library page: in the upper left, click Upload Forms.
  2. If you previously exported the signature template form in , select the Use exported definitions checkbox.
  3. In the upper left, click Add files.
  4. In the Open dialog, browse to the folder and select the signature template form for the supplemental document type, then click Open.
  5. Click Start Upload to upload the file.
  6. When finished uploading the form, click the Back button to return to the Form Library page.
  7. On the Form Detail tab, select the Is Template Form checkbox.

  8. Select the document tag in Document Tag. This is the associated document type for the template form. For example, select IllustrationWithSignature to indicate the required document is an illustration with a signature. Hexure sets up the Document Tags for your organization. Contact your Strategic Account Manager to have the tags added. Be sure to use descriptive names for your document tags. In a distributor environment, the carrier document tag types include only carrier template document tag types that are currently on a form in the application. CarrierOnly visibility can be added to the carrier's List Profile and document tags so carrier-only uploaded documents can arrive at their proper destination. DistributorOnly visibility can be added to the distributor's List Profile and document tags so distributor-only uploaded documents can arrive at their proper destination.

    NOTE: Needs determination documents are attached as NeedsDetermination document tags, illustration as LinkedIllustration, and Account Opening documents as AccountOpening. All other attached documents coming from one activity to the next activity come over with a document tag of Other.

  9. Select the States where the required form (e.g., Illustration with Signature) is approved.
  10. At the bottom of the Form Library page, click the Form Designer button to go to the form on the Form Designer page.

  11. Add signature controls for the placement of signatures on the document page.

  12. On the navigation bar, click Product Config and then click Packages.

  13. On the Forms in Package tab, add the template form to the applicable packages and make it a required form.

  14. Click the Save Package button.

    A dialog displays the following message in after the user clicks the Create Application button: "This Application has one or more Template Form with Document Tags [IllustrationWithSignature]. Please upload related document from Other Actions > Documents." Click Close to close the dialog.

  15. On the navigation bar, click Utilities and then click Deployments to deploy the forms and packages to the applicable environments. See Deploy Data.
