Add and Update Form Detail

  1. On the lower left of the Form Library page, click the Form Detail tab and add or update the Form Detail information for the selected form. 

    Form Detail Tab Settings
    Setting Description
    Form Number In the Form Number text box, enter the form number, which is the unique identification number for the form. This is a required field. Keep the State Identifier to make it easy to know which form you are working with.
    Form Name This is the unique name for the form. This is a required field. Leave the State Identifier off to make enabling/disabling rules easier on the Designer page.
    Form Description This is a brief description of the form (for example, "VA Application").
    Edition This is the edition number of each form (for example, "1/2016" or "v2"). Edition is used as a tag to assist in defining versions.
    Label This is a free-form string field that can be used by the integration provider to determine which forms to get definitions for and obtain item IDs from the form definition. It can also be used for specifying additional version/edition information for a form.
    Attachment TC Sets the ACORD TC value for the attachment type in the first text box, and the second text box pre-fills with the associated attachment type name. The attachment type value is available for the providers to use in the ACORD outbound transaction.
    File Size Displays the size of the form file.
    Form URL (Print Forms activity only) Defines the form URL that non-FireLight users can enter in a browser to access and download the PDF form.
    This Form Begins On Sets the date the form becomes effective for use in FireLight®. Add the form to the applicable packages.
    This Form Expires On Sets the expiration date of the form in FireLight. When the form expires, remove it from all packages in which it resides.
    Effective Time Zone Sets the time zone to be used for the form beginning and expiration dates.
    Will Be Replaced By

    If not already set, this field is disabled. Use the This Form Expires On field to set the date when the form expires. Use the This Form Begins On field on the replacement form to designate the day when the replacement form begins and place the replacement form in the applicable packages.

    NOTE: The setpackagestatebyname rule causes an exception if a form has the Will Be Replaced By property set. The logic that loads package names for the application forms does not take into account forms that are replaced by another form, leading to the exception due to a null package name. A workaround is to use This Form Expires On property for the form to be replaced and use This Form Begins On property for the new form instead of using the Will Be Replaced By property.
    Exclude From Back Office

    The Exclude From Back Office setting does not allow the form to be submitted electronically (print only). This is useful for forms like product prospectus or other marketing forms.
    Is Report Designates that the form is rendered as a report only—not for use with the application.
    Remove Excluded Forms From Print For forms that are conditional in a package, hides the form from the print dialog if the form has not been included by the rules.
    Signature Order Sets the signature capture order of Normal, First, Last. Normal: Use the order specified within the order of forms as defined within the package definition. First: Override the package order and place the form first in the list. Last: Override the package order and place the form last in the list.
    Is Template Form Designates that the form is a template form. For details, see Add Template Forms. Also select a Document Tag.
    Document Tag

    Selects the document type of the template form (Wet Signed Signatures, Illustration With Signature, etc.). Hexure sets up the document tag types for your organization. Contact your Strategic Account Manager to have the provider List Profile updated and the document tags added. Be sure to use descriptive names for your document tags.

    CarrierOnly visibility can be added to the carrier's List Profile and carrier's document tags so carrier-only uploaded documents can arrive at their proper destination.

    In a distributor environment, the carrier document tag types include only carrier template document tag types that are currently on a form in the application.

    DistributorOnly visibility can be added to the distributor's List Profile and distributor's document tags so distributor-only uploaded documents can arrive at their proper destination.

    Users can select the document tag type from a list in the Other Actions menu, Documents dialog when the associated template forms are included in the forms package. See Add Template Forms for instructions on using templates.
    Uploaded Displays the date and time the form was uploaded.
    Last Updated Displays the date and time the form was last updated.
    State Approved grid Select checkboxes to designate the states/jurisdictions in which the form is approved.
    Check /Uncheck All Selects or clears all checkboxes in the State Approved grid. This checkbox switches between selecting or clearing all of the State Approved checkboxes.

  2. Click the Save button to save Form Detail entries.

  3. On the Deploy Data page, deploy the forms and packages to the applicable environments.
