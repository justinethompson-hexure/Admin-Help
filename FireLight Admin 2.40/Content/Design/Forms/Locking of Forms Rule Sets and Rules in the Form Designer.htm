Locking of Forms, Rule Sets, and Rules in the Form Designer

Locking of Forms

The FireLight® system automatically locks a form for the first user who starts to make changes to the form. This prevents other users from editing the form at the same time.

For other users who attempt to open the form while it is locked by the first user, the system displays a "User <user name> is currently working on the form" message on the form page and disables the Rules Window link, Form Properties link, Form History link, and SAVE buttons. Other users can only view, navigate between the pages or close the form while the form is locked.

Example: Default lock time is one hour (based on what is set in the organization's database).

  • If the user clicks the Rules Window link, lock time for the wizard resets to that of the rule set timeout so that both wizard and rule set timeout are the same.

  • After 55 minutes of lock time, when the user continues to edit the wizard, lock time automatically rolls over to one more hour.

  • After 55 minutes of lock time, when the user is not continuing to make changes to the wizard, the system alerts the user to save data. On the Alert box, the user can do any of the following:

    • Click the SAVE NOW button to save the changes and release the lock.

    • Click the EXTEND button to continue working and extend the lock for one more hour.

    • Does not click a button. After 5 minutes of prompt, the system releases the lock without saving any changes, the lock icon turns gray, and the following text appears below the lock icon: "The application is inactive."

  • When the user clicks the SAVE button, navigates to another page, or exits the form, FireLight unlocks the form. Another user can then edit the form.

If another user loads the form without locking it and another user updates that form, the system prompts the initial user to reload the form to view latest changes.

Locking of Rule Sets and Rules

The FireLight system automatically locks a rule set for the first user who starts to make changes to the rule set. This prevents other users from editing the rule set at the same time.

For other users who attempt to open the rule set while it is locked by the first user, the system displays: "User <username> is currently working on the Rule Set" and disables the New, Update, Create Multiple, Delete All, and the delete (red X) action buttons on the Rules Window. Other users can only view and export rules or close the window.

If two users attempt to edit the rule set at the same time, the time lapse when both users start to make changes could be in microseconds; the system ensures that no two users lock the rule set together, and there is no edit lock entry already there for the rule set.

When edit lock on the rule set is released by the first user, the user who has the rules window already open for edit is prompted to reload the window if they attempt to lock the Rule Set. An Alert message displays: "This Rule Set is changed since this window is opened. Reload Rules Window to view latest version of the Rule Set."

Example: Default lock time is one hour (based on what is set in the database).

  • After 55 minutes lock time, when the user continues to make changes to the rule set without saving the rule set in between, the system automatically extends the lock time to one more hour.

  • If the user is not continuing to make changes to the rule set, the system alerts the user after 55 minutes (5 minutes before the lock time expires) to save their work. The alert message displays: "Edit lock will release automatically after 5 mins. Save your changes to avoid losing work. Saving will reset lock time for another hour."

    On the Alert box, the user can do any of the following:

    • Click the EXTEND button to extend the lock another hour.

    • Click the SAVE NOW button to save the changes and release the lock.

    • Does not click a button. After 5 minutes of prompting, the system releases the lock without saving any changes, the lock icon turns gray, and the following text appears below the lock icon: "The application is inactive."

  • When the user clicks the Update button on the Rules Window, the system checks whether the selected rule set has been opened by another user for edit. If not, the system will lock the rule set for editing.

  • When the user closes the Rule Set window, an alert prompts the user to Save Changes. On the confirmation box, the user can:

    • Click Yes on to save the changes and release the lock from the rule set.

    • Click No to release the lock without saving any changes.

  • When the user loads another page from the menu, a confirmation box appears. On the confirmation box, the user can:

    • Click Leave to release the lock from the Rule Set and load the selected page without saving any changes to that Rule Set.

    • Click Cancel to have the current rule set remain loaded as-it-is.
