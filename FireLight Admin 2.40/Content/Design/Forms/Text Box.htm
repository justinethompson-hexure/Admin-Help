Text Box

Use the Text Box control for text fields where the user fills in names, amounts, etc.

  1. On the left of the Form Designer page under Add Control to Form, click Text Box.
  2. Drag the Text Box control from the upper left of the form to a text box location on the form. For example, drag a text box to the OWNER_FIRST_NAME field.
  3. Drag the right side of the Text Box control to fill the form field space on the form.

  4. On the left under Input Control, click Field Properties.

  5. Enter or change settings in Field Properties.

    Soft Limit

    The addendums are generated based only on the soft limits and not the maximum character limit. Any overflow text is added to the addendum page for the form. The addendum reference appears in the PDF text box. Add a Soft Limit if you select the Multi Line checkbox.

    The system generates an individual addendum document for each of the forms present in the activity, at the time of the Print PDF action. All soft limit fields with overflow text present on a form are placed into a separate addendum document specific to that form. Each individual addendum generates directly after the corresponding form and not at the end of the package.

    The Addendum document for each form includes the form name in the title in the format "Addendum - <Form Name>." Multiple versions of the same form (form indexing) are easily identified with a numeric reference following. Example: “Addendum – <Form Name>”, “Addendum – <Form Name> 2”, etc.

    Both overflow fields and their overflow content Addendum references are numerically applied from left to right, top to bottom and not defined by tab order of the form. Column heading for the field is "Reference ID" and column heading for the content is "Overflow Content."

    All the signature fields placed on a form also appear after the addendum text ends on the last page of the associated addendum document. The addendum document is also presented for signature along with its associated form.

    This impacts Print PDF functionality and Signatures for all the activity types, except for the Print Forms activity.

    NOTE: Carrier forms may need to be set up/evaluated for the Addendum reference order if a specific order is desired. Addendum references are numerically applied from left to right, top to bottom and not defined by tab order of the form.

    Mask

    SSN Format Select the level of masking to apply to the SSN, TIN, or Canadian SIN identification number during data entry: None (not a SSN, TIN or SIN field), Visible (all numbers are visible, e.g., 123456789), Partial (last four numbers are visible, e.g., XXXXX6789), or Full (no numbers are visible, e.g., XXXXXXXXX). The default value is None, which indicates the field is not a SSN, TIN or SIN field. The Mask and Is Password field properties cannot be used when the SSN Format option is other than None. To secure the value, SSN mask formatting is applied on the server so it is not visible in the browser. NOTE: Global Scope does not apply for SSN format. To have formatting on all fields with the same dataItemID, set the SSN Format in the Field Properties for each of the controls.

    Is Password Set whether the control is a hidden entry text box. When this checkbox is selected, the control appears similar to a password entry text box where the text entry is hidden on the form. Is Password cannot be used when SSN Format is set to other than None.
    Show Character Count This field property becomes available when a Max Length is added. Select the Show Character Count checkbox to display the character countdown on the lower right of the Text Box field. In the combined wizards+forms UI (in React), a character countdown displays on the lower right of the Text Box field as the user types in the field. For example, "42/100" shows the text box contains 42 characters of a possible 100 characters per the 100 Max Length setting in the field properties. When a user hovers over the character countdown value, a tool tip is displayed as "Maximum characters allowed xxx." Requires a Max Length entry. NOTE: The legacy forms UI does not support this feature. It is available only in wizards UI and the wizards+forms UI.

  6. Click the Update button to update the field properties and lock the form for editing.

  7. Click the Close button to close the Field Properties dialog.
  8. Click the SAVE button on the upper left of the Designer page to save the controls and field properties, which unlocks the form for editing by other users.
