Copy Fields to a Form from a Form or Wizard

Use the Copy Fields feature to copy fields from one form to another form or from a wizard to a form. To copy fields on the same form, use the Toolbox (see Align, Copy, and Move Controls).

System Rules for the Copy Feature

The following system rules apply.

  • Matching fields copied together stay together.
  • Any and all fields in an existing fillable form can be copied to a new non-fillable form uploaded as a non-fillable PDF or XPS file.
  • Original fields uploaded with a fillable PDF form cannot be copied to other forms uploaded as Fillable PDF files.
  • Fields added to a form in the designer can be copied to any other form.

Copy Fields

  1. On the upper left, select the target form that receives the copied fields.
  2. On the left under Input Control, click Copy Fields.

  3. Click the Copy From field and select the source form or wizard to copy the fields from. Or type a form or wizard Name, Number, or Edition to filter the list of forms and wizards and then select a form or wizard from the filtered list.

    NOTE: In the Copy From list, wizards have "Wizard-" preceding the wizard name so wizards go to near the end of list.

    Copy Fields from Wizard Guidelines:

    • The Page column displays the wizard page name instead of displaying a page number as is shown in forms.

    • The "Keep Original Field Coordinates" checkbox is cleared and disabled. Wizard data items do not have coordinates assigned to them.

    • The "Copy all Selected Fields to the Active Page" is auto-selected and disabled.

    • Any Custom Action button controls using the Custom Action Type "External Source" or "Milliman" are excluded from the list of controls available to be copied from a wizard to a form destination. While these buttons might be available for copying from a wizard to a form, the functionality tied to these Custom Actions is not functional in a forms environment.

    • No matter the size of the control as it exists in the Wizard, the Default size of the controls in the Form environment is used for the fields/controls upon landing on the form. Controls can then be moved and adjusted as necessary to fit form requirements. The default size for each control is listed below:

      • Combo Box (equates to wizard Dropdown List), Text Box, Calendar (equates to wizard DatePicker), and Address Autocomplete controls appear on the form with the default size of 50x18.

      • Check Box and Radio Buttons appear on the form with the default size of 15x15.

      • Button (equates to wizard Custom Action) appear on the form with the default size of 20x20.

    • All copied wizard fields land on the form beginning at the top left of the form, moving left to right across the page and then down to the next line. If there are existing data items on the form, the fields being copied over from a wizard may overlay those existing controls.

    • If more fields are copied from a wizard than there is space on the form to receive them, after all lines on the form have been filled, all "excess" data fields are located at the bottom left corner of the form and are staggered so that the user can see there are multiple data fields.

    • Where there are one-to-one relationships between wizard data field properties and form data field properties, the data field retains the same field properties on the destination form as existed on the source data item from the wizard.

      Retained data field properties from the wizard field data source:

      • Text Box retained field properties: Field Name, Common Tag Name, Error Placement, Input Type, Placeholder Text, Text Mask, SSN Format, Data Group, Data Group Property, Max Length, Read Only, and Office Use Only.

      • Combo Box (wizard Dropdown List) retained field properties: Field Name, Common Tag Name, List Values, Data Group, Data Group Property, Read Only, Office Use Only, Error Placement, and Help Text.

      • Check Box and Radio Button retained field properties: Field Name, Common Tag Name, Export Value, Group Name, Error Placement, Help Text, Display Style / Border Style, Data Group, Data Group Property, Read Only, and Office Use Only.

      • Button (wizard Custom Action) retained field properties: Field Name, Common Tag Name, Help Text, Data Group, Custom Action, Custom Action Type, and Action Parameter.

      • Calendar (wizard DatePicker) retained field properties: Field Name, Custom Tag Name, Error Placement, Help Text, Data Group, Data Group Property, Text Mask, and Read Only.

      • Address Autocomplete retained field properties: Field Name, Common Tag Name, Placeholder Text, Help Text, and Data Group.

    • When the fields from the source wizard are displayed in the Copy Fields dialog and data items do not exist on the destination form, the checkboxes adjacent to the Field Name/DataItemId column are selected (checked) and the checkboxes under the Already Exists column are cleared (unchecked).

    • Any "Masks" added in the "Organization Settings" under the Activity tab are available for mapping when a data item to be mapped has one of the Masks selected.

    • Any "custom" mask that exists in the wizard and is not a part of the mask created under the Organization Settings is not supported in Forms. If a data field to be copied from wizard to form has a "custom" mask, the data field is available for copying but the mask setting in the Field Properties is blank for the copied field.

  4. In the grid, ensure all the fields to be copied are selected and then clear the checkboxes for the fields that will not be copied.

    • Select all fields - Select the checkbox in the Field Name column of the list header. Clear the checkbox to clear all fields.
    • Search for a particular field - Type the field name in the Search field and click Go to display fields that match the search text.
    • Search for all fields on a particular page - Type Page:[#] in the Search field, such as “Page:2”.
    • Clear the search results - Clear the text in the Search field and click Go to clear the search results.
    • Sort the grid columns - Click a column title name to sort the grid by the selected column. Click the same column title to sort the grid in the reverse (descending or ascending) order.

    The entries in the Page, Field Type, In PDF, DataItemID, and Already Exists columns are read only.

  5. Below the grid, select the following checkboxes that apply.

    Keep Original Field Coordinates retains the field coordinates from the source form.

    Copy All Selected Fields To The Active Page copies all of the selected fields to the active page in the target form.

    Copy All Selected Fields and Pages copies the selected fields and the pages the fields exist on to the active page in the target form.

    Copy Page Names copies the page names.

    Delete All Existing Fields Before Copy deletes all of the existing fields in the target form before copying fields from the source form.

  6. Click the Copy Selected Fields button to copy the fields and rules to the form that is active on the Designer page and lock the form for editing.
  7. Click SAVE on the upper left of the target form page to save the copied fields, which unlocks the form for editing by other users.
