Combo Box

Add a Combo Box Control

NOTE: When a user opens a legacy form in the forms only view, by default all Combo Box dropdown lists are auto filled with the first value in the lists. When the user changes any other field (e.g., selects a Check Box), all Combo Box dropdown lists automatically become blank. This is normal behavior for Combo Box controls in the Forms Only view. A blank item can be added to a combo box List Name Values field property for the combo box to be blank when the user opens the form, which is the same behavior as the form and wizard UI. See Differences between Combined Wizard+Forms UI and Legacy Forms UI for other differences between the Forms Only view and the Wizard + Forms UI view.

Use the Combo Box control to limit answers to list items. In this example, a combo box control is used for selecting a Gender.

  1. On the left under Add Control to Form, click Combo Box.

  2. Drag the Combo Box from the upper left of the form to the applicable field on the form and re-size it to fit the field space.

  3. On the left under Input Control, click Field Properties.

  4. Enter or change settings in Field Properties.

    List Name Values

    Create the List Name Values by doing one of the following, depending on whether the Common Tag Name property is set to a common tag name or blank.

    The following tooltips are available in the List Name Values dialog title row:

    Tip: To quickly move a checked row to the top or bottom of the list, hold Ctrl and click the up or down arrow icon in a row.

    There are three scenarios when adding a list to a combo box.

    • Common Tag Name property is set to a common tag name:

      1. Enter or select a common tag in the Common Tag Name property.
      2. Click the Edit List Name Values link.

      3. The system automatically generates the stored list values and caption/rules values that are associated with the Common Tag Name. All list items are automatically selected. In the Move column, click the up and down arrows on the right of the rows to re-order the list items. In a row, clear the checkbox to hide the row from the list the user sees.

        The List Value column always contains a value if there is an associated Common Tag Name and is always blank on user-created rows.

        The list has a blank row below the last row. Select the checkbox to create a new user-created row where you can add a Caption/Rules Value and optional Rules Override value. The OK button is disabled until you add a Caption/Rules Value for the new row. There is no limit to the number of rows you can add. When the last blank row is checked, a new blank row is added.

        Optionally, use the Caption Override and Rules Override columns to add caption and rules overrides for the list name values. The Caption Override shows a different value in the list the user sees. The Rules Override allows you to set a different value to be used in the rules than the Caption/Rules Value.

        The following example shows the List Name Values when adding a Field Name and Common Tag Name of Owner_Gender for a combo box control and then making some modifications to the List Name Values. Select the checkbox in the last row and add a Caption/Rules Value of Binary so the user sees the option "Binary" in the list of options. (User-created rows do not have access to the Caption Override column. Enter the Caption/Rules Value and optional Rules Override if applicable.)

        In the following example, add a Caption Override for each of the common tag list name values. Enter the first letter of each applicable option in Caption Override. For example, enter "F" in Caption Override so the user sees "F" instead of "Female" in the list and so on. Add "B" in Caption so the user sees the option "B" for the user-created row instead of "Binary."

        To add a blank option to the list:

        1. Select the checkbox in the last blank row of the list to create a new blank row.

        2. Enter {blank} in the Caption/Rules Value.

          To move the blank row to the top of the list, press and hold Ctrl and click the up arrows icon on the right of the row as shown below.

          The blank row is now at the top of the list.

          To move a row to the bottom of the list, press and hold Ctrl and click the down arrow icon on the right of the row.

          Click the up arrow icon or down arrow icon to move the row up or down in the list one row at a time.

      4. Click OK to save the list and return to the Field Properties dialog.

    • Common Tag Name property is blank.

      1. Click the Edit List Name Values link to open the List Name Values dialog. The List Name Values dialog contains one blank row. If the list already exists, the dialog contains the previously added rows.

      2. To enter a new row, select the checkbox for the blank row. Enter the Caption/Rules Value and the Rules Override value for the option to be displayed in the list. Add a new row for each option to be included in the list. Following is an example for a list of options added for the Life_RiderOptions field that does not have a common tag name in the Common Tag Name field property. The List Value column does not contain values for the manually added rows. Optionally, enter values in the Rules Override column for the applicable rows.

      3. Click OK to save the entries and return to the Field Properties dialog.

    • Common Tag Name property is selected with an existing list.

      1. Select a common tag name in the Common Tag Name field property after the dropdown list is created.

      2. The system displays a Confirm dialog. Click OK to close the dialog and accept the changes to the list.

      3. The system merges the Common Tag Name list with the existing list, retains the selected original rows, and adds unselected rows associated with the Common Tag Name at the end of the List Name Values dialog.

      4. Select the checkbox to include, or clear the checkbox to exclude, applicable rows in the list. Optionally re-order the list items using the up and down arrows. Remove user-created rows by clicking the trash can icon.

      5. Click OK to save and return to the Field Properties dialog.

    Select a Data Group if applicable. This is where you select a Data Group for fields associated with a custom action Button field. For example, when Action Type in a Button is set to CRM, set Data Group to the type of party (e.g., Owner, Annuitant, Beneficiary, etc.). When Action Type is set to CRM and the CRM Type is set to 1035YellowPages, set Data Group to the 1035 Exchange Ceding Company number (1035 Exchange Ceding Company 1, 1035 Exchange Ceding Company 2, etc.). NOTE: Custom List does not apply to Combo Box controls.
    Data Group Property

    Select an option in Data Group Property when a field is associated with a custom action Button. When Action Type is set to CRM, select the type of data to pull in the Data Group Property for the selected Data Group and dataitem (e.g., for the Owner_FirstName field, select Owner in Data Group and select FirstName in Data Group Property). NOTE: Custom List does not apply to Combo Box controls.
