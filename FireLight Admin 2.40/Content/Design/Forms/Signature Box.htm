Signature Box

NOTE: Within FireLight®, most dates and times in the UI are displayed in the user's time zone (as configured in Preferences). Signatures are different. Signature timestamps on forms are shown in UTC. Signature dates and times elsewhere in the UI, including dates obtained from the Signature Date control, are converted to the time zone of the signer’s state of residence.

Use the Signature Box control to add an e-signature to the form.

  1. On the left under Add Control to Form, click Signature Box.

  2. Drag the Signature Box control from the upper left of the form to the signature area on the form and re-size it to fit the field space.

  3. On the left under Input Control, click Field Properties.

  4. Enter or change settings in Field Properties.

    Group Name

    (Optional) Type the name for the group of signatures. The Group Name is used by the system to assemble a collection of forms into a document set that can be signed together. For example, you could define one group named “1Consent” to include only the consent form, and another group called “Application” for the rest of the forms. FireLight arranges the document sets first by number and then alphabetically. To review the consent form first, the Group Name is set to "1Consent" and the Group Name fields for the forms in the application package are set to "Application." Doing this, FireLight leads the signer through two separate signing processes; starting with the E-Consent form and followed by the Application.
    Signer Type

    Select the type of signer in the list. The system uses Signer Type to determine who signs the form in the signature capture process.

    NOTE: Activities containing forms that require to be signed by a Trustee displays the text "TTEE" as a suffix on the Trustee signature field at the following places:

    • Form/application

    • Addendum document

    • Application PDF

    • Application Audit Report

    • Applications created via Embedded APIs/IXN

    Trustee signature controls should be higher than size 160x30, so that TTEE is not overlapped with the signature on it.

    Text "TTEE" appears suffixed on the Trustee signature field on the Application, Application PDF, and Application Audit for the following cases:

    • For all the trustee signers such as: Trustee, Trustee 2, Trustee 3, Trustee 4, etc.

    • When multi-selection is done at the time of signing for "Trustee" signer along with "Owner" signer type.

    State Match Warning

    Select the checkbox to warn about solicitation state/signing state mismatch. If a state mismatch, a Signature Warning dialog warns the agent of the state mismatch. The agent can choose to submit the application and FireLight records an audit, "Signer solicitation state and signer state mismatch acknowledged by agent." State Match Warning does not work when the Enable Auto Submit checkbox is selected in Organization Configuration/Groups on the App tab in the Submit section. State Match Warning works only when there is a post signature review queue.

    The "State Signing Error" differs from the existing "State Match Warning," as a final error is provided on the fifth attempt and the user can no longer proceed with the signing ceremony in that activity. The "State Match Warning" provides a warning to the agent if there is a post signature review queue but allows the activity to proceed. These two settings cannot be used in conjunction and are not allowed to be selected in the admin Forms Designer together. The setting selection boxes act like a toggle in the signature control, only allowing one to be selected at a time.

    Use LexisNexis Select the checkbox to use LexisNexis third-party client authentication. For more information, see LexisNexis Client Identification Verification During E-Signature in Features.
    Use SMS Text Select the checkbox to use SMS authentication for the signer role. You can write rules on the signature tags per signer and account for workflow when the client does not have a mobile number. When entering the client mobile number during data entry, you can activate signature tags that have the Use SMS Text option selected. If a client mobile number is not available, you can deactivate signature tags with the Use SMS Text option selected, and instead, activate standard signature tags. For more information, see SMS Text Authentication in Features.
    Review Use Only Select the checkbox to allow a form to be included for review purposes only in the signing ceremony. Signature is not applied to these controls. If a review control has been added only for a signer without the corresponding signature control, the review control is ignored as there is no signing ceremony.
