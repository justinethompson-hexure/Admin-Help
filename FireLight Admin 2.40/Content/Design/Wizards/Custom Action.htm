Custom Action

IMPORTANT! In parent/child activities, you may need to write a rule to disable the use of custom action buttons that could change data in the child activity when coming from a pre-sale activity.

Purpose

Provides a button field to initiate a custom action on a wizard. The button and associated fields often belong to a unique Data Group. Clicking or tapping the button field initiates the custom action, for example, opening a custom list, uploading a custom file, calling a CRM to get data, initiating a carrier-specific custom action, etc. Associated fields display the results of the custom action, for example, custom list data.

FireLight supports triggering a custom action call from any page within the same package, regardless if the button or fields to trigger that call are on that same page. The call can only occur when a custom action button is enabled. If the button is disabled, the call cannot occur. Additionally, a call can only occur on a single activity, meaning that the call can't occur from a presale when the custom action button is in an application. The clickcustomaction rule can reside on any form or wizard in the package and trigger a custom action button.

NOTE: The value of the Custom Action click count is incremented with each click. See "Add a Rule to Disable a Custom Action Button After Three Attempts" below for a sample rule to control the number of times the Custom Action button can be clicked.

  1. Drag a Custom Action control from the right pane to the wizard page.
  2. On the page, double-click the Custom Action to open the Custom Action Properties dialog.

  4. Enter the remaining field properties and click OK to save your changes.
  5. On the upper left, click SAVE WIZARD to save the wizard, which unlocks the wizard for editing by other users.

Data Group

Select a Data Group for the custom action button and associated fields that require a Data Group. Set Data Group and Data Group Property values in each field that is associated with a CRM or Custom List action button. When Custom Action is set to Custom List, set Data Group to the Custom List number (Custom List 1, Custom List 2, etc.), and set Custom List to the name of a Custom List that is available in the Custom Lists page in Organization Configuration. NOTE: Custom File Upload does not require a Data Group selection.

CRM

 

Set the custom action button to open a Customer Relationship Management (CRM) program, such as Redtail, Salesforce, Whitepages, 1035YellowPages, etc., or a custom CRM from your organization. You must select a Data Group and a CRM Type.

  • Select a Data Group for the type of party (e.g., Owner, Annuitant, Beneficiary, etc.) and set the CRM Type to define the CRM (Redtail, Salesforce, etc.) from which to pull the data.

    For Cooperative Technologies 1035 YellowPages, set Data Group to the 1035 Exchange Ceding Company {#} number (1035 Exchange Ceding Company 1, 1035 Exchange Ceding Company 2, etc.).

  • Select a CRM Type option in the list:

    Preferences (pulls data from the CRM set up on the user's Preferences page in FireLight), Redtail, Salesforce, Whitepages, 1035YellowPages (Cooperative Technologies 1035 YellowPages), Custom, or NPIS (National Provider Identifier Standard database for looking up a health care provider or office).

Set Data Group and Data Group Property values in each field that is associated with a CRM.

Custom Action

Set the button to initiate a custom action for your organization. Some Custom Actions are coded by the provider specifically for your organization. You must select a Custom Action Type and Custom Action Parameter.

Custom Action Type select a custom action type in the list:

  • Regular Provider Action initiates a custom provider action that allows the Provider to send the binary PDF data as well as the document tag associated with the PDF to add a document or replace an existing uploaded PDF during the data entry process. You must enter a value in Custom Action Parameter for the action and a document tag value (e.g., Upload Sample Doc) that is i the provider for your organization. Audit history shows the initial addition and when changes occurred to the document. NOTE: To add this ability to your organization's Provider, contact your Strategic Account Manager.
  • Print initiates a print action. Enter Print in Action Parameter to print all forms with the same attachment TC type as the current form. Multiple TC values (separated by commas) can be passed, and in this case, all forms with matching Attachment Type TC values will be included (and no special logic for 73 will happen if included in the list). For example: Print|71,72,73,99,101.
  • See Custom Action Parameter for more information.
  • Quick View launches Quick View for illustration calculations.
  • GIACT initiates a web service call to GIACT Systems for banking account verification and authentication. See GIACT Services Integration in Features for setup instructions.
  • LexisNexis initiates a web service call to LexisNexis for client identity validation or risk classifier information. For client identity validation, enter LexisNexis in Custom Action Parameter. For risk information, enter LexisNexisRisk in Custom Action Parameter. See LexisNexis Client Identification Verification During Data Entry and LexisNexis Risk Classifier Integration in Features for setup instructions and feature information.

  • Multiple Action initiates a sequence of actions. You must enter parameters in Custom Action Parameter. See Create User, Transfer, Email Custom Action (Concierge Paper Workflow) topic in Features for detailed information.

  • APPS initiates a web service call to the APPS Scheduled4Real paramed online ordering system. Enter the applicable Custom Action Parameter for the action taken by the APPS custom action button: APPSCheckAvailability, APPSMakeReservation, or SubmitParamedOrder. Ensure Data Group is set to Insured or Other Insured as applicable.

  • Data Entry Review initiates a web service call to the Data Entry Review process that includes transmission of PDF documents and corresponding metadata.

  • ExamOne initiates a web service call to the ExamOne ScheduleNow paramed online ordering system. Enter the applicable Custom Action Parameter for the action taken by the ExamOne custom action button: ExamOneGetMobileTimes, ExamOneConfirmMobileTimes, ExamOneGetInCenterLocations, ExamOneGetInCenterTimes, ExamOneConfirmInCenterTime, SubmitParamedOrder or SubmitHealthPiQtureOrder.

    Ensure Data Group is set to Insured or Other Insured as applicable.

  • Launch Morningstar initiates a call to Morningstar Intelligence Tool.

Custom Action Parameter Add an action parameter for the selected Custom Action Type.

  • Regular Provider Action – When Custom Action Type is set to Regular Provider Action, enter an action and a document tag value (e.g., enter Upload Sample Doc) that is in the provider for your organization. This allows the Provider to send the binary PDF data as well as the document tag associated with the PDF to add a document or replace an existing uploaded PDF during the data entry process.
  • Print – When Custom Action Type is set to Print, leave Custom Action Parameter blank to print all forms. Type Print in Custom Action Parameter to print all forms with the same Attachment TC type as the current form (this is the Attachment TC setting on the form in the Form Library). To print all forms with a specific Attachment TC type, type Print|[Attachment TC type]. For example, to print forms in the Form Library with an Attachment TC type of 73 (in ACORD 1035 Exchange Forms, Attachment TC type = 73), type Print|73 in Custom Action Parameter. For an example, see the Cooperative Technologies 1035Yellow Pages topic in Features.
  • GIACT – When Custom Action Type is set to GIACT, ensure GIACT is in Custom Action Parameter.
  • LexisNexis – When Custom Action Type is set to LexisNexis for client identity validation, ensure LexisNexis is in Custom Action Parameter.
  • LexisNexisRisk – When Custom Action Type is set to LexisNexis for risk classifier information, ensure LexisNexisRisk is in Custom Action Parameter.
  • Multiple Action - When Custom Action Type is set to Multiple Action, select or enter the applicable action parameters. The format for entering parameters is "parameter 1," "parameter 2," etc. As the sample naming implies, the parameters identify the action and the entry order identifies the action sequence. It is not necessary to number parameters, e.g., CreateUser, TransferActivity, EmailUser. See the Create User, Transfer, Email Custom Action (Concierge Paper Workflow) topic in Features for detailed information.
  • APPSCheckAvailability - When Custom Action Type is set to APPS, enter APPSCheckAvailability in Custom Action Parameter for the button that checks for paramed exam availability.
  • APPSMakeReservation - When Custom Action Type is set to APPS, enter APPSMakeReservation in Custom Action Parameter for the custom action button that makes the paramed exam reservation.
  • DataEntryReview - When Custom Action Type is set to DataEntry, enter DataEntryReview in Custom Action Parameter for the custom action button that makes Data Entry Review available.
  • ExamOneGetMobileTimes - When Custom Action Type is set to ExamOne, ensure ExamOneGetMobileTimes is in the Custom Action Parameter for the custom action button that gets the times available for mobile paramed exams.
  • ExamOneConfirmMobileTimes - When Custom Action Type is set to ExamOne, enter ExamOneConfirmMobileTimes in the Custom Action Parameter for the custom action button that confirms the time available for the ExamOne mobile paramed exam.

  • ExamOneGetInCenterLocations - When the Custom Action Type is set to Exam One, enter ExamOneGetInCenterLocations in the Custom Action Parameter for the available ExamOne In Center locations.

  • ExamOneGetInCenterTimes - When the Custom Action Type is set to Exam One, enter ExamOneGetInCenterTimes in the Custom Action Parameter for the available ExamOne In Center times.

  • ExamOneConfirmInCenterTime - When the Custom Action Type is set to Exam One, enter ExamOneConfirmInCenterTimes in the Custom Action Parameter for confirmation of the ExamOne In Center paramed exam time.

  • SubmitParamedOrder - When the Custom Action Type is set to APPS or ExamOne, enter SubmitParamedOrder in the Custom Action Parameter for the custom action button that submits the paramed exam order.

  • SubmitHealthPiQtureOrder - When the Custom Action Type is set to ExamOne, enter SubmithealthPiQtureOrder in the Custom Action Parameter for the custom action button that submits the HealthPiQture order.

  • For Morningstar custom action button, set Custom Action Parameter to Morningstar. See Morningstar Integration in Features for setup instructions.

Custom File Upload

Set the button to launch a Documents dialog in . In Documents, if the document type is not found in the Document Type list, it is automatically enabled regardless of the Admin setting. Also use the Custom File Upload setting to upload image files to the Documents dialog if image files are allowed. See the Documents [User Instructions] dialog in Other Actions for user instructions.

Enter the Type of Doc to Upload and determine whether to select the following options:

  • Allow Non PDF File – If non-pdf image files and .xps documents are allowed, select this checkbox. The following image file types are allowed: .jpg, .tiff, .png, .gif, and .bmp. After upload, the image files are converted into .pdf files and stored the same as other files.

  • Disable Change Document Type – Select the checkbox to not allow the agent to change the Document Type. Clear the checkbox to allow the agent to change the Document Type.

Custom List

Set the custom action button to access a Custom List that was uploaded for your organization. You must select a custom list in Custom List and select a Custom List number in Data Group.

  • In Data Group, select the Custom List {#} number (Custom List 1, Custom List 2, etc.).

  • In Custom List, select the Custom List Name for the button to access. See "Example: Custom Action with Custom List for Occupations" below. The list options are the names of the custom lists that exist on the Custom Lists page in Organization Configuration. See Custom Lists for guidelines and instructions for adding custom lists to FireLight Admin.

You can add up to nine fields (e.g., Text Box controls) for displaying custom list column data. In each field that receives the custom list data, be sure to set the Data Group to the same custom list number as the Custom List field and select the Data Group Property. See Custom Lists in Organization Configuration for instructions to add, update and delete custom lists in Admin.

External Source

Set the custom action button to designate an external source that is used. The External Source option is available only in wizards. Select an External Source Type.

External Source Type ties the selected API to the custom action button that is used to launch the underwriting interview.

  • Select Aura Next in External Source Type. See RGA Aura Next Integration in Features for setup information.

    • Enter the External Source Project ID and External Source Interview ID for Aura Next. The Project ID and Interview ID are used to create the Aura Next case. These can be retrieved from your Aura Next configuration portal.

    • Enter an External Source Locale if other than U.S. English, e.g., fr_CA, es_MX, etc. If the locale is not set on the custom action button, then the default locale English (en_US) is used. This can be retrieved from your Aura Next configuration portal.

  • Select Magnum in External Source Type, optionally select an Outbound Mapping List and Inbound Mapping List, and enter an External Source Parameter for Magnum, which is the Magnum rulebase name. See Swiss Re Magnum Integration in Features for detailed information.

    • Select the Outbound Mapping List to pass to Magnum. Outbound Mapping List data is passed to Magnum from FireLight when Magnum is opened. Updated Outbound Mapping List data is passed to Magnum from FireLight whenever Magnum is opened on an existing activity. The Outbound Mapping List is stored in Organization Configuration on the Custom Lists page.

    • Select the Inbound Mapping List for Magnum. Inbound Mapping List data is passed to FireLight from Magnum when the Magnum interview dialog is closed. Inbound Mapping List data is mapped to applicable FireLight field data items. The Inbound Mapping List is stored in Organization Configuration on the Custom Lists page.

    • Enter an External Source Parameter (required) for Magnum. This is the rulebase name for Magnum.

    • Enter an External Source Locale if other than U.S. English, e.g., en_GB, fr_CA, es, etc. If the locale is not set on the custom action button, then the default locale English (en_US) is used.

Multiple Activities Auto-Fill

Set the button to designate a Party role for a pre-sale Multi-App activity that collects Party information of individuals and entities (trusts) that may be associated with specific roles in the multiple product applications that may be generated from a pre-sale Multi-App activity. Create a separate Custom Action button for each Party role (Owner, Joint Owner, Insured, Joint Insured, Trust, etc.). This custom action button can be associated with role specific fields through selection or entry of a role prefix.

FireLight requires a Party role entry in the Auto-Fill Prefix field property, which auto-completes the entry as you type if the entry is a common tag.
Example: Data Entry Review

Data Entry Review allows for an incomplete PDF package to be sent, along with metadata, prior to Client Fill and Sign, meaning the activity and documents will not be at 100% before going through the established Pre-Signature/Pre-Submit Review Queue. Any clients who choose to implement data entry review may require provider setup and should consult their project manager.

Submit Application to Data Entry Review on Custom Action Call

  • When the custom action Data Entry Review is clicked, the data entry review process begins.

  • The application PDF, documents and corresponding metadata are available for the provider to send.

  • Application is shown in READ ONLY mode.

  • The application enters the Pre-Submit Review Queue.

  • During Data Entry Review, the following are not available from the Other Actions menu: Request Client to Fill App, Request Client to Fill & Sign, Unlock, Share, Transfer

The Delete Button on the All Activities page is hidden while the application is in Data Entry Review.

Add a Rule to Require a Button to Be Clicked

The following sample rule is for a text box field (with the same data item ID in the Name field property as the Custom Action button) that displays the number of times that the button has been clicked. The rule requires the button to be clicked at least once.

Add a Rule to Require a Button to Be Clicked

 **Used "Required Field" default template.** <if> <condition> <not> <isnullorwhitespace> <diget dataitemid="ButtonName" /> </isnullorwhitespace> </not> </condition> <removemessage dataitemid="ButtonName" /> <else /> <postmessage dataitemid="ButtonName"> <const value="You must click the ButtonName at least once." /> </postmessage> </if>

Call a Rule by a Custom Action

For the most part, rules run based on data item changes. Coordinating the changes can trigger a rule to run after a Custom Action. It doesn’t re-run all rules, but it does update any data items returned by the provider, which triggers rules to run for any related rules. Rules are triggered only when an activity is opened and/or when the data item value changes.

Clear a Grid by Custom Action Rule

 <Block> <if> <condition> <compare op="=="> <diget dataitemid="Button_ClearAll" type="Double" /> <const value="1" type="Double" /> </compare> </condition> <table type="Grid" list="VA SubAccounts" name="subAccountGrid" dataitemid="SubAccount_Grid"> foreach name="row"> <get name="subAccountGrid" /> <cellset prop="Value" name="Allocation" type="double" failvalue="0" parseformatting="true"> <get name="row" /> <const value="0" /> </cellset> </foreach> </table> </if> <diset dataitemid="Button_ClearAll"> <const value="0" type="Double" /> </diset> </Block>

 EXAMPLE: Custom Action with Custom List for Occupations

EXAMPLE

NOTE: You must first add a custom list to the Custom Lists page in Organization Configuration before you add the custom action control. See Custom Lists for guidelines and instructions for adding a custom list to Admin. For this example, a "Peak Life - Occupations" custom list with two columns named "Letter" and "Occupation" was added to the Custom Lists page.

1. Add a Custom Action control for the custom list named “Peak Life - Occupations”.
In the Custom Action Properties dialog for the custom action control, the following properties are set:
Field Name = Button_Occupation (button data item name)
Text = Select Primary Occupation (display name of the button)
Custom Action = Custom List (type of custom action for button)
Data Group = Custom List 1 (indicates that this is the first custom list button control set up on the wizard—each custom list button control needs to have a unique Custom List number)
Custom List is set to Peak Life - Occupations (name of the custom list)

2. Add an associated Text Box control to receive the data from the custom list. The custom action button and the associated text box must be on the same page of the wizard.

In the Text Box Properties dialog for this example, the following properties are set:
Field Name = Owner_EmpDetails_Occupation (tag for the Owner Occupation)
Text = Occupation (display name of the text box control)
Data Group = Custom List 1 (must match the Data Group setting in the custom action button control)
Data Group Property = Occupation (Data Group Property list contains the column heading names of the selected Custom List—in this example, it is set to the "Occupation" column name from the custom list)

The FireLight user clicks the button and selects an occupation from the Custom List dialog.

The selected occupation populates the Occupation text box on the wizard in the FireLight UI.
