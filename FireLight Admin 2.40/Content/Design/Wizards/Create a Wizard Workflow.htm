Create a Wizard Workflow

NOTE: Hexure provides a Template Wizard (Annuity) and associated Rule Set that you can download from Admin Help and use as a starting point for creating a wizard of your own. See Add a Template Wizard and Rule Set for instructions.

Maximum Wizard Size: The maximum size for a single wizard is 10MB.

Create a wizard workflow in FireLight®. The wizard facing presentation layer is in addition to the existing form-based solution, creating an alternative experience for the agent. Using this workflow, the PDFs do not display to the end user during data collection, but are filled out based on the wizard entry presented during the signing ceremony. Using the Admin wizard designer, firms are able to adjust the wizard real-time. NOTE: Wizards are not available on the FireLight Console.

  1. Create the wizards for the product. Create a wizard for each navigation stop in the wizard workflow. For example, you may want to create wizards for personal information, beneficiary, coverage options, health questionnaire, etc., for a Term Life product. See Wizard Library topics for more detailed information.

    1. On the upper left of the Designer/Wizard Library page, click Create Wizard.
    2. On the Wizard Detail tab, fill in the Wizard Number and Wizard Name, select the States Approved, and fill in any additional wizard details. Click Save at the bottom of the tab.
    3. When finished creating the wizards, click Wizard Designer at the bottom of the tab.

  2. Add pages, section breaks, containers, and fields to the wizards. See the Add and Delete Pages and Section Breaks and Add Containers and Fields topics for more detailed information.

    1. Near the top of the left pane of the Designer/Wizard Designer page, select a Wizard in the list.
    2. In the left pane, click Add New Page, enter the Name, Title, Subtitle, and Navigation Text for the page and click OK. The Name uniquely identifies the page for usage in rules. The Title and Subtitle are displayed when the wizard is rendered. Add all applicable pages to the Wizard. For example, add an owner page and an insured page to the personal information wizard for a Term Life product.
    3. To group fields together, drag a container (panel or group) from the Add Field pane on the right to the wizard page. Double-click the container to set the properties.
    4. Drag fields from the Add Field pane to the container or page. Double-click each field and set the field properties.
    5. (Optional) Add one or more section breaks to the pages.On the left, click Set Page Workflow. In the Page Workflow dialog, click ADD SECTION BREAK. Drag the Section Break icon to the desired location among the pages in the wizard. A section break requires that all previous fields must be 100% complete before the user can proceed to the next page in the activity.
    6. When finished adding containers and fields, click Save Wizard on the upper left.

  3. Add or edit rule sets for the wizards. See Rule Sets topics for more detailed information.

    NOTE: You can use the same rule set for the wizard that you use for the associated form. Use the following instructions if you would like to add wizard-specific rules, such as hiding or showing a panel based on a selection made by the user.

    1. In the Wizard Designer, select a field on the wizard page and click EDIT RULES in the left pane.
    2. On the upper left of the Rules window in Filter Rule Sets, begin typing the first letters of the rule set for the wizard and click the rule set name in the filtered list to open the rule set.
    3. On the upper right of the Rules window, click NEW RULE.
    4. Enter a Rule Name, select a Rule Scope and select a pre-built rule Template from the list. Click CREATE RULE on the upper right.
    5. On the Rules window, click SAVE RULE SET on the lower right.
  4. To view whether a field has Required or Validate Input properties checked, select the field and then click AutoGen Rules. Clicking AutoGen Rules without any field selected lists system-generated rules for all wizard data items with the Required and/or Validate property checked.

  5. Add the product for the wizards and associated forms. See Products topics for more detailed information.

    1. On the navigation bar, click Product Config/Products.
    2. On the upper left of the Products page, click Add New Product.
    3. Enter the Product Name, select the Product Type, select the Product Category if the product is a Non-Insurance Product, select the applicable State Approved checkboxes. Enter any additional product information and click Save Product Config.

  6. Add the package for the product, forms, wizards and rule sets. See Packages for more detailed information.

    1. On the navigation bar, click Product Config/Packages.
    2. On the upper left, click Add New Package.
    3. On the Package Detail tab, enter the Package Name and select an ACORD Type and Area Type for the wizards and forms. If the package is available for only specific role codes, enter the role codes in Role Codes.
    4. On the Forms/Wizards In Package tab under All Forms/Wizards, press and hold the Ctrl key and select the wizards to include in the package along with the associated forms in the package. Click the right arrow to move the wizards to the Forms/Wizards in Package list. Use the up and down arrows to move the wizards into the display order.
    5. On the Rule Sets In Package tab under All Rule Sets, select one or more rule sets for the package. These rule sets contain the rules for the wizards and forms.
    6. Click Save Package.

  7. Add a sales agreement for the package. See Sales Agreements for more detailed information.

    1. On the navigation bar, click Product Config/Sales Agreements.
    2. Select a Product in the Product Agreement grid.
    3. Select a Selling Org and select the States Approved checkboxes.
    4. In the Available Packages list, select the package for the product with wizards and forms and click the right arrow to move it into the Assigned Packages list for the product.
    5. Click Save Sales Agreement.

  8. Grant access to the product. See Product Access in Organization Configuration/Groups for more detailed information.

    1. On the navigation bar, click Organization Config/Groups.
    2. In the grid, select a TC Role Code.
    3. On the Product Access tab, select the checkbox for the new Product.
    4. Click Save at the bottom of the Groups page.

  9. Deploy the product to the applicable environments. See Deploy Data for more detailed information.

    1. On the navigation bar, click Utilities/Deployments.
    2. In Synchronization Scope, select All Configuration.
    3. In Reason, type a reason for the deployment, for example, type New Product or New Wizard.
    4. In the list, select the checkbox for the product.
    5. Click Deploy Now.
