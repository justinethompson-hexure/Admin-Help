Copy Fields to a Wizard from a Wizard

In the Page Options section of the Wizard Designer, use the Copy Wizard Fields feature to copy fields to a wizard from a wizard.

  1. Go to the Designer/Wizard Designer page.

  2. In the lists, select a Wizard and select a Page.

  3. On the lower left of the wizard page in the Page Options section of the page, click the Copy Wizard Fields link to open the Wizard Copy Fields dialog.

    Search for a wizard and field name.

    1. On the upper left of the dialog under Copy From, type a search word or phrase for a wizard name. For example, type "Annuity" to search for wizards with the word "Annuity" in the wizard name.

    2. In the list of wizard names, click the name of a wizard.

    3. Below the title row of the grid, enter any or all of the following wizard data: Field Name, Page Name, Field Type, and Data Item ID to further filter the fields on the selected wizard.

  4. Select the checkboxes for the individual wizard fields to copy to the wizard. Or, select the checkbox in the title row of the Wizard Copy Fields dialog to select all the fields displayed on the dialog to copy to the wizard.

    • When mapping from a source wizard to a destination wizard page, if data items from the source wizard are not located anywhere within the destination wizard, the checkbox column on the Wizard Copy Fields dialog adjacent to the "Field Name" is checked and the checkboxes in the "Already Exists" column is not checked.

    • If a data item on the source wizard already exists in the destination wizard, the checkbox column adjacent to the "Field Name" is unchecked and the checkboxes in the "Already Exists" column are checked. This is the case even when a copy from a source wizard to a destination wizard has been completed but the wizard has not yet been saved. The copy process sees that these fields already exist on the destination wizard. They can be copied again but the boxes have to be checked as they are not auto selected for mapping.

  5. Select either the Copy All Selected Fields To The Active Page checkbox to copy just the selected fields from the source wizard to the active page or select Copy All Selected Fields and Pages checkbox to copy the selected fields and the pages the selected fields exist on to the destination wizard.

  6. On the lower right, click the Copy Selected Fields button. FireLight copies the selected wizard fields to the open wizard page in the Wizard Designer.

  7. On the upper left of the wizard, click the Save Wizard button to save the wizard.

System Rules for the Copy Wizard Fields Feature

  1. All fields present at the root level of the source wizard can be copied over to the destination wizard. Parent controls such as Panel and Group that hold other controls in them are copied along with all their child controls. The child controls are not listed while copying.

  2. The following form controls are not available for copying from a wizard to a wizard: Signature Box, Signature Initials, and BarCode.

  3. When a "Copy Wizard Fields" process completes, the source wizard fields display on the destination wizard as follows:

    1. The size of the fields copied to the destination wizard is the same as from the source wizard.

    2. All copied source wizard fields land on the destination wizard page beginning at the top left of the wizard, moving left to right across the page, and then down to the next line until all fields selected for copying are listed on the current wizard page.

    3. Where there is a one-to-one relationship between destination wizard data field properties and source wizard data field properties, the data field is to retain the same field properties on the destination wizard as they existed on the data item from the source wizard.

  4. When mapping from a source wizard to a destination wizard page, if data items from the source form are not located anywhere within the destination wizard, the checkbox column on the Wizard Copy Fields dialog adjacent to the "Field Name" are checked and the checkboxes in the "Already Exists" column are not checked.
