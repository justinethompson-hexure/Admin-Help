Add a New Rule from a Pre-Built Rule Template

  1. On the navigation bar, click Designer/Wizard Designer.

  2. On the upper left of the Wizard Designer page, select a wizard name in the list.
  3. On the wizard, click in a field. For example, click the Date of Birth field for the owner. In the Text Box Properties, the Name is the Owner_DOB dataitem name. Rules are tied to dataitem names.

  4. In the left pane, click EDIT RULES to open the Rules window.

  5. In Filter Rule Sets, begin typing the name of the rule set to filter the list, then click the rule set name in the list.
  6. The List only rules for [dataitem] checkbox is selected by default. The rules list contains only rules for the selected dataitem on the wizard page.
  7. (Optional) Clear the List only rules for [dataitem] checkbox to view all the rules for the wizard page in the list. To search for rules associated with a single dataitem, type the dataitem name in Filter Rules with Field.
  8. Click NEW RULE on the upper right.

  9. Select a Rule Scope in the list.

  10. Select a pre-built rule in the Template list. For example, select Date Past.

  11. If applicable, select an option in dataitemid.
  12. Enter a Rule Name for the new rule. For example, add the dataitem name to the rule, as in Date Past: Owner_DOB.

  13. On the upper right, click CREATE RULE.

  14. On the lower right, click SAVE RULE SET to save the rule set, which unlocks the rule set for editing by other users.

    NOTE: If the Validate Rule Name Uniqueness checkbox is selected in Organization Configuration/Organization Settings on the Admin tab, the new rule cannot be saved unless it has a unique name. Click OK on the Rule name must be unique dialog.

  15. Click CLOSE to exit the Rules window.

  16. After adding new rules and saving a rule set, add the rule set to the applicable Packages.
