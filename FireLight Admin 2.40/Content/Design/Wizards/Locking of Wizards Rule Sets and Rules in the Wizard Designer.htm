Locking of Wizards, Rule Sets, and Rules in the Wizard Designer

Locking of Wizards

The FireLight® system automatically locks a wizard for the first user who starts to make changes to the wizard. This prevents other users from editing the wizard at the same time.

For other users who attempt to open the wizard while it is locked by the first user, the system displays a "User <user name> is currently working on the wizard" message on the upper left of the wizard page above the Wizard and disables the EDIT RULES and SAVE WIZARD buttons. Other users can only view, navigate between the pages, or close the wizard while the wizard is locked.

Example: Default lock time is one hour (based on what is set in the organization's database).

  • If the user clicks the EDIT RULES button, lock time for the wizard resets to that of the rule set timeout so that both wizard and rule set timeout are the same.

  • After 55 minutes of lock time, when the user continues to edit the wizard, lock time automatically rolls over to one more hour.

  • After 55 minutes of lock time, when the user is not continuing to make changes to the wizard, the system alerts the user to save data. On the Alert box, the user can do any of the following:

    • Click the SAVE NOW button to save the changes and release the lock.

    • Click the EXTEND button to extend the lock time to one more hour.

    • Does not click a button. After 5 minutes of prompt, the system releases the lock without saving any changes, the lock icon turns gray, and the following text appears below the lock icon: "The application is inactive."

  • When the user clicks the SAVE WIZARD button, navigates to another page, or exits the wizard, FireLight unlocks the wizard. Another user can then edit the wizard.

If another user loads the Wizard without locking it and another user updates that Wizard, the system prompts the initial user to reload the Wizard to view latest changes.
