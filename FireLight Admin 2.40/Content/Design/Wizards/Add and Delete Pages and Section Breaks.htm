Add and Delete Pages and Section Breaks

A wizard page comprises its properties and one or more elements on the page, which includes section breaks, containers and fields.

About Section Breaks

If an activity has a wizard with a section break, the user cannot jump from one page to another or from one form/wizard to another unless data entry on pages prior to the section break are 100% complete. The data entry flow occurs from top to bottom (first page of the first form/wizard to the next). For the design document, see CMR 239391. "CMR 239391" in the Admin Help.

  • A Section Break must have a page prior to it.

  • Section Breaks cannot be next to each other.

  • When adding a new Section Break, it is by default added after the last page in the wizard. If there is already a Section Break present at that position, the new Section Break is not added, and FireLight alerts the admin user to change position of the previous Section Break.

  • The admin user changes position of a Section Break by using the drag and drop function.

  • During data entry, pages after the first section break are disabled or non-editable (greyed out) and once all those pages prior to the section break are 100% complete, FireLight enables the Next button to allow movement to the next page/form/wizard.

  • For FireLight embedded users, the API for embedded that returns the wizard pages and navigation graph show the section breaks and pages within the section breaks.

  • During data entry, when errors are on one or more pages within a section break, the red triangle alert icon displays at the left side of the section name in the Page List. Also, on the last page of the section break with errors, the Next button is disabled. On remaining pages within that section, the Next button is enabled for the user to move among the section pages for data entry. For user instructions, see "Page List" in the Page Navigation Page Navigation topic in the FireLight Platform User Guide.

  • When data entry is 100% complete in all the pages within a section, a green check mark icon appears to the left of the section name in the Page List. In this case, the green check mark icon also appears to the left of page names.

  • Wizards that do not contain section breaks allow the user to navigate to any page without the requirement to complete a previous section.

Add Pages and Section Breaks

  1. On the navigation bar, click Designer/Wizard Designer.
  2. Near the top left of the Wizard Designer page, select a wizard in the list.
  3. To add a page, click Add New Page on the left bar. In the Page Properties dialog, enter the Name, Title, and Subtitle for the page. Optionally, add Navigation Text for page, which replaces the Previous and Next button names when navigating to the page. The Name uniquely identifies the page for usage in rules. The Title and Subtitle are displayed when the wizard is rendered. Click Edit Page on the left bar to open the Page Properties dialog and change any of the settings.
  4. To begin editing a page, select the page you want to edit in the Page dropdown.
  5. To edit the page properties, click Edit Page on the left bar.
  6. To show dataitem names for the fields, select the Show Tags checkbox on the upper right of the page.
  7. To delete a page, click Delete Page on the left bar. The designer automatically opens the last page in the wizard if it applies.
  8. To copy a page, click Copy Page on the left bar. The designer opens the new copy.

  9. To set the page workflow, click Set Page Workflow on the left bar. In the Page Workflow dialog, click and drag each page into the desired order.

    (Optional) Add a section break to the page workflow. A Section Break requires that all fields prior to the section break must be 100% complete before proceeding to the next page or section in the activity.

    1. Click the ADD SECTION BREAK button to add a section break to the page workflow.

    2. (Optional) Click the information icon on the upper right for Section Breaks guidelines.

    3. Drag the Section Break icon to the desired location among the pages in the wizard. A Section Break can be placed between two pages or after the last page. It cannot be placed before the first page.

    4. Double-click the Section Break icon to open the Section Break Properties dialog.

      Enter a new section break data item ID in Name and enter new display text in Text. The maximum length is 75 characters for Name and for Text. The Name and Text fields cannot be empty. Do not duplicate section heading names within the wizard.

      Click OK to save the section break. Click REMOVE SECTION BREAK to remove the section break from the Page Workflow. Click CANCEL to cancel changes to section break properties.

    5. Click the OK button once the page workflow is correct, or click the CANCEL button to ignore any changes made on the dialog.

  10. Click SAVE WIZARD, which unlocks the wizard for editing by other users.

Delete Pages and Section Breaks

  1. On the navigation bar, click Designer/Wizard Designer.
  2. Near the top left of the Wizard Designer page, select a wizard in the list.
  3. On the left bar, select a Page in the list.
  4. On the left bar, click Delete Page.

  5. If a Section Break is impacted by the deleted page, a "Check Page Workflow after deletion to review potential removal of a Section Break." Alert message appears.

    NOTE: If deleting a page invalidates a section break condition, the section break containing the page is also deleted along with the page. For example, if a Section Break is after the first Page in a Page Workflow, and the first Page is deleted, the Section Break following the first page is also deleted because a Section Break cannot be before the first Page.

  6. Click Close on the message dialog to close the dialog and perform the delete/removal actions based on the Page and Section Break setup.
