Add a Custom Rule for a Selected Field

Basics for Authoring a Rule in the Rules Window

  • Starting from a template is the easiest.
  • Review the lower pane of the Rules window for validation errors. Be sure to enclose multiple actions within a <block> node, unless the actions are within a <switch> or <if> node, which treat actions within the body or after the <else/> node as if a <block> node is present. “There can only be one root node” error usually indicates that the action is not within a block.
  • On the upper right of the Rules Window, click the Node Help link for help with creating rules.

Modify an Existing Rule Template to Create a Custom Rule

Following is an example of modifying an existing Required Field rule to add a date field format for the Owner_DOB calendar control. This ensures the owner’s date of birth is formatted in MM/DD/YYYY format on the application.

  1. On the navigation bar, click Designer/Wizard Designer.
  2. On the upper left of the Wizard Designer page, select a wizard in the list.

  3. Click in a field on the wizard.

    For example, click in the Owner_DOB field on the wizard.

  4. On the left, click EDIT RULES.
  5. In the Filter Rule Sets box, type the name of the rule set if known or scroll down the list to find the rule set and click the rule set name in the list.

  6. On the left, ensure the checkbox is selected for List only rules for [Owner_DOB] (the name of the selected field on the wizard page). On the upper right of the Rules window, click the NEW RULE button.

  7. In the Template list, select the Required rule template and click CREATE RULE on the upper right.
  8. At the top of the Rules window in Rule Name, enter Required Field Date Formatted: Owner_DOB.

  9. On the right of the Rules window, add XML for the date format as follows and enter Required Field Date Formatted: Owner_DOB in Rule Name.

    <diset dataitemid="Owner_DOB">

    <format format="MM/dd/yyyy">

    <diget dataitemid="Owner_DOB" />

    </format>

    </diset>

  10. In Rule Scope, select an option to indicate when the rule is executed.

  11. On the lower right, click SAVE RULE SET, which unlocks the rule set for editing by other users.

    If the Validate Rule Name Uniqueness checkbox is selected in Organization Configuration/Organization Settings on the Admin tab, the new rule cannot be saved unless it has a unique name.

  12. Click CLOSE on the lower right to close the Rules window.

  13. When you re-open the Rules window and select the rule set, the new rule appears in the list of rules for the Owner_DOB control.

  14. After adding the new rule and saving the rule set, add the rule set to the applicable Packages.
