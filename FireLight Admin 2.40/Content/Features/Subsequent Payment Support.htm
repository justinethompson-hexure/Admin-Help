Subsequent Payment Support Feature
Description, Setup, User Experience, and Additional Information

Description

The Subsequent Payment (Sub Pay) feature adds the framework to support the collection of very basic Subsequent Premium data and the submission of the DTCC Sub file (30-35) as an outbound feed. FireLight® will have generic data items that can be used when carriers create their own SubPay forms. A template form can be provided if requested by the client. SubPay e-submissions can be processed with DTCC Money Settlement if both firms are settling members.

NOTE: See implementation details for DTCC Money Settlement.

Client/FireLight Setup

Clients need to work with their Strategic Account Manager to update their back end feeds.

  • There is not an additional license fee; however, there is a new (lower) transaction fee to support subsequent premiums. If a client already has Annuity Line of Business active and would like to add Sub Pay transactions, contact your Strategic Account Manager to get the pricing details and complete an addendum to your contract to support this new feature.

  • The inbound SSO will need to be updated to allow for additional contract level details to be pre-filled into the application.
  • The SSO will have the 508 ACORD TransType to launch directly into a Sub Pay order.
  • Carriers will need to set up Sub Pay forms and establish some rules at the product and state level for when the Sub Pay changes are allowed.
  • Clients need to work with the integration team to change their DTCC file to support the DTCC file changes.

Admin Setup

  1. Designer/Form Library page: upload a Sub Pay form on the Form Library page. has a generic form that can be used or carriers can create their own form to define the requirements.

  2. Designer/Form Designer page: open the Sub Pay form on the form Designer page and ensure the following data elements are included on the form:

    • Policy Number
    • Owner Name
    • Product Type and Jurisdiction (optional)
    • Qualified Account indicator, Tax Year (if qualified money)
    • Additional Amount (carriers can code minimum and maximum rule on this field)
    • Funding sources (ACH, Wire, Brokerage Account)

    If money settlement is being used, you need the DTCC Member ID for the settling firm in Organization Configuration/Organization Settings on the Organization tab and money settlement setup (see the DTCC Money Settlement implementation details).

  3. Organization Configuration/Groups page/Activity Access tab: select the Sub Pay checkbox for the transaction type for products that will support Sub Pay. (If the group is also allowed to create new business applications, ensure the Application checkbox is selected for the group.) If you created a generic Subsequent Payment product, select the Subsequent Payment checkbox on the Product Access tab.

  4. Product Configuration/Activities page:

    1. In the grid at the top of the page, select the Sub Pay activity.

    2. Ensure ACORD Type is set to Sub Pay (508).

    3. Update the settings for your organization.

    4. Enter the Continue/Apply Label, for example, enter Apply.

  5. Product Configuration/Products page:

    1. Select a Sub Pay product in the upper grid.
    2. In the States Approved grid, select the checkboxes in the Approved column for each state where Sub Pay is allowed for the selected product.

  6. Product Configuration/Packages page: add the subsequent payment forms to a package and set the ACORD Type to SubPay (508).

  7. Product Configuration/Sales Agreements page: add subsequent payment maintenance forms to a product by adding the subsequent payment package as part of a specific maintenance type package for the product.

    You can also create a Sub Pay only product with Subsequent Payment as the only approved Transaction Type on the Products page. Then add a package and sales agreement for the product with only the subsequent payment forms included in the assigned package.

  8. Utilities/Reports page/Admin Reports: the Agent Report and Agent Activity Report list the transaction type to allow for tracking of application, subsequent payment, and post issue transactions.

    Activity report updates include:

    • Product Line Submission Summary report now contains counts only for new business applications

    • Detailed Submission Report by Channel now contains transaction specific counts

    • An additional transaction column has been added to the detail reports - Detail Submission, DTI Detail Submission, Detail Decline Sign, Detail Submission by Channel, Detail Submission by Carrier, Pending Orders, Pending Deletion, Pending Purge, and Purged Applications.

    • Additional submission summary report listing submission counts by transaction type added

User Experience

  1. On the FireLight Home, click Subsequent Payment on the right.

    If the order initiates on the client side and the SSO into FireLight has the 508 ACORD TransType to launch directly into a Sub Pay order, skip steps 1, 2 and 3.

  2. On the Create New Subsequent Payment page, select a Jurisdiction and Product Type in the lists and then click a product name.

  3. On the product Required Forms/Optional Forms page, select the optional forms to include and click the Create button. On the Create dialog, type a name for the application and click the Create button.
  4. Fill in updated information on the application/subsequent payment forms and continue the normal application process through signature capture and submission.
