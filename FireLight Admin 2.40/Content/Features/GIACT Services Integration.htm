GIACT Services Integration

GIACT Services Integration
Description, Setup, User Experience, and Additional Information

Description

FireLight® can integrate with GIACT Systems, LLC full service payment solutions services for banking account verification and authentication. These services can be used for one-time and recurring ACH payments.

gVerify is GIACT’s bank account verification service that verifies the status of bank accounts in real time and reports any negative history associated with the account. Inquiries on currently assigned routing numbers will return the bank name assigned to the routing number.

The gVerify service verifies the account and returns an account response code indicating:

  • Open and valid account
  • Account in NSF status or excessive returns
  • Closed or frozen account
  • Account not located
  • Non-demand deposit account that does not allow posting of ACH debits

  • Special routing numbers like Government institutions

  • American Express Travelers Cheque accounts

  • Stop payment orders on a specific check number or check number range

gAuthenticate is GIACT’s bank account authentication service that takes the gVerify one step further to authenticate the customer data against the account. This service works with both personal and business bank accounts.

The gAuthenticate service will authenticate:

  • Person name or business name
  • Tax ID Number (SSN/ITIN/EIN)
  • Date of Birth
  • Address
  • Phone Number

The information returned from GIACT's services is available to be leveraged in the FireLight rules engine for additional configuration of the service. For example, clients can write rules to limit the number of attempts to validate the account before enforcing another form of payment. Clients can configure that after three attempts an error message that requires the user to select a different funding source is presented.

Clients will contract directly with GIACT to use their services.

GIACT provides the gVerify and gAuthenticate response codes. See the "Additional Information" section at the end of this topic for the Design Document, Report, and Test resources that are available in Admin Help.

FireLight Setup

There is no additional cost within FireLight. Time and materials for implementation specific rules still apply.

Admin Setup

  1. Organization Configuration/Organization Settings page, Admin tab: set up the Third-Party Service Config.

    1. To the right of Third-Party Service Config, click the Click to Edit link to open the Third-Party Service Configuration dialog.

    2. Click the Add New link at the top of the dialog.

      1. In Config Name, enter Password.
      2. In Config Value enter the password provided by GIACT.
      3. Select GIACT in Service Type and click the Add Config button.

    3. Click the Add New link at the top of the dialog.

      1. In Config Name, enter Username.
      2. In Config Value enter the username provided by GIACT.

      3. Select GIACT in Service Type and click the Add Config button.

        NOTE: Config Group is an optional configuration used by the DocuSign Integration integration.

        Examples:

    4. Close the Third-Party Service Configuration dialog.
    5. Click the Save Organization button on the Organization Settings page.

  2. Design/Form Designer or Wizard Designer page: do the following to set up a custom action button control and data items on the form or wizard.

    IMPORTANT! FireLight now supports triggering a custom action call from any page within the same package, regardless of whether the button or fields to trigger that call are on that same page. See Add Multiple Copies of a Form for use on multiple-instance forms (forms only—not wizards).

    1. Add a Button control to the form or add a Custom Action button to the wizard. Be sure to name it something that you can easily identify in rules. In the Field Properties:

      Forms:

      1. In Action Type, select Custom Action.
      2. In Action Parameter, type GIACT.
      3. In Data Group, select an EFT Validation property for the button to map to the GIACT web service. For example, select the EFT Validation 1 option if this the first EFT information validation section on the form.

      Wizards:

      1. In Custom Action, select Custom Action.
      2. In Custom Action Type, select GIACT.
      3. In Custom Action Parameter, enter GIACT.
      4. In Data Group, select an EFT Validation property for the button to map to the GIACT web service. For example, select EFT Validation 1 if this is the first EFT information validation section on the wizard.

    2. In the fields that will receive the data, open the Field Properties and select the Data Group that matches the Data Group for the GIACT button.

      For example, set Data Group to EFT Validation 1. Then select the Data Group Property that matches the field entry. For example, select FirstName in Data Group Property for the account holder's First Name field. Ensure that Data Group and Data Group Property are selected on all fields that need to be filled in with GIACT.

      When you set Data Group to an EFT Validation option, the available Data Group Property list contains options that are specifically for GIACT. Create the applicable form or wizard controls for the following Data Group Property options.

      Data Group Property Options for GIACT
      GIACT Button Data Group Properties
      *Indicates fields that are required for the GIACT button to be enabled.
      • Routing Number*
      • Account Number*
      • First Name*
      • Last Name*
      • Business Name*
      • gVerify Account Response Code
      • gVerify Verification Response
      • Bank Name
      • Bank Account Added Date
      • Bank Account Last Updated Date
      • Bank Account Closed Date
      • Check Number
      • gAuthenticate Customer Response Code
      • gAuthenticate Verification Response
      • Tax ID Number (SSN/ITIN/EIN)
      • Date of Birth
      • Address
      • Phone Number

      *Business Name is required only if not providing the First Name and Last Name.

User Experience

  1. Begin an application or other activity.

  2. Fill in information for the following required fields for GIACT:

    • First Name
    • Last Name
    • Business Name (required only when not providing First Name and Last Name)
    • Account Number
    • Routing Number

  3. Click the GIACT custom action button.

    When the web services calls complete, the GIACT information populates the appropriate fields on the form or wizard:

    • gVerify Account Response Code
    • gVerify Verification Response
    • Bank Name
    • Bank Account Added Date
    • Bank Account Last Updated Date
    • Bank Account Closed Date (if applicable)
    • gAuthenticate Customer Response Code
    • gAuthenticate Verification Response
  4. Continue with the application or other activity.
