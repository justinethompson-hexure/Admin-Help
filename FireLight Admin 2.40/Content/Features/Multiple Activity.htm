Multiple Activities
Description, Setup, User Experience, and Additional Information

Description

Multiple Activities is an activity that simplifies the creation of multiple activities in a single sale scenario. Multiple Activities serves as a "Parent" activity that is linked to "Child" activities. Multiple Activities supports a generic collection of party data in the Parent once, then allows that data to be leveraged in all of the Child activities without reentering that information multiple times. Because of the unique characteristics of this workflow, it is a unique activity/transaction type of 1204.

NOTE: Multiple Activities must be generated from a wizard or wizard and form Parent activity. Multiple Activities can pre-fill to a forms only Child application that is forms because the Custom Action Fill button is available on the forms and wizard, but pre-filling from a forms only Parent activity is not supported.

When multiple applications/activities are going to be taken during a single sales appointment process, the Multiple Activities pre-sale allows users to:

  • Enter demographic and potentially medical information on participant(s) one time and be able to apply that information to multiple activities/applications without reentering that information.

  • Identify multiple new activities/applications that they want to create.

  • When all party information has been entered and product applications are identified, users can select Create/Update to pass party and related data to the child activities/applications.

Admin Setup

  1. In Designer on the Form Library and/or Wizard Library page create a Multiple Activity form and/or wizard.

  2. In Designer, on the Form Designer and/or Wizard Designer page:

    1. Add controls to the Multiple Activities form or wizard for entering demographic and potentially medical information on participants. Common tagging for field names should begin with "Party#_[FieldName]," e.g., "Party1_FirstName."

    2. For the forms or wizards that will receive the party information from the Multiple Activities, add a Button (form) or a Custom Action button (wizard) control in each area of the forms and wizards where party information is entered from the Multiple Activity.

      In the field properties for the button/custom action:

      1. In Custom Action, select Multiple Activities Auto-Fill in the list.
      2. In Auto-Fill Prefix, enter the party type that will receive the information from the Multiple Activities (Owner, Insured, Annuitant, Joint Owner, etc.).
      3. Save the forms and/or wizards.

  3. In Product Configuration on the Activities page:

    1. On the upper left, click Add New Activity and name it Multiple Activities, for example.
    2. In ACORD Type, select Multiple Activities (1204).
    3. Click the link and click a Font Awesome Icon to use for Multiple Activities. Enter the name of the Font Awesome Icon class in the field (for example, enter fas fa-ellipsis-h for a horizontal ellipsis).
    4. In Continue/Apply Label, enter the label name for the Continue button (e.g., enter New Activity, Application, etc.).
    5. Fill in any other applicable settings and click the Save Activity button.

  4. In Product Configuration on the Products page:

    1. Add a Multiple Activity product.
    2. Select a Product Type and Product Category and select the states where it is approved.
    3. Fill in any other applicable settings and click the Save Product Config button.

  5. In Product Configuration on the Packages page:

    1. Add a Multiple Activity package.
    2. In ACORD Type, select Multiple Activities (1204) in the list.
    3. In Area Type, select Activity.
    4. Add the Multiple Activity form or wizard and any applicable rule sets to the package.
    5. Fill in any other applicable settings and click the Save Package button.

  6. In Product Configuration on the Sales Agreements page:

    1. In the grid at the top of the page, select the Multiple Activity product.
    2. On the left, fill in applicable settings and select states in State Approvals.
    3. Under Available Packages, select the Multiple Activity package and click the right arrow to move it to the Assigned Packages list.
    4. Click the Save Sales Agreement button.

  7. In Organization Configuration on the Groups page:

    1. In the grid at the top of the page, select a group Name/TC Role Code.
    2. On the App tab in the Other section, select the Enable Multiple Open Activities checkbox.
    3. On the Activity Access tab, select the Multiple Activities checkbox.
    4. On the Product Access tab, select the Multiple Activity checkbox.
    5. Click the Save button.

User Experience

NOTE: In this example, the child activities are applications.

  1. On the Home page under Start New, click Multiple Activities or on the navigation bar, click New Activity > Multiple Activities.
  2. Select a Jurisdiction and click a Multiple Activities name in the list.
  3. Select the Multiple Activities checkbox and click the Create button.
  4. Enter a name for the Multiple Activities and click the Create button.
  5. Fill in the information for each Party. Parties are numbered as Party1, Party2, etc., so a single Party can be designated, for example, as an Owner/Annuitant in one child activity and as a Beneficiary in another child activity.
  6. When finished adding party information, click the Next button.
  7. Select an activity type in the list and click the Add Activity button.
  8. In the lists, select a Jurisdiction, Locale (if applicable), Product Type, and Carrier (if applicable). Then select a product in the grid.
  9. Click the Create button.
  10. Enter a name for the activity and click the Create button to create a child activity. The activity appears in the Multiple Activity grid.

  11. Add another child activity. Select an activity in the list and click the Add Activity button. Repeat the process for creating each child activity under the Multiple Activity parent activity.

    To remove an activity, click the trash can icon in the Remove column of the Multiple Activities grid.

  12. When all child activities are added, click the Create button on the upper right to create the child activities.
  13. On the Open Activities menu on the navigation bar, click the Open Activities down arrow and click a child activity in the list.
  14. Click the custom action button and click a party name to pre-fill the information from the Multiple Activity into the open application.

  15. To add or change information in the Multiple Activity and update the child activities:

    1. On the navigation bar in the Open Activities menu, click the name of the Multiple Activity.
    2. Add or change field entries in the Multiple Activity.
    3. Click the Update button on the upper right to update the information in the Multiple Activity and associated activities.

  16. Continue filling in the application information and go through the normal steps to complete the application process for each child application.
