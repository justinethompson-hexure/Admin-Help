Carrier Testing as a Distributor
Description, Setup, User Experience, and Additional Information

Description

FireLight® provides a means for a carrier to test inside of a distributor environment. This allows carriers that have an active sales agreement with a distributor to have visibility of how their carrier forms, wizards, and rules function inside the distributor's environment, which could possibly avoid potential issues caused by rule conflicts between carrier forms and distributor forms.

In FireLight Admin, the distributor controls the level of carrier access to the distributor environment through a Carrier Test Group role code and activates the carrier testing view via organization settings.

Guidelines:

  • Carrier testing as a distributor is available in all environments except Production.
  • Carrier Single Sign On (SSO) testing is not available in the distributor environment.
  • Carrier access is based on active sales agreements, which means carrier users with access to the distributor environment can access only their own carrier activities and products in the distributor's carrier testing environment. They do not have access to any other carrier's activities or products.
  • When the carrier user logs in, they have the carrier role code, but the distributor role code is what manages product and activity access.
  • Using the distributor carrier test group role code, carrier users can create activities and submit orders from the distributor environment to test outbound feeds (App/Sub or 103 or Name Value Pairs is sent). This allows carriers to do full end-to-end testing in the distributor environment and make sure the outbound files are correct.

Admin Setup

Distributor:

Allow carriers to test in your distributor environment. Do the following to add a carrier test group with a TC Role Code defined for the group and activate carrier testing in the distributor environment.

  1. In Organization Configuration on the Groups page, add a carrier test Group with a Name and TC Role Code entered for the group. Then select the group options for the carrier test Group.

    NOTE: This example uses "Carrier Test Group" for a sample name. If you already have a carrier test group set up, you do not need to create a new carrier test group Name and TC Role Code.

    1. On the Activity Access tab, select the applicable activity checkboxes for the Carrier Test Group. Ensure all appropriate activities are selected for the Carrier Test Group.
    2. On the Product Access tab, select all the product access checkboxes per Carrier selling organization for the Carrier Test Group.
    3. Set any additional group options for the Carrier Test Group.
    4. At the end of the page, click Save.
  2. In Organization Configuration on the Organization Settings page, Activity tab, select the Enable Carrier Testing View checkbox and select the Carrier Test Group in the dropdown list.

Carrier:

Allow carrier users to test your carrier products, forms, wizards, and rules in the distributor environment.

  1. Ensure your carrier products, packages, and sales agreements are deployed to the distributor's UAT or Staging environment using the Distributor All Configuration synchronization scope. See Deploy Data for deployment information.
  2. In Organization Configuration at the top of the Groups page, select the carrier role that will do the testing in the distributor's environment.
  3. On the App tab in the Testing section, select the Enable Distributor Testing View checkbox.
  4. At the end of the page, click Save.

User Experience

When a carrier user (with the distributor testing view enabled) logs in to FireLight, a Distributor Testing View list appears above the Recent Activity list. This list contains only the distributors that have the carrier testing view enabled and have active selling agreements with the carrier.

  1. On the FireLight Home page, select a distributor name in the Distributor Testing View list.

    FireLight transitions from the logged in Carrier Testing group Name/TC Role Code to the Distributor Testing group Name/TC Role Code in the distributor environment.

  2. Create an activity in the selected distributor environment.
  3. Complete the activity in the distributor environment.
  4. Submit the activity from the distributor environment to test outbound feeds (App/Sub or 103 or Name Value Pairs is sent). The submission should work just like it is a distributor submitting the activity.
  5. To return to your carrier testing view: go to the FireLight Home page in the distributor view and select your carrier organization in the dropdown list above the Recent Activity list.
