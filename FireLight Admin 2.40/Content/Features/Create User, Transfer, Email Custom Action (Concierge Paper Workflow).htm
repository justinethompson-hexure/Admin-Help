Create User, Transfer, Email Custom Action (Concierge Paper Workflow)
Description, Setup, User Experience, and Additional Information

Description

Organizations that are using FireLight® can extend their service to include a "concierge" workflow to convert paper transactions being submitted into digital transactions using FireLight Admin. The only optional functionality requiring Hexure involvement would be branding. This service is supported for both SSO and FireLight Access organizations.

Admin Setup

The Custom Action functionality for Concierge Workflow may vary based on organization and specific workflow. One premise of the Concierge Workflow is to convert an application submitted on paper to a digital application in FireLight.

  1. Create a multi-parameter configurable Custom Action Button in a form or wizard, set the following required field properties for the button, and save the button and form or wizard.

    • Name - enter the data item name of the button.
    • Text - enter the display name of the button.
    • Action Type - select Custom Action in the list.
    • Custom Action Type - select Multiple Action in the list.
    • Action Parameter (form), Custom Action Parameter (wizard) - select or enter the applicable action parameters. The format for entering parameters is "parameter 1," "parameter 2," etc. As the sample naming (CreateUser, TransferActivity, EmailUser) implies, the parameters identify the action and the entry order identifies the action sequence. It is not necessary to number parameters (e.g., CreateUser, TransferActivity, EmailUser).

  2. Create New User: If one of the custom action parameters is to create a new user in FireLight, add the following required fields/data items to the form or wizard for collecting user creation and authentication information.

    • CompanyProducerID - to be collected in field tagged FLI_NEW_USER_MAP_CODE
    • RelationRoleCode - to be collected in field tagged FLI_NEW_USER_ROLE_CODE
    • EMailAddress - to be collected in field tagged FLI_NEW_USER_EMAIL
    • FullName - to be collected in field tagged FLI_NEW_USER_NAME

    If the organization intends to use the last four of Government ID and DOB for authentication into FireLight, also add the following fields/data items to collect those data in the form or wizard:

    • NewUserDOB - to be collected in field tagged FLI_NEW_USER_DOB
    • NewUserSSNLast Four - to be collected in field tagged FLI_NEW_USER_SSN_LASTFOUR
  3. Transfer: If one of the custom action parameters is to transfer the activity from the back office to a specified FireLight user, the transfer action is triggered on selection of the custom action button. A valid FireLight User ID must exist before the back office user clicks the custom action button—even if the user was created milliseconds prior to triggering the transfer action by clicking the custom action button.
  4. Email: If one of the custom action parameters is to send an email to the agent or advisor, ensure the Transfer Activity Request and Transfer Activity with Direct Login Link concierge email templates are set up in the Organization Configuration > Email > Transfer Activities Email CategoryTransfer Activities Email Category.

User Experience

Back Office/Case Manager Workflow:

  1. Back office user enters FireLight and keys in the paper application.

    NOTE: Some organizations are exploring alternative entry methods (i.e., OCR scanning, etc.). This step need not be manual, but that will depend on the back-office process defined by each organization. The significant aspect is that the application is initiated by someone other than the writing agent.

  2. Back office user submits In Good Order (IGO) applications.

  3. If the application is Not In Good Order (NIGO) and the agent is not an existing FireLight user, the back office user clicks a custom action button on a form or wizard in FireLight to initiate creation of the agent in FireLight, transfer ownership of the application to that agent, and notify and provide access to the application through an email to the agent.

    A popup message displays, "Activity has a pending transfer of ownership." If the agent/user already exists as a user in FireLight, based on the User ID entered, the activity will still be transferred to that user.

Agent/Advisor Experience:

  1. Agent receives an email with verbiage defined by the organization.
  2. After clicking the Login link in the email, the agent is taken directly to Data Entry for the specific application that requires their attention. If desired, a branding code could be passed in the login to modify FireLight.
  3. Agent completes the requested changes and follows the particular workflow defined by the Organization.
