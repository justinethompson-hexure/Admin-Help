LexisNexis Risk Classifier Feature
Requirements, Description, Integration, Setup, User Experience, and Additional Information

Contracting and Legal Requirements

IMPORTANT!

All clients that use the LexisNexis service are required to contract directly with LexisNexis for their IDs. Please note LexisNexis imposes reporting and auditing requirements based on data returned from MVR, ScriptCheck, Experian, and any other service utilized by LexisNexis Risk Classifier. It is up to the client to understand their requirements and work with LexisNexis to ensure they are meeting their contractual requirements. We have highlighted a few of the requirements as they pertain to FireLight.

Following are a few areas that are the responsibility of the client:

  • To ensure that firms have the data to meet the reporting requirements, firms will need to receive the signed approvals for completed orders (if done in FireLight) and pending orders that hit the service. Firms will work with their provider to receive the documents. See the Provider section below.

  • Clients that implement LexisNexis Risk Classifier are held to regulatory requirements to gather client approval prior to hitting the FCRA-regulated services that LexisNexis Risk Classifier will call. Hexure recommends achieving this within a presale workflow. Please contact your Strategic Account Manager to discuss this further.

  • Clients that implement LexisNexis Risk Classifier must use the Reason codes in the XML Response to generate any potential Adverse Action Letter for their users.

  • Clients that implement LexisNexis Risk Classifier must use the Reason codes in the XML Response to generate any potential address discrepancy, Consumer statement, subject deceased, or any other sensitive situation that needs to be communicated to their users.

  • FireLight can use rules to display messages to clients per reason or error code that is received back from the LexisNexis Risk Classifier. However, these rules will need to be customized per client, and can be implemented by either the FireLight Implementation team, or the clients.

Description

The LexisNexis Risk Classifier service provides clients the ability to call into LexisNexis to access their Risk Classifier functionality. In FireLight, this requires a Custom Action call for this service that can be used prior to the activity reaching 100% completion.

Upon clicking this button, the custom action call sends back scores, descriptions, and general information from the Motor Vehicle Department and Consumer Credit bureaus on the user that can be used by the company's underwriting department to return a faster approval/rejection rating than what occurs today.

The Reason Code and Description are sent back to the client both in the name/value pair XML and in a PDF format as part of the application package through the outbound process. This can be added to the application package via a form template that utilizes document types.

These scores and general information will also be available within the application's audit history and report, which can be referenced until the application's purge from FireLight.

LexisNexis Usage Detailed Activity Report

This report assists users in triaging any activity associated with the LexisNexis services: including, but not limited to, the RDP Quiz, Instant ID, and the Risk Classifier. Following are items specifically available in the LexisNexis Usage Report to help clients triage their LexisNexis activities. See Detailed Activity Reports in Admin Help for a sample report.

  • Instead of showing the "Status during time of call" on the LexisNexis Usage Report, the column is now renamed to the "Service Type" column. The Service Type column shows "Instant ID," "Quiz," or "Risk Classifier" to distinguish among the various LexisNexis web services.
  • The new "Quoteback ID" column shows the QuoteBack sent in the XML response in the following order: Application ID/Selling Org/Product Org/Application Name. This value is up to 60 characters long.

LexisNexis Rick Classifier Admin Report

This report provides an Admin report that includes LexisNexis Risk Classifier Ref ID, Quoteback ID, Result Code(s), Risk Score, and Reason Code(s) during the dates specified. See Admin Reports in Admin Help for a sample report.

Submission Endpoint

The LexisNexis implementation requires the client to ensure their submission endpoint contains the necessary data points from the LexisNexis call to satisfy the reporting requirements outlined in the LexisNexis contracting. Due to those contracting requirements, clients also need to design into their workflow the ability to receive the LexisNexis data points and any returned documents for incomplete orders that were never transmitted. Please see the Provider Integration for steps to ensure you receive the necessary data.

FireLight Provider/Integration Setup

Provider Work needs to be completed to use the LexisNexis Services. Please contact your Strategic Account Manager to contact the Implementation team to complete this work.

For LexisNexis Risk Classifier, FireLight creates a hook from the provider into the base system of the service request table to be able to send the Name/Value pair XML to the client. This hook limits the information that it is pulling back, so as not to create any memory issues or size limitations on what is sent to the client.

Admin Setup

  1. In Organization Configuration on the Organization Settings page, Admin tab: set up the LexisNexisRisk credentials. LexisNexis supplies the configuration AccountID, ClientID, and values. NOTE: A Provider Work Needed popup message appears when you click the Add Config button. Click the OK button to close the popup. The Provider work is referring to the development work for data in the datafeed for processing in the backoffice. Contact your Strategic Account Manager for assistance.

    Click the Click to Edit link next to Third-Party Service Config and enter the LexisNexisRisk service credentials (configuration IDs and values):

    1. Click the Add New link at the top of the dialog. In Config Name, type AccountID. In Config Value, type the LexisNexis AccountID value. In Service Type, select LexisNexisRisk and click the Add Config button.

    2. Click the Add New link at the top of the dialog. In Config Name, type ClientID. In Config Value, type the LexisNexis ClientID value. In Service Type, select LexisNexisRisk and click the Add Config button.

    3. Click the Add New link at the top of the dialog. In Config Name, type CarrierCode. In Config Value, type the LexisNexis CarrierCode value. In Service Type, select LexisNexisRisk and click the Add Config button.

    4. Click the Add New link at the top of the dialog. In Config Name, type Password. In Config Value, type the LexisNexis Password value. In Service Type, select LexisNexisRisk and click the Add Config button.
    5. Click the Add New link at the top of the dialog. In Config Name, type Username. In Config Value, type the LexisNexis Username value. In Service Type, select LexisNexisRisk and click the Add Config button.
    6. Click the Add New link at the top of the dialog. In Config Name, type Workflow. In Config Value, type the LexisNexis Workflow value. In Service Type, select LexisNexisRisk and click the Add Config button.

    NOTE: Config Group is an optional configuration used by the DocuSign Integration integration.

  4. Add applicable form rules.

    FireLight will associate form rules with client-defined dataitems through data groups and data group properties. For example, the client can use the Score in the “background” to determine an underwriting decision. FireLight uses the values sent back from LexisNexis to trigger error messages. The client can take this information and code within the form rules a message to return to the user on the error and what the next steps are to take.

User Experience

Prior to the activity reaching 100% completion in the FireLight data entry process for a signer type with LexisNexis Risk Classifier enabled:

  1. The agent enters the LexisNexis Risk Classifier required (and recommended if available) signer information into the associated signer information fields and the policy issue state and policy face amount on the form or wizard.
  2. The agent clicks the LexisNexis Risk custom action button to initiate a call to the LexisNexis Risk web service.
  3. LexisNexis Risk Classifier service returns Risk Scores, Result Codes, up to 4 Reason Codes and 4 Descriptions, a Result Message, and an Order Number in the associated fields on the form or wizard.
  4. The company's underwriting department uses the risk scores, descriptions, and general information from the Motor Vehicle Department and Consumer Credit Bureaus on the signer to return back a faster approval/rejection rating.

  5. All of the actions taken with the LexisNexis Risk Classifier are included in the audit history and the report that is returned back to the client.

    For example, "FireLight made a web service call on behalf of Hexure to LexisNexis Risk Classifier for the `Owner` for activity `test riskclassifier - APM Test LN Service` and received score_result, national_credit_file, motor_vehicle_report pdf reports with Order Number HJ4654."

    If an error occurs and no files are sent, a "no" is added to show no reports were received. For example, "FireLight made a web service call on behalf of Hexure to LexisNexis Risk Classifier for the `Owner` for activity `LN RC test - JB Forms Only` and received no PDF reports with Order Number KI4657."

    If an error occurs during the LexisNexis Call, no order number is added to the audit. For example, "FireLight made a web service call on behalf of Hexure to LexisNexis Risk Classifier for the `Owner` for activity `LN RC test - JB Forms Only` and received no pdf reports."

NOTE: When the custom action call has occurred, the application is able to be copied, but the LexisNexis values (Risk Score, Result Code, Result Message, Reason Code 1-4, and Reason Messages 1-4) will not be copied over to the next application.
