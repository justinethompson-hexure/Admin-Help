ExamOne Paramed Exam Order Integration
Description, Setup, User Experience, and Additional Information

Description

This Integration will allow for a Paramed exam to be scheduled through FireLight by sending basic proposed insured’s demographics and account information to the ExamOne ScheduleNow Paramed online ordering system. Based on the proposed insured’s demographics and if the insured prefers their exam to be completed at an In-Center (ExamOne Office) or Mobile (Personal Location), it will open up the location(s) availability. Upon choosing the location, the dates and times calendar will open for the insured to choose the desired slot. If the desired location, date and time is available, then the appointment is locked in for the insured. This new functionality is managed via a custom action button within the activity. All clients that want to use this service are required to contract directly with ExamOne.

Impacts:

  • Activities: Adding checkbox to enable Paramed ordering

  • Admin: Third-Party Configuration - Adding a new Service Type

  • Data Group Properties

  • Email Templates: different statuses on Paramed ordering

  • UI Changes: Statuses

  • Common Tags: life insurance-related

  • API Structure: send required insured data and account info, order ID and availability will open and save in the database. We will then submit the order to ExamOne with the same tracing ID and lock the order with the schedule

  • Forms: Data Group Properties

  • Reports: Usage

  • Audits: Addition of new codes needed for ACORD 1122

NOTE: Paramed ordering is currently not available with E-Signature in React for FireLight embedded clients.
