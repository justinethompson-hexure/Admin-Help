Redtail Integration Feature
Description, Setup, User Experience, and Additional Information

Description

FireLight® can integrate with Redtail to pull in client data from the Redtail CRM.

Currently, you can map the following Redtail properties to the corresponding Data Group Property settings within FireLight fields:

  • FirstName
  • MiddleName
  • LastName
  • Occupation
  • MarStat
  • Gender
  • BirthDate
  • Age
  • Citizenship
  • City
  • AddressState
  • Zip
  • AddressCountry
  • YearsAtAddress
  • CountyName
  • AreaCode
  • DialNumber
  • SSN
  • Email
  • Business Name

There are two ways to pull in the user ID and passwords for this integration. The best solution depends on how your firm has contracted with Redtail.

  1. FireLight can collect the user ID and password during application data entry and use those credentials to connect to Redtail or users can enter their user ID and password in the CRM Preferences section of the Preferences page and FireLight will automatically supply their credentials to Redtail. This is best for user/seat licenses within Redtail.

  2. FireLight can pull credentials from the organization configuration setting for firm wide contracts.

All contracting is done directly with Redtail. Redtail will supply the user name and password credentials.

Admin Setup

IMPORTANT! FireLight now supports triggering a custom action call from any page within the same package, regardless if the button or fields to trigger that call are on that same page. See Add Multiple Copies of a Form for use on multiple-instance forms.

  1. In the Form Designer or Wizard Designer, the following changes are needed to add a button control (form) or custom action button (wizard) and CRM data into the form (e.g., a coversheet form).

    • Add a Button to the form or wizard. In the Field Properties, select CRM in Action Type. Select a CRM Type option in the list: select the Redtail option to pull data from the Redtail CRM. Select the Preferences option to pull data from the CRM set up on the user's Preferences page in FireLight. In the list, select a Data Group for the button. For example, select the Owner option to populate owner information in the associated fields.

    • In the associated fields, select the Data Group that matches the Data Group for the CRM button and select the Data Group Property that matches the field entry. For example, in the Owner_FirstName field, select Owner in Data Group and select FirstName in Data Group Property. Ensure the Data Group and Data Group Property are selected in all fields that need to be filled in.

  2. Set up the Redtail credentials at either the organization level or at the advisor level.

    Organization level:

    In Organization Configuration on the Organization Settings page, Admin tab, set up the Redtail credentials for the organization. (Note: It will use the form's Organization if set up, not necessarily the distributor's or carrier's Organization.)

    1. Click the Click to Edit link next to Third-Party Service Config.

      The following inputs are case sensitive.

      • Click the Add New link at the top of the dialog. In Config Name, type Username. In Config Value, type the Username Redtail provided for your organization. In Service Type, select Redtail CRM and click the Add Config button.
      • Click the Add New link at the top of the dialog. In Config Name, type Password. In Config Value, type the Password Redtail provided for your organization. In Service Type, select Redtail CRM and click the Add Config button.

      • Click the Add New link at the top of the dialog. In Config Name, type Workflow. In Config Value, type the Workflow credential that Redtail provided for your organization. In Service Type, select Redtail CRM and click the Add Config button.

        NOTE: Config Group is an optional configuration used by the DocuSign Integration integration.

    2. Click the Save Organization button on the Organization Settings page.

    Advisor level:

    NOTE: To enable Advisors to enter their Redtail CRM credentials in FireLight, select the Enable CRM Preferences checkbox in Organization Configuration/Groups on the App tab.

    Advisors add their login credentials and FireLight will pull those credentials each time they connect to Redtail.

    1. Open the Preferences page.
    2. In the CRM Preferences section in CRM, select Red Tail.
    3. Enter your User Name and Password that Redtail provided and click Save.

  3. In Product Configuration on the Packages page, add the form with the new button and data mapping to the applicable product packages.
  4. In Utilities on the Deployments page, deploy the Organization Configuration and deploy the Product Configurations or Package Configurations that include the forms or wizards with a Redtail CRM button.

User Experience

  1. Begin an application or other activity.

  2. Click the Redtail CRM button.

  3. On the upper right of the My Contacts dialog, click the first letter of the contact's last name or commercial name to filter the list.

  4. In the list of names on the left, click a name to view the contact's information on the right.

  5. Click the Select button to populate the contact's information on the form.
