Enrollment Activity
Description, Setup, User Experience, and Additional Information

Description

The purpose of the Enrollment activity is simplification of the Application build and Participant signature aspects of the enrollment process. This new functionality improves the workflow experience for Agents and Case Managers (CM). The streamlining of this functionality allows users to manage the workflow much more efficiently and allow the sponsoring carrier to enter a market involving larger group/enrollment scenarios.

Major aspects of this activity include the following:

  • Case Managers are the primary FireLight users in the workflow. The Case Manager accesses FireLight through their existing SSO.

  • Case Managers start the new enrollment activity by selecting it from the FireLight home page. On selection, the activity launches and presents 3 data collection screens/views.

    • “Plan” page – Plan information is collected on the first screen. Plan information may include but is not limited to Employer ID (required), Employer Name, Ownership, Plan Type, etc. Full definition of information collected to be defined. This information is passed via common tag to linked/child applications that is created for each participant. Data collected/required will vary by organization. It may be necessary to assure fully completed plan information exists prior to allowing navigation to the Participant page. Currently, the Enrollment process supports only the Primary Owner and the Primary Insured.

    • “Participant” page – The participant page will allow addition, deletion, and update of participants in the plan. The primary method of adding participant information is through census upload. The Participant page is presented in a grid view with columns to be defined. The grid view must support validation to identify situations where required information is not present or is formatted incorrectly.

    • Census Upload – The Census includes participant demographic information and may include product/benefit information. Product information included is used for the purpose of stubbing out product applications for each participant. For this reason, the product/benefit fields included in the census upload vary by Product Type.

  • Manual addition, deletion, and update of records/rows of the participant page grid should be supported. For smaller groups, the user may choose to simply add/edit participant information directly. In this scenario, there needs to be functionality to trigger stubbing out of new or edit of existing applications.

  • This page displays a link for each application “stubbed out” through the census and manual entry process.

  • Case Managers use existing FireLight functionality to share the Enrollment activity with the agent of record on the case.

  • Information of the agent of record on the case appears on the application and not information for the Case Manager.

  • Verify that agents have visibility of all child/linked applications associated with the enrollment/parent activity.

  • Case Managers are able to send multi-life/batch signature requests to participants: The system supports multi-life/batch:

    • “Signature Only”

    • “Client Fill and Sign”

  • Type of signature required is driven by the signer type/role in relation to the application. Agents have visibility of their cases, and only their cases through the share initiated by the Case Manager.

  • Agents are not required to interact with or sign Applications.

  • Permissions of what the agent can do with the applications are defined by group/role permissions.

  • The audit from the Enrollment is sent to the next activity. The audits on both the Enrollment and the next activity will include an audit record of the linkage.

  • A new activity report shows the enrollment process. This report is a detailed activity report and is available to be sent to the organization via email on a daily, weekly, or monthly basis. See the "FireLight Enrollment Report Active Applications" in Detailed Activity Reports.

Admin Setup

Create a new Enrollment activity.

  1. On the upper left of the Product Configuration/Activities page, click Add New Activity.

  2. On the Activity Detail tab, enter or select the following settings.

    Activity Name = Enrollment (this is editable)

    ACORD Type = Enrollment (1204-13000)

    Activity Type = Pre-Sale

    Branding Icon = selected Font Awesome Icon

    Continue/Apply Label = Apply (client-defined)

  3. Click Save Activity.

Create an Enrollment wizard and Plan Design wizard page.

  1. On the upper left of the Designer/Wizard Library page, click Create Wizard.

  2. On the Wizard Detail tab, enter the following settings:

    Wizard Number = Enrollment (this is editable)

    Wizard Name = Enrollment (this is editable)

    State Approved list = select the states where the Enrollment product is approved

  3. Click Save.

  4. Click Wizard Designer to go to the Wizard Designer.

  5. In the Enrollment wizard, click Add New Page. This page captures the information needed to tie participants to a batch enrollment.
    In the Page Properties dialog, enter the following settings.

    Name = [name of the data item for the page, e.g., PlanDesign, etc.]

    Title = [display name of the wizard page, e.g. Plan Design, etc.]

    Click OK to save and close Page Properties.

  6. Drag panels and fields from the right column onto the wizard page. The page must contain a required Group ID field, which ties the application back to the enrollment process and a product CUSIP ID. Following is an example layout of the required Group ID field, required CUSIP ID field, and some optional fields that it could also contain.

    Add an Agent Information Panel with the following fields, especially if the agent is not participating in capturing the information on each participant.

    Agent Name, Text Box

    Agent Email, Text Box

    Add a Plan Information Panel with the following fields.

    • Group ID = Text Box, Required, Field Name and Common Tag Name = FLI_COMPANY_ID

    • Company Name = Text Box, Field Name and Common Tag Name = Insured_EmpDetails_EmployerName

    • Owner Name = Text Box, Optional, Field Name and Common Tag Name = Owner_FullName

    • Owner Email = Text Box, Optional, Field Name and Common Tag Name = Owner_Primary_EmailAddress

    • Type of Coverage = Dropdown List or group of Radio Buttons, Optional, Field Name = Enrollment_ProductSelection

    • Child Application CUSIPID = Text Box, Required, Field Name = Enrollment_CUSIPID

  7. Click Save Wizard.

Add Enrollment rules.

  1. On the Product Configuration/Rule Sets page, click Add New Rule Set on the upper left.

  2. Enter or select the following settings for the new rule set.

    Rule Set Name = the rule set name, e.g., Enrollment Rules

    Rule Set Type = User Interface

    State Approved = select the states in which the Enrollment rule set is approved

  3. Add any other settings as needed.

  4. Click Save Rule Set.

  5. Click the View All Rule Definitions link to open the Rule Definitions Window. Manually add the rules and click Update. Optionally, add rules from the Enrollment wizard page in the Wizard Designer.

Add an Enrollment product.

  1. On the upper left of the Product Configuration/Products page, click Add New Product.

  2. On the Product Details tab, enter or select the following settings.

    Product Name = Enrollment

    Product Type = Non-Insurance Product

    Product Category = Non-Insurance Product

    Product Type Filter (UI) = Non-Insurance Product

    Billing Code = Normal

    Product Locale = (select applicable locale)

    State Approved list = select the states where the Enrollment product is approved

  3. Click Save Product Config.

Add an Enrollment package.

  1. On the Product Configuration/Packages page, add an Enrollment package.

  2. On the upper left, click Add New Package.

  3. On the Package Detail tab, enter or select the following settings.

    Package Name = Enrollment

    ACORD Type = Enrollment (1204-13000)

    Area Type = Activity

  4. On the Forms/Wizards in Package tab, drag each Enrollment form and wizard from the All Forms/Wizards list to the Forms/Wizards in Package list.

  5. Click Save Package.

  6. On the Rule Sets in Package tab, drag each Enrollment rule set from the All Rule Sets list to the Rule Sets in Package list.

  7. Click Save Package.

Add the Enrollment sales agreement.

  1. On the Product Configuration/Sales Agreements page, add an Enrollment sales agreement.

  2. In the grid at the top of the page, select the Enrollment product.
  3. On the left, fill in applicable settings and select states in State Approvals.
  4. Under Available Packages, select the Enrollment package and click the right arrow to move it to the Assigned Packages list.
  5. Click the Save Sales Agreement button.

Grant access permissions to the Enrollment activity.

  1. In the grid at the top of the Organization Configuration/Groups page, select a group Name/TC Role Code.

  2. On the Activity Access tab, select the Enrollment checkbox.

  3. On the Product Access tab, select the Enrollment checkbox.

  4. Click the Save button.
