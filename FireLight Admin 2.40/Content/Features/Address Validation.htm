Address Validation - Lob.com API Integration
Description, Setup, User Experience, and Additional Information

Description

NOTE: This service is not Health Insurance Portability and Accountability Act (HIPAA) compliant.

This feature integrates FireLight with the Lob.com address validation API to accomplish address validation on a set of fields without the user needing to trigger the validation via a button, etc.

NOTE: Only the Production environment uses the live instance. Our LOB.com account has a key and endpoint defined for a test region and a live region. Production is the only environment that uses the live environment.

The API to be used is described at https://lob.com/docs/java#us_verifications.

Admin Setup

  1. Designer/Form Designer or Designer/Wizard Designer page: On the forms or wizards, add the data items that will capture the information needed for the rule: Line 1, Line 2 (optional), City, State, and Zip (first 5 characters) address input fields and a field to display address validation error or warning message text (e.g., lob_Warning_Text data item). Save the forms and wizards.

  2. Product Configuration/Rule Sets page: Add the rule for the address validation. See the AddressValidation rule node help for a sample rule. Save the rule set.

    The rule returns a string value so that any resulting behavior can be set up in response. For example, the resulting behavior can be an error/validation message, forcing the user to correct the issue before being able to submit the activity, or it could be a warning/validation message instead.

  3. Product Configuration/Packages page: Add the rule set with the address validation rule to the applicable packages that contain the forms and wizards that will use the address validation. Save the packages.
  4. Utilities/Deployments page: Deploy the Forms/Wizards, Rule Sets, and Packages changes to the applicable environments.

User Experience

NOTE: Requests made in the test environment always validate request arguments, simulate live environment behavior, and enforce rate limits. They never print, mail nor, for verification services, verify addresses. Using this feature in any FireLight test environments must follow the input rules specified at https://docs.lob.com/#us-verification-test-environment.

  1. Create a new activity that includes address validation on the form or wizard.
  2. Alerts could appear for fields that have the address validation rule applied. For example, click Alerts to display a "Please enter a full address" alert message.
  3. Fill in the applicable address fields for the address validation rule.
  4. If an address field is not complete or the data entry does not match a USPS valid address, an Address Validation popup message appears with a corrected address. Click Accept to accept the corrected address or click Decline to decline the corrected address.
  5. Continue with data entry.
