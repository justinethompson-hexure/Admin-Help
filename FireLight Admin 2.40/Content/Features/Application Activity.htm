Application Activity

Application Activity
Description, Setup, User Experience, and Additional Information

Description

The application activity allows users to create an e-Application for submission to back office processing.

Application activity requires a licensing component. Contact your Strategic Account Manager for details.

Admin Setup

When you license the Application activity, Hexure adds the Application activity to the grid at the top of the Activities page in Product Configuration. Do the following to set up FireLight Admin for the Application activity.

  1. Upload Forms and Create Wizards:

    Add or create forms and wizards for the applicable activities in FireLight.

    • Upload existing forms (in PDF or XPS format), add form detail, and view form history using the Designer/Form Library page. See Upload Forms and Add and Update Form Detail for instructions. NOTE: Forms with existing named tag fields as added through Adobe Acrobat is present on the uploaded forms. Be sure to highlight the existing fields on the forms in Adobe Acrobat before importing the forms to FireLight.

    • Create Wizards on the Designer/Wizard Library page. For a shortcut, download and copy the Annuity Template Wizard and rule set provided by Hexure and then modify the wizard and rule set for your organization. See Add a Template Wizard and Rule Set for instructions.

  2. Design Forms and Wizards:

    • Design forms using the Designer/Form Designer page by selecting master settings and adding input controls, field properties, and rule sets or rules.

    • Design wizards using the Designer/Wizard Designer page by adding pages, containers, fields, and rule sets or rules to each wizard.

      NOTE: You need to create rule sets on the Product Configuration/Rule Sets page first. See "Add a Rule Set and Detail" in Rule Sets for instructions. Then you can add the rules either on the Rule Sets page or from the Rules Window link on the Form Designer page or from the Rules Window link on the Wizard Designer page. You can use the same rule set for both wizards and forms.

  3. Set Up Product Configurations:

    1. Set Application activity options on the Product Configuration/Activities page. In the grid at the top of the page, select the Application activity. Update the settings for your organization. To add a pre-submission review step to the application process, select either the Allow Pre-Signature Review or Allow Post-Signature Review checkbox depending on whether the pre-submission review is before or after the signature step in the application process.
    2. Add the products on the Product Configuration/Products page. See "Add a Product" in Products.
    3. Add form, wizard, and rule set packages for the products on the Product Configuration/Packages page. See "Add and Edit a Package" in Packages.
    4. Add sales agreements for the products and packages on the Product Configuration/Sales Agreements page. See "Add and Edit Sales Agreements" in Sales Agreements.
    5. Optionally add and edit fees and expenses repository information and quote for Term Life products. See Fees, Expenses, and Commissions Repository and Quote in Features for instructions.

  4. Set Up Organization Configurations:

    1. Add and edit the organization settings on the Organization Configuration/Organization Settings page. See the Help topics for the Organization Settings tabs (Activity, Admin, Maintain Devices, Mobile Configuration, Organization, Security, Signature) for descriptions of the available settings and instructions.
    2. Define FireLight user access groups and role codes for your organization on the Organization Configuration/Groups page. See Create and Edit Groups for instructions. For each user group, select the group in the grid at the top of the Groups page and use the tabs below the grid to set the applicable access options for a selected user group role. See the Help topics for the Groups tabs (Activity Access, Admin, App, Case Access, Debugging Tools, Needs Filter, Product Access, Review Queue, Training) for descriptions of the available settings and instructions.
    3. Add the custom lists for your organization on the Organization Configuration/Custom Lists page. These are the custom lists that are generally accessed from a button control on forms and wizards. See Custom Lists for instructions.
    4. Edit the default email templates as applicable for your organization on the Organization Configuration/Email page. See Email for instructions.
    5. Set up post-submission review queues and the review queue script on the Organization Configuration/Review Queue page. See Review Queue for instructions.
    6. Map data item IDs on the Organization Configuration/Mapping page. For example, distributors can use the DataItem Synch tab on the Mapping page to link data items between the carrier and distributor. See Mapping for instructions.
    7. (Optional) If using the optional Tele-Sign feature for capturing signatures, add the Tele-Sign scripts on the Organization Configuration/Tele-Sign page. See Tele-Sign Scripts and the Tele-Sign feature for instructions.

    NOTE: Some Admin changes may take 20–30 minutes to take effect in the FireLight application.

  5. Test the Forms, Wizards, and Rules: Log in to the FireLight application and test the forms, wizards, and rules. Ensure you can create the applications and complete the application process for each product. See "Application" in the New Activity section of the FireLight Platform User Guide FireLight Platform User Guide for instructions.

  6. Deploy the Forms, Wizards, Rule Sets, Product Configurations, and Organization Configuration Settings:

    In FireLight Admin, deploy the products, forms/wizards, packages, rule sets, organization, groups, etc., configurations to staging and production using the Utilities/Deployments page. Use the applicable All Configuration scopes to deploy the top level of data: Activities, Products, Forms/Wizards, Packages, Sales Agreements, and Rule Sets. Deploy the applicable Custom Lists, Groups, Mapping, Review Queue, and Organization configurations. See Deploy Data for instructions.

  7. View and Add Reports:

    Use the Utilities/Reports page to generate and view Admin reports for your organization. You can add activity reports that run on a scheduler. Activity reports can be scheduled using the Activity Report Scheduler or by JIRA ticket request. See Reports for instructions and sample Admin and Activity reports.

  8. Optional: Export Forms, Wizards, and Rule Sets:

    Export selected forms or wizards with metadata (fields) using the applicable Form Library or Wizard Library page. Use the Export feature to back up forms and wizards (in case forms are deleted in the system), if planning to use the forms and wizards in multiple environments, or to use the same form or wizards multiple times; for example, when requiring a proof of citizenship form or wizard for joint or additional insured. Export rule sets using the Product Configuration/Rule Sets page.

User Experience

  1. On the Home page under Start New or on the New Activity menu, click Application.

    NOTE: If you enter FireLight through your organization’s CRM system and FireLight launches directly into a new business application, skip steps 1 through 4. The client data associated with the individual selected in the CRM will post over to FireLight.

  2. For a single carrier implementation, select a Jurisdiction and Product Type from the drop-down lists.

  3. For a multi-carrier implementation, select a Jurisdiction, Product Type, and Carrier from the drop-down lists.

  4. Click a product name in the grid to begin an application for the product. Products sold by a subsidiary carrier may show the subsidiary carrier’s name depending on your organization’s settings.
  5. FireLight displays a list of Required Forms for the application. Any Optional Forms appear below the list of Required Forms. You can select one or more optional forms to include within the application.
  6. Click Create Application.
  7. Make any changes to the default application name and click Create.  
  8. Fill in the information on the application form or wizard. Click CONTINUE when Data Entry reaches 100%.
  9. Optional: If your organization implemented a pre-submission review queue, select Request Review in the action menu, select the applicable reviewers on the Preferences page, and click the Send Request to Reviewer(s) button. Otherwise, click Submit on the action menu to skip the pre-submission review. The review step can occur before or after the signing process depending on the Allow Pre-Signature Review or Allow Post-Signature Review settings on the Product Configuration/Activities page in Admin. Following reviewer approval in the post-signature review process, the application is auto-submitted. Note that this occurs without use of the EnableAutoSubmit Admin Groups option nor the FLI_AUTO_SUBMIT dataitem.

  10. Complete the signing process.

    1. Click Use E-Signature to electronically sign the documents.
    2. In the List of Required Signers, click a signer type button (e.g., click Owner).
    3. Click Sign Now to sign on this device.
    4. Fill in the Identification Verification fields and click Verified.
    5. The signer reviews the documents, selects the checkbox for I have reviewed and agree with the terms expressed within this document, and clicks Sign.
    6. The signer enters their signing information (Signer Full Name and City) and clicks I Consent to complete their signature capture.
    7. In the List of Required Signers, click Agent. You review the documents and click Sign.
    8. Enter your Agent On-Site Signature information and click I Consent.
    9. Click CONTINUE to submit the new application.
