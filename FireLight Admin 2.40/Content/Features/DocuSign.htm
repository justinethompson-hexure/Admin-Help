DocuSign Integration

DocuSign Integration
Description, Setup, User Experience, and Additional Information

Description

The FireLight® integration with DocuSign allows the option for clients to use the DocuSign services for their e-signature captures.

  • DocuSign must be honored any time it is enabled regardless of the organization initiating the activity. When carriers enable DocuSign and deploy the activity to distribution, FireLight must honor DocuSign for the distribution as well as honor it in the carrier organization.

  • DocuSign will support Remote Sign only.

  • FireLight hands off the signing process for remote sign when DocuSign is enabled (email notifications and signing are completed by DocuSign).

    When the activity has been signed, Firelight receives back the signature on the original documents. These documents include the signatures of the user, and each document is individually hashed as to secure the authentication of these signatures.

    A full audit recorded with the DocuSign signatures is captured as a signature certificate. This is not appended to the FireLight audit report but is sent as a separate document to carriers along with the Audit report. Additionally, the DocuSign signature certificate is listed in the Application Packet Definition to identify the additional certificate to the organization.

    The Envelope ID is added to the Audit report to be able to tie the FireLight Audit Report and the DocuSign Signature certificate together.

    When the signatures are canceled through DocuSign, the Signing Certificate is deleted automatically, and the new Signing Certificate is uploaded.

    When the documents are received, a FireLight audit is made within the audit history and audit report. The display status is "DocuSign Signatures Complete" along with the timestamp and the session user of "DocuSign." The FireLight Audit message says "All parties have signed {0} through DocuSign." {0} is the Application Name. The DocuSign Signing Certificate is attached to the FireLight Audit Report. The FireLight Audit and DocuSign Signing Certificate are available for review when the application is submitted. This includes the FireLight Admin Report, Review Queue, and the outbound via the provider. The FireLight Audit message within the audit history can be localized.

  • The DocuSign service can be used at an organizational level or set by a data item for a more detailed process (i.e., for a specific product, state, etc.).

  • When DocuSign is enabled, the Signature Audit is not captured within the FireLight Signing audit. However, FireLight receives a signature audit back from DocuSign that is attached to the application and sent within the client's payload.

  • FireLight needs the signed documents with the digital signature applied and the DocuSign signature audit returned 100% complete to FireLight. The clients that need DocuSign implemented have the DocuSign digital signature, tamper evidence, and audit approved.

  • FireLight is notified of status changes via the DocuSign Webhook. If the call to notify FireLight that signatures are complete fails, FireLight will remain in “Signatures” status. However, DocuSign does resend the notification after 24 hours of the failure. DocuSign also provides the ability to manage failed notifications from their portal. This is a manual process and cannot be managed by FireLight support. It can be managed only by a client organization admin with authority to access the DocuSign portal. For information about viewing errors in DocuSign Connect that may have occurred when DocuSign attempts to communicate with FireLight, see the DocuSign Connect Failures.pdf DocuSign Connect Failures.pdf in Admin Help.

  • A user can cancel the signature from FireLight and from DocuSign up until the last signer has signed.

  • The user must have authentication when sending Personally Identifiable Information (PII) to DocuSign, so FireLight will still send the last 4 digits of the SSN and DOB as well as the mobile phone number of each party to DocuSign.

  • SMS Text Verification is used to verify each signer, including the agent.

  • Supplemental documentation is hidden from DocuSign signers if a signature control is not present. If a Signature control is present, the Admin user needs to set the form protection to "Signer" so only the applicable signer sees this document. Supplemental documentation is sent back to FireLight and attached to the application to be submitted to the carrier organization.

  • The Third-Party Usage report includes a Config Group column for DocuSign. See Detailed Activity Reports in Admin Help for a description and sample report.

Impacts:

  • FireLight E-Signatures - Specifically the "Send Email Request" or Remote Signing capability

  • Organization Settings - Enabling the DocuSign option.

  • Third-Party Configuration Table - Service type for DocuSign, new column for sub firm "Control Group."

  • App Changes - New UI page for "DocuSign Signing Ceremony" information that can be entered and initiate the call to DocuSign.

  • Reports - The Third-Party configuration report is altered to show the DocuSign signed applications and add the Control Group.

  • Audits - New audits is created to capture the calls to DocuSign, any cancellations, and the completion of the signature process.

  • Display Statuses - New display statuses show when the signature process is in process, canceled, or completed through DocuSign.

  • Transfer - Transfer is disabled when DocuSign is used, even if it is selected within the Groups settings. Share is not impacted.

  • Client Fill and Sign - Client Fill and Sign is disabled when DocuSign is used, but also notifies the user of this through a tool tip.

DocuSign Services Setup

To use the DocuSign services, contracting needs to be established between your organization and DocuSign. Please reach out to DocuSign to start this process.

To validate and authenticate a user through FireLight using DocuSign, SMS Text Verification is used. SMS verification incurs a per use fee and must be enabled throughout the account’s recipient security settings through DocuSign. For more information, please review https://support.docusign.com/guides/ndse-user-guide-recipient-authentication.

NOTE: Use of SMS Text Verification requires the user to have a mobile phone to receive the SMS text. This is currently enabled only for U.S. numbers.

FireLight Setup

Clients wanting to use the DocuSign service will need to contact Hexure to enable the service. Contact your Strategic Account Manager to create an addendum to the contract to support the DocuSign integration. Once the licensing agreement is completed, the DocuSign features in the Admin tool is available for setup.

Admin Setup

NOTE: See the DocuSign Setup GuideDocuSign Setup Guide in Admin Help for more detailed setup information.

  1. To use DocuSign at an organization level:

    • In Organization Configuration, Organization Settings on the Signature tab: in the Signature System list, select DocuSign to enable DocuSign at the organization level. This forces use of DocuSign at the organization level. This setting is honored even if a carrier's products are deployed into a distributor's environment.

    • Ensure the Allow E-Signature checkbox is selected. DocuSign requires e-signature. NOTE: If "FLI_DECLINED_ESIGN=true, this data item overrides the "Use DocuSign Signature" checkbox and the user will need to wet sign

  2. In Organization Configuration, Organization Settings on the Admin tab, add the Third-Party Configuration settings for DocuSign. DocuSign provides the configuration IDs and values. For more information on the setup, please see the DocuSign Setup GuideDocuSign Setup Guide provided in the home dashboard of the Admin Help.

    NOTE: For each DocuSign credential, adding the optional Config Group value provides the distributor a way to have credentials associated to certain carriers and can even write rules enabling DocuSign if a certain subfirm value is passed through.

    1. Click the Click to Edit link next to Third-Party Service Config to open the Third-Party Configuration dialog.
    2. Click the Add New link at the top of the dialog. In Config Name, type ClientID. In Config Value, type the Client ID that DocuSign provided. If applicable, add an optional Config Group. In Service Type, select DocuSign and click the Add Config button.
    3. Click the Add New link at the top of the dialog. In Config Name, type HMACKey. In Config Value, type the HMAC Key that DocuSign provided. If applicable, add an optional Config Group. In Service Type, select DocuSign and click the Add Config button.
    4. Click the Add New link at the top of the dialog. In Config Name, type RSAPrivateKey. In Config Value, type the RSA Private Key that DocuSign provided. If applicable, add an optional Config Group. In Service Type, select DocuSign and click the Add Config button.
    5. Click the Add New link at the top of the dialog. In Config Name, type UserID. In Config Value, type the User ID that DocuSign provided. If applicable, add an optional Config Group. In Service Type, select DocuSign and click the Add Config button.

    6. Click the Add Config button.

Designer/Forms

Forms with signature controls need to have the Protection option set to Signer in the Form Properties so it is visible only to the applicable signer. If there are multiple signers, those signers will see the document as well. Any document that they don't need to sign will have its visibility determined by the form Protection property. See Set Form Properties for instructions.

All documents are sent to DocuSign, even in cases where no signers can see the document. Any document that no-one needs to sign is a supplemental document that will display in a collapsed view for people who can still see it.

Rules

Distributors can use the FLI_DOCUSIGN_ESIGN to write rules that force the use of the DocuSign functionality if the carrier does not have the organization setting selected. However, this data item will not be able to turn off the use of DocuSign.

  • If FLI_DOCUSIGN_ESIGN = true, the distributor who has this set is able to force the use of the DocuSign functionality for that Product or Products that contain the rule set.

  • If FLI_DOCUSIGN_ESIGN = false, this will not produce any values.

When turning this functionality on, the distributors will need to use the following rule template.

FLI_DOCUSIGN_ESIGN

 <diset dataitemid="FLI_DOCUSIGN_ESIGN"> <const value="true" type="String" /> </diset>

In addition, when a distributor wants to associate the config group value (a unique identifier much like Firm ID) with a specified organization, they can use the following rule for that value to be used with the DocuSign credentials.

For example, if Distributor 1 has a firm ID of "ABCD" for a carrier that needs to use DocuSign, then within the third-party configuration, this Distributor can set up this firm with the DocuSign credentials, and then write a rule that when this FIRM ID is sent in via the SSO, that the DocuSign credentials are used associated with the same firm ID.

FLI_DOCUSIGN_ESIGN with FLI_FIRM_ID

 <if> <condition> <compare op="=="> <diget dataitemid="FLI_FIRM_ID" /> <const value="ABCD" /> </compare> </condition> <diset dataitemid="FLI_DOCUSIGN_ESIGN"> <const value="true" type="String" /> </diset> <diset dataitemid="qwq"> <const value="ABCD" type="String" /> </diset> </if>

Use the FLI_THIRDPARTY_CONFIG_GROUP data item to associate the config group value within the third-party configuration table inside rules. This is helpful when a distributor wants to associate the DocuSign credentials to a specific carrier organization but needs to identify that org using the unique identifier that is stored within the third-party configuration table. If FLI_THIRDPARTY_CONFIG_GROUP has a wrong value, then no organization is associated with the DocuSign and is used universally across all carrier organizations (get a 500 error). Currently, the FLI_THIRDPARTY_CONFIG_GROUP is used only by DocuSign.

The following rule can be used to associate the Config Group found in the third-party configuration table to the DocuSign functionality. The FLI_THIRDPARTY_CONFIG_GROUP data item can be used within any rule template or rule set within FireLight.

FLI_THIRDPARTY_CONFIG_GROUP

 <if> <condition> <compare op="=="> <diget dataitemid="FLI_FIRM_ID" /> <const value="ABCD" /> </compare> </condition> <diset dataitemid="FLI_DOCUSIGN_ESIGN"> <const value="true" type="String" /> </diset> <diset dataitemid="FLI_THIRDPARTY_CONFIG_GROUP"> <const value="ABCD" type="String" /> </diset> </if>

NOTE: Communication between the activity and DocuSign (web service calls, initiation, cancels, their response, etc.) can be viewed within the API Insights tool. This ensures the ability to help troubleshoot for the clients that is using the DocuSign functionality.

SMS Text Verification Setup

See SMS Text Authentication for setup instructions (includes Validate Identity Verification setup). We recommend also using the Validate Identity Verification, as it would be an additional validation method when using the DocuSign services.

User Experience

  1. User clicks CONTINUE on the requirements notification or clicks the CONTINUE button on the upper right to access Signatures.

  2. User clicks Use E-Signature to start the E-Signature process in DocuSign. Optionally, user clicks Decline E-Signature to have the client sign paper copies of the application.

  3. User fills in Agent and Client signer type information, including the signer's name and email, agent's name and email, and any authentication for the signer, which includes the last 4 of the SSN, DOB, and the Mobile Phone Number. The Agent information will always show at the top. User clicks Initiate DocuSign. If there is a failure with the call, a message states, "An error occurred while attempting to reach DocuSign. Please try initiating this service again." The user can then select the Initiate DocuSign button again to make another call to DocuSign.

  4. The DocuSign Signatures page displays the signer status as Pending Signatures for each signer type.

    In FireLight, the signatures are in Pending Signatures or Signatures status until the signatures are completed.

    • Optionally, the user can click Cancel DocuSign Signatures to cancel the DocuSign Signatures and return to the Data Entry step. The signers receive an email from DocuSign that the signatures were voided.

    • Optionally, the user can click Unlock Application on the upper right of the Signatures page and click Unlock Application on the Unlock Application dialog to cancel the signature process, unlock the activity, and return to the Data Entry step.

    • Optionally, the user can click Back to Application on the upper right of the Signatures page to return to the activity in the data entry view without leaving the Signatures step. The user clicks CONTINUE to return to the Signatures step.

  5. The agent and client signers each receive an email from DocuSign that provides the names of the documents for them to sign, DocuSign instructions, and a Review Documents button they can click to proceed to the signing ceremony in DocuSign. In the email, the signer clicks the Review Documents button to proceed to the signature capture in DocuSign.

  6. The signer receives an email for SMS Authentication. The signer verifies the phone number and clicks SEND SMS. The signer enters the code they received in the SMS message and clicks CONFIRM CODE.

  7. The DocuSign home page opens. The signer does the following.

    1. Signer clicks the Electronic Record and Signature Disclosure link to read the disclosures and then selects I agree to use electronic records and signatures checkbox. The CONTINUE button becomes available.

    2. The signer clicks the CONTINUE button.

    3. On the upper left, the signer clicks START.

    4. The signer scrolls down to review each document and clicks the Sign button where applicable for their signer role.

    5. On the Adopt Your Signature dialog, the signer confirms their name, initials, and signature, views the Preview, and clicks the ADOPT AND SIGN button to complete the signature. Optionally, the signer can click CANCEL to cancel the signature. The Adopt Your Signature dialog appears only once for the first signature adoption.

    6. The signer reviews the next document and clicks the Sign button to automatically fill their signature into the signature blocks until no Sign buttons remain in the documents. The signer may also need to enter the city and state where they are signing the documents.

    7. After reaching the end of the documents, the signer clicks the FINISH button.

    8. In the Save a Copy of Your Document dialog, the signer can use the controls on the upper right to download and print their documents.

      Signer clicks NO THANKS to close the dialog without signing up for a free DocuSign account.

      For downloaded documents, a dialog opens. In the notification bar at the bottom of the window, the signer uses the browser settings to open or save the document. Signer clicks CLOSE to close the DocuSign dialog.

      For printed documents, DocuSign generates a PDF file and opens the file. Signer uses the PDF controls to print or save the file and closes the PDF file.

      Optionally, the signer can sign up for a free DocuSign account by entering their Password and Confirm Password, selecting their country, and clicking the SUBMIT button. Signer clicks NO THANKS to close the dialog without signing up for a free DocuSign account.

    9. DocuSign displays the "You're All Done!" page. Signer will receive a copy once everyone has signed. Signer closes the browser to exit DocuSign.

    10. When all parties have completed the required document signatures, each signer receives an email from DocuSign with PDF documents attached. The signer can open and save the PDF file attachments or click the VIEW COMPLETED DOCUMENTS button to view the documents in a browser window or tab.

  8. When the Signatures are completed within DocuSign, they is sent back to FireLight with the documents. In the Recent Activity list and in All Activities, the Display status will show "DocuSign Signatures Complete" and in the audit, the timestamp and session user of "DocuSign Signatures Complete." The message will say "All Signatures have been successfully completed through DocuSign for {0}" where the {0} is the Application Name.

  9. The user can access the application documents by opening the application from either the Recent Activity list on the Home page or from All Activities. On the Other Actions menu, the user clicks Documents to open, view, save, and print the application documents.

  10. Optionally, the user can unlock the application, cancel all signatures, and return to the Data Entry step. On the Other Actions menu, the user clicks Unlock Application. On the Unlock Application dialog, the user clicks Unlock Application to cancel the signature process, unlock the activity, and return to the Data Entry step.

  11. The user clicks the CONTINUE button to proceed to the next step after Signatures (Review or Finalize).
