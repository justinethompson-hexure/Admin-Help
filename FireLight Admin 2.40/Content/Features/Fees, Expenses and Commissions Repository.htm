Fees, Expenses, and Commissions Repository

Fees, Expenses, and Commissions Repository Feature
Description, Setup, User Experience, and Additional Information

Description

This feature provides a repository of fees, expenses, and commissions data in FireLight® for use in fees, expenses, and commissions disclosure forms. stores and maintains a data repository along with common tags for each data item. FireLight supports the industry-defined formats for the data.

  • DTCC Fee & Expense Transmittal (TXLife 1241) Data Dictionary (PPfA Lite)

  • DTCC Commission Schedule Transmittal (CST) Record Layouts

There is a licensing component for both the carriers and distributors to support this feature. Clients interested in participating should contact their Strategic Account Manager for pricing.

Carriers are required to tag their applications with data items for each Rider that will have a fee and set the export value to match the PPfA. Carriers need to add rules in their application form to define when the rider/feature/arrangement and funds are selected and which ProductCode to use to pull the fees from the repository.

NOTE: FireLight can support additional formats, on a custom basis, as needed.

Data Delivery

Data Delivery via DTCC

The distributors will license the data with the DTCC and request that Hexure is a destination. Carriers can send the PPfA and the CST files for these licensed clients through the DTCC and the files for the licensed clients will route to Hexure.

FireLight is permitted to use only the data for the DTCC participant or participants who have requested that their data be shared with FireLight. In this scenario, FireLight receives a separate transmission of the data for each distributor so it knows who is a licensed client of Insurance Profile.

The license could be at the top firm level or at a group level.

Data Delivery via a Direct Upload to FireLight

FireLight also supports a manual upload using the same industry standard formats. Firms choosing to use direct upload to FireLight can manually upload the files in the Admin.

NOTE: For non-street versions, will support a file upload for unique versions.

FireLight Setup

Carriers and distributors pay a licensing fee to use the repository. Contact your Strategic Account Manager to set up a Work Order for licensing, form rules implementation, etc.

  • Carriers set up rules and/or mapping for selection of features and arrangements that will incur a fee and set their export values to the PPfA ProductCode. Carriers must tag each option in with the Common Tags for each RiderType TypeCode (TC) ) that applies to the product. The Export Option needs to match the ProductCode/FeatureOptProductCode in the PPfA. For variable products, the FundCode also needs to be defined in the carrier form.
  • Distributors submit a Fees and Expenses disclosure form to pull in the fees for the application. The form needs to be tagged with the common tags per RiderType TypeCode, Policy Level Fee, and Fund Fee. Rules also need to be written to run the query to pull in the rules. The FireLight Forms team can complete the tagging as part of a Work Order. Note that the disclosure form is a static PDF, but the content pulled for each fee is dynamic per the selection in the application. Form fields for fees that do not apply is left blank.

Admin Setup

  1. Design/Form Library page: Upload the fees, expenses, and commissions disclosure forms that contain the common tags and rules. NOTE: In the "Additional Information" section below, you can view/download a sample disclosure form. This form includes the common tags and rules. Distributors, you can load it into FireLight and use the Copy Fields and Rules to move the tags and rules into your firm’s disclosure form. Carriers, you can use this form for your testing along side your application tagging.
  2. Designer/Form Designer page: On the disclosure form, create a field to hold the fee and add a queryfee rule in the Rules Window to pull data from the repository. See the "|queryfee" rule under "|GetSet" in Advanced Rules Category and Element Definitions Reference for a detailed description and examples.
  3. Organization Configuration/Organization Settings page: Carriers, enter the DTCC Member ID that matches the PPfA DTCCMemberID. Distributors, enter the DTCC Member ID provided by the DTCC for your participant. Or if you are not a DTCC member, Hexure will need to make up a value, and that is the value the carriers will need to use in their upload.
  4. Organization Configuration/Groups page: In the upper list select a TC Role Code for users who will have access to the repository. On the Admin tab, select the Enable View Fees checkbox. This allows access to the Fee Inquiry page in Product Configuration that supports the manual upload and download of fee files.
  5. Product Configuration/Fee Inquiry page: Upload fees, expenses, and commissions files (PPfA Lite, CST) by clicking the Upload File button. Optionally, download files for backup or review. Note that the PPfA DTCC Member Code, ProductCode, and CUSIP must match the DTCC Member Code and the CUSIP ID and Product Code in the Product Profile. Carriers then need to add rules and/or forms tagging to define the rider and arrangement options and set the Export Value to the FeatureOptProduct.ProductCode in the PPfA. NOTE: When you upload an updated version of a file, it overwrites the existing same-named file.
  6. Product Configuration/Products page: If a product has a different DTCC ID, such as a product variation for a unique state filing, enter the product DTCC ID number in Product DTCC Member Id. Carriers must update the Product with the CUSIP ID or Carrier Code and Product Code for each product with fees that is queried.
  7. Product Configuration/Packages page: Distributors add the fees and expenses disclosure form to the product application packages. This form needs to be tagged with the Common Tags and the rules to query for the data.

User Experience

  1. If required, complete a Needs Determination before creating an application.
  2. Create an application.
  3. Fill in the information on the application. The data on the fees and expenses disclosure form will auto-complete based on the carrier’s tagging and rules.
  4. When Form Entry is 100%, continue to the electronic signature process.
