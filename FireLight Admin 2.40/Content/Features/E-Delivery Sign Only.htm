E-Delivery Sign Only
Description, Setup, User Experience, and Additional Information

Description

The E-Delivery Sign Only workflow provides sign only of client contracts. An e-Delivery Sign Only request initiates with FireLight® receiving an ACORD DocSend (TX121 and a TransSubType typecode of 12122 with processing instruction = Agent Directed or Client Directed) message that drives the E-Delivery Sign Only workflow. The organization's provider can have the ability to access the TXLifeRequest object so the provider can add and manipulate data items that are sent in the 121. The provider can be able to call the TXLifeRequest object for all agent directed and client directed cases. During the process, FireLight provides status messaging back to the carrier to keep track of the contract delivery.

NOTE: The initial e-Delivery emails for client pick-up can be resent with the trigger being a 121 with an ACORD TransMode 10 (OLI_TRANS_MODE_SUBRESPONSE). To trigger the resending of the emails, the 121 must contain the same TransRefGuid and PolNumber so FireLight can find and process the correct case. In the event a child case reaches a terminal status that causes the parent case to change to any status other than "Sent To Client" (e.g., one party declines e-sign, thus the application is now a paper app), no initial emails is resent for any child cases. If a child case has been completed, it’s initial email will not be resent.

NOTE: The EXTERNAL role code governs the behavior of the signing ceremony for the signature request. The DEFAULT role code will be in effect for agents coming in a) without a role code, or b) with an invalid role code.

FireLight supports two other E-Delivery workflows. Click a link to go to the workflow feature topic.

  • E-Delivery Deliver Only

FireLight Setup

Contact your Strategic Account Manager to create an addendum to the contract to support the E-Delivery workflow type, set up the integration, and accept the transaction fees. Also contact the Strategic Account Manager for assistance with creating and designing wizards and rule sets for e-Delivery.

NOTE: E-Delivery is available only with the Wizard user interface.

Admin Setup

Following is a synopsis of the E-Delivery Sign Only setup process. Refer to the FireLight e-Delivery GuideFireLight E-Delivery Guide for detailed implementation instructions.

  1. Designer/Wizard Library page: Create one or more wizards to be included in the E-Delivery Sign Only package.
  2. Designer/Form Library page: The PDF forms to be sent in the TX121-12122 and presented in FireLight (example the Policy) are sent in the TX121-12122 with a document tag identifying the form. See the FireLight e-Delivery Guide for details.
  3. Designer/Wizard Designer and Form Designer page: Design the wizards and/or forms to be included in the E-Delivery Sign Only package and add the rules in the E-Delivery Sign Only rule set. Firms can have more than one wizard and/or form. There is different wizards for the different E-Delivery product types (Deliver Only, Fill and Sign, and Sign Only).
  4. Product Configuration/Rule Sets page: Add the rules for E-Delivery Sign Only including the corresponding rules to enable/disable the signature controls based on the OLIFE role.

  5. Product Configuration/Activities page: In the grid at the top of the page, click Add New Activity and enter the E-Delivery Sign Only product name in Activity Name. Activity name determines the name in the app portal. If e-delivery fill and sign is not used, the name for e-delivery button in manage section is determined by the E-Delivery sign only product name. Select E-Delivery Sign Only (121-12122) in ACORD Type. Enter settings applicable for your organization and click Save Activity.

    Note the following:

    • Complete Purge Policy The number of days after an E-Delivery Sign Only case was finished (this includes Expired, Canceled, Re-Issued and Completed cases). The default is 30 days and can be changed only by Hexure. The default is much lower because E-Delivery cases use more space than other Activities and we need to limit the space used. Once the 30-day time period has been exceeded the Purge process will purge the activity.
    • Incomplete Purge Policy The number of days since an incomplete E-Delivery Sign Only case was last updated. The default is 30 days and can be changed only by Hexure. The default is much lower because E-Delivery cases use more space than other Activities and we need to limit the space used. Once the 30-day time period has been exceeded the Purge process will purge the activity.
  6. Product Configuration/Products page: Add the E-Delivery Sign Only product as a Non-Insurance Product in Product Type, Product Category, and if applicable, Product Type Filter (UI). Configure product settings. Add a unique Product Code that can be used to delineate the different packages to use for E-Delivery Sign Only. Save the product config.
  7. Product Configuration/Packages page: Add the E-Delivery Sign Only package. In ACORD Type, select E-Delivery Deliver Only (121-12123). Add the E-Delivery Sign Only wizards and rule sets to the package, and save the package. NOTE: The rules are run only once before the case is locked. Agent/client receive the case in a locked state.
  8. Product Configuration/Sales Agreements page: Add the sales agreement for the E-Delivery Sign Only product, assign the E-Delivery Sign Only package to the sales agreement, and save the sales agreement.

  9. Organization Configuration/Organization Settings/Activity tab: Be sure to set the following options and click Save Organization:

    • In E-Delivery Reminder Frequency, select the frequency (Never, Daily, Weekly) that FireLight will send reminder e-mails to the agent to complete outstanding e-Delivery cases. This is set by the requested date of the first requirement info node in the 121. E-Delivery reminder emails run on the scheduler with the reminder set at intervals from the time of the application reaching the E-Delivery status. "Daily" frequency emails go out at 6:00 AM UTC. "Weekly" frequency emails go out on Sunday at 7 AM UTC.
    • In E-Delivery Expiration Warning Threshold, enter the number of days prior to the expiration of a pending e-Delivery request an email is sent to the agent and client. This email will warn of the impending expiration of the client signature(s), case expiration, etc. If 0 is entered, no email is sent. Go to Organizations > Email > E-Delivery Email CategoryE-Delivery Email Category to see the associated e-Delivery request expiration email templates for the client and agent.

  10. Organization Configuration/Groups page:

    1. Select a TC Role Code in the grid.
    2. Activity Access tab: Select the E-Delivery Sign Only checkbox to enable E-Delivery Sign Only for the user group.
    3. Product Access tab: Select the checkbox for the E-Delivery Sign Only product name.
    4. Save the Groups page.

  11. Utilities/Deployments page: When testing is completed, deploy the E-Delivery Sign Only product to the applicable environments. See Deploy Data.

    1. Synchronization Scope: Select All Configuration.
    2. Reason: Type the reason for the deployment.
    3. Deployment tab: In the Deploy column, select the checkbox for the E-Delivery Sign Only product.
    4. Set whether to Deploy Now or Schedule Deployment and whether or not to use Force Deploy.

User Experience

The E-Delivery Sign Only case goes into FireLight from the ACORD DocSend (TX121-12122) message through the firm's Provider.

Client Directed Processing Cases:

  1. The Back Office creates a new E-Delivery Sign Only case that is locked and ready for signatures. The status is E-Delivery Sign Only Ready for Signatures.

  2. The Back Office sends an email to each client to request the client to sign the E-Delivery requirements within a client-directed processing workflow. The information comes from the 121 party info. The name and email address fields are disabled and pre-filled from the 121. The party fields must be mapped in the Admin Mapping tool to correctly map name and email address. The status is Pending Signatures - E-Delivery Sign Request Sent for each client request email sent. FireLight sends the E-Delivery status of Pending Signatures - E-Delivery Status Sent to the carrier for reach client request.

  3. The client enters the Last 4 Digits of SSN/Government ID and their Birth Date or a Passcode sent to them via a separate email (if the passcode option is enabled), and clicks Enter to log in to FireLight.

  4. On the Electronic Signatures page, the client clicks one of the following:

    • Review & Sign button to review and sign the E-Delivery documents.

    • Contact Agent button to send a message to the agent.

  5. On the Signature page, client clicks each document to review and then selects the checkbox for I have reviewed and agree with the terms expressed within this document.

  6. On the Capture Electronic Signature page, client enters their Signer Full Name and City and signs by clicking the I Consent button, signing on the signature pad with a mouse or stylus, or signing with the Topaz signature tablet.

  7. On the Congratulations page, client clicks a button.

    • Client clicks Document Download button to download the E-Delivery Sign Only documents.

    • Client clicks OK button to exit FireLight. Client receives an email to download or print the completed E-Delivery Sign Only documents.

      1. Client clicks the URL and enters the Last 4 Digits of SSN/Government ID and Birth Date on the Welcome page and clicks Enter.

      2. In Download PDF Documents, client clicks the Download All link to download all of the activity documents in a single PDF file and or the client clicks the Request Print button to send a request to the carrier to provide a paper copy of the documents.

  8. On the top navigation bar, the client can click the Request Print button to send a request to the carrier to provide a paper copy of the activity documents. This can be used instead of downloading the documents or in addition to downloading the documents. FireLight sends a message to the carrier to provide a paper copy. Client closes the browser window to exit FireLight.

  9. When a client signs and completes the E-Delivery Sign Only process, FireLight submits the case to the back-office systems for final processing. The status changes to Pending Signatures - Submitted. The status then changes to Complete. When back office processing is complete, the status changes to Complete - Back Office Message.

  10. FireLight sends a Complete - E-Delivery Status Sent status to the carrier.
