Needs Determination Activity
Description, Setup, User Experience, and Additional Information

Description

The Needs Determination activity collects the client data ahead of any sale transaction to allow the agent/producer/rep to make a best interest recommendation based on unbiased questions. The distributor and the financial institution will define the questions they want to ask and supply that to FireLight® for tagging and workflow setup. The data collected during the Need Determination can be used in the product filter to eliminate products that do not match the client need. The process and completed PDFs is part of the audit record. The completed PDFs is sent to any subsequent applications to be included in the e-Signature and review process if a Needs Determination template form is added to the package. This entire workflow will create a defensible process for clients.

The Needs Determination activity does the following:

  • Captures questions via the standard PDF with tags and rules and stores the answers as new data items. These data items can be defined into filter rules to apply to the available product selection in the electronic application process.
  • Has its own audit report, output, and reporting.

  • Uses the rules engine and has access to the same group option functionality as the application, post issue, and sub-pay activity types. This will allow for different Need questionnaires depending on the group role code.

To support this feature, there is a licensing component for the distributors and for forms tagging. Distributors will need to work with their Strategic Account Manager for pricing.

FireLight Setup

NOTE: Workflows using the FireLight Needs Determination activity can enable the product filtering feature to filter the list of available products based on entries in the Needs Determination. See "Needs Determination Product Filtering" on this Help page.

Determine the types of Needs Determination workflows for your organization. Submit a Work Order with your Strategic Account Manager to update your SSO, applicable database tables and rules for your organization and for any custom Upload button work, etc.

Workflow Options:

  • SSO directly into the FireLight® Needs Determination activity. SSO opens a new Needs Determination activity with the SSO pre-filled data. Use the ACORD activity type 116. Submit a Work Order with your Strategic Account Manager to update your SSO. When the application process begins, data items from the Needs Questionnaire are pre-filled in the product application as read-only fields. To make the pre-filled application fields editable, include the requested editable fields in your Work Order submitted to your Strategic Account Manager. Also have an additional DocType of Needs Determination with a Document Tag of NeedsDetermination added to your organization's List Profile, so a Needs Determination template form can be added to the application package, which allows the user to print and view the Needs Determination within the application package. If a template is not added, the document will only be visible only by viewing it in Other Actions/Documents and would then be included last in the final document package for submission to the provider.

    Click the following thumbnail image to view the DOL workflow for a Needs Determination transaction.

  • SSO directly into a new application with the data from an outside Needs Determination activity. Use the ACORD activity type 103. The Needs Determination completed outside of FireLight can be sent as an attachment with the description and document tag of OutsideNeedsDetermination. This type of template cannot be deleted when sent in the post over.

    Submit a Work Order with your Strategic Account Manager to update your SSO and an update to your organization's Upload Document types in the list profile database table.

    Click the following thumbnail image to view the DOL workflow using SSO with a pre-filled Needs Determination.

  • Create a Needs Determination in FireLight using the FireLight Needs Determination activity type available from the New Activity list. When the application process begins, data items from the Needs Questionnaire are pre-filled in the product application as read-only fields. To make the pre-filled application fields editable, include the requested editable fields in your Work Order submitted to your Strategic Account Manager. Also have an additional DocType of Needs Determination with a Document Tag of NeedsDetermination added to your organization's List Profile, so a Needs Determination template form can be added to the application package, which allows the user to print and view the Needs Determination within the application package. If a template is not added, the document is visible only by viewing it in Other Actions/Documents and would then be included last in the final document package for submission to the provider.

    NOTE: Needs determination documents is attached as NeedsDetermination document tags, illustration as LinkedIllustration, and Account Opening documents as AccountOpening. All other attached documents coming from one activity to the next activity will come over as a document tag of Other.

  • When an advisor starts a new application in FireLight (or starts a new application via SSO), but rules identify that a Needs Determination is required, Organizations need to define in a Work Order whether they want to support an upload or stop the application process. Note that if manual upload is supported, the FireLight Needs Determination product filtering will not occur.

    Upload option: The organization can add a template form so FireLight will know that a Needs Determination is required in the application. If a template form requires a Needs Determination and one was not sent in the SSO or pre-filled from FireLight, the system will look at rules to determine if the advisor can proceed and upload the Needs Determination.

    Stop option: Rules need to support presenting an error message if the advisor must stop and begin with the Needs Determination activity. A global dataitem FLI_NEEDS_BASED_APP can be set to True defining that the application is based on a Need activity. Firms can use this dataitem in rules to enforce workflow that requires a Need activity prior to an Application. A rule can be set to ensure the value is true, and if not, provide notification to the advisor that a Needs Determination must be completed and they should exit the application to fulfill that requirement. Submit a Work Order with your Strategic Account Manager for the error message and rules.

Admin Setup

NOTE: If you would like to rename the Needs Determination activity for your organization, contact your Strategic Account Manager to submit a work order for changing the Needs Determination activity name in the UI system wide, the audit history, emails and reports.

FireLight Needs Determination Activity

  1. Design/Form Library page: Upload the client submitted Needs Questionnaire PDF to FireLight. Also upload a Needs Determination Template form that is added to the product packages. (In the Need Determination to Application Apply process the template form is replaced by the completed Needs Questionnaire form.) For the template form, select the Is Template Form checkbox and in Document Tag, select the NeedsDetermination Document Tag. Select the States where the Needs Determination is approved and click the Update button. NOTE: Contact your Strategic Account Manager to have the implementation team add an additional DocType of Needs Determination with a Document Tag of "NeedsDetermination" added to your organization's List Profile.

  2. Designer/Form Designer page: Add any custom tags and rules to the Needs Questionnaire form. The data items in the Needs Questionnaire can be defined into filter rules if the product filtering feature on the Organization Configuration/Groups page is enabled. We recommend using radio button controls with export values instead of checkbox controls, which have their export value defaulted to "true."

    NOTE: FireLight recognizes two FireLight Global data items in the new activity workflow.

    • FLI_NEXT_ACTIVITY_PRODUCT_CUSIP – the CUSIP configured on the Product
    • FLI_NEXT_ACTIVITY_JURISDICTION – the numeric jurisdiction code

    The two data items can be used in a Needs Determination new activity form to skip the standard FireLight new activity workflow and begin the next activity. The source activity (Needs Determination) form can include input controls where the user can select the FLI_NEXT_ACTIVITY_JURISDICTION and FLI_NEXT_ACTIVITY_PRODUCT_CUSIP values. These input controls will generally be either dropdowns or custom lists. NOTE: These lists cannot be retrieved from the database or filtered by the provider. This means the lists that get built by the forms designer may have products that the user is not configured to sell.

    The logic in the New Activity workflow determines if the source activity contains the above well known data items and attempts to create the next activity based on these values when the user clicks the APPLY button after completing the Needs Determination questionnaire. If successful, the user is redirected to the activity without needing to make selections in the standard FireLight new activity workflow. If the user is not allowed to sell the selected product in the selected jurisdiction, the user is shown a popup explaining why the selection is not valid and the user must go through the standard FireLight new activity workflow.

  3. Product Configuration/Activities page: In the grid at the top of the page, select the Needs Determination activity. Ensure ACORD Type is set to Needs Determination (116-11600). Update the settings for your organization. Enter the Continue/Apply label for the Needs Determination activity. On the Next Activities tab, move the possible next activities after the Needs Determination is complete from the All Activities list to the Next Activities list and click the Save Activity button.

  4. Product Configuration/Products page: Add a Needs Determination product and select Preneed or Non-Insurance Product in Product Type. In the State Approved grid, select the Approved checkbox for each state where the Needs Determination is approved.

  5. Product Configuration/Packages page:

    1. Add a Needs Determination product package. Set ACORD Type to Needs Determination (116-11600). Add the Needs Questionnaire form to the package and save the package.
    2. For each product that will require a Needs Determination, add the Needs Determination Template form to the product package and save the package.
  6. Product Configuration/Sales Agreements page: Select the Needs Determination product in the upper list, move the Needs Determination package from the Available Packages to the Assigned Packages list. Set Package Duty to Required. Some clients will use a cover page to determine if a Need Determination is required and if that workflow is being implemented, the Need Determination package is conditional and rules will need to be defined in the cover page for when to trigger the Need.

  7. Organization Configuration/Groups page:

    1. Select a TC Role Code for a group of users in the upper list who will use the FireLight Needs Determination activity.

    2. On the Activity Access tab, select the Needs Determination checkbox. NOTE: This group code does not need to be enabled for users who SSO into FireLight with a pre-filled needs questionnaire and do not use the Needs Determination activity in FireLight.

      Additionally, to require certain role codes to always start with a Needs Determination before an application, clear the Application checkbox. This removes the option of manually selecting New Business Application from the New Activity list. The user must begin with a Needs Determination and then progress to an application through the Apply functionality from the Needs Questionnaire or from the All Activities page.

    3. On the Product Access tab, select the Needs Determination product for each role code that will use that activity. This ensures that the advisor has access to the package. This also allows a firm to define different need determination questionnaires per role code.
    4. Repeat for each group of users that will use the FireLight Needs Determination activity.

Outside Needs Determination Attachment in Post-Over

  1. Designer/Form Library page: Upload a template for the Outside Needs Determination form. Select the Is Template Form checkbox. In Document Tag, select Outside Needs Determination. Select the States where the Needs Determination is required and click the Update button.
  2. Product Configuration/Packages page: For each product that will require a Needs Determination, add the Needs Determination Template form to the product package and save the package.

Outside Needs Determination Document Upload

You can add an outside needs determination template form to your application package. This will allow and/or force the upload of a Needs Determination completed outside of FireLight. Rules can be placed on this template to define if a Needs Determination is required in the application. The system will look at rules to determine if the advisor can proceed to 100%. This template will need to be set to a Document Tag for Outside Needs Determination on the Forms page. The Outside Needs Template form could include a signature control. Contact your Strategic Account Manager for a Work Order to implement the necessary rules and Document Tag.

IMPORTANT!

The fields that pass from a Needs Determination to an Application are read-only fields. However, a custom action button has the ability to change the data in read-only fields by design. If the needs determination Application has a custom action button on it, an admin user can disable that custom action button by using a rule to hide the custom action button so users cannot change needs determination data within the associated Application. Following is a sample rule.

Hide a Custom Action Button

 <if> <condition> <compare op="=="> <const value="true" type="String" /> <diget dataitemid="FLI_NEEDS_BASED_APP" /> </compare> </condition> <setvisibility dataitemid="someCustomActionBtn" > <const value="false"/> </setvisibility

 

User Experience

FireLight Needs Determination Activity

  1. SSO directly into the Needs Determination activity or from the FireLight Home page or navigation bar, click New Activity, and click Needs Determination.
  2. In the Create New Needs Determination dialog, click Needs Determination.
  3. In the Required Forms dialog, the Needs Questionnaire is selected and locked. Click Create.
  4. In the Create dialog, type a name for the Needs Determination activity and click Create.

  5. Complete the required information on the Needs Questionnaire form and when form entry is 100%, click the Continue button.

    • If a signature control is on the Needs template form in the application package, the Needs Questionnaire will proceed without signatures. The signature is collected in the application signature process.
    • If a signature control is on the Needs Questionnaire, proceed through the signature process and click CONTINUE on the navigation bar. On the Confirmation Dialog, click Yes. On the transmitted successfully confirmation message, click Close.

  6. Click APPLY on the navigation bar to proceed with creating a new product application. If the product filtering feature is enabled, the list of available products on the Create New Application dialog is filtered based on the entries made in the Needs Questionnaire form. The Needs Questionnaire is included in the application forms. Completed Needs Questionnaire form fields with data items that match application form field data items will pre-fill into the matching application form fields as read-only.

  7. When the application is submitted, the Needs Determination Questionnaire is included in the application documents on the Other Actions menu in Display/Print PDF and Documents listings. On the All Activities page, the Application and related Needs Determination are visible in the list. At the top of the All Activities page in Activity type, select Needs Determination to filter the page view by FireLight Needs Determination activities. A Needs Determination that is tied to an application cannot be deleted unless you first delete the unlocked application and then delete the Needs Determination.

Outside Needs Determination Attachment in Post-Over

The Needs Determination is completed outside of and is sent as an attachment with the description and document tag of Needs Determination with the SSO into . The outside Needs Determination form is included in the application like "Other Documents," but it cannot be deleted.

Outside Needs Determination Document Upload

  1. Complete the Needs Determination outside of .
  2. Start a New Business application in .
  3. FireLight displays a message, for example: "This application has one or more Template forms with Document Tag [Outside Needs Determination]. Please upload related documents from Other Actions/Documents."
  4. On the Other Actions menu, click Documents. In Document Type, select Other, browse for the Outside Needs Determination document, and click Upload.

The document is added to the list of Application Documents. The uploaded Needs Determination document is like "Other Documents" in that you can delete and replace the Needs Determination document. The document is included in the Other Actions menu, Display/Print PDF documents list, and in the Documents for Application list that the client can download and print.
