Post-Issue Services Support
Description, Setup, User Experience, and Additional Information

Description

The Post-Issue Services feature implements new menu options to support the ability to complete and submit post issue servicing forms in FireLight®.

FireLight Setup

Contact your Strategic Account Manager.

  • For single sign on, the SSO needs to be updated to indicate that the transaction is a post issue transaction rather than a new business transaction. The transaction type should be set to ACORD 113 Policy Admin for all post issue transactions. The transaction subtype will then be used to indicate what type of administrative changes is made: ACORD 113 transaction for administrative changes (name changes, address, email, phone, owner change, beneficiary change, etc.), ACORD 105 transaction for withdrawal requests, and ACORD 107 transaction for program additions/changes (automatic withdrawal services, rebalancing, DCA, etc.).

  • Provider changes are needed only if the client would like to use the new outbound ACORD.

  • There is a licensing addendum to support post issue.

Admin Setup

  1. Product Configuration/Rule Sets page, Rules Window: use global data items FLI_TRANSTYPE and FLI_TRANSSUBTYPE in rules on the service forms to enable/disable forms and create validation rules based on the transaction and subtransaction types sent in the SSO request. For direct login users, add checkboxes on the cover sheet for the types of subtransactions, e.g., change of beneficiary, etc.
  2. Organization Configuration/Groups page Activity Access tab: select the Post Issue checkbox for the transaction type for products that will support Post Issue service forms. (If the group is also allowed to create new business applications, ensure the Application checkbox is selected for the group.)

  3. Product Configuration/Activities page:

    1. In the grid at the top of the page, add or select the Post Issue activity.

    2. Ensure ACORD Type is set to Generic Post Issue (113).

    3. Update the settings for your organization.

    4. Enter the Continue/Apply Label, for example, enter Generic Post Issue.

  4. Product Configuration/Products page:

    1. Add a new Post Issue Services product.
    2. Set Product Type, Product Type Filter (UI) and Billing Code to Service Forms.
    3. In the States Approved grid, select the checkboxes in the Approved column for each state where the Post Issue Service product is allowed.
    4. Click Save Product Config.

  5. Product Configuration/Packages page: add a Post Issue Service Forms package.

    1. Set the ACORD Type to Generic Post-Issue (113). Set Area Type to Activity.

    2. On the Form/Wizards in Package tab, set the post issue service forms inclusion option to Conditional or Optional as applicable. Be sure to add at least one required form (e.g., a Cover Sheet, Contract Change Request, etc.) to the package. The package must have at least one required form.

  6. Product Configuration/Sales Agreements page: add post issue service forms to a product by including the post issue service package as part of a specific maintenance type package for the product.

    You can also create a Post Issue only product with Post-Issue Services as the only approved Transaction Type on the Products page. Then add a package and sales agreement for the product with only the post issue service forms included in the assigned package.

  7. Utilities/Reports page, Admin Reports: the Agent Report and Agent Activity Report list the transaction type to allow for tracking of application, subsequent payment, post issue and other transactions.

User Experience

  1. On the Home page in , click Post-Issue Services on the right (direct login users only).

    If the order initiates on the client side, and the SSO into FireLight has the 113 ACORD TransType, the product ID and the product jurisdiction to launch directly into a Post-Issue order, skip steps 1, 2, and 3.

  2. On the Create New Post-Issue Services page, select a Jurisdiction and Product Type in the lists and then click a product name.

  3. On the product Required Forms/Optional Forms page, select any optional forms to include and click the Create button. On the Create dialog, type a name for the application and click the Create button.
  4. Fill in updated information on the applicable application/post-issue service forms and continue the normal application process through signature capture and submission.
