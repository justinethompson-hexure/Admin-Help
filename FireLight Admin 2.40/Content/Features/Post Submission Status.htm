Post Submission Status Workflow
Description, Setup, User Experience, and Additional Information

Description

FireLight® supports post submission status changes and post submission agent notifications of the status change. The workflow includes the Provider polling an outside source to return the current status (e.g., "1035 Exchange Initiated," "1035 Exchange Outstanding," "1035 Exchange Complete," "Contract Issued," etc.). The system first looks to the distributor's Provider and if there is no code for polling, it will then look to the carrier's Provider.

FireLight updates the Display status and External status in the Recent Activities and All Activities and notifies the agent of the status change. Notification to the agent occurs with the status change via an onscreen icon, message center update, and an email with the details of the status.

NOTE: Review queue statuses reside in the Display Status reflecting if the activity has been approved, declined, or needing additional information.

Provider Setup

Contact your Strategic Account Manager to arrange for the necessary Provider setup work to add Provider polling to bring back post-submission statuses for the post submission workflow.

Admin Setup

In Organization Configuration on the Email page in the System category, review and update the Post Submission Pending Case Status email template as applicable for your organization.

User Experience

  1. The agent completes the required application data entry and submits the application to the carrier for processing.

  2. Behind the scenes, the integration Provider polls an outside source to return the current status (e.g., "1035 Exchange Initiated," "1035 Exchange Outstanding," "1035 Exchange Complete," "Contract Issued," etc.).

  3. When a notification comes through the Provider polling call, a message center "envelope" icon displays next to the post-submission activity on the Home page in the Recent Activity list and on the All Activities page in the activities list. If you have the Send Message Center Emails checkbox selected in Preferences, you also receive the message via email.

    The following post-submission statuses display according to the polling call: In Carrier Review, Carrier Review Complete, Approved (Carrier status), Declined (Carrier status), and Further Review (Carrier status).

  4. Click the icon to go directly to the message in the FireLight message center. After you review the message, the icon changes to to indicate the message was reviewed.

    NOTE: The messages can also be accessed by the Message Center icon on the FireLight navigation bar. However, this will show all messages related to that user.

    The post submission message contains the agent name, case name, status, and carrier along with any Provider specific information, such as instructions for the agent.

  5. To review the post submission Audit History for the activity: open the activity and click History on the Other Actions menu, or go to the activity in All Activities and click the display status (e.g., click In Carrier Review) or click the View History link to open the History dialog.

    The History dialog displays the post submission audit history, which includes any polling comments listed within the status change, such as time stamps and additional details pertinent to the audit from the post-submission polling call. NOTE: Any references to the Provider are shown as "Back Office."
