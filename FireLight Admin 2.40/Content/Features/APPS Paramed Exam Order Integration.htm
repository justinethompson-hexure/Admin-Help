APPS Paramed Exam Order Integration
Description, Setup, User Experience, and Additional Information

Description

This Integration will allow for a Paramed exam to be scheduled through FireLight by sending the proposed insured’s parameters to APPS Schedule4Real to return Paramed exam availability. APPS will provide a list of dates and times that are available to schedule the exam and it will include the Examiner ID #. Upon the selection, the carrier will pass the same insured demographics and the selected Examiner ID of the requested date and time. APPS will pass back a message to the carrier with either a Confirmation Number or a message that the time/examiner selected is no longer available. This new functionality is managed via a custom action button within the activity. All clients that want to use this service are required to contract directly with APPS.

Impacts:

  • Activities: Adding checkbox to enable Paramed ordering

  • Admin: Third-Party Configuration - Adding a new Service Type

  • Data Group Properties

  • Email Templates: different statuses on Paramed ordering

  • UI Changes: Statuses

  • Common Tags: life insurance related

  • API Structure: send required insured data and account info, order ID and availability will open and save in the database. We will then submit the order to ExamOne with the same tracing ID and lock the order with the schedule

  • Forms: Data Group Properties

  • Reports: Usage

  • Audits: Addition of new codes needed for ACORD 1122

NOTE: Paramed ordering is currently not available with E-Signature in React for FireLight embedded clients.
