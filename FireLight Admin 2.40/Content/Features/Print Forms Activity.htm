Print Forms Activity

Print Forms Activity / Forms Repository
Description, Setup, User Experience, and Additional Information

Description

The Print Forms activity allows a group of users to print numerous forms from a carrier's forms repository within FireLight®, which can simply be printed as a package without filling out applications, etc., or submitting to a carrier. The carrier sends users directly to the new Print Forms activity via SSO, by clicking the Print Forms button on the Home page, or by selecting the Print Forms option in New Activity on the navigation bar in FireLight. The Print Forms activity gives agents an easy way to access and print the forms the carrier has loaded into FireLight. It can also provide non-FireLight users access to the forms for organizations that license the Print Forms activity.

Print Forms activity:

  • Supports only Wizard data entry for the Print Forms activity wizard. Forms and wizards can be edited in the Print Forms package.
  • The transaction is accounted for the first time the Print button is clicked.
  • NOTE: A Form URL path can be added to each form in the Form Library to allow non-FireLight users access to the forms outside of FireLight. This Form URL field is available only to organizations with the Print Forms activity licensing.

  • After the activity is printed it is available for editing of the inputs. The Print button will still be enabled for additional printing. All fields is editable outside of FireLight, regardless of form and wizard data/fields edited while in FireLight. If data is populated in fields within FireLight, the fields will remain editable outside of FireLight and will not be locked down or stamped on the form.

    NOTE: If the forms have a dynamic dropdown list/combo box/checkbox, rules, etc., that drive form field selections, the available selections is based on the options available at the time the forms are saved outside of FireLight. Read Only field properties are not supported and will remain fillable outside of FireLight.

  • All fields is editable outside of FireLight, regardless of data/fields edited while in FireLight. This could result in the appearance of “double tags” due to tag formatting based on selection. Example: SSN vs. TIN text box based on ownership type; form may appear to have two text boxes for the SSN/TIN entry. Best practice for a Configuration Specialist in this instance would be to have the fields slightly askew for the user to select the appropriate formatted field for their entry.

  • If data is populated in fields within FireLight, the fields will remain editable outside of FireLight and will not be locked down or stamped on the form.

  • The Read Only field property is not supported by Editable PDF Forms.

  • The following properties are supported by Editable PDF Forms:

    Text Box

    • Max Length

    • Font Size

    • Font Family

    • Font Color

    • Text Alignment

    • Background

    • Is Password

    • Mask (phone number, SSN, DOB only)

      If the associated field has a mask configured, and if:

      • The mask contains "phone" in the dataitem name, then the format is (XXX)XXX-XXXX and be 10 digits

      • The mask contains "SSN" in the dataitem name, then the format is XXX-XX-XXXX and be 9 digits

      • The mask contains "DOB" in the dataitem name, then the format is mm/dd/yyyy

      For Dates, this should be the formatting:

      12092005 -> 12/09/2005

      2-3-2005 -> 2/3/2005

      12/31/2005 -> 12/31/2005

      NOTE: It is not date aware to the level of ensuring the date is valid for the given month, e.g., 2-30-2005 -> 2/30/2005. If something is outside the general range (for instance, month > 12, day > 31), it will leave the entered value as-is, i.e., 23-40-1960 -> 23-40-1960.

    If no custom font color, size, style, etc. are selected, the font properties is FireLight defaults.

  • Print Form cases are hidden and do not appear in the Home page or All Activities page. Once the case is closed it will not be accessible.
  • Print Form cases follow the standard purge policy.
  • Reporting is adjusted to account for Print Forms transactions.

FireLight Setup

Contact your Strategic Account Manager. Implementation of this feature requires an addendum to the contract as there are fees associated with printing the forms.

Admin Setup

The setup in Admin is similar to other activities in that access to the Print Forms activity is controlled by settings in Organization Configuration/Groups and you need to add Product, Sales Agreement, and Package with Wizard, Forms and Rules.

  1. Designer/Form Library page:

    1. Upload any additional PDF forms you would like to add to the forms repository in FireLight. The fillable PDFs can include FireLight field properties and rules.

    2. On the Form Detail tab, select the approved states for each form. The Print Forms activity requires a jurisdiction selection to ensure the correct forms are accessed for printing.

    3. On the Form Detail tab, FireLight automatically adds a URL path in the Form URL field for each form when it is displayed. The URL path does not change because it is based on the form guide and org. This allows access to the form for users outside of FireLight. To the right of the Form URL field click the Copy button to copy the URL link for convenience.

      This URL has no authentication associated with it so users outside of FireLight can access the form. The URL path is unique for each organization, contains the org partition code, and either the Form ID or Form Name. It is structured similar to the following example:

      https://www.firelighteapp.com/egapp/unsecure/PrintForm?formid=NQ1234&org=pli (least secure)

      NOTE: The fillable PDFs can include FireLight field properties and rules.

  2. Designer/Wizard Library page: create the wizard for the Print Forms activity.
  3. Designer/Wizard Designer page: add the fields needed to control the inclusion/exclusion of forms to the wizard.
  4. Product Configuration/Rule Sets page: add a rule set and create rules for the inclusion/exclusion of forms for the wizard.
  5. Product Configuration/Activities page: add a new Print Forms activity. Set ACORD Type to Print Forms (310). Enter a Font Awesome Icon in Branding Icon. Update the settings for your organization. Click Save Activity.
  6. Product Configuration/Products page: add a new Print Forms product and select the Product Type. Because FireLight requires a product in the setup and it does not apply to Print Forms, set Product Type to Non-Insurance Product and set Product Category to Non-Insurance Product. In the States Approved grid, select the states where the Print Forms product is approved.
  7. Product Configuration/Packages page: add a new Print Forms package and set ACORD Type to Print Forms (310). On the Forms/Wizards in Package tab, add the Print Forms wizard and the applicable forms to the Forms/Wizards in Package list. For the Print Forms wizard, set the inclusion option to Required. On the Rule Sets in Package tab, add the rule sets for Print Forms to the Rule Sets in Package list.
  8. Product Configuration/Sales Agreements page: add a sales agreement for Print Forms. Enter details, select the approved states, add the Print Forms package to Assigned Packages, and set Package Duty to Required.
  9. Organization Configuration/Groups page: select the Print Forms checkbox on the Activity Access tab and select the Print Forms (Print Forms product name) checkbox on the Product Access tab for the applicable role codes.
  10. Utilities/Deployments page: for the Print Forms activity, deploy the All Configuration (includes Product, Package, Forms, Wizard, Rule Sets, Sales Agreement) and the Group Configuration to the applicable environments.

User Experience

Print Forms activity users in FireLight:

  1. Start a new Print Forms activity in FireLight via SSO post-over or on the FireLight Home page under Start New or on the navigation bar under New Activity, click Print Forms.

  2. On the Create New Print Forms page, select a Jurisdiction, Product Type, and Organization (if applicable) in the lists, then click the Print Forms link. The product and jurisdiction (and Organization, when applicable) help to identify the correct forms.
  3. On the Required Forms page, click Create.

  4. Fill in the information on the Print Forms wizard and the forms and wizards in the package. On the Print Forms navigation bar, click Print.

    The Print button is enabled when no validation errors exist (Alerts check mark is green) or the organization has Allow NIGO Printing enabled in Organization Configuration/Organization Settings on the Activity tab.

  5. In the Select Documents to Print dialog, select the forms to print and click the Print Selected Documents button.

    FireLight generates and opens a PDF file with all selected forms. You can save the file to your computer. Close the file when finished.

  6. Use the buttons on the FireLight navigation bar to go to the Home page, Log Off, or open the Other Actions list and select an option:

    • Click Rename/Summary to view a Summary of the Print Forms activity.
    • Click Display/Print PDF to open the Select Documents to Print dialog and select forms to print. Print the forms, then save the printed forms.

    NOTE: After the activity is printed, it is unlocked and available for editing of the inputs. The Print button is still available for additional printing. Print Form cases are hidden and do not appear in the Home page or All Activities page. Once the case is closed it will not be accessible in FireLight.
