Fee Inquiry Activity

Fee Inquiry Activity Feature
Description, Setup, User Experience, and Additional Information

Description

The fee repository queryfee rule and repository manager are enhanced to support querying fees out of the repository to fill a blank fee worksheet. This is a new option that is an alternative to querying fees based on a predefined PDF that is tagged with data items for each rider type. The new query option uses a number attribute (number=”1” for the first fee, “2” for the second fee, etc.) to pull the nth fee in the list and put it in the worksheet.

The Fee Inquiry activity type allows users with access to the Fee Inquiry activity type to query the selected product fee values and display them in a dynamic fee worksheet. This activity type is available via SSO and through manual selection of the Fee Inquiry activity. The fee inquiry activity is a one-time view into the fees for the selected product. The Fee Inquiry is not saved and will not be available in Recent History or All Applications. The only action available for this inquiry is Print/download. There are no other actions available, including save, signatures, review, send to client, or submit. The fees can now be pulled based on the order of the data in the PPfA and do not require a static placeholder in the PDF. Only the fees that are in the PPfA for the selected product are displayed, which creates a more user-friendly fee display. To do this, the enhancement added a fee attribute called number (number="1" for the first fee, "2" for the second fee, etc.). The Fees and Expense License or DOL License is required to support the Fee Inquiry activity. If you are already licensed for the repository, there is not an additional license and there are no transaction fees to use the inquiry. If you are not licensed, please contact your Strategic Account Manager for an addendum to your contract to support the repository.

FireLight® Setup

  1. The Fees and Expense License or DOL License is required to support the new transaction. If you are already licensed for the repository, there is not an additional license and there are no transaction fees to use the inquiry. If you are not licensed, please contact your Strategic Account Manager for an addendum to your contract to support the repository.

  2. Clients can utilize a SSO with 1241 Transaction Type to launch directly into the jurisdiction and product selection for the Fee Inquiry. In the post-over, add the transaction type 1241 (OLI_TRANS_TRNFEEEXPENSE). This will bring the user to the jurisdiction and product selection for the Fee Inquiry. Additionally, clients can include the jurisdiction (optional) and the product type (CUSIP ID) (optional) to the XML request to post over into the Fee Worksheet for the pre-filled product and jurisdiction.

Admin Setup

  1. Designer/Form Library page: upload the fee inquiry form to be used with the Fee Inquiry transaction. Use the FireLight pretagged form with rules and use the copy fields and rules to finish the form setup. Click this CMR 161212-2 link to open the design document that contains sample queryfee rules. Click this Sample Form link to open a sample form with the tags and rules. The query rules have been enhanced to support a dynamic form pulling all of the fees in the PPfA and CST in the order of the PPfA and not a pretagged static form. (Please note, clients may elect to have a Fee Inquiry form per product type to eliminate sections that do not apply. For example, a FA will not need a fund section.)
  2. Product Configuration/Activities page: select the Fee Inquiry activity. Ensure ACORD Type is set to Fee Inquiry (1241). Update the settings for your organization Update the settings for your organization and click Save Activity.

  3. Product Configuration/Products page: select an existing product. Be sure the CUSIP and Carrier code are defined in the product and match what is loaded in the Fees Profile.

  4. Product Configuration/Packages page: add a new maintenance service package. In ACORD Type, select Fee Inquiry (1241) and set the form inclusion option to Required. Add the fee inquiry form to the package. (Please note, if clients have a Fee Inquiry form per product type, you will need to correctly associate your packages to the selling agreements.)

  5. Product Configuration/Sales Agreements page: add the Fee Inquiry package to the sales agreement for the applicable products.

  6. Organization Configuration/Groups page, Activity Access tab: select the Fee Inquiry checkbox for each role code that will have access to the Fee Inquiry transaction. On the Product Access tab, select the applicable product checkboxes.

  7. Carriers will have to load the Fees and Expenses into the repository. See Fees, Expenses and Commissions Repository for details on adding fees to the repository.

User Experience

  1. Enter FireLight through either a direct login or one of the external services (SSO, New Business, or Alt New Business).

    Direct Login

    1. Click the New Application button and click Fee Inquiry from the drop-down list.

    2. Select the Jurisdiction and the Product Type to create a Fee Inquiry transaction.

      NOTE: The application name dialog does not exist for fee inquiry transactions.

    External Login

    From the external login, FireLight launches into the edit transaction page or the jurisdiction and product selection page.

    If the incoming SSO request contains the transaction type 1241 (OLI_TRANS_TRNFEEEXPENSE) and the jurisdiction and the product type (CUSIP ID) are empty, the user is brought to the jurisdiction and product selection for the Fee Inquiry. Additionally, clients can include the jurisdiction (optional) and the product type (CUSIP ID) (optional) to the XML request to post over into the Fee Worksheet for the pre-filled product and jurisdiction.

  2. Click the Display Print PDF button to view or print the fee inquiry form.

NOTE: In Fee Inquiry transactions:

  • The Display Print PDF button replaces the Other Actions menu.
  • The Save button is removed and you are not prompted to save when leaving the fee inquiry.
  • The fee inquiry transaction does not display in Recent Applications on the Home page or on the All Applications page.
