Salesforce Integration Feature
Description, Setup, User Experience, and Additional Information

Description

NOTE: All contracting is done directly with Salesforce.

FireLight® can integrate with the Salesforce CRM that allows data to be pulled from Salesforce into FireLight using a custom action button on a form or wizard.

Currently, you can map the following Salesforce properties to the corresponding Data Group Property settings within FireLight fields:

  • First Name
  • Last Name
  • Birthdate
  • Phone
  • Mailing Street
  • Mailing City
  • Mailing State
  • Mailing Postal Code
  • Mailing Country

Salesforce Setup

You must first set up an account with Salesforce.

Organizations with a firm-wide Salesforce account and individuals with a single user account need to do the following to create their FireLight Integration connected app in Salesforce and get their Salesforce Client ID (Consumer Key) and Client Secret (Consumer Secret) for the connected app.

Admin Setup

  1. Designer/Form Designer or Wizard Designer page: do the following to add a button control that will pull CRM data into the form or wizard.

    IMPORTANT! FireLight now supports triggering a custom action call from any page within the same package, regardless if the button or fields to trigger that call are on that same page. (See Add Multiple Copies of a Form for use on multiple-instance forms.)

    Form:

    1. Add a Salesforce CRM Button control. In the Field Properties, select CRM in Action Type. Select a CRM Type option in the list: select the Salesforce option to pull data from the Salesforce CRM or select the Preferences option to pull data from the CRM set up on the user's Preferences page in FireLight. In the list, select a Data Group for the button. For example, select the Owner option to populate owner information in the associated fields on the form.

    2. In the associated form fields, select the Data Group that matches the Data Group for the Salesforce CRM button and select the Data Group Property that matches the field entry. For example, in the Owner_FirstName field, select Owner in Data Group and select FirstName in Data Group Property. Ensure the Data Group and Data Group Property are selected in all fields that will pull data from Salesforce.

    Wizard:

    1. Add a Salesforce Custom Action field to the wizard. In the Custom Action Properties, select CRM in Custom Action. Select a CRM Type option in the list: select the Salesforce option to pull data from the Salesforce CRM or select the Preferences option to pull data from the CRM set up on the user's Preferences page in FireLight. In the list, select a Data Group for the button. For example, select the Owner option to populate owner information in the associated fields on the wizard.
    2. In the associated wizard fields, select the Data Group that matches the Data Group for the CRM button and select the Data Group Property that matches the field entry. For example, in the Owner_FirstName field, select Owner in Data Group and select FirstName in Data Group Property. Ensure the Data Group and Data Group Property are selected in all fields that will pull in data from Salesforce.

  2. Set up the Salesforce login credentials at either the organization level or at the advisor level.

    There are two ways to pull in the login credentials for this integration. The best solution depends on how your firm or individual user has contracted with Salesforce.

    Organization level:

    FireLight can pull credentials from the organization configuration setting for firm wide contracts.

    In Organization Configuration on the Organization Settings page, Admin tab, set up the Salesforce credentials for the organization. (Note: It will use the form's Organization if set up, not necessarily the distributor's or carrier's Organization.)

    1. Click the Click to Edit link next to Third-Party Service Config.

      • Click the Add New link at the top of the Third-party Service Configuration dialog. In Config Name, type Username. In Config Value, type the Username credential for your organization’s Salesforce account. In Service Type, select Salesforce and click the Add Config button.
      • Click the Add New link at the top of the dialog. In Config Name, type Password. In Config Value, type the password credential for your organization’s Salesforce account. In Service Type, select Salesforce and click the Add Config button.

      • Click the Add New link at the top of the dialog. In Config Name, type ClientID. In Config Value, type the Consumer Key value for your organization’s FireLight Integration app that you set up in Salesforce. In Service Type, select Salesforce and click the Add Config button.

      • Click the Add New link at the top of the dialog. In Config Name, type ClientSecret. In Config Value, type the Consumer Secret value for your organization’s FireLight Integration app that you set up in Salesforce. In Service Type, select Salesforce and click the Add Config button.

      • Close the Third-Party Service Configuration dialog.

        NOTE: Config Group is an optional configuration used by the DocuSign Integration integration.

        Example:

    2. Click the Save Organization button on the Organization Settings page.

    Advisor level:

    NOTE: To enable Advisors to enter their Salesforce CRM credentials in FireLight, select the Enable CRM Preferences checkbox in Organization Configuration/Groups on the App tab in FireLight Admin.

    Advisors with a user-specific/seat license with Salesforce enter their Salesforce login credentials in Preferences and FireLight will automatically pull their credentials to connect to Salesforce. Advisors need to set up an account in Salesforce to get their login User Name and Password and go through the Salesforce Setup instructions above to get their Salesforce Consumer Key and Consumer Secret.

    1. Open the Preferences page.
    2. In the CRM Preferences section in CRM, select Salesforce.

    3. Enter the Salesforce User Name, Password, Consumer Key, and Consumer Secret and click Save.

  3. Product Configuration/Packages page: add the forms or wizards with the new Salesforce button and fields to the applicable product packages.
  4. Utilities/Deployments page, deploy the Organization Configuration and deploy the Product Configuration and Package Configuration for each product/package that includes the forms and wizards with a Salesforce CRM button.

User Experience

  1. If you have a user-specific license, enter your Salesforce login information in CRM Preferences on the Preferences page once and skip this step in future activities. If the firm has supplied a corporate license, skip this step.

  2. Begin an application or other activity.

  3. Click the Salesforce CRM button.

  4. Near the top of the dialog in My Contacts, enter the first few letters of the contact's last name or commercial name and click the search button to filter the list.

  5. In the list of names, click a name to view the contact's information.

  6. Click the Select button to populate the contact's information in the associated fields on the form or wizard.
