Whitepages.com Integration
Description, Setup, User Experience, and Additional Information

Description

FireLight® can integrate with Whitepages.com. FireLight clients contract directly with Whitepages.com. Whitepages.com provides the Reverse Phone API Key access credential.

Whitepages.com integration does not require any changes to the Provider for your organization.

Admin Setup

  1. Organization Configuration/Organization Settings page, Admin tab: set up the Third-Party Service Config.

    1. Click the Click to Edit link to the right of Third-Party Service Config.

    2. Click the Add New link at the top of the dialog. In Config Name, enter ReversePhoneAPIKey, in Config Value enter the API Key provided by Whitepages, select WhitePages in Service Type and click the Add Config button.

      NOTE: Config Group is an optional configuration used by the DocuSign Integration integration.

    3. Click the Save Organization button on the Organization Settings page.

  2. Design/Form Designer or Wizard Designer page: do the following to set up the Button control and data items on the form or wizard.

    IMPORTANT! FireLight now supports triggering a custom action call from any page within the same package, regardless of whether the button or fields to trigger that call are on that same page. (See Add Multiple Copies of a Form for use on multiple-instance forms.)

    1. Add a Button control to the form or add a Custom Action field to the wizard. In the Field Properties, select CRM in Action Type. Select a CRM Type option in the list: select the Whitepages option to pull data from the Whitepages CRM or select the Preferences option to pull data from the CRM set up on the user's Preferences page in FireLight. In the list, select a Data Group for the button. For example, select the Owner option to populate owner information in the associated fields on the form or wizard.

    2. In the fields that will receive the data, open the Field Properties and select the Data Group that matches the Data Group for the Whitepages button. For example, set the Data Group to Owner. Then select the Data Group Property that matches the field entry. For example, select FirstName in Data Group Property. Ensure that Data Group and Data Group Property are selected on all fields that need to be filled in.

      The following fields can be mapped for the Whitepages:

      • FirstName
      • MiddleName
      • LastName (FullName for Businesses)
      • Gender
      • DiaNumber
      • AreaCode (LineType is mapped in this field)
      • Line1
      • Line2
      • City
      • AddressStateTC
      • Zip

User Experience

  1. Begin an application or other activity.

  2. Click the Whitepages button.

  3. On the upper right of the dialog, type the contact's telephone number and press Enter or click the search button.

  4. On the right, click the Select button to populate the contact's information on the form.
