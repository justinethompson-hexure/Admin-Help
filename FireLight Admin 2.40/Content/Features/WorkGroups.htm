Work Groups

NOTE: The Work Groups section is available when the "Allow Work Groups" option is enabled in FireLight Admin.

Use Work Groups to create inter and intra organization workflows allowing users to have shared visibility into an activity, receive notifications for the activity life cycle, and allow members from cross organizations to have the same visibility.

With this feature, users can set up work groups and add members in the preferences view of the app portal. The Work Group section with the Manage Work button is visible in the user’s Preferences view only if the user’s organization has enabled the option in the Organization section of the Admin Tool. With work groups, activities can be shared with users within the same organization without using the case sharing option in the group’s options of the Admin tool.

The work group functionality allows for greater refinement of sharing activities, because each work group can be limited by the activity type and product type. The members of a work group share can also be set to read-only for the activities in the work group share. In addition to allowing share access to the selected product types and activity types, the members of the group will be cc’d on any agent communications. If the member does not wish to receive the agent emails, the member can check the option to disable notifications for that group.

Guidelines

Workflow participants can:

  • Create activities to which all workflow participants have access/visibility.

  • Access status information on activities.

  • Receive email and FireLight messaging related to activities.

  • Have group/role based permissions regarding functionality allowed on activities.

  • Mark an activity as "Private" if they wish to exclude it from workflow participant visibility.

  • Delegate access to individuals outside standing workflow participants. This may be done with functionality similar to current sharing functionality.

  • Receive all agent emails sent for an activity to all Members of the work group, as well as the Admin and work group Producer. All participants in the work group can choose to Disable Notifications by selecting the checkbox in the row with their Participant ID.

  • Provide case access to a select group.

Following are examples of workflows that would be facilitated by inter-organization transfer of activities:

  • Sales Desk/Case Manager Interaction

  • Post-Issue Changes/Requests

  • NIGO Correction Process

"Admin" roles of the work group have different permissions than "Members" of the workflow.

  • The Admin user who creates the Work Group is automatically assigned as the Admin of the group.

  • The Admin user is required to search for a Producer. This is the Producer/Agent who will become secondary Admin for all activities in the work group. The Admin cannot have the same user role as the Producer. They must be different.

  • Admin users can make edits to Work Group Name, Participants (Remove), Disable Notifications for an individual participant (selecting this option disables all notification types and cannot be refined by specific notification), and enable/disable Read-Only Access for individual members.

  • Permissions are either read-only or edit rights. If read-only, the read-only rights are followed at the group code for the activity. If not read-only, then the Group's session permissions are followed.

Members of the work group can:

  • Disable Notifications.

    Selecting this option will disable all notification types for the individual member and cannot be refined by specific notification.

  • See their edit/read-only access setting, but cannot make changes.

  • Remove themselves from the working group.

  • See who is in the work group with them but cannot make edits to other members or have the option to delete them.

  • Create a new work group and become Admin for that work group.

Members of the work group cannot:

  • Change the Work Group Producer, so they will not see the "Work Group Producer" search box.

  • Add additional members to work groups where they are not the Admin, so they will not see the "Member" search box in their view.

Work Group Email

An email is sent when a Member is added to the work group. The default text for the addition is “You have been added as <ROLE> of the work group <WORK_GROUP_NAME>. If you do not wish to receive notifications for the activities created by the members in this group, you my “Opt Out” by signing into FireLight and navigating to “Manage Work Groups” from your preferences page.”

An email is sent when a member is removed from the work group. The default text for the removal is “You have been removed as <ROLE> of the work group <WORK_GROUP_NAME> . If you feel you have been removed in error, please contact your work group Admin.

An email is sent when a Member opts out of Notifications using the Disable Notifications option in Work Group in the Preferences view. The default text for the Opt Out email is “You have opted out of notifications for activities as a member of the work group <WORK_GROUP_NAME>. If you wish to opt in, you can reverse the “Opt Out” selection by signing in to FireLight and navigating to Manage Work Groups from your Preferences page.

Enabling and Using Work Groups

  1. Enable Work Groups in the Admin tab of Organization settings in the Admin app option on by selecting the Allow Work Groups check box.

  2. In the App, select Preferences from the toolbar menu and click Manage Work Groups.

    3. If the user has the group option for “Allow Creation of New Work Group” checked in the Groups view of the Admin tool, the user can click Create New to create a new work group.

    Users without the ability to create work groups can view their group access and change their notification settings.

    When a work group is created, the user is added to the group as the Admin.

  3. Rename the work group and add a producer. The producer can be added by typing the name in the producer text field and clicking the search icon or just selecting the search icon.

    4. The selected name from the user list populates the Producer field.

    Once the producer field has been filled, the Activities tab and Product tab are populated.

  4. Add members to the group following the same steps.

  5. Check the Read Only and Disable Notifications for each member and the activities and product types you want to share for this group.

  6. When finished, save the changes made by clicking the Save Work Group button at the bottom of the screen.

Manage Work Groups

Add a Work Group

  1. If you have access to Work Groups, click the Manage Work Groups button.

  2. On the upper left of the Manage Work Groups dialog, click the Create New link.

  3. The user role who creates the work group is automatically the Admin Assignment. The Admin enters a Work Group Name, selects a Work Group Producer, and selects a Member to add to the work group. The Admin adds all applicable members to the work group. If a Member creates a new work group, the Member cannot see the User List to search for Work Group Producer or Member.

  4. On the Participants tab, select checkboxes to Disable Notifications and/or grant Read-Only Access for the applicable Participants. All participants in the work group can choose whether to Disable Notifications to avoid receiving all agent emails and notifications.

    NOTE: If the individual user has a group setting for Read-Only pre-defined by role code in FireLight Admin, their rights within the work group is set to Read-Only and cannot be changed within the work group by the work group admin. In FireLight Admin, Groups, App tab under User Rights Home/All Activities, the "Enable Read-Only Activity" setting would need to be changed by the user's FireLight Admin for that member to have Edit rights.

  5. On the Activities tab, an Admin user selects the checkboxes for the activities the work group can access. Activities are limited to the activities available to the work group Producer. Adding and removing activities is for the entire work group and not on an individual basis. Member roles have Read-Only access to the Activities tab. Members must contact the Admin of their work group for consideration of adding/removing an activity. E-Delivery activities are not supported. NOTE: Activities is accessible in "All Activities" from the FireLight home page when an activity is created for the activity and product for those within the work group participant list. The ability to view or read-only after opening the activity is dependent on the settings in Work Groups for the individual. In the event the user is a participant of another work group, the filter functionality can be used on that tab to filter by the owner of the activity.

  6. On the Product Types tab, and Admin user selects the checkboxes for the products the work group can access. Product types are limited to the product types available to the work group Producer. Adding and removing product types is for the entire work group and not on an individual basis. Member roles have Read-Only access to the Product Types tab. Members must contact the Admin of their work group for consideration of adding/removing a product type.

  7. Click the Save Work Group button below the tabs.

  8. Click the X on the upper right of the dialog to close it.

Delete a Work Group or Participant

  1. In the upper grid of Manage Work Groups dialog, click the red X on the right of the Work Group name to delete the work group.

  2. On the Participants tab of the Manage Work Groups dialog, click the red X on the right of a Participant name to delete the participant from the work group.

  3. Click Yes on the Delete Confirmation dialog to delete the work group or participant. The user will no longer see the deleted group or member once the deletion is confirmed.

View or Edit an Existing Work Group

  1. If you have access to Work Groups, click the Manage Work Groups button.

  2. In the Manage Work Groups dialog, the upper grid lists the work groups in which you are a participant.

  3. Click a work group Name in the upper grid.

  4. Depending on whether you are an Admin or Member, view or make edits to the work group on the Participants, Activities, and Product Types tabs.

  5. If applicable, click the Save Work Group button after making edits.

  6. Click the X on the upper right of the dialog to close it.
