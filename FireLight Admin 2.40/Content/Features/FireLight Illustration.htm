FireLight Illustration

FireLight Illustration Activity
Description, Setup, User Experience, and Additional Information

Description

FireLight® Illustration adds basic illustration components (user interface wizards, rule sets, illustration engine, and output), which provide the Guided Illustration and Illustration activities in FireLight. FireLight Illustration currently uses the ForeSight® illustration engine to provide the calculations and output for illustrations. (Hosting of third-party engines in FireLight Illustration is available in a future release.) The FireLight Illustration feature supports only wizards (no forms) for data entry.

This first release of FireLight Illustration includes base Guided Illustration wizards and base sample Illustration wizards and accompanying rule sets for Fixed Annuity (FA), Variable Annuity (VA), and Term Life product types. In the initial release of the FireLight Illustration feature, the FireLight base team maintains the Guided Illustration wizards that are included, which means clients cannot copy, create, or modify the Guided Illustration wizards.

Prior to creating an illustration, an advisor can optionally enter data in a Guided Illustration activity. The Guided Illustration provides a consistent user experience for entering data in each product type and filters the list of available products before continuing to a product illustration. Data gathered in the Guided Illustration activity transfers to the product Illustration activity. Guided Illustrations remain in Data Entry mode and are used repeatedly to gather preliminary data for product Illustrations.

If the client organization also has the Application activity implemented in FireLight, the advisor or agent can choose to start an Application activity in FireLight when the product Illustration is completed. Applicable data gathered in the Illustration activity transfers to the product Application activity along with the PDF output as an attachment.

FireLight Illustration Setup

FireLight Illustration can be implemented for FireLight-only clients and for FireLight clients who are also ForeSight clients.

  • FireLight Illustration requires illustration component licensing fees in addition to usage fees and a one-time implementation fee. Contact your Strategic Account Manager for details.

  • FireLight Illustration feature requires development effort for setup and testing of illustration calculations, output, interfaces, and any integration work that may need to be done.

    IMPORTANT! All organizations calling Illust services must have a CalcRouter config in all environments to correctly set up communication with the ForeSight Illust Services layer.

  • Existing FireLight clients who are also ForeSight clients can contact their Strategic Account Manager to update or upgrade their ForeSight system to use the FireLight Illustration feature. This includes mapping for FireLight Illustration for each ForeSight product module that is in use.
  • Support is available for using the FireLight Illustration content in an API for outside sources to consume the Illustration. Contact your FireLight PM for assistance.
  • A multi-product SaaS web version of the TestBench test tool is available to support regression testing of illustration values and report output for ForeSight and FireLight Illustration.
  • A ForeSight Admin Tool (FSAT) is available to support illustration rate updates, etc.

FireLight Admin Setup

The FireLight team provides sample base illustration wizards for supported product types. The carrier can create and self-manage the product Illustration wizards or the carrier can contact their Strategic Account Manager to have the FireLight team create and design their customized Illustration wizards.

If the organization would like to have the illustrations included in application packages with the ability to print and view illustration PDF documents from the application package, have the FireLight team add a DocType of Illustration with a Document Tag of LinkedIllustration to the organization's list profile. Then add an Illustration template form to the applicable application packages. If a template is not added, the document is visible only by viewing it in Other Actions/Documents and would then be included last in the final document package for submission to the provider.

NOTE: Needs determination documents is attached as NeedsDetermination document tags, illustration as LinkedIllustration, and Account Opening documents as AccountOpening. All other attached documents coming from one activity to the next activity will come over as a document tag of Other.

  2. Designer/Wizard Designer page: create the pages, containers, and fields for the illustration wizards. Click Edit Rules and add new rules to the rule sets using the Rules Window in the Wizard Designer. See Wizard Designer.

    Subfirm management:

    • FLI_AGENT_ID_NUMBER is now required. It is the agent identification number from the TX1228 used in the postover.

    • FLI_SUB_FIRM_NAME and FLI_SUB_FIRM_ID are now required in the FireLight Illustration API. FLI_SUB_FIRM_NAME is the name of the subfirm accessing the system and FLI_SUB_FIRM_ID is the unique ID of the subfirm accessing the system. A contracted firm (the firm setup in the FireLight organization) may have many subfirms under them.  The FLI_SUB_FIRM_ID is used to identify the subfirm. The DTCC ID is used for firms that are DTCC members. When the subfirm is not a DTCC member, FireLight will assign a unique ID. Add a rule to control the validation at the subfirm level.

      Following is a sample rule that checks for FLI_SUB_FIRM_ID. If it does not have a value then a validation message is posted. The same can be done for FLI_SUB_FIRM_NAME. If you do not have a FLI_SUB_FIRM _ID you can put a dummy ID in there. You can do a lot with this rule, which includes looking for specific values, including a generic dummy ID. Or, you can just look for the presence of a value. The following scenario is just looking for a value, not an exact ID.

      Look for a value, not an exact ID

       <if> <condition> <isnullorwhitespace> <diget dataitemid=”FLI_SUB_FIRM_ID" addifnotfound="true" /> </isnullorwhitespace> <postmessage dataitemid=”FLI_SUB_FIRM_ID"> <const value="Sub Firm ID must be provided." /> </postmessage> <else /> <removemessage dataitemid="FLI_SUB_FIRM_ID" /> </if>

      NOTE: Carriers have two options to implement the Sub Firm rules.

      • Assign the FLI_ApiRoleCode to the package

        Using FLI_ApiRoleCode is a heavy-handed approach because if a package is assigned FLI_ApiRoleCode, then only a package with FLI_ApiRoleCode will have the rule. Packages that do not have a Role set will not be loaded. This is very useful when users with a Role get one package and all others get the standard package. Be careful when using this feature.

      • Use FLI_API_INDICATOR DataItem in the rule

        Using the FLI_API_INDICATOR gives them much more granular control. They can use it in the logic of an individual rule. Basically the rule logic would be:

        If (FLI_API_INDICATOR is set) Then

        Run My Rule logic.

User Experience

  1. Start a new Illustration activity in FireLight via post-over or from the Home page under Start New or on the navigation bar under New Activity, click Illustration.

  2. On the Create New Illustration page, select a Jurisdiction and Product Type in the lists, then click a Product Name. (If Localization is enabled for your organization, select an option in the Locale list if the locale is different from the default locale.)
  3. Fill in the information on the illustration wizard.
  4. Click Illustrate on the toolbar to view illustration calculations in a Quick View dialog. On the lower right, click Close to exit Quick View.
  5. Click Print on the toolbar to view the illustration reports in a separate tab or window.

  6. When no Alerts are present, the product illustration is complete.

  7. To save the illustration, click Save on the navigation bar, enter the Illustration Name in the Save Illustration dialog, and click Save. If you do not save the illustration, it will not be added to the Recent Activity list on the Home page or to the All Activities page.

  8. (Optional) If your organization has the Agent Illustration PDF Output option enabled and you are sales desk, assistant, etc., click the Send Email button next to the Print button on the illustration toolbar and fill in the information on the Send Email dialog. If a “|CLIENT_NAME|” tag appears in the Subject and Message, replace it with the client’s name. Click Send to send an email to the agent with a link to the illustration PDF output. Repeat for each agent to receive the email. This allows the agent to click the link within the email and quickly print the illustration PDF output.

  9. (Optional) If your organization has the Application activity implemented, you can click Apply on the toolbar to begin a new application for the illustrated product. FireLight pre-fills data entered on the illustration into matching fields on the application. The Apply button is unavailable when Alerts are present.
