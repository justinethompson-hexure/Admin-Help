Account Opening Activity
Description, Setup, User Experience, and Additional Information

Description

The Account Opening activity allows users to create a pre-sale account. Account Opening is a submitted activity separate from other activity types (e.g., application, illustration, etc.) for reporting and downstream processing. Account Opening has its own Post Over to send users directly into the Account Opening activity or users can click Account Opening on the FireLight Home page. The initial Account Opening screen includes the Jurisdiction list and Custodian list (if more than one Custodian exists). Account Opening has its own output file for downstream processing. The Account Opening activity can be used as a launch point into other supported FireLight activities, but the Account Opening, because it was submitted, cannot be edited once complete. Account Opening activities have their own transactions within the Reports.

FireLight Setup

Contact your Strategic Account Manager to create an addendum to the contract to support this new Account Opening activity type and accept the small transaction fees.

If the organization would like to have the Account Opening activities included in application packages with the ability to print and view account opening PDF documents from the application package, have the FireLight team add a DocType of Account Opening with a Document Tag of AccountOpening to the organization's list profile. Then add an Account Opening template form to the applicable application packages. If a template is not added, the document is visible only by viewing it in Other Actions/Documents and would then be included last in the final document package for submission to the provider.

NOTE: Needs determination documents are attached as NeedsDetermination document tags, illustration as LinkedIllustration, and Account Opening documents as AccountOpening. All other attached documents coming from one activity to the next activity will come over as a document tag of Other.

Admin Setup

  1. Design/Form Library or Wizard Library page: upload the account opening form or create a wizard, enter details, and select the approved states. Add an account opening template form with a signature control with the Document Tag set to Account Opening.
  2. Product Configuration/Rule Sets page: add or update a rule set with rules for the account opening form or wizard.
  3. Product Configuration/Activities page: select Account Opening in the grid or add an Account Opening activity. Ensure ACORD Type is set to Account Opening (106). Verify the other settings on the page. See Activities for detailed information.
  4. Product Configuration/Products page: add a new account opening product and select the Product Type. Because FireLight requires a product in the setup and it does not apply to Account Opening, we recommend that you set Product Type to Non-Insurance Product and, if available, set Product Category to Non-Insurance Product.
  5. Product Configuration/Packages page: add a new account opening package for the account opening form/wizard and set ACORD Type to Account Opening. On the Forms/Wizards in Package tab, add the account opening form/wizard to the account opening package and set the inclusion option to Required. On the Rule Sets in Package tab, add the rule set for the account opening form/wizard.
  6. Product Configuration/Sales Agreements page: add a sales agreement for account opening. Enter details, select the approved states, add the account opening package to Assigned Packages, and set Package Duty to Required.
  7. Organization Configuration/Groups page: select the Account Opening checkbox on the Activity Access tab and select the Account Opening checkbox on the Product Access tab for the applicable role codes.
  8. Deploy forms/wizards, product configurations, and organization configurations using the Utilities/Deployments page. See Deploy Data for instructions.

User Experience

  1. Start a new Account Opening activity in FireLight via post-over or from the Home page.

    • FireLight accepts a new ACORD TransType of 106 for an account opening. You can also pass a New Business Policy Number Submission OLI_TRANS_NBPOLNUM. Upon receiving this value in the SSO, FireLight puts the user in the account opening activity.
    • On the FireLight Home page under Start New or on the navigation bar under New Activity, click Account Opening.
  2. On the Create New Account Opening page, select a Jurisdiction and Custodian in the lists, then click or tap an account opening. (The Custodian list will not appear if only one Custodian exists.)
  3. On the Required Forms page, click Create. In the Create dialog, edit the Name for the Account Opening, select an option from the On Behalf of list if applicable, and click Create.
  4. Fill in the information on the new account form or wizard. When Data Entry reaches 100%, click CONTINUE.

  5. If signatures are required: on the Electronic Signatures page, click a signing option: Use E-Signature to sign the forms electronically or Decline E-Signature to wet sign paper forms.

    If using e-signature: in the List of Required Signers, select a signer type and click Sign Now to sign the account opening on this device or click Send Email Request to have the user sign later using another device. Repeat until all signer types have signed the forms.

  6. Submit the account opening.
