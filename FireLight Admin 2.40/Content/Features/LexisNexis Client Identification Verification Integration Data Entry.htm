LexisNexis Client Identification Verification During Data Entry
Description, Setup, User Experience, and Additional Information

Description

FireLight can integrate with LexisNexis, a third-party client authentication vendor. FireLight clients contract directly with LexisNexis to obtain their access credentials. Organizations can configure LexisNexis for client identity verification during the data entry process and during the e-Signature process. Organizations work directly with LexisNexis to configure the number of questions required for a pass score for client authentication during the e-Signature process. This topic covers the LexisNexis client identity verification during the data entry process.

NOTE: For LexisNexis client identity verification during the signature process, see LexisNexis Client Identification Verification During E-Signature.

Through rules, organizations can limit users in how many custom action button clicks they are allowed to make per signer. For an example, see "Disable Custom Action Button After Three Attempts" below.

Guidelines

Currently, FireLight:

  • Creates an audit record for the initial web service call to LexisNexis and a second audit at the conclusion of the authentication process. The CVI, NAS, NAP, status, score, and diversionary values is included in the application audit report within the Identity Authentication section and is available for use within the Provider.
  • Provides an activity report with LexisNexis audits for all applications (including active applications not yet submitted) to aid in the reconciliation process with the details LexisNexis provides for billing.

LexisNexis Usage Report:

This report assists users in triaging any activity associated with the LexisNexis services, including but not limited to the RDP Quiz, Instant ID, and the Risk Classifier. Following are items specifically available in the LexisNexis Usage Report to help clients triage their LexisNexis activities. See Detailed Activity Reports in Admin Help for a sample report.

  • Instead of showing the "Status during time of call" on the LexisNexis Usage Report, the column is now renamed to the "Service Type" column. The Service Type column shows "Instant ID," "Quiz," or "Risk Classifier" to distinguish among the various LexisNexis web services.

  • The new "Quoteback ID" column shows the QuoteBack sent in the XML response in the following order: Application ID/Selling Org/Product Org/Application Name. This value is up to 60 characters long.

LexisNexis Instant ID Admin Report

This report provides an Admin report that includes LexisNexis Ref ID, LexID/Order Number, CVA Score, NAS Score, IID Risk Indicator(s), and Pass/Fail Score during the dates specified. See Admin Reports in Admin Help for a sample report.

FireLight Setup

Provider work needs to be completed to use the LexisNexis Services. Please contact the Implementation team to complete this work.

Admin Setup

Admin Setup for LexisNexis Client Validation During the Data Entry Process

  1. In Organization Configuration on the Organization Settings page, Admin tab: set up the LexisNexis credentials. LexisNexis supplies the configuration IDs and values.

    NOTE: The credentials will always be the carrier credentials. If a carrier is going to deploy their product and forms to a distributor, the credentials from the carrier is used. For distributors, if credentials exist in both the carrier and distributor environments, FireLight uses the carrier's credentials.

    1. Click the Click to Edit link next to Third-Party Service Config and enter the LexisNexis credentials (configuration IDs and values): Username, Password, AccountID, and Workflow. AccountID and Workflow are used to create the service endpoint; therefore, slashes(/) are not allowed.*

      • Click the Add New link at the top of the dialog. In Config Name, type Username. In Config Value, type the accountID/username (e.g., 12345/LexisNexisUserID) LexisNexis provided. In Service Type, select LexisNexis and click the Add Config button.

      • Click the Add New link at the top of the dialog. In Config Name, type Password. In Config Value, type the password LexisNexis provided. In Service Type, select LexisNexis and click the Add Config button.

      • Click the Add New link at the top of the dialog. In Config Name, type AccountID. In Config Value, type the accountID LexisNexis provided. In Service Type, select LexisNexis and click the Add Config button.

      • Click the Add New link at the top of the dialog. In Config Name, type Workflow. In Config Value, type the workflow ID LexisNexis provided for your organization. In Service Type, select LexisNexis and click the Add Config button.

        *Config Group is an optional configuration used by the DocuSign Integration integration.

        Examples:

      NOTE: A Provider Work Needed pop-up message appears when you click the Add Config button. Click the OK button to close the pop-up. The Provider work is referring to the forms development work of the CustomAction button and DataGroup for the PARTY (Insured, Owner, Payor, etc.). You will probably want the data in the datafeed for processing in the backoffice. Contact your Strategic Account Manager for assistance.

      To update a config value: clear the current value, enter a new value, and click the Update link next to the value. To delete a config value: click the Delete link next to the value and click OK on the confirmation dialog.

    2. Click the Save Organization button on the Organization Settings page.

    NOTE: When in UAT or STAGING, use the staging credentials from LexisNexis. LexisNexis PRODUCTION credentials will not work in the FireLight UAT and STAGING environments. When testing in UAT or STAGING, use the test data provided by LexisNexis in the spreadsheet within the Admin Help here: Insurance_Technologies_TestSeeds_December2019.xlsx.

  2. Designer/Form Designer and Wizard Designer (if wizard is applicable):

    Add a custom action button to the form/wizard that will initiate a web service call to LexisNexis for each signer type and add the associated fields for the data items.

    IMPORTANT! FireLight now supports triggering a custom action call from any page within the same package, regardless of whether the button or fields to trigger that call are on that same page. LexisNexis cannot be used on multiple-instance forms. (See Add Multiple Copies of a Form for a description of multiple-instance forms.)

    Form:

    1. Add a Button control to the form.
    2. Double-click the Button control to open the Field Properties dialog.
    3. Enter the dataitem name for the button in Name.
    4. Select the signer type (e.g., Owner) in the Data Group list.
    5. Select Custom Action in the Action Type list.
    6. Enter LexisNexis in Action Parameter and click Update.

    Wizard:

    1. Add a Custom Action control to the wizard.
    2. Double-click the Custom Action control to open the Custom Action Properties dialog.
    3. Enter the dataitem name for the custom action button in Name.
    4. Select the signer type (e.g., Owner) in the Data Group list.
    5. Select Custom Action in the Custom Action list.
    6. Select LexisNexis in the Custom Action Type list.
    7. Enter LexisNexis in Custom Action Parameter and click OK.

  3. Add Text Box field controls for the data items and set the Data Group and Data Group Property of each data item used by LexisNexis for each signer type in the Forms. Allowed signer types include Owner, Joint Owner, Insured, Annuitant, Joint Annuitant, and Payor. The First Name, Last Name, Street Address, City, State, Zip, DOB, SSN, and Phone Number data items are required to make the web service call to LexisNexis.

    If using Wizards to set the Data Group and Data Group Property of each data item, FireLight requires that the same Data Group and Data Group Property of each data item must also be set in the Form for the mapping to work properly.

User Experience

In the FireLight data entry process for a signer type with LexisNexis validation enabled, the agent completes the data entry for the client First Name, Last Name, Street Address, City, State, Zip, DOB, SSN, and Phone Number for the signer type (Owner, Insured, Annuitant, Joint Owner, Joint Annuitant, Payor) on the form or wizard and clicks the custom action button on the form for LexisNexis to validate the client.

When the agent clicks the custom action button, FireLight automatically saves any data that was entered on the activity. This includes any data group and data group properties that are populated (i.e., Owner First Name, Owner Last Name, etc.).

In All Activities, FireLight disables the Delete button for activities with LexisNexis validation.

NOTE: When the custom action call has occurred, the application is able to be copied, but the LexisNexis values (LexID, NAS Score, NAP Score, CTI Score, Risk Indicators, and Pass/fail Score) will not be copied over to the next application.
