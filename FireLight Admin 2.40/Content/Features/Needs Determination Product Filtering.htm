Needs Determination Product Filtering

Needs Determination Product Filtering
Description, Setup, User Experience, and Additional Information

Description

This feature provides a filter from the FireLight® Needs Determination activity to the available products to select in an application. NOTE: At this time, product filtering is not available in workflows that use outside needs determinations.

In FireLight Admin, the Needs Filter tab on the Organization Configuration/Groups page allows a group code to have rules and/or product filter categories defined for when to include a product in the available product options for the application selection. For example, Accumulation, Income, Protection, Death Benefit, and Protection of Capital can be questions asked in the Needs Determination and based on the answers to these questions and others (financial profile, risk tolerance, time horizon for access to the funds), FireLight filters the list of available products so that only the products that meet the rules are listed in the available product options for the application product selection.

Additionally, a product level definition needs to be added to identify whether the product supports Non-Qualified and Qualified money, and whether the product is a Fee Based Product or Commission Based Product. The carrier needs to define these settings at the product level, similar to how states are defined. These settings can be part of the distributor logic in their filter rules.

The product filter leverages rules written in the Needs Determination transaction, the Product Category defined in Admin, and a query to define the filter.

FireLight Setup

  • Licensing is required to support this feature. Distributors need to work with their Strategic Account Manager for pricing. Additionally, each client needs to submit a Work Order to define their specific filter criteria for the products on their shelf. This needs to be done in conjunction with your Needs Determination questionnaire or SSO.

  • Each financial institution needs to define their own product categories and rules to apply for the filter.

    Because some firms have different distribution channels set up in FireLight, the rules can be written to apply to group roles.

    Using the filter rules, FireLight can use weightings and scorings to include or exclude products in the filter. Clients using the Needs Determination transaction need to define the rules, scoring, and weighting in the rules engine for the Needs Determination questionnaire. Upon launching into the Application, the rules need to be evaluated and applied to further filter the product availability in the application product selection. Submit a Work Order for the exact rules needed. For filter examples to provide in your work order, click the "CMR 147397" design document link in the "Additional Information" section of this feature.

  • When using the FireLight Needs Determination Transaction to capture questions that are stored as new data items, the data items are defined into filter rules to apply to the available product selection in the FireLight application. Submit a Work Order for the definition of the product filter. Rules written in the Needs Determination transaction will assign data points based on items selected in the Needs Determination transaction. These data items are used to query the Needs Filter table.
  • When using an Outside Needs Determination, the SSO contain the data items that are defined in the filter. Submit a Work Order for creating the SSO linking to the product filter.

Admin Setup

  1. On the Organization Configuration/Groups page, select a TC Role Code in the upper list.

  2. On the Activity Access tab, select Needs Determination.

    Additionally, to require the User Group to always do a Needs Determination before an application, also select the Disable Create Application checkbox.

  3. On the Needs Filter tab, do one of the following:

    • Click the Upload button to upload a CSV or XLSX file with the product categories for the products to which the selected group has access. If an uploaded data set is too large, the group filter definition table will not render on the Needs Filter tab and a "Data set too large - unable to render table" message appears under the Upload button. The data is saved, but you are unable to view it on the Needs Filter tab. See the "Additional Information" section below for links to the design document and sample XLSX template.

    • Manually add the product categories as follows.

      1. Select an Organization in the list. This is the organization whose products is filtered.
      2. Click the New Category button.
      3. On the right, type a name for the filter category.
      4. In the list, select a filter type: String, ValueGreaterThanX, ValueLessThanX, NumericRange, StringOverrideFilter, ValueContainsString, or NotEqualString. See the page for filter type descriptions and accepted inputs.
      5. Type the Data Item ID from the Needs Questionnaire form fields or rules.
      6. On the left, select Products in the list to which filters will apply.
      7. Enter the applicable filter value for each product. If the filter category does not apply to a product, leave it blank for that product.

      8. Click the Save button.

  4. Repeat adding product filter categories for each applicable User Group.

User Experience

  1. SSO directly into the Needs Determination Transaction or from the FireLight Home page or navigation bar, click New Application, and click Needs Determination.
  2. In the Create New Needs Determination dialog, click Needs Determination.
  3. In the Required Forms dialog, the Needs Questionnaire is selected and locked. Click Create.
  4. In the Create dialog, type a name for the Needs Determination application and click Create.

  5. Complete the required information on the Needs Questionnaire form and when form entry is 100%, click the Continue button.

    • If a signature control is on the Needs template form in the application package, the Needs Questionnaire will proceed without signatures but ensures the signature is collected in the application signature process.
    • If a signature control is on the Needs Questionnaire, proceed through the signature process and click CONTINUE on the navigation bar. On the Confirmation Dialog, click Yes. On the transmitted successfully confirmation message, click Close.
  6. Click APPLY on the navigation bar to proceed with creating a new product application. The list of available products on the Create New Application dialog is filtered based on the entries made in the Needs Questionnaire form. The Needs Questionnaire is included in the application forms. Data items from the Needs Questionnaire are pre-filled in the product application as read-only fields.
