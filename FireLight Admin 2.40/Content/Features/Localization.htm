Localization

Localization Feature
Description, Setup, User Experience, and Additional Information

Description

Enable localization for languages/locales other than the English (en-us) default language.

FireLight Setup

Contact your FireLight® Strategic Account Manager for the following:

  • Have the FireLight team provide a spreadsheet of the FireLight UI labels, lists, and messages in English and in the initial translation of the localized language. You can update the spreadsheet with your preferred translations and return the spreadsheet to the FireLight team for implementation.
  • Provide the available jurisdictions for the language/locale. This requires a database update. Several database updates are required for localization.

  • Your organization's Provider may also require changes in logic or where it needs to send data. When a product is localized and two different formatted versions of a value (e.g., localized date and currency formats) exist, the integration developer may need to use the Product locale to determine the expected format.

    Example Date Time: March 13th 2024 3:19:42 PM

    en-US - 3/13/2024 3:19:42 PM

    en-CA - 2024-03-13 3:19:42 PM

    fr-CA - 2024-03-13 15:19:42

Admin Setup

  1. On the Organization Configuration/Organization Settings page:

    1. On the Organization tab, click the Localization Click to Edit link. In the Localization dialog, select the Languages/Locales for your organization and save the organization. The default is English (en-US) and cannot be cleared from the list of approved languages/locales.
    2. On the Activity tab in Edit Masks, click the Add link and add the date and currency formats for the approved locales if different from the default en-US locale. For example, you could add a mask for the SIN government identification number using the SSN edit mask and give it a new name, such as "SIN Edit Mask."

  2. On the Designer/Form Library or Designer/Wizard Library page:

    1. Upload forms or create wizards in the new language/locale.
    2. For each form or wizard: add detail, select the States Approved, and save the form or wizard.

  3. On the Designer/Form Designer or Designer/Wizard Designer page:

    1. Design the forms or wizards pages and fields using localized display text.
    2. Save the forms or wizards.

  4. On the Product Configuration/Rule Sets page:

    1. Add a new Rule Set for the localized product.

    2. Update the rules for currency, dates, UI messages, etc., that require the localized language and save the rule set. (Data items are not translated.)

      NOTE: The rule set can be updated from the Form Designer or Wizard Designer by clicking the Designer/Form Designer, Rules Window link or the Designer/Wizard Designer, Edit Rules link to open a Rules window.

  5. On the Product Configuration/Products page:

    1. Add a separate product for the additional locale. For example, for product ABC that is in the English (en-us) language/locale, add a product ABC-fr-CA for the French Canadian locale.

    2. Select the applicable language/locale in the Product Locale list.

      Retain the same CUSIP ID as the product that is in the English (en-US) locale.

    3. Select the States Approved.
    4. Save the product.

  6. On the Product Configuration/Packages page:

    1. Add a new package for the new localized product.
    2. Add the localized forms and rule sets to the package and save the package.

  7. On the Product Configuration/Sales Agreements page:

    1. Add a new sales agreement, assign the localized package and set Package Duty to Required.
    2. Select the State Approvals and save the sales agreement.

  8. On the Organization Configuration/Email page:

    For each email template used by your organization, select the Locale for the template, add the applicable translated text, and save the email template. Do not translate <VARIABLE_NAMES> that appear in brackets. FireLight will match the email template Locale with the Product Locale.

  9. On the Organization Configuration/Groups page:

    1. In the list, select a group.
    2. On the Product Access tab, select the checkbox for the new product to grant group access to the product.

  10. On the Deployments page:

    Deploy the product, organization, localization and group configurations to the appropriate environment.

User Experience

NOTE: In Other Actions, the Documents dialog used in Wizard activities uses the standard file upload controls, which cannot be localized. An alternative is being developed for v2.19. This will affect clients using Wizards and a language other than English. All other text on the dialog can be localized.

Users can change the language/locale in FireLight.

  1. When you log in to FireLight for the first time, FireLight prompts you to select your preferred language (locale) and changes the user interface (UI) language to the selected locale. The FireLight user interface labels, lists, and messages display in the language associated with the selected Locale.

    NOTE: You can change the default Locale and the Jurisdiction in Preferences.

  2. When you create a new activity, FireLight filters the Products list for the Locale on the Create New page. To view other products, you can select a different Locale on the Create New page. FireLight changes to match the locale of the selected product. The Signature process, Review Queue, etc., will use the same locale as the product.

  3. FireLight remains in the same locale as the product application until you create an application in a different locale, you change the Locale in Preferences, or you log off FireLight. When you log back on, FireLight reverts to your selected Locale on the Preferences page.

    NOTE: If you copy a localized activity on the All Activities page and make updates to the copy, you may want to update the copy of the activity in the same locale as the original activity to avoid some potential issues with localized dates, currencies, etc.
