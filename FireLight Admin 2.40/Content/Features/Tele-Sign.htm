Tele-Sign
Description, Setup, User Experience, and Additional Information

Description

FireLight® provides support for integration with Interactive Voice Response (IVR) providers to add Tele-Sign as an option within FireLight to satisfy client signature requirements without requiring email services. This is especially useful when applications are taken over the phone and clients do not have access to email.

For clients that enable Tele-Sign, there are minor impacts to the FireLight UI, Admin tool, and the Audit Report. Implementing this feature will require an integration with an IVR platform.

Two new web service calls are created. The first verifies the identity of the Applicant/Payor entered in FireLight and is executed by a REST based call/response to validate inputs provided by the client when they initiate the IVR phone call. This web service is initiated by the IVR service. The second web service call is also initiated by the IVR service and provides the status of the signature request to FireLight. Upon receiving the completion web service call, FireLight applies a new “TeleSigned by: xx” signature stamp in the application.

Tele-Sign Setup

Client organizations contract directly with the Interactive Voice Response (IVR) provider for Tele-Sign capability.

Contact your Strategic Account Manager for the FireLight API setup work.

Organization Configuration/Groups, App tab:

  1. In the Signing: Tele-Sign section, grant access to each user group that will have access to Tele-Sign.

    • Select the Enable Tele-Sign checkbox to enable the Tele-Sign feature.

    • Select the Enable Tele-Sign Email Link checkbox to enable the ability for users to select to send the "Tele-Sign Document Delivery" Tele-Sign Document Delivery email to the client to view and download the Tele-Sign PDF forms. The signer will need to verify the last 4 digits of their ID and their Birth Date to view and download the PDF documents.

    • Select the Enable Tele-Sign Download Forms checkbox to enable and show the Download Forms button to users for downloading Tele-Sign PDF forms for mailing to the signer.

    • Select the Enable Tele-Sign Generate Link checkbox to enable and show the Generate Link button to users. The Generate link process creates a URL link that a signer can open to access the PDF forms. The signer will need to verify the last 4 digits of their ID and their Birth Date to view and download the PDF forms.

  2. Allow a single signer to sign for multiple signer types. When a form has signature fields for multiple signer types (e.g., Insured and Owner are the same person), under Signing: General, select the Enable Single User to Sign Multi Signer Types checkbox. Tele-Sign requires only one signature capture for the non-agent signer types.

Designer/Form Designer or Wizard Designer page:

Add the following four data items to the form or wizard (either as fields or set them via a rule):

  • FLI_ENABLE_TELESIGN – set this to True if you want the Tele-Sign option to be available.

  • FLI_TELESIGN_APP_ID – set this to any number (recommended at least 4 digits – must be unique per application).

  • FLI_TELESIGN_PIN – set this to any 4-digit number (does not need to be unique).

  • FLI_TELESIGN_DOB – set this to the signer's birth date.

Product Configuration/Sales Agreements page:

Select the Tele-Sign scripts for the applicable products in the sales agreements.

  1. In the grid at the top of the page, select a product that will use Tele-Sign.
  2. On the lower left, click the Options tab.
  3. In the Tele-Sign Script list, select the Tele-Sign script for the selected product.
  4. Click Save Sales Agreement.

Utilities/Deployments page:

  1. In Synchronization Scope, select the Tele-Sign Script Configuration scope.
  2. Type a Reason for the deployment.
  3. In the grid, select the checkboxes for the Tele-Sign products to deploy.
  4. On the right, click Deploy Now or schedule a deployment to deploy the Tele-Sign scripts into the next environment. (There are no Distributor Only or Carrier Only versions of the Tele-Sign Script Configuration.)
  5. In Synchronization Scope, select the applicable All Configuration scope to deploy the updated Sales Agreements.
  6. Type a Reason for the deployment.
  7. In the grid, select the checkboxes for the Tele-Sign products Sales Agreements to deploy.
  8. On the right, click Deploy Now or schedule a deployment.

User Experience

  1. Create a new application (using a form containing the Text Box fields listed above).
  2. Enter appropriate Tele-Sign data into the four fields listed above.
  3. Complete data entry on the application.
  4. Click Continue.
  5. Click Use E-Signature.
  6. In the List of Required Signers, select the non-agent signer. Only one non-agent signer can sign using Tele-Sign. If the organization has Enable Single User to Sign Multi Signer Types option enabled, and the application has more than one signer type (e.g., Insured and Owner), and both signer types are the same person, select both signer types for Tele-Sign to work.
  7. On the Client Signature Choice page, click the Tele-Sign button on the right (this button will not be available unless Enable Tele-Sign is selected in Group options and FLI_ENABLE_TELESIGN data item is set to True on the form).
  8. On the Tele-Sign: Signer Information page, fill in the Signer Information at the top of the page (Signer Full Name, City, and State). At the bottom of the page, fill in the Signer Email, Signer Birth Date and Signer Last 4 Digits of SSN/Government ID to have an email with a link to the signed documents be sent to the signer after the signature is completed. Click Continue.

  9. On the Tele-Sign: Form Access page, select a Delivery Type in the list (Download, Generate Link, or Send via Email).

    NOTE: If any of the following delivery options are disabled for the user's role, the disabled options do not appear in the list. You do not need to select one of these options to complete Tele-Sign.

    1. Download a copy of the application forms: In the Delivery Type list, click Download. Print the forms and provide the printed forms to the client via mail or other form of delivery.

    2. Generate a URL link to give to a signer so they can view the application PDF online:

      1. In the Delivery Type list, click Generate Link.
      2. Enter the Last 4 Digits of SSN/Government ID, enter the Signer Birth Date in mm/dd/yyyy format (with slashes), and click Generate Link.
      3. Provide the link to the signer so they can access the application PDF.
      4. The signer goes to the URL in a web browser, enters their Last 4 Digits of SSN/Government ID and their Birth Date to view the application in PDF format.
      5. The signer uses the PDF viewer controls to download and print the PDF file.

    3. Send an email to the client signer to access the PDF forms:

      1. In the Delivery Type list, click Send via Email.
      2. Enter the Signer Name, Signer Email, Signer Last 4 Digits of SSN Government ID, and Signer Birth Date.
      3. Click Send Email.
      4. The client receives an email and clicks the URL link in the email to go to the URL in a web browser. Client enters Last 4 Digits of SSN/Government ID and their Birth Date to view and download the PDF application documents.
  10. Click Continue.
  11. Read the Tele-Sign script and follow the instructions to verify the signer’s identity and gain their Tele-Sign voice signature.

  12. OPTIONAL: Click the Cancel Tele-Sign button to clear the signature and start over.

  13. Click the Done button to continue.

    • If the Tele-Sign signature was completed but declined, FireLight returns to the application in Data Entry mode.

    • If the signature has not yet been completed, FireLight returns to the List of Required Signers page. A "(Tele-Sign in Progress)" indicator appears next to the client signature name for any signer types using Tele-Sign.

      • To return to the Agent Instructions page, click the client Tele-Sign signature button. Click Done on the Agent Instructions page to return to the List of Required Signers page.
      • To cancel a Tele-Sign signature capture, click the Cancel Tele-Sign link under the client signer name and click OK on the confirmation dialog.

    • If the Tele-Sign signature was completed and approved, FireLight returns to the application view to complete the review process if Review is enabled. If Review is not enabled, the application returns to the List of Required Signers if additional signatures need to be collected, or the application can be submitted either manually or automatically if the Enable Auto-Submit option is enabled on the Organization Configuration/Groups page, App tab.

      NOTE: The Agent can sign in a default order, first or last, depending on what is set in the Agent Signing Order list in Organization Configuration/Organization Settings page on the Signature tab.

  14. FireLight sends a message center notification that specifies either an approved or declined signature.
