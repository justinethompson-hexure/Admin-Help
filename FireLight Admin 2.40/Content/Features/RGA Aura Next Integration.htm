RGA Aura Next Integration
Description, Setup, User Experience, and Additional Information

Description

NOTE: The RGA Aura Next integration is available for Wizard only.

With the Aura Next integration, Firelight® leverages the Aura Next APIs within an activity to gather underwriting data for life applications.

The Aura Next APIs are embedded in FireLight and shown in a dialog when opened by clicking a Custom Action button added to a wizard page in the activity.

When Aura Next is opened, data is passed from the FireLight activity to set up the correct interview questions in Aura Next.

When Aura Next is closed, the data is saved and passed to FireLight, which can be mapped to the appropriate PDF form fields so the data is submitted with the activity.

FireLight queries an Aura Next API for a status. Once the data is complete in the Aura Next interview, the status is set to complete. FireLight leverages the status to recognize when the activity can be set to 100% complete when data entry is finished. After the Aura Next interview is complete, it can be reopened and updated prior to activity submittal.

FireLight also retrieves and stores the case-level underwriting decision when the interview is complete. Additionally, the full underwriting decision tree is stored in JSON format and made available to the provider for added customizations.

NOTE: Aura Next setup is handled by the carrier, not the distributor. When a distributor runs a carrier product activity, the carrier custom action button and provider are called after the relevant distributor components.
