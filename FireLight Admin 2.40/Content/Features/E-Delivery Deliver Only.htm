E-Delivery Deliver Only
Description, Setup, User Experience, and Additional Information

Description

The E-Delivery Deliver Only workflow provides delivery only of client contracts. Deliver Only cases require no signing and arrive in a locked status. An e-Delivery Deliver Only request initiates with FireLight® receiving an ACORD DocSend (TX121 and a TransSubType typecode 12123 with processing instruction = Agent Directed or Client Directed) message that drives the E-Delivery Deliver Only workflow. The organization's provider can have the ability to access the TXLifeRequest object so the provider can add and manipulate data items that are sent in the 121. The provider can be able to call the TXLifeRequest object for all agent-directed and client-directed cases. During the process, FireLight provides status messaging back to the carrier to keep track of the contract delivery.

NOTE: The initial e-Delivery emails for client pick-up can be resent with the trigger being a 121 with an ACORD TransMode 10 (OLI_TRANS_MODE_SUBRESPONSE). To trigger the resending of the emails, the 121 must contain the same TransRefGuid and PolNumber so FireLight can find and process the correct case. In the event a child case reaches a terminal status that causes the parent case to change to any status other than "Sent To Client" (e.g., one party declines e-sign, thus the application is now a paper app), no initial emails is resent for any child cases. If a child case has been completed, it’s initial email will not be resent.

FireLight supports two other E-Delivery workflows. Click a link to go to the feature topic.

  • E-Delivery Sign Only

FireLight Setup

Contact your Strategic Account Manager to create an addendum to the contract to support the E-Delivery workflow type, set up the integration, and accept the transaction fees. Also contact the Strategic Account Manager for assistance with creating and designing wizards and rule sets for e-Delivery.

NOTE: E-Delivery is available only with the Wizard user interface.

Admin Setup

Following is a synopsis of the E-Delivery setup process. Refer to the FireLight e-Delivery GuideFireLight E-Delivery Guide for detailed implementation instructions. NOTE: The parties that carriers use for deliver only cases must be updated by Hexure in the signer list profile. Deliver only does not have signature controls and must be set up through list profile.

  1. Designer/Wizard Library page: Create one or more wizards to be included in the E-Delivery package. Although the e-delivery workflow uses only the wizard interface, it can use forms that are converted to wizard view. Deliver Only is also presented in a locked view and does not support fill. It can be pre-filled by the 121 and rules is run before the case is locked.
  2. Designer/Form Library page: The PDF forms to be sent in the TX121 and a TransSubType typecode of 12123 and presented in FireLight (example the Policy) are sent in the TX121-12123 with a document tag identifying the form. Upload one or more template forms with a DocumentTag that correctly associates the PDF in the TX121-12123 with the form requirements in FireLight. See the FireLight e-Delivery Guide for details.
  3. Designer/Wizard Designer or Form Designer page: Design the wizards or forms to be included in the E-Delivery package and add the rules in the E-Delivery rule set. It doesn’t need to be wizards as it is pre-filled only from 121 and not user entered. Firms can have more than one wizard or form. There is different wizards for the different E-Delivery product types (Deliver Only, Sign Only, Fill and Sign).
  4. Product Configuration/Rule Sets page: Add the rules for E-Delivery Deliver Only. Rules is run on each case before it is locked. The Agent/client receives the case in a locked state.

  5. Product Configuration/Activities page: In the grid at the top of the page, click Add New Activity, enter the name of the E-Delivery Deliver Only product in Activity Name. If it is the only e-delivery product enabled, it will also be used in the e-delivery dashboard button in the Manage section of the Home page. Select E-Delivery Deliver Only (121-12123) in ACORD Type. Enter settings applicable for your organization. Click Save Activity.

    Note the following purge dates:

    • Complete Purge Policy The number of days after an E-Delivery Delivery Only case was finished (this includes Expired, Canceled, Re-Issued, and Completed cases). The default is 60 days and can be changed only by Hexure. Once the 60-day time period has been exceeded the Purge process will purge the activity.
    • Incomplete Purge Policy The number of days since an incomplete E-Delivery Delivery Only case was last updated. An Incomplete activity has been updated. The default is 60 days and can be changed only by Hexure. Once the 60-day time period has been exceeded the Purge process will purge the activity.
  6. Product Configuration/Products page: Add the E-Delivery Deliver Only product as a Non-Insurance Product in Product Type, Product Category, and if applicable, Product Type Filter (UI). Configure product settings. Add a unique Product Code that can be used to delineate the different packages to use for E-Delivery Delivery Only. Save the product config.
  7. Product Configuration/Packages page: Add the E-Delivery Deliver Only package. In ACORD Type, select E-Delivery Deliver Only (121-12123). Add the E-Delivery Deliver Only wizards and rule sets to the package, and save the package.
  8. Product Configuration/Sales Agreements page: Add the sales agreement for the E-Delivery Deliver Only product, assign the Deliver Only package to the sales agreement, and save the sales agreement.

  9. Organization Configuration/Organization Settings/Activity tab: Be sure to set the following options and click Save Organization:

    • Select the E-Delivery Reminder Frequency (Never, Daily, Weekly) of an agent reminder email for E-Delivery based on the case expiration date. E-Delivery reminder emails run on the scheduler with the reminder set at intervals from the time of the application reaching the E-Delivery status. "Daily" frequency emails go out at 6:00 AM UTC. "Weekly" frequency emails go out on Sunday at 7 AM UTC.
    • In E-Delivery Expiration Warning Threshold, enter the number of days prior to the expiration of a pending E-Delivery an email should be sent to the agent and/or client. This email will warn of the impending expiration of the E-Delivery. If 0 is entered, no email is sent.

  10. Organization Configuration/Groups page:

    1. Select a TC Role Code in the grid.
    2. Activity Access tab: Select the E-Delivery Deliver Only checkbox to enable E-Delivery Deliver Only for the user group. (Optional) Near the end of the page, select Disable Approve Button to prevent the agent/advisor from approving the E-Delivery. For example, if the carrier allows visibility to multiple agents, they can use this option to disable the Approve button. They will still have the ability to reject the case.
    3. Product Access tab: Select the checkbox for the E-Delivery Deliver Only product name.
    4. Save the Groups page.

  11. Utilities/Deployments page: When testing is completed, deploy the E-Delivery Deliver Only product to the applicable environments. See Deploy Data.

    1. Synchronization Scope: Select All Configuration.
    2. Reason: Type the reason for the deployment.
    3. Deployment tab: In the Deploy column, select the checkbox for the E-Delivery Deliver Only product.
    4. Set whether to Deploy Now or Schedule Deployment and whether or not to use Force Deploy.

User Experience

The E-Delivery case from the carrier goes into FireLight from the ACORD DocSend (TX121-12123) message through the firm's Provider.

Agent Directed Cases:

  1. The E-Delivery Delivery Only workflow is locked and ready for download. The status is Pending Agent Review.

  2. FireLight sends a request for agent review email to the agent.

  3. Agent opens the E-Delivery Deliver Only case.
    • On the left of the FireLight Home page, agent clicks an E-Delivery Deliver Only case in the Recent Activity grid.
    • On the right of the Home page under Manage, agent clicks the E-Delivery button to go to all the E-Delivery cases in All Activities. Agent clicks the View button to open an E-Delivery Deliver Only case in the list. NOTE: In the Manage section on the Home page, the counter on the E-Delivery button shows the total number of E-Delivery cases that are in the Pending Agent Review status. Pending Agent Review statuses will increase the Agent's dashboard counter.

  4. Agent reviews the information. If it is accurate and complete, agent clicks the Approve button. If the information is not accurate, the agent cannot change, but must use cancel e-delivery.

  5. FireLight sends an email to the client to request the client to download the E-Delivery documents. The case is locked and ready for download. The information comes from the 121 party info. The name and email address fields are disabled and pre-filled from the 121. The party fields must be mapped in the Admin Mapping tool to correctly map name and email address. The status changes to Pending E-Delivery. An email request is sent to each client in the parent case and child cases. FireLight sends the Pending E-Delivery status to the carrier.

  6. The client clicks the link in the email, enters their login information and clicks Enter. Once the client authenticates, the case is considered complete. The completion process is initiated and an 1122 is sent.

  7. FireLight opens with two buttons.

    Contact Agent – Client clicks the Contact Agent button to compose and send a message to the agent.

    Review Documents – Client clicks the Review Documents button to access the individual documents for review. On this page, the Client can also do the following:

    • Click the Document Download button to open another view and click a Download All link to download all of the E-Delivery Deliver Only documents. Display status changes to Download Complete and a 1122 is sent to the carrier. The audit report is also regenerated to include the download audit. Depending on the browser view, Client can click the Open, Save and Cancel buttons for the downloaded E-Delivery Deliver Only documents PDF file.

    • Click Request Print to request the carrier to print the documents and mail them to the client. Upon selecting Request Print, the Display status changes to Request Print for the child case and the parent case. FireLight does not delete the cases because these cases were completed and the Request Print can be available post submit. FireLight retains the audit history.

    • Click Contact Agent to compose and send a message to the agent.

    • Click Log Off to log off FireLight.

  8. If all child cases are locked and transitioned into E-Delivery Complete status, FireLight submits the E-Delivery Deliver Only case to the back-office systems for final processing. The status changes to Pending E-Delivery - Submitted and then to Complete on the main agent case.

  9. When back office processing is complete, FireLight sends a Complete status to the carrier.
