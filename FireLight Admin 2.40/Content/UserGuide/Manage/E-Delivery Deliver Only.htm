E-Delivery Deliver Only [User Instructions]

E-Delivery Deliver Only workflow is available if your organization has implemented the E-Delivery Deliver Only feature.

E-Delivery Deliver Only workflow includes delivery of activity/application documents to the clients in the E-Delivery Deliver Only parent case and associated child cases.

FireLight® supports two E-Delivery Deliver Only processing workflows that drive what FireLight does with the case when it is received. Cases can either be agent directed or client directed.

  • Agent directed cases go into a dashboard view for the agent to manage. The agent must approve the case before it is sent to the clients.
  • Client directed cases go directly to the clients and do not require an agent to approve the case.
Agent Directed E-Delivery Deliver Only Case

Agent Directed E-Delivery Deliver Only Case

  1. The agent receives an email that notifies the agent they have a pending e-Delivery Deliver Only transaction for review. The e-Delivery case is in E-Delivery Pending Agent Review status.

    The agent will also receive an email that notifies the agent when all child cases are complete. The E-Delivery cases are always auto submitted (ignores the Auto Submit group option in Admin).

  2. The agent logs in to FireLight. The agent opens the case by doing one of the following:

    • On the FireLight Home page, the agent clicks the E-Delivery Deliver Only case in the Recent Activity grid on the left.

      NOTE: On the Home page, the counter on the right of the E-Delivery button in the Manage section increases any time an agent directed case is in a “Pending Agent Review” status. Count is based on pending agent review, so any action that causes it to set to pending agent review (i.e., new agent directed case, client change request) will add it to the count and it will not decrease until the case moves to another display status.

    • On the lower right under Manage, the agent clicks the E-Delivery button to open All Activities filtered by E-Delivery cases and then clicks the View button for the E-Delivery Deliver Only case.

  3. The case is locked down. The agent cannot unlock the case. On the E-Delivery navigation bar, Agent can click Cancel E-Delivery to cancel the E-Delivery Deliver Only, or the agent can click the Approve button to continue. The cancel is only available while the case is not complete.

    NOTE: The carrier has an option in FireLight Admin to show or hide the Request Print button. When the button is shown and the agent clicks Request Print, a request for the carrier to send a paper copy is sent to the carrier. Once Request Print is selected, the order is effectively ended. The agent view has no active buttons. The clien view has not active buttons. FireLight revokes the original email links sent to the client and FireLight sends to the clients the email called Revoke E-Delivery Request. This email is also sent if the client does a request change or a decline sign.

  4. Agent ensures the client information is correct and clicks the Send Email button to send the Request Clients Complete their E-Delivery Requirements emails to the clients.

  5. The Pending Requests dialog shows the E-Delivery Deliver Only pending requests for the parent and child cases. The agent can click links to send reminders or passcodes to the clients. Agent clicks Close.

  6. Agent can click the link icon to view the Linked Activities and status.

  7. Clients receive the E-Delivery Deliver Only email. Each client clicks the URL link within the email and enters their login credentials to open FireLight. Following are sample client request emails, passcode email (if passcode is enabled), and a login page.

    NOTE: The case is complete when the client login authenticates. For E-Delivery Deliver Only cases, the requirement is only that the policy was delivered, not that the clients reviewed the documents.

  8. FireLight opens to the Electronic Documents page. Client clicks one of the following:

    • Review Documents button to review and download documents as shown in step 8.

    • Contact Agent button to open a dialog and send a message to the agent.

  9. FireLight displays the E-Delivery Deliver Only documents list. Client can click a document button to review the document. Client cannot make any changes because as soon as the client logs in, the E-Delivery Deliver Only case is complete.

  10. Client can click the Document Download button to open the Download PDF Documents page.

    Client can download the activity documents or request a printed copy from the carrier by doing one of the following.

    • Click the Download All link. On the notification bar, client clicks an option to Open, Save menu, or Cancel the downloaded PDF documents.

    • NOTE: The carrier has an option in FireLight Admin to show or hide the Request Print button.

      When the button is shown, click the Request Print button to request a printed copy of the documents from the carrier. The following message appears.

  11. Client clicks the Log Off button on the upper right of the page to exit FireLight.

  12. After the client completes the case, the agent's view of the E-Delivery Deliver Only case in FireLight displays the following message for the case. If the Request Print is available, the agent can click Request Print to request printed copies of the documents from the carrier.

  13. (Optional) In All Activities, click the link icon to view the status of the child cases.

    Click the View History link to view the audit History for the E-Delivery Deliver Only case.

    NOTE: Actions taken on the child case will show in the child case audit. Not all audits are shown in parent case history.

Client Directed E-Delivery Deliver Only Case

Client Directed E-Delivery Deliver Only Case

  1. Clients in the child cases receive the E-Delivery Deliver Only email that indicates their documents are ready to be downloaded. The client receives a separate email with the login Passcode if passcode is enabled. The E-Delivery Deliver Only case is in Pending E-Delivery status.

    If the agent opens the case in the agent's view, the Pending Requests dialog shows the E-Delivery Deliver Only pending requests for the cases. The Delivery Status is In Progress. The agent can click links to send reminders or passcodes. The agent clicks Close.

  2. The client clicks the URL link in the email and logs in to FireLight using either the Last 4 Digits of SSN/Government ID and Birth Date or the Passcode if passcode is enabled.

    NOTE: The case is complete when the client login authenticates. For E-Delivery Deliver Only cases, the requirement is only that the policy was delivered, not that the clients reviewed the documents.

  3. FireLight opens to the Electronic Documents page. Client clicks one of the following:

    • Review Documents button to review and download documents as shown in step 8.

    • Contact Agent button to open a dialog and send a message to the agent.

  4. FireLight displays the E-Delivery Deliver Only documents list. Client can click a document button to review the document. Client cannot make any changes because the E-Delivery Deliver Only case is locked and complete.

  5. Client can click the Document Download button to open the Download PDF Documents page.

    Client can download the activity documents and/or request a printed copy from the carrier by doing one of the following.

    • Click the Download All link. On the notification bar, client clicks an option to Open, Save menu, or Cancel the downloaded PDF documents.

    • (Optional) The carrier has an option in FireLight Admin to show or hide the Request Print button. When the button is shown, the client can click the Request Print button to request a printed copy of the documents from the carrier. The following message appears. The case is in Request Print status.

  6. If the client does not Request Print, client clicks the Log Off button on the upper right of the page to exit FireLight.

  7. (Optional for Agent) In All Activities, agent clicks the link icon to view the status of the parent and child cases.

    If the E-Delivery case is still pending for one of the clients, the status for that client is Pending E-Delivery.

    Click the View History link to view the audit History for the E-Delivery Deliver Only case.

    After the clients complete the case, the agent's view of the E-Delivery Deliver Only case in FireLight displays the following message for the case. If the Request Print is shown, the agent can click Request Print to request printed copies of the documents from the carrier.
