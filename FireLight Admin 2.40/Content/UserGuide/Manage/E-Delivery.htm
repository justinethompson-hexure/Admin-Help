E-Delivery (Fill and Sign) [User Instructions]

E-Delivery (fill and sign) workflow is available if your organization has implemented the E-Delivery fill and sign feature.

E-Delivery fill and sign is the workflow that includes taking the agent and client through reviewing, approving, signing, and submitting a policy electronically.

FireLight® supports two E-Delivery fill and sign processes that drive what FireLight does with the case when it is received. Cases can either be agent directed or client directed.

  • Agent directed cases go into a dashboard view for the agent to manage. The agent must approve the case before it is sent to the clients.
  • Client directed cases go directly to the clients and do not require an agent to approve the case.
Agent Directed E-Delivery (Fill and Sign) Case

Agent Directed E-Delivery (Fill and Sign) Case

  1. The agent receives an email notification to log in and review the E-Delivery case. The case is in a Pending Agent Review status.

  2. On the FireLight Home page, the agent selects the e-Delivery case in the Recent Activity grid on the left or on the lower right under Manage, clicks the E-Delivery product name button to open All Activities filtered by e-Delivery cases. The counter on the right of the E-Delivery button on the Home page increases any time an agent directed case is sent in and is in a “Pending Agent Review” status. The counter only shows cases that have not been opened yet. Count is based on pending agent review, so any action that causes it to set to pending agent review (i.e., new agent directed case, client change request) will add it to the count and it will not decrease until the case moves to another display status.

  3. The agent reviews the E-Delivery case and fills in required agent information, for example, Agent ID. The agent uses the E-Delivery case navigation bar to Approve and Sign an E-Delivery case, Request Reissue, view Pending Requests and Alerts, and to Cancel E-Delivery. The available actions on the navigation bar depend on the current e-Delivery workflow status.

  4. On the navigation bar, the agent clicks Approve.

    The Request Clients to Complete their E-Delivery Requirements and Sign dialog opens. The agent clicks Send Email to send the request email to the clients.

    The Pending Requests dialog opens. The agent clicks Close.

  5. The clients receive an email notification to access the policy, review the content, complete any remaining requirements, and sign. The client also receives a Passcode email if the Passcode feature is enabled.

  6. The client enters the Last 4 Digits of SSN/Government ID and their Birth Date or a Passcode sent to them via a separate email, and clicks Enter to log in to FireLight.

  7. The client reviews each page of the e-Delivery package and enters client information where applicable.

  8. The client clicks the Approve & Sign button and clicks Lock Activity to lock the E-Delivery case for signing.

  9. Client clicks Sign Activity and completes the standard FireLight e-signing process in step 10. Or the client can click the Contact Agent button to send an email to the agent or click the Review Documents button to open and review the documents in the package.

  10. On the Signature page, client clicks each document to review and then selects the checkbox for I have reviewed and agree with the terms expressed within this document.

  11. On the Capture Electronic Signature page, client enters their Signer Full Name and City and signs by clicking the I Consent button, signing on the signature pad with a mouse or stylus, or signing with the Topaz signature tablet.

  12. On the Congratulations page, client clicks OK or clicks the Log Off button on the upper right to log off and exit FireLight.

    Before logging off, the client can click the Contact Agent button to open a message, enter message text for the agent and click Send to send the message to the agent.

  13. When each client approves and signs an e-Delivery request, the agent receives an email notification that the client has signed and approved the E-Delivery case.

  14. If the agent’s signature is required for the e-Delivery case: when clients have reviewed, approved and signed the e-Delivery client requests, the agent receives an email similar to the following.

  15. Agent logs on to FireLight and opens the e-Delivery case.

  16. On the right of the e-Delivery navigation bar, the agent clicks the Sign Now button.

  17. In the List of Required Agent Signatures, the agent clicks a button to begin the agent signature process.

  18. The agent completes the standard FireLight signature capture process for the agent.

    When all signatures are complete, the agent receives an email similar to the following.

  19. The clients receive an email notification to download or print the completed E-Delivery.

  20. Client clicks the URL and enters the Last 4 Digits of SSN/Government ID and Birth Date on the Welcome page and clicks Enter.

    Client clicks the Download All link to download all of the case documents in a single PDF file.

    On the notification bar, client clicks an option to Open, Save menu, or Cancel the downloaded PDF documents. This view is browser dependent.

    Click clicks the Log Off button to log off and exit FireLight.

  21. (Optional for Agent) In All Activities, agent clicks the link icon to view the status of the parent and child cases.

    Agent clicks the View History link to view the audit History for the E-Delivery Fill and Sign case.

    After the clients complete the case, the agent's view of the E-Delivery Sign Only case in FireLight displays the following message for the case.

    The agent can print and save the E-Delivery case documents by clicking Display/Print PDF on the Other Actions menu, selecting documents and clicking the Print Selected Documents button.

Client Directed E-Delivery (Fill and Sign) Case

  1. The clients receive an email notification to access the policy, review the content, complete any remaining requirements, and sign. The client also receives a Passcode email if the Passcode feature is enabled.

  2. The client enters the Last 4 Digits of SSN/Government ID and their Birth Date or a Passcode sent to them via a separate email, and clicks Enter to log in to FireLight.

  4. The client clicks the Approve & Sign button and clicks Lock Activity to lock the E-Delivery case for signing.

  5. Client clicks Sign Activity and completes the standard FireLight e-signing process in step 6. Or the client can click the Contact Agent button (see "Contact Agent") to send an email to the agent or click the Review Documents button to open and review again the documents in the package.

  6. On the Signature page, client clicks each document to review and then selects the checkbox for I have reviewed and agree with the terms expressed within this document.

  7. On the Capture Electronic Signature page, client enters their Signer Full Name and City and signs by clicking the I Consent button, signing on the signature pad with a mouse or stylus, or signing with the Topaz signature tablet.

  9. When each client approves and signs an e-Delivery request, the agent receives an email notification that the client has signed and approved the E-Delivery case.

  10. The client receives an email notification to download or print the completed E-Delivery.

  11. Client clicks the URL and enters the Last 4 Digits of SSN/Government ID and Birth Date on the Welcome page and clicks Enter.

    Client clicks the Download All link to download all of the case documents in a single PDF file.

  12. Client clicks Log Off on the upper right to log off FireLight.

Alerts

The following example shows an Alert for missing data.

  1. On the toolbar, click the alert icon next to Alerts to display the validation message. Click X on the lower left to close the message.

Attach Documents

Attach Documents

The client can upload and attach documents to the E-Delivery case.

  1. After logging in to the E-Delivery case in FireLight, the client clicks Attach Documents on the upper right.

  2. In the Documents dialog, the client selects a Document Type in the list and clicks the Choose File button.

  3. The client selects a document in the Open dialog and clicks Open.

  4. Client clicks the Upload button.

    The document appears in the Uploaded Documents list. Client can click the View link to open and view the document.

    Client can choose another file and upload or click Close to exit the dialog.

Cancel E-Delivery

Cancel E-Delivery

Use this option when the client wants to cancel the electronic delivery. This is a terminal status and prevents further action by the agent or the clients on the case.

  1. The agent logs in to FireLight and opens the E-Delivery case the client wants to cancel.

  2. On the E-Delivery toolbar, the agent clicks Cancel E-Delivery to cancel the E-Delivery case.

  3. To proceed with the cancellation, the agent clicks Cancel E-Delivery on the Cancel E-Delivery Case dialog.

  4. The following message appears on the open E-Delivery case.

  5. FireLight sends a status back to the carrier letting them know that the client has elected to cancel the E-Delivery case.

Contact Agent

Contact Agent

  1. Before logging off, the client can click the Contact Agent button on the upper right to send a message to the agent.

  2. In the dialog, the client enters message text and clicks Send to send the message to the agent.

 Request Change

Request Change

The client can request a change to the E-Delivery.

  1. Click Request Change on the E-Delivery toolbar.

  2. In the Request Change dialog, enter the reasons for the changes, and select the checkbox for I have completed my review. No further edits is allowed.

  3. Click a button. Click Save & Close to close the dialog without submitting the requested change. Click Submit to submit the change and log out of FireLight.

  4. When the agent views the audit History for the case in All Activities, the status is Change Requested by Client and "A Next Step Activity was deleted."

Request Reissue

Request Reissue

If applicable, the agent can request a reissue when changes need to be made to the policy/contract.

  1. Agent opens the E-Delivery case.

  2. On the E-Delivery toolbar, the agent clicks Request Reissue.
  3. In the Request Reissue dialog, select an option in the Reason list: Issued Incorrectly, Client Requested Change, or Other.

  4. Enter one or more detailed reasons in the Reason Description text box. Ensure you have thoroughly reviewed and listed all reasons for the request for reissue.

  5. Near the bottom of the dialog, read the information next to the checkbox and select the checkbox.

  6. Click the Save & Close button to save and close the E-Delivery case or click the Submit button to submit the request for reissue.

  7. A Log Off message appears. Agent closes the message or clicks the Log Off link to log off FireLight. The case status changes to Change Requested by Agent.

 E-Delivery Statuses

E-Delivery Statuses

The following E-Delivery statuses are available in FireLight.

Canceled – For other than Agent directed E-Delivery Fill and Sign cases, when an E-Delivery email is undeliverable, the status is "Canceled" (so a reversal can be issued) with a sub status of "Other" for the parent and child cases. The status for the child case with the undeliverable email is "Canceled" with a sub status of "Undeliverable Email."

Change Requested by Agent – If the agent clicks “Request Reissue” for any reason, the case is set to the terminal status of “Change Requested by Agent”. A newly generated, corrected case will need to be sent in by the carrier to proceed.

Change Requested by Client – In an agent directed workflow, the child case(s) are set to this status if a client clicks “Request Change.”

In a client directed workflow, the parent and child cases are set to this status if the client clicks “Request Change”. This is considered a terminal status in a client directed workflow. A newly generated, corrected case will need to be sent in by the carrier to proceed.

Complete – When all child cases are complete, the parent case is updated to this status. The case is delivered to the carrier for processing.

Expired – The requested date for the parent case in requirement info (<RequestedDate>2018-10-05</RequestedDate>) sent in the 121 will determine the expiration date. Once this date has passed, if the case is not complete it is set to a terminal status of expired. A newly generated case would need to be sent in by the carrier to proceed.

In Agent Signatures – When the agent begins the signing process, the case display status transitions to “In Agent Signatures,”

In Signatures - This status triggers on the child cases when any party has started the signature process.

Pending Agent Review – In an agent directed workflow, the parent case is set to this status when it is sent into FireLight. It is also set on the parent case if a client clicks “Request Reissue.” This status will not appear in a client directed workflow and will repeat anytime a policy is sent back to the agent for review.

Pending Agent Signatures – If agent signatures are required on either the agent portion or the client portion of the e-Delivery case, after the client signatures are captured, the display status changes to “Pending Agent Signatures.”

Pending Client Review – Once child cases are sent, the parent and child cases are set to Pending Client Review. This status will remain for the parent case until all child cases are complete or a reissue is requested.

Policy Canceled – If the agent clicks “Cancel E-Delivery” for any reason, the case is set to the terminal status of “Policy Canceled”. A newly generated, corrected case will need to be sent in by the carrier to proceed if applicable.

Signatures Complete – Indicates that the child case has been signed and no further actions are required for that child case

Submit Requested – Once the parent case is in a complete status, the child cases will show this status.

Undeliverable Email – The sub status of the child case with the undeliverable email. In the "Request Reissue" dialog (shown when an agent clicks the "Request Reissue" button in an Agent Directed E-Delivery Fill and Sign case, an "Update Client Email Address" option is added to the Reason list.
