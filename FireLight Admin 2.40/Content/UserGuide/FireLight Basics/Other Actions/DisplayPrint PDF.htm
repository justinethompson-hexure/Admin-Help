Display/Print PDF [User Instructions]

Use the Display/Print PDF task on the Other Actions menu to view, save, and print selected documents from the activity.

NOTE: For an Illustration activity, click Display/Print PDF on the Other Actions menu to open the illustration in a separate tab or window where you can view, save or print the illustration pages.

The following example is for an Application activity.

  1. On the navigation bar, click Other Actions > Display/Print PDF.

    In the following example, an addendum document contains overflow text for the Application document.

  2. On the left, select the checkbox for each document to print.

    NOTES:

    • The system displays addendum documents for each of the applicable forms in the same sequence of the forms. (This does not impact the Print Forms activity.)

    • A Signing Ceremony Only document type is included in the signing ceremony but does not contain signature controls (for example, disclosures or privacy notices). This document type is included here as an optional print document, but it is not visible in the edit activity process and it is not presented to the back office for processing with the activity package.

  3. On the right, click the blue up arrows or blue down arrows to rearrange the print order. When an activity package is in locked status, the blue arrows are disabled and the documents will print in the default order set in FireLight® Admin.

  4. Click Print Selected Documents to open the selected forms in a separate PDF viewer window.

    When an addendum document is present for a form, the associated form field in the form displays the reference ID and the Addendum document name (Addendum - <Form Name>) in the field where the overflow text exceeded the number of characters allowed for the field. Both overflow fields and their overflow content Addendum references is numerically applied from left to right, top to bottom and not defined by tab order of the form.

    In the following example, the Employer Name field in the VA Two Application form displays "See #1, Addendum - VA Two Application" to provide the user with the reference ID (#1) and the addendum document name (Addendum - VA Two Application) where the overflow text will appear.

    The Addendum - VA Two Application document displays the Reference ID and the Overflow Content from the Employer Name field in the VA Two Application.

    All the signature fields placed on a form will also be shown after the addendum text ends on the last page of the associated addendum document. To show signatures on Form addendum documents, the addendum document will have the signature field along with the Signed Date added of the same signer type(s) as that of its original form. These signature fields are stacked from top to bottom and are placed after the addendum text ends on the last page of the document. The signature fields are labeled to identify the signer type based on the form signature control. The signed date fields are adjacent to the signature field. If the Addenedum does no have enough space remaining on the page to show signature(s) on the page, the signatures will overflow onto another addendum page. The addendum document is also presented for signature along with its associated form.

    In case multiple copies are generated for the form using the Form Index property, all the addendum documents of each form copy will show the same signature fields as the corresponding form.

    In the Review Queue, Application History and Audit Report links show audit history of signatures on the addendum document like other forms at the time of the signing ceremony.

  5. Use the scroll bar on the right to view all of the forms.

  6. Use the PDF controls for sizing, viewing, saving and printing forms.

  7. Close the PDF viewer window.

  8. In the Select Documents to Print dialog, click Back to Application to return to the application package.
