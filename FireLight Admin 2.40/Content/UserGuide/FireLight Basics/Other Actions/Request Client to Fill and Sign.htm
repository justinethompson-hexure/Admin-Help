Request Client to Fill & Sign [User Instructions]

This is an optional feature that is available only when your organization enables it in FireLight® Admin.

User the Request Client to Fill & Sign feature to request the client to supply missing information for required form fields and to sign the completed application in one session. This feature eliminates the need to send separate client requests for filling in missing field information and for signing the completed applicaiton.

This is an optional feature that is available only when your organization enables it in FireLight Admin.

CONDITIONS

  • Request Client to Fill & Sign feature is not available when Pre-Signature Review is enabled.

  • Activities containing forms that require to be signed by a Trustee will display the text "TTEE" as a suffix on the Trustee signature field at the following places:

    • Form/application

    • Addendum document

    • Application PDF

    • Application Audit Report

    • Applications created via Embedded APIs/IXN

    Text "TTEE" will show suffixed on the Trustee signature field on the Application, Application PDF and Application Audit for the following cases:

    • For all the trustee signers such as: Trustee, Trustee 2, Trustee 3, Trustee 4, etc.

    • When multi-selection is done at the time of signing for "Trustee" signer along with "Owner" signer type.

BEFORE USING THIS OPTION

  • Owner basic information is required before sending the request.

  • Agent required information must be completed by the agent before sending the request because once the client signs the application, it is locked from editing.

    On a form, FireLight designates agent-required fields by a green background. On the DATA ENTRY tab, click the percent sign (%) to view a list of errors/required fields. In the list, agent required fields display “(OFFICE USE ONLY)” after the form page name as shown below.

  • The Request Client to Fill & Sign option becomes available on the Other Actions menu after required agent fields are completed. Only the primary signer receives an email. When the primary signer completes the information and signes the application, FireLight sends client sign requests for the remaining signers. Secondary signers cannot add information to the forms.

Request Client to Fill & Sign

  1. In an open activity on the navigation bar, click Other Actions > Request Client to Fill & Sign. The system populates the matching fields with data that was added to the application.

  2. Fill in the information (SSN/Government ID, Birth Date, etc.) for a signer.

    (Optional) If your organization uses SMS Text Authentication for signatures, an additional Mobile Number field appears below the Birth Date field. Fill in the signer's Mobile Number.

    NOTE: If the organization uses Passcode Only Validation, enter the information. The Last 4 Digits of SSN/Government ID and Birth Date are not required. A message will also display below the client section, *Client Authentication via Passcode*.

    NOTE: If the organization uses Client Authentication via Passcode and SMS Text Authentication, enter the information including the client's Mobile Number. The Last 4 Digits of SSN/Government ID and Birth Date are not required.

  3. On the right, add any additional information to the message for the client fill and sign request.

  4. Click Send Email to send the request to the recipient. (Click Cancel to cancel the request and return to the application.)

    Click OK on the Request Sent dialog.

  5. On the Pending Request dialog, click a Send Reminder link or click a Send Passcode link to send the Passcode to the client.

    FORMS

    • To send a Passcode to the client’s cell phone, click the Send Passcode to [Client’s] Cell Phone link, enter the Client’s mobile number and click the Send SMS button.

    • To send a Passcode to the client’s email, click the Send Passcode to [Client’s] Email link or Send button (email) as applicable.

    WIZARDS

  6. The client receives an email similar to the following and does the following:

    1. Clicks the FireLight URL link.

    2. Client types their Passcode or types the Last 4 Digits of SSN/Government ID and their Birth Date and clicks the Enter button.

      NOTE: If SMS Text Authentication is enabled, the client must enter the Last 4 Digits of SSN/Government ID and their Birth Date and then click the Send Passcode button. FireLight sends a Passcode to the client’s mobile phone.

      NOTE: If the organization uses Client Authentication via Passcode the following dialog appears. Enter the Passcode provided by the Passcode Email or the agent and click Enter.

      NOTE: If the organization uses Client Authentication via Passcode and SMS Text Authentication, SMS Text is the method of authentication. Click Send Passcode.

    3. Client enters the Passcode and clicks Verify.

    4. In FireLight, the client fills in the required information on the form or wizard.

    5. (Optional) The client can attach documents or required template forms to the application. On the navigation bar, client clicks Attach Documents, selects the Document Type, browses to and selects the document, and clicks Upload to attach the uploaded document to the application.

      NOTE: Client cannot attach documents after the Data Entry is 100% and the application is locked. See step f for details.

    6. Client clicks CONTINUE on the “Data Entry has met the requirements” dialog.

    7. Client does one of the following, depending on whether the application is in form or wizard format. This will lock the application and allow no additional data or changes.

      FORMS – Click OK.

      WIZARDS – Click Lock Activity.

      NOTE: If the client finds changes after the activity locks, they can click Contact Agent on the Electronic Signatures page to send a request for the agent to unlock the application and close the web browser to exit.

    8. (Optional) Client clicks the Review Document button to review the documents for the activity in a separate browser window or tab.

    9. Client clicks the Sign Activity button.

    10. Client completes the electronic signature capture process and clicks OK on the dialog.

    11. Client receives a validation message that they have successfully logged out of the application.

    12. Agent receives a message that the client has signed and approved the application.

    13. FireLight sends a signature request via email to any other signers as applicable.

Undeliverable Emails

When SendGrid is enabled, FireLight handles undeliverable Request Client to Fill App and Request Client to Fill & Sign emails as follows.

  1. The agent creates a new activity. On the Other Actions menu, the agent initiates a Request Client to Fill App or a Request Client to Fill & Sign, enters the information on the request dialog, and clicks Send Email Request with an email address that is undeliverable.

  2. SendGrid returns the invalid email. A "Failure delivering email" message is posted to the Message Center.

    The agent can click a link to Resend this message, Send new email to [Agent Name], or Send Passcode to [Agent Name]'s Cell Phone. After clicking a link, the agent can enter a correct email address and resend the message, etc.

  3. If the agent has the "Send Message Center Emails" checkbox selected in Preferences [User Instructions], the agent receives an email with the message information.
