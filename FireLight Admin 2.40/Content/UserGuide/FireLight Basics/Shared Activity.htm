Shared Activity Tab [User Instructions]

NOTES:

  • Only individually shared activities appear on the Shared Activity tab and not when all activities are shared with a user. This list includes any activity for which your user ID accepted the single activity share request by clicking the link in the Accept Single Activity Share Request email. The Shared Activity tab is always available—it is not dependent on Admin Group settings for activity share.

  • If a user group in the Admin tool has the Group setting enabled for Case Access by Firm ID for their role code, those users can access shared activities on the Shared Activity tab based on the firm ID/rep code passed over during the SSO login process.

Impacts:

  • Shared Activity tab displays shared cases when "Case Access by Firm ID" Group setting is selected

Use the Shared Activity tab on the Home page to easily identify and access individually shared activities.

  1. On the left of the Home page in the Shared Activity tab, click an activity name to open an activity that you recently accessed.

    NOTE: If no shared activities exist yet, the Shared Activity tab displays a "No Shared Activities" message instead of a list of shared activities.

  2. (Optional) In the View list, select a user ID or All to filter the list.

  3. If the Admin tool Groups option for your user group is set to "Disable View Activities," the Shared Activity tab displays only the status and date of shared activities in a read-only view. Clicking an activity in the list will not open the activity. Additionally, on the All Activities page, the View button will not be visible for each activity in the list. The user cannot view the activity from the All Activities page.

Shared Activity Guidelines:

  • Only individually shared activities (single activity share) appear in the Shared Activity tab.

  • Shared requests that are not accepted do not appear in the Shared Activity tab.

  • Shared requests that are cancelled/removed by the originator no longer appear in the Shared Activity tab.

  • Expired share requests do not appear in the Shared Activity tab.

  • The user who originated the share request continues to see the activity in their Recent Activity tab.
