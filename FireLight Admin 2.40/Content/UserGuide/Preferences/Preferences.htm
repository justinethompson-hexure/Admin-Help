Preferences [User Instructions]

Use My Preferences to set your preferred default settings for the FireLight® system.

On the navigation bar, click Preferences. My Preferences page contains the following sections for setting your default options.

General

General

  1. Enter your User Full Name and User Email Address.

  2. Select your personal FireLight default options in the drop-down lists for Default Jurisdiction, Default Product Type, Default Time Zone, Locale, and Page Size for My App.List (maximum is 100). NOTE: In Default Product Type, select the All option to view a list of all available products when creating a new activity.

    For example, click the Locale list and select a different language/locale from the list. See Localization [User Instructions] for how FireLight adapts to the selected language/locale. Click the reset button to the right of the Locale list to reset FireLight to the default locale after you open an activity in a locale that is different from the locale selected here and then close the activity.

  3. Select the Send Message Center Emails checkbox to have FireLight send emails to your email address for any messages posted in the FireLight message center. This includes email failure notifications from the SendGrid email delivery system.

  4. (Optional) If your organization set Enable Use Agent Email for your role code and you (the agent/activity owner) want all of the assistant's actions to use your agent email for outgoing communications, select the Use Agent Email for All Communications checkbox. Clear the checkbox to use the logged-in user (assistant, agent, etc.) email address for outgoing communications.

  5. (Optional) Rule Tracing is an optional feature that your organization can implement for specific users to provide rule tracing for a rule debugging tool in FireLight Admin. To trace all rule runs for your current session, select the Enable Rule Tracing For Session checkbox. You need to select the checkbox each time you log in and want to create rule tracing. When you create or edit an activity, a red pause button appears at the top of the screen. Click the button to pause the rule trace. To resume the rule trace, click the green continue button.

    Copy the activity ID:

    1. Click the Copy button to the right of the red pause button or green continue button.
    2. (Optional) If an Internet Explorer security dialog appears, click Allow access on the security dialog.
    3. Click OK on the Alert dialog.
    4. Provide the ID to the Admin user who will view the rule trace data.

    This option is available only when the Enable Creation of Rule Traces option is enabled in FireLight Admin for your user role. Admin users who have the Enable Viewing of Rule Traces enabled for their role can then view the rule trace information that you record for rule debugging and troubleshooting in FireLight Admin.

  6. Click the Save button to save your settings.

 Work Groups

Work Groups

NOTE: The Work Groups section is available when the "Allow Work Groups" option is enabled in FireLight Admin.

Use Work Groups to create inter- and intra-organization workflows, allowing users to have shared visibility into an activity and receive notifications for the activity life cycle and allow members from cross organizations to have the same visibility.

Guidelines

Workflow participants can:

  • Create activities to which all workflow participants have access/visibility.

  • Access status information on activities.

  • Receive email and FireLight messaging related to activities.

  • Have group-/role-based permissions regarding functionality allowed on activities.

  • Mark an activity as "Private" if they wish to exclude it from workflow participant visibility.

  • Delegate access to individuals outside standing workflow participants. This may be done with functionality similar to current sharing functionality.

  • Receive all agent emails sent for an activity to all Members of the work group as well as the Admin and work group Producer. All participants in the work group can choose to Disable Notifications by selecting the checkbox in the row with their Participant ID.

  • Provide case access to a select group.

Following are examples of workflows that would be facilitated by inter-organization transfer of activities:

  • Sales Desk/Case Manager Interaction

  • Post-Issue Changes/Requests

  • NIGO Correction Process

"Admin" roles of the work group have different permissions than "Members" of the workflow.

  • The Admin user who creates the Work Group is automatically assigned as the Admin of the group.

  • The Admin user is required to search for a Producer. This is the Producer/Agent who will become secondary Admin for all activities in the work group. The Admin cannot have the same user role as the Producer. They must be different.

  • Admin users can make edits to Work Group Name, Participants (Remove), Disable Notifications for an individual participant (selecting this option disables all notification types and cannot be refined by specific notification), and enable/disable Read-Only Access for individual members.

  • Permissions are either read-only or edit rights. If read-only, the read-only rights are followed at the group code for the activity. If not read-only, then the Group's session permissions are followed.

Members of the work group can:

  • Disable Notifications.

    Selecting this option disables all notification types for the individual member and cannot be refined by specific notification.

  • See their edit/read-only access setting but cannot make changes.

  • Remove themselves from the working group.

  • See who is in the work group with them but cannot make edits to other members or have the option to delete them.

  • Create a new work group and become Admin for that work group.

Members of the work group cannot:

  • Change the Work Group Producer, so they will not see the "Work Group Producer" search box.

  • Add additional members to work groups where they are not the Admin, so they will not see the "Member" search box in their view.

Enable Work Groups

  1. In the Admin tool, in the Admin tab under Organization Settings, select Allow Work Groups.
  2. From the toolbar menu, select Preferences, then click Manage Work Groups.

    • If Allow Creation of New Work Group is selected in the Groups view of the Admin tool, users can click Create New to create a new work group.

    • Users without the ability to create work groups can view their group access and change their notification settings.

    • When a work group is created, the user is added to the group as the Admin.

  3. Rename the work group and add a producer. The producer can be added by typing the name in the Producer field and clicking the Search icon.

    • The selected name from the User list populates the Producer field.

    • Once the Producer field has been filled, the Activities and Product tabs are populated.

  4. Add members to the group following the same steps.
  5. Select Read Only and Disable Notifications for each member and the activities and product types you want to share for this group.
  6. Click Save Work Group to save your changes.

Manage Work Groups

Add a Work Group

  1. If you have access to Work Groups, click the Manage Work Groups button.

  2. On the upper left of the Manage Work Groups dialog, click the Create New link.

  3. The user role who creates the work group is automatically the Admin Assignment. The Admin enters a Work Group Name, selects a Work Group Producer, and selects a Member to add to the work group. The Admin adds all applicable members to the work group. If a Member creates a new work group, the Member cannot see the User List to search for Work Group Producer or Member.

  4. On the Participants tab, select checkboxes to Disable Notifications and/or grant Read-Only Access for the applicable Participants. All participants in the work group can choose whether to Disable Notifications to avoid receiving all agent emails and notifications.

    NOTE: If the individual user has a group setting for Read-Only pre-defined by role code in FireLight Admin, their rights within the work group are set to Read-Only and cannot be changed within the work group by the work group admin. In FireLight Admin, Groups, App tab under User Rights Home/All Activities, the "Enable Read-Only Activity" setting would need to be changed by the user's FireLight Admin for that member to have Edit rights.

  5. On the Activities tab, an Admin user selects the checkboxes for the activities the work group can access. Activities are limited to the activities available to the work group Producer. Adding and removing activities is for the entire work group and not on an individual basis. Member roles have Read-Only access to the Activities tab. Members must contact the Admin of their work group for consideration of adding/removing an activity. E-Delivery activities are not supported. NOTE: Activities are accessible in "All Activities" from the FireLight home page when an activity is created for the activity and product for those within the work group participant list. The ability to view or read-only after opening the activity depends on the settings in Work Groups for the individual. In the event the user is a participant of another work group, the filter functionality can be used on that tab to filter by the owner of the activity.

  6. On the Product Types tab, an Admin user selects the checkboxes for the products the work group can access. Product types are limited to the product types available to the work group Producer. Adding and removing product types is for the entire work group and not on an individual basis. Member roles have Read-Only access to the Product Types tab. Members must contact the Admin of their work group for consideration of adding/removing a product type.

  7. Click the Save Work Group button below the tabs.

  8. Click the X on the upper right of the dialog to close it.

Delete a Work Group or Participant

  1. In the upper grid of the Manage Work Groups dialog, click the red X to the right of the Work Group name to delete the work group.

  2. On the Participants tab of the Manage Work Groups dialog, click the red X to the right of a Participant name to delete the participant from the work group.

  3. Click Yes on the Delete Confirmation dialog to delete the work group or participant. The user no longer sees the deleted group or member once the deletion is confirmed.

View or Edit an Existing Work Group

  1. If you have access to Work Groups, click the Manage Work Groups button.

  2. In the Manage Work Groups dialog, the upper grid lists the work groups in which you are a participant.

  3. Click a work group Name in the upper grid.

  4. Depending on whether you are an Admin or Member, view or make edits to the work group on the Participants, Activities, and Product Types tabs.

  5. If applicable, click the Save Work Group button after making edits.

  6. Click the X on the upper right of the dialog to close it.

How to Enable and Use Work Groups

1. In the Admin Tool, navigate to the Admin tab of Organization settings, Admin tab, and check Allow Work Groups.

2. In the App, select Preferences from the toolbar menu and click Manage Work Groups.

3. If the group option for “Allow Creation of New Work Group” is checked in the Groups view of the Admin tool, the user can click “Create New” to create a new work group.

4. Users without the ability to create work groups can view their group access and change their notification settings.

5. When a work group is created, the user is added to the group as the Admin.

6. Rename the work group and add a producer. The producer can be added by typing the name in the producer text field and clicking the search icon or by just clicking the search icon.

7. The selected name from the user list populates the producer field.

8. Once the producer field has been filled, the activities tab and product tab populate.

9. Add members to the group following the same steps.

10. Check the Read Only and Disable Notifications for each member and the activities and product types you want to share for this group.

11. When finished, save the changes made with the Save Work Group button at the bottom.

 

 Mobile Access

Mobile Access

IMPORTANT! FireLight supports Console and iPad only for forms. It does not support Console and iPad for wizards.

This section is included only if the optional FireLight Console for Windows (Console) feature is enabled in FireLight Admin, which allows you to work on activities while disconnected from the Internet.

NOTE: iPad users can access FireLight from their iPads and follow the instructions in steps 1 through 4 below. Refer to the FireLight eApp™ for iPad Guide FireLight eApp™ for iPad Guide for instructions about using FireLight eApp™.

Install the FireLight Console or FireLight eApp

Use the following instructions to install the Console (or FireLight eApp™ for iPad) to your device (tablet, laptop, etc.).

  1. Click or tap the Add New Device button to get an Access PIN.

  2. Click or tap the Send PIN button.

  3. Enter your Name and Email and click or tap the OK button to send the Access PIN to your email address.

  4. (iPad Only)

    1. If using the iPad for steps 1 through 3, tap the Download on the App Store button to go to the FireLight eApp page in the iOS App Store and then install the FireLight eApp from there.

      NOTE: Download on the App Store button is only available in the Production environment. It is not available in the testing environments (UAT/Staging).

    2. After installing FireLight eApp, enter the Access PIN (see steps 2 and 3) and a Device Name (e.g., MDavid2) and tap Continue.

    3. Log in to FireLight eApp:

      1. If your organization requires a username, enter your designated username in the Enter User Name box.

      2. Enter a password in the Enter Password box, re-enter the Password in the Re-Enter Password box, and tap Continue. Remember this password for future logins to FireLight eApp.

      3. FireLight eApp starts and displays your active applications on the My Applications window.

  5. (Console Only) Click the Windows Install button.

    Using Internet Explorer: In the notification bar at the bottom of the FireLight window, click Run.

    Using Chrome: In the notification bar on the lower left, click the setup.exe up arrow and click Open.

  6. (Console Only) On the Security Warning dialog, click the Install button (Internet Explorer) or chick the Run button (Chrome).

  7. (Console Only) Enter the Access PIN (sent to your email) and the Device Name (name you would like to use for your tablet, laptop, etc.) and click the Continue button.

  8. If the following error message appears, click OK.

  9. Enter a valid Access PIN or go back to the My Preferences page in FireLight to create a new mobile device.

  10. (Console Only) Enter a Password with the designated number of characters or digits, re-enter the Password, and click the Login button to start the Console.

    Password Requirements:

    • The Console prevents the re-use of any passwords created in the past x-number (3 times Password Expiration) days. This means if the Console is set to expire a password in 60 days, you cannot use passwords created in the last 180 days.
    • Each FireLight client organization configures password requirements for the Console. The login page contains the specific requirements.

    Your organization may also require a login Username entry.

    NOTE: The Password is required to log on to the Console.

  11. If an Invalid Password dialog appears, click OK to close the dialog and then enter a valid Password, re-enter the Password and click the Login button.

    12. The FireLight Console starts and displays your active applications in the On Server section of the Console.

  12. Refer to the FireLight Console for Windows Guide FireLight Console for Windows Guide for instructions about using the Console.

  13. When ready to exit the Console, click the menu button on the upper left and click Exit.

    NOTE: In FireLight, the Mobile Access section displays the Console status and provides buttons to enable you to Wipe Device (wipe the data from the Console) and to Reset Password for the Console.

Add Multiple Devices for the Console (Optional)

If you are an agent who sells for multiple carriers, you can segregate your applications in the Console by carrier by using multiple device profiles (one for each carrier). First, add a new device using the FireLight web program and then activate the new device profile using the Console. Add a device for each carrier FireLight login as needed.

  1. In the Mobile Access section, click the Add New Device button to get an Access PIN.

    Click the Send PIN button.

  2. Enter a unique device Name, enter your Email address, and click the OK button to send the Access PIN for the new device profile to your email address. DO NOT CLICK the Download on the App Store button or Windows Install button.
  3. Log on to the Console to complete the new device profile activation. Refer to the “Console Settings” section in the FireLight Console for Windows Guide for instructions on how to complete the device profile activation.
  4. Repeat the process for adding a new device in FireLight and then activate the new device profile in the Console for each carrier.

Reset Password for the Console

If you forget your Console Password or the Password expires, you can initiate a Reset Password from FireLight.

  1. Click the Reset Password button.

  2. Click OK on the Password Reset dialog.

    You need to reset your Password the next time you log on to the Console. Refer to the FireLight Console for Windows Guide for Password reset instructions.

Wipe Data from (Deactivate) the Console or FireLight eApp

Use the Wipe Device button to wipe out (delete) all updates you made to all downloaded applications from FireLight to the Console or FireLight eApp on the iPad. This deactivates the Console or FireLight eApp. To use the Console or FireLight eApp again, you need to install the Console or FireLight eApp. See Install the FireLight Console for instructions.

  1. Click the Wipe Device button.

  2. Click OK on the Deactivate Device dialog to wipe the data from and deactivate the Console.
