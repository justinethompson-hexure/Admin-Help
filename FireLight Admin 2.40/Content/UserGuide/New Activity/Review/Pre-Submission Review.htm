Pre-Submission Review Step [User Instructions]

NOTE: If the organization uses a Pre-Signature Review Queue, the Request Client to Fill & Sign option is disabled on the Other Actions menu. The Request Client to Fill option is still available.

The pre-submission review step in the application process is an optional step where the Agent sends a request for review to various reviewers (based on the organization’s preferences) prior to submitting the application package for processing.

When an activity is locked by the reviewer (status = "In Pre-Submit Review" or "In Review") and the application is opened by the agent/user in FireLight®, a message appears with text, "Application is locked for review." The agent/user cannot unlock the application. The Unlock option on the Other Actions menu and the Unlock button on the Rename/Summary dialog on the Other Actions menu are disabled while the activity is in review.

One of the following occurs when a reviewer approves an application order in the pre-submission review queue:

  • If the pre-submission review occurs after signatures are completed and your organization does not have a post-submission review process, FireLightl automatically submits the application to the back office for processing.
  • If your organization has a post-submission review process and pre-submission review occurs after signatures are completed, FireLight first inserts the application into a post-submission review queue, and, upon reviewer approval, submits the application order to the carrier for processing.
  • If the pre-submission review occurs before signatures are completed, upon the PreSubmit reviewer approval the application goes to the PreSubmit Review Complete status in FireLight. You then need to open the application, capture signatures, and submit the application. If your organization has a post-submission review process, FireLight first inserts the application into a post-submission review queue, and, upon reviewer approval, submits the application order to the carrier for processing.

If a reviewer rejects an application order, the application returns to the Recent Activity list with a “Review Declined” status. Copy the application, make the corrections noted in the Review Rejected message, and resubmit the application.

REVIEWERS NOTE: See the FireLight Review Queue Guide for pre-submission and post-submission reviewer instructions.

Depending on your organization, the pre-submission review step can occur before or after the signatures step.

NOTE: If your organization requires applications to go to pre-submission review prior to application submission, the Send Email To Reviewer(s) page will open instead (go to step 3).

Step through the following instructions for pre-submission review features.

Pre-Signature Review

  1. When you complete the DATA ENTRY step, click the CONTINUE button on the right as shown below.

  2. On the Electronic Review page, click a button to determine the type of review.

    Clicking the Send Request to Reviewer(s) button initiates the electronic review process. You will collect all reviews and approvals for the application electronically. Go to step 3.

    Clicking the Decline E-Review button declines electronic review. Click OK on the Electronic Review Declined page to return to the application. You can then proceed to the Signatures step without requesting a review.

  3. If electronic review, enter review information on the Send Email to Reviewer (s) page.

  4. Select the checkbox next to each Reviewer Name who will receive the pre-submission review request.

  5. Do any of the following:

    • Request another reviewer—click the Add Reviewer link, type the Reviewer Name and Reviewer Email, then select the checkbox for the reviewer.
    • Omit a reviewer—clear the checkbox next to the Reviewer Name.
    • Clear the selected reviewers and begin again—click the Reset link.

  6. Click Send Email Request to send the review request via email.

    NOTE: Passcode for Reviewers is an automated unique login code for the reviewers to access and approve the application package on a secure website. If your organization enables Auto-Email Passcode in FireLight Admin, FireLight automatically sends the passcode to the reviewer via a separate email. If this feature is not enabled for your organization, you need to provide this code to the reviewers via mobile phone or in a separate email.

  7. Click Back To App on the Email Requests Sent page to return to the application.

Post-Signature Review

  1. When you complete the DATA ENTRY and SIGNATURES steps, click the CONTINUE button on the right as shown below.

  2. On the action menu, click Request Review to proceed with the pre-submission review process.

    On the action menu, click Submit to skip the pre-submission review and submit the application. Click OK on the Confirmation Dialog.

  3. If requesting review, on the Electronic Review page click a button to determine the type of review.

    Clicking the Send Request to Reviewer(s) button initiates the electronic review process. You will collect all reviews and approvals for the application electronically. Go to step 4.

    Clicking the Decline E-Review button declines electronic review and submits the activity. Click Close on the Application is pending transmission... message to close the dialog.

  4. If you requested review, enter review information on the Send Email to Reviewer (s) page.

  5. Select the checkbox next to each Reviewer Name who will receive the pre-submission review request.

  6. Do any of the following:

    • Request another reviewer—click the Add Reviewer link, type the Reviewer Name and Reviewer Email, then select the checkbox for the additiona reviewer.
    • Omit a reviewer—clear the checkbox next to the Reviewer Name.
    • Clear the selected reviewers and select again—click the Reset link.

  7. Click Send Email Request to send the review request via email.

    NOTE: Passcode for Reviewers is an automated unique login code for the reviewers to access and approve the application package on a secure website. If your organization enables Auto-Email Passcode in FireLight Admin, FireLight automatically sends the passcode to the reviewer via a separate email. If this feature is not enabled for your organization, you need to provide this code to the reviewers via mobile phone or in a separate email.

  8. Click Back To App on the Email Requests Sent page to return to the application.

    Following reviewer approval in the post-signature review process, the application is auto-submitted.

Review Multi-Life Applications (Reviewer Instructions)

The review process is slightly different for multi-life applications, which are reviewed in a batch.

  1. As a reviewer, click the URL link within the email you received from the agent.

  2. Type the Passcode the agent sent in a separate email and click Enter. Click the Questions and Support link for additional information.

  3. On the Review Applications page, click the View button to open an application for review.

  4. View each multi-life application. If needed, click the Email Agent button to email the agent for more information about an application.

  5. You can approve or reject each application one at a time by clicking the Approve button or Reject button for each application in the grid.

    To save time, batch approve or reject multiple applications. Go to steps 6 and 7.

  6. Approve multiple applications:

    1. On the Review Applications page, select the checkbox for each application to approve and click the Approve All Selected button above the grid.

      NOTE: The buttons above the grid become available when you select multiple applications.

    2. (Optional) To upload an additional document (e.g., reviewer notes, etc.), click the uploaded link near the top of the dialog and browse or choose a PDF file.

    3. Click the I Consent button to approve the applications.

    4. Click OK on the confirmation dialog.

  7. Reject multiple applications:

    1. On the Review Applications page, select the checkbox for each application to reject and click the Reject All Selected button above the grid.

    2. Enter a Reason for Rejection. You must enter a reason before the I Reject button becomes available.

    3. Click the I Reject button to continue.

      A confirmation dialog appears. Click OK to close the dialog and return to the previous page.

  8. A Review Applications message states, “You have finished reviewing all the applications in this request. Thank you. Click the Log Off button on the upper right.
