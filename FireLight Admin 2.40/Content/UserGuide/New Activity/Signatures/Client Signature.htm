Client Signature [User Instructions]

NOTE: In FireLight Admin, the Groups/App tab settings for "Disable Send Email Request for Client" and "Disable Sign Now for Client" settings determine whether Send Email Request or Sign Now is available for client signers.

NOTE: The EXTERNAL role code governs the behavior of the signing ceremony for the signature request. The DEFAULT role code will be in effect for agents coming in a) without a role code, or b) with an invalid role code.

After completing the first client signature, the same process repeats for all remaining required client signers regardless of signer type.

NOTE: Activities containing forms that require to be signed by a Trustee will display the text "TTEE" as a suffix on the Trustee signature field at the following places:

  • Form/application

  • Addendum document

  • Application PDF

  • Application Audit Report

  • Applications created via Embedded APIs/IXN

Text "TTEE" will show suffixed on the Trustee signature field on the Application, Application PDF and Application Audit for the following cases:

  • For all the trustee signers such as: Trustee, Trustee 2, Trustee 3, Trustee 4, etc.

  • When multi-selection is done at the time of signing for "Trustee" signer along with "Owner" signer type.

Capture a client signature:

  1. Click the applicable signer button (Owner, Joint Owner, Annuitant, Authorized Signer, Insured, Trustee, etc.) in the List of Required Signers on the Signatures page to begin the e-signature process for the Client.

    OPTIONAL: If the signer is more than one signer type and the organization enabled the multi-signer type feature in FireLight® Admin, click each signer type button that applies to them and then click Proceed to begin a single signing process for all selected signer types. In the following example, the Annuitant is also the Owner.

    NOTE: The organization can require the agent to sign last as shown above. The agent button is disabled until all client signatures are completed.

  2. Choose a method to obtain the Client’s signature.

    • If the Client is with the agent, click Sign Now. If only one signer is selected, the signer type and name appear on the Client Signature Choice page.

    • If the Client is not present and would prefer to sign using a secure web site, click Send Email Request.

  3. Follow the instructions below for "Sign Now" or "Send Email Request - Generate Link Without Email."

Sign Now

NOTE: If the organization implemented the Tele-Sign feature, see "Tele-Sign" below for instructions to capture client signatures via telephone.

  1. After clicking Sign Now on the Client Signature Choice page, enter information in the following Agent and Client Identification Verification sections. Select a Form of Identification in the list and then enter client information accordingly. The Birth Date field requires the client birth date in the format mm/dd/yyyy.

  2. After filling in the information, click Verified to proceed. Click Cancel to return to the Signatures page without the Client signature. Click Back to Application on the upper right to return to the application without capturing signatures.

    NOTE: If the organization uses Passcode Only Validation, the Last 4 Digits of SSN/Government ID and Birth Date field entries are not included in the Client Identification Verification. A *Client Authentication via Passcode* message appears above the option to Send Email Passcode. Enter information and click the Send Email Passcode button.

    The client receives an email similar to the following.

    The Client enters the Passcode from the email into the Verify Client dialog and clicks the Submit button.

    Click Verified to proceed.

  3. If initials are required in a document, the Client clicks the form and buttons to move through the document and clicks in the initials text box with a gray border.

    The Client enters his or her initials in the text box on the Please enter your initials dialog and clicks OK. Optionally, the Client can click Cancel to not capture initials and halt the document review and consent process.

  4. The Client clicks a document button to open the document. The Signature page displays the first form within the order's package of forms that requires the Client's signature. The Client reviews all of the pages in the document by clicking the forward and back buttons to move to the next and previous pages in the document. The Client selects the checkbox for I have reivewed and agree with the terms expressed within this document.

    A green check mark appears next to the reviewed and consented documents.

    The Client repeats the review and consent process for all remaining documents.

  5. When the Client has reviewed and approved all documents, the Capture Electronic Signatures page opens. The Client enters their information.

    The organization can limit the selections in the State list to only the approved states on the Sales Agreement. If so, the signer selects an approved state from the State list. The organization can also leave the State blank, so the signer is required to select the correct state for the signing ceremony.

    If the organization enabled script fonts, the Client can select a script font in the Select Font list. If the organization did not enable script fonts, FireLight uses the standard block font.

    To override the text script, the Client can click in the signature box and sign on the pad using a mouse or stylus.

    The Client clicks one of the following:

    • I Consent to authorize the signature.
    • I Decline to decline the signature.
    • Cancel to cancel the signature process and return to the List of Required Signers page.

    • Clear Signature to clear the signature and use the text script.

    State Signing Error: If the Signature Box control for any signer type on the form has the State Signing Error option enabled in Admin and the state selected is not the state where the activity is being signed, a message similar to the following Signing Error message appears. Client clicks OK to close the message and selects another State in the list.

    The organization can set the number of state not matching attempts to be unlimited in Admin or the organization can set a limit on the number of attempts for state not matching.

    If the number of state mismatch attempts is limited, on the final attempt to sign with the state not matching, a message like the following Signing Error message appears.

    The following message appears in FireLight.

    The agent receives a message and email with content similar to the following.

    An audit message is written out for the Signing Ceremony for the signer type each time the warning message is displayed during the signature process. If the final configurable base message is presented to the user, a final audit record is also written out.

    Optional: If the Client would like to review and re-sign the application, do the following:

    1. Click the Re-Sign link next to the applicable signature type (Annuitant, Owner, etc.).

    2. Click OK on the Re-Sign Signature verification dialog.

  6. Click a required signer in the List of Required Signers to continue the electronic signature process. Have remaining required signers electronically sign the document package using the same process as the first signer.

  7. A notification appears when the required signatures are completed. Click the CONTINUE button on the upper right to go to Review or to go to Finalize. This is a fully configurable message by organization.

  8. Go to Agent Signature [User Instructions] if the agent has not yet signed.
  9. Go to Pre-Submission Review Step [User Instructions] or go to Submit the Activity [User Instructions] depending on whether your organization has a post-signature pre-submission review step.

Generate Link Without Email

The Generate Link Without Email feature allows the agent to generate a Passcode and link to a FireLight website that clients who do not have personal access to email can use to log in to FireLight and sign their application.

  1. Click the Send Email Request button on the Client Signature Choice page, the system auto-populates the Client Name, Client Last 4 Digits of SSN/Government ID, and Client Birth Date. Optionally, click Cancel to go to the List of Required Signers page.

  2. Click the Generate Link Without Email button to generate a Passcode and link to a FireLight website that clients who do not have personal access to email can use to log in to FireLight and sign their application. If the organization does not use the Passcode feature, the Client can enter the Last 4 of SSN/Government ID and Birth Date to log in to FireLight.

    In the following example, the Passcode is ko6b1jcs and the link is https://firelight.insurancetechnologies.com/JMN.

    NOTE: If the organization uses Passcode Only Validation, fill in the information and click the Generate Link Without Email button. A *Client Authentication via Passcode* message appears below the client section.

  3. Provide the Passcode and link to the client via phone or text. Click Back to Signers at the end of the page to return to the List of Required Signers page.

  4. The client logs in to the website using the provided URL link and the Passcode or Last 4 of SSN/Government ID and Birth Date.

    If the organization uses Passcode Only Validation, enter the Passcode. The Last 4 of SSN/Government ID and Birth Date will not display as a validation option if the FireLight Admin setting is selected to use Passcode Only Validation.

    The Electronic Signatures page opens.

  5. On the Electronic Signatures page, the Client clicks a button to do any of the following:

    • Sign Activity – opens the Owner Signature page to review the application pages and begin the signing process. Go to step 6.

    • Review Documents – opens document pages in a separate tab or window. Client scrolls through the pages, uses the PDF controls to print or save the documents to the client’s PC, and closes the tab or window to exit the documents review.

    • Contact Agent – opens a message box to send a request for information to the Agent. On the message box, the client enters a message and clicks Send. The signer needs access to an email account entered in the From Address field to receive a reply to the message.

  6. After clicking the Sign Activity button, the client clicks a document button, reviews the document and selects I have reviewed and agree with the terms expressed within this document checkbox below the documents names.

    The Client repeats the review and consent process for all remaining documents.

  7. When the Client signer has reviewed and approved all documents, the Capture Electronic Signature page opens.

    On the Capture Electronic Signature page, the client enters their information.

    The organization can limit the selections in the State list to only the approved states on the Sales Agreement. If so, the signer selects an approved state from the State list. The organization can also leave the State blank, so the signer is required to select the correct state for the signing ceremony.

    Client selects a font in Select Font (if the feature is enabled), and clicks the I Consent button to submit their signature. Client can click Clear Signature to clear the system signature and sign in the signature block using a mouse, finger or stylus. Optionally, the client can click I Decline to decline signing.

    State Signing Error: If the Signature Box control for any signer type on the form has the State Signing Error option enabled in Admin and the state selected is not the state where the activity is being signed, a message similar to the following Signing Error message appears. Client clicks OK to close the message and selects another State in the list.

    The organization can set the number of state not matching attempts to be unlimited in Admin or the organization can set a limit on the number of attempts for state not matching.

    If the number of state mismatch attempts is limited, on the final attempt to sign with the state not matching, a message like the following Signing Error message appears.

    The following message appears in FireLight.

    The agent receives a message and email with content similar to the following.

    An audit message is written out for the Signing Ceremony for the signer type each time the warning message is displayed during the signature process. If the final configurable base message is presented to the user, a final audit record is also written out.

    After the Client clicks, I Consent, the following verification page appears.

    The client clicks one of the following:

    • Ok on the message or clicks Log Off on the navigation bar to log off and exit FireLight.

    • View Activity PDF on the navigation bar to view the activity documents.

    • Contact Agent on the navigation bar to send an email to the agent.

  8. The agent receives an email that the client has signed and approved the application.

  9. If agent signatures are required, the activity status is In Signatures. The agent opens the activity and completes the agent signing process.

  10. Go to Pre-Submission Review Step [User Instructions] or go to Submit the Activity [User Instructions] depending on whether the organization has a post-signature pre-submission review step.

Send Email Request

  1. Click the Send Email Request button on the Client Signature Choice page.

    The system auto-populates the Signer Name, Signer Email, Signer Last 4 Digits of SSN/Government ID, and Signer Birth Date. Optionally, click Cancel to go to the List of Required Signers page.

  2. Click Send Email Request to send the email signature request to the client via email.

    NOTE: If the organization uses Passcode Only Validation, the Signer Last 4 Digits of SSN/Government ID and Signer Birth Date are not available. Enter the Signer Name, Signer Email, Your Name and Your Email, and click Send Email Request. A *Client Authentication via Passcode* message appears below the client section.

    After sending the email request to the client, FireLight assigns an access Passcode and sends it to the user via email. Click Back to Signers to close the signature request.

    NOTE: You can send a reminder and a Passcode by clicking links within a message in My Messages [User Instructions].

  3. FireLight sends the client request via email. The Client is instructed to click a URL link, enter the last four digits of their SSN/Government ID and Birth Date or a Passcode to access and complete the e-signature process for the application. The URL link is valid for a set amount of time determined by the organization.

  4. The Client clicks the URL on the email and enters the Passcode from the Passcode Email or Agent, or enters the Last 4 Digits of SSN/Government ID and Birth Date, and clicks Enter to go to electronic Signatures in FireLight. A message appears if the Client enters invalid or expired login Last 4 Digits of SSN/Government ID, Birth Date, or Passcode.

    NOTE: If the organization uses Passcode Only Validation, the client enters the Passcode from the Passcode Email or provided by the agent.

  5. On the Electronic Signatures page, the Client clicks a button to do any of the following:

    Sign Activity – opens the Owner Signature page to review the document set pages and begin the signing process. Go to step 6.

    Review Documents – opens document pages in a separate tab or window. Client scrolls through the pages, uses the PDF controls to print or save the documents to the Client’s PC, and closes the tab or window.

    Contact Agent – opens a message box to send a request for information to the Agent. On the message box, the Client enters a message and clicks Send.

  6. After clicking Sign Activity, if the Client signer’s initials are required in the document, an "Initials are required in this document set" appears below the document buttons.

    The signer goes to the document page, clicks the initials text box on the document, and enters their initials and clicks OK on the Please enter your initials dialog.

    After entering initials, the signer clicks a document button to begin review of the document.

    When the Client signer has reviewed and approved a document, the Client selects I have reviewed and agree with the terms expressed within this document checkbox on the lower left. The Client repeats the document review and consent process for all documents within the package.

  7. When the Client reviews and approves all documents, the Capture Electronic Signature page opens. The Client enters their information.

    The organization can limit the selections in the State list to only the approved states on the Sales Agreement. If so, the signer selects an approved state from the State list. The organization can also leave the State blank, so the signer is required to select the correct state for the signing ceremony.

    If the organization enabled script fonts, the Client can select a script font in the Select Font list. If the organization did not enable script fonts, FireLight uses the standard block font.

    To override the text script, the Client can click in the signature box and sign on the pad using a mouse or stylus.

    The Client clicks one of the following:

    • I Consent to authorize the signature.

    • I Decline to decline the signature.

    • Clear Signature to clear the signature and use the text script.

    State Signing Error: If the Signature Box control for any signer type on the form has the State Signing Error option enabled in Admin and the state selected is not the state where the activity is being signed, a message similar to the following Signing Error message appears. Client clicks OK to close the message and selects another State in the list.

    The organization can set the number of state not matching attempts to be unlimited in Admin or the organization can set a limit on the number of attempts for state not matching.

    If the number of state mismatch attempts is limited, on the final attempt to sign with the state not matching, a message like the following Signing Error message appears.

    The following message appears in FireLight.

    The agent receives a message and email with content similar to the following.

    An audit message is written out for the Signing Ceremony for the signer type each time the warning message is displayed during the signature process. If the final configurable base message is presented to the user, a final audit record is also written out.

  8. After signing and consenting, the following verification screen appears.

    The client clicks one of the following:

    • Click OK on the message or click Log Off on the navigation bar to log off.

    • Click View Activity PDF on the navigation bar to view the activity documents.

    • Click Contact Agent on the navigation bar to send an email to the agent.

  9. A notification appears when the required signatures are completed. Click the CONTINUE button on the upper right. A message appears. This is a fully configurable message by organization.

  10. Go to Pre-Submission Review Step [User Instructions] or go to Submit the Activity [User Instructions] depending on whether the organization has a post-signature pre-submission review step.

Tele-Sign

If your organization implemented the Tele-Sign feature, do the following to capture client signatures.

  1. Enter the appropriate data into the following Tele-Sign fields on the form or wizard:

    • Enable Tele-Sign – enter True
    • Tele-Sign App ID – enter any number (recommended at least 4 digits – must be unique per application)
    • Tele-Sign PIN – enter any 4-digit number (does not need to be unique)
    • Tele-Sign DOB – enter the signer’s birth date

  2. When the activity is complete, click CONTINUE to proceed to signatures.

  3. On the Electronic Signatures page, click Use E-Signature.

  4. In the List of Required Signers, click the non-agent signer(s) and click Proceed.

    NOTE: If the non-agent signer is more than one signer type on the form and the organization enabled the multi-signer type feature in FireLight Admin, click each signer type button that applies and then click Proceed to begin a single signing process for both signer types. For example, the Payor can also be the Applicant.

  5. On the Client Signature Choice page, click Tele-Sign.

  6. On the Tele-Sign Signer Information page:

    1. Enter the signer information for Signer Full Name, City, and State in the upper section.

    2. If the signer will receive an email with a link to the signed documents, enter the information for the Signer Email, Signer Birth Date and Signer Last 4 Digits of SSN/Government ID in the lower section.

    3. Click Continue.

  7. On the Tele-Sign: Form Access page, select a Delivery Type in the list (Download, Generate Link, or Send via Email). On the Form Access section, choose a document delivery method.

    NOTE: Your organization may not have all options available. A delivery method does not need to be selected to complete the Tele-Sign signature.

    • Download: In the Delivery Type list, select Download and click the Download Forms button to download a copy of the PDF application forms that can be mailed to the client. Click Continue.

    • Generate Link: In the Delivery Type list, select Generate Link, enter the signer information for Last 4 Digits of SSN/Government ID and Signer Birth Date, and click the Generate Link button to generate a URL link to give to the signer so they can view and download the application PDF file on the Web. The link text appears below the Generate Link button as shown below. Click Continue.

    • Send via Email: In the Delivery Type list, select Send via Email. Enter the Signer Name, Signer Email, Signer Last 4 Digits of SSN Government ID, and Signer Birth Date. Click the Send Email button as shown below. The client receives an email and clicks the URL link in the email to go to the URL in a web browser. Client enters the Last 4 Digits of SSN Government ID and their Birth Date to view and download the PDF application documents. Click Continue.

  8. On the Agent Instructions page, read the Tele-Sign script and follow the instructions to verify the signer’s identity and gain their Tele-Sign voice signature.

  9. (Optional) Click Cancel Tele-Sign to clear the Tele-Sign signature and start over.

  10. Click the Done button to continue.

    • If the Tele-Sign signature was completed, but declined, FireLight displays a message that the Tele-Sign was declined and returns to the application in Data Entry mode.

    • If the signature has not yet been completed, FireLight returns to the List of Required Signers page. A "(Tele-Sign in Progress)" indicator appears next to the client signature name for any signer types using Tele-Sign.

      • To return to the Agent Instructions page, click the client Tele-Sign signature button. Click Done on the Agent Instructions page to return to the List of Required Signers page.

      • To cancel a Tele-Sign signature capture, click the Cancel Tele-Sign link under the client signer name and click OK on the confirmation dialog.

    • If the Tele-Sign signature was completed and approved, FireLight returns to the application view if Review has been enabled. If Review has not been enabled, FireLight returns to the list of signers if additional signatures need to be collected, or the application can be submitted, either manually or automatically if your organization uses auto submit.

      NOTE: The Agent can sign in a default order, first or last, depending on what is set in the organization settings.

Optional Features:

Lexis Nexis Client Identification Verification

FireLight incorporates the LexisNexis Instant ID and Instant Authentication services into the client verification/authentication process during the signing process for both Sign Now and Sign Later (external signing). FireLight supports the LexisNexis Instant ID process, sending the client's name, address, date of birth, and SSN for validation. FireLight includes the application ID and DTI number in the message to use as the customer reference values within the LexisNexis system.

In the Sign Now process, the initial call to LexisNexis occurs at the time of client verification after the agent submits the verification screen for the selected signer. In the Sign Later process, FireLight initiates the web service call to LexisNexis after the SSN/DOB or passcode validation occurs.

If your organizaiton added LexisNexis client identity verification/authentication duruing the e-siqnaure process, a LexisNexix Client Authentication page appears. The signer selects an answer for each multiple-choic question and clicks the Submit button. Based on the responses to the initial questionnaire, LexisNexis may return a bonus question. The signer selects an answer for the multiple-choice bonus question and clicks the Submit button.

Regardless of the authentication score by LexisNext, after clicking the Submit button, the signer continues with the electronic signature signing the documents. FireLight creates an audit record for the initial web serive call to LexistNexis and a second audit at the conclusion of the authentication process.

Require Signature Pad for Signature Capture

If your organization requires a signature on the signature pad or an external Topaz signature pad, the following message appears when you click the I Consent button. Click OK to close the message and then sign on the signature pad using a mouse, your finger or a stylus.

Revoke a Signature Request

  1. Click the Revoke Request link on the signer type button to revoke the signature request.

  2. Click OK on the confirmation dialog.
SMS Text Authentication

FireLight® supports SMS Text Authentication and validation for all signature workflows including Sign Now, Remote Sign (Send Email Request), and Client Fill and Sign. This provides an additional layer of validation because both the email and the phone number would have to be compromised to forge the signature.

Sign Now

  1. On the Client Identification Verification page, FireLight pre-fills the Mobile Number field with the mobile number provided in the SSO or by data entry. The agent fills in client identity information and clicks the Send SMS Text button.

    NOTE: If your organization uses the Validate Identity Verification feature, the last 4 digits of the SSN/Government ID number, Birth Date, and Mobile Number are blank and need to be filled in during signer verification. The entered values must match values entered in the application.

    NOTE: If the organization uses Passcode Only Validation, SMS Text is the method of authentication in lieu of the passcode email. Enter the information including the Mobile Number and click Send SMS Text.

  2. Enter the Passcode sent to the client’s mobile number in the dialog and click the Submit button.

  3. FireLight verifies the mobile number and replaces the Send SMS Text button with a Verified button. Click the Verified button and proceed with the normal Sign Now process. The SSN/Government ID and Birth Date are not included when the organization uses Passcode Only Validation along with SMS Text Authentication as shown below.

Send Email Request

  1. On the Send Email Request workflow, enter the client identification data, including the Signer Mobile Number, and click the Send Email Request button.

    NOTE: If your organization uses Passcode Only Validation along with SMS Text Authentication, the Signer Last 4 Digits of SSN/Government ID and Signer Birth Date fields are not included.

  2. The client receives an email request similar to the following and clicks the URL link.

  3. The client clicks the link in the email request to open the login dialog in FireLight. Client enters Last 4 Digits of SSN/Government ID and Birth Date and clicks the Send Passcode button.

    NOTE: If the organization uses Passcode Only Validation, the following dialog appears. The Client clicks the Send Passcode button.

  4. Client enters the Passcode for the recent request sent to their mobile number and clicks the Verify button to proceed with the normal Send Email Request process.

Client Fill and Sign

  1. On the navigation bar, click Other Actions > Request Client to Fill & Sign. The system populates the matching fields with data that you added to the activity.

  2. Fill in the signer information (SSN/Government ID, Birth Date, etc.).

    An additional Mobile Number field appears below the Birth Date field. Fill in the signer's Mobile Number.

    NOTE: If the organization uses Passcode Only Validation in addition to SMS Text Authentication, fill in the information including the signer's Mobile Number.

  3. On the right, add any additional information to the message for the client fill and sign request.

  4. Click Send Email to send the request to the client. (Click Cancel to cancel the request and return to the activity.)

    Click OK on the Request Sent dialog.

  5. Click Back To Signers to return to the List of Required Signers.

  6. On the Pending Request dialog in forms, click a Send Reminder link or click a Send Passcode link to send the Passcode to the client.

    • To send a Passcode to the client’s cell phone in forms, click the Send Passcode to [Client’s] Cell Phone link, enter the Client’s mobile number and click the Send SMS button.

    • To send a Passcode to the client’s email in wizards, click the Send Passcode to [Client’s] Email link or Send button (email) as applicable.

  7. The client receives an email similar to the following and does the following:

    1. Client clicks the FireLight URL link.

    2. Client types their Passcode or types the Last 4 Digits of SSN/Government ID and their Birth Date and clicks the Enter button.

      The client must enter the Last 4 Digits of SSN/Government ID and their Birth Date and then click the Send Passcode button. FireLight sends a Passcode to the client’s cell phone.

      Client enters the Passcode and clicks Verify.

    3. After the system verifies the SMS Passcode, the client can proceed with the fill and sign request.

Sign with Topaz Signature Pad

If your organization uses the Topaz external electronic signature pad, you can use the Topaz signature pad as another option for capturing electronic signatures in FireLight as shown below.

Connect the Topaz Signature Pad

Be sure to connect the Topaz Signature Pad to the device and enable it as shown in the following examples.

  • If using the Internet Explorer 11 or later browser: open the Tools menu and click Manage add ons. On the left under Show, select All add-ons in the list. On the right, scroll down to Topaz Systems, Inc. If it is set to Disabled, right-click SigPlus NPAPI plugin and then click Enable.
  • If using the Chrome browser: click on the upper right, click Settings, scroll down and click Show advanced settings. Scroll down to Privacy and click the Content settings button. Scroll down to Plug-ins and click the Disable individual plug-ins link. Scroll down to SigPlusWebPlugin. Click the Enable link and select the Always allowed checkbox. Close the Plug-ins tab and click Done on the Content settings dialog. Close the Settings tab.

Sign with the Topaz Signature Pad

  1. Click the Sign with Topaz Signature Tablet link above the signature pad. Then sign on the Topaz signature pad instead of using a mouse or touch screen.

  2. If you would prefer not to use the Topaz signature tablet, click one of the following options:

    Clear Signature clears the signature and uses the text script for the signature.

    Sign with Computer Mouse or Touch Screen link signs using the computer mouse or touch screen.

Topaz Signature Pad Setup

Benefits of Topaz

The Topaz® signature pad provides a high quality signature capture. It supports modern browsers (Chrome, FireFox, Opera, Edge, and Internet Explorer). Topaz signature pad is recognized for its signature integrity and forensic authentication. For more information about the benefits of Topaz and supported browsers, visit the Topaz website, http://topazsystems.com/.

Topaz in FireLight

NOTE: The organization needs to contact their FireLight® Strategic Account Manager to have the CSS modified to display the option to sign with the Topaz signature tablet.

Install the Topaz Plug-In

  1. Note the model number on the bottom of the Topaz tablet.
  2. Connect the Topaz tablet to your device (laptop, etc.).
  3. Create a new application in FireLight.
  4. Complete Form Entry and continue to Signatures.
  5. Click Agent in the List of Required Signers and review and approve the application.
  6. Click Sign to open the Agent On-Site Signature page.

  7. Click the information icon on the right of the Sign with Topaz® signature tablet link.

  8. In the dialog, click the link to install the SigWeb® Software.

  9. The Topaz Systems website opens. Click the link to the SigWeb Installation Help Guide for step-by-step installation instructions.

    IMPORTANT! If the Topaz HTTPS certificate expires, the Topaz signature pad will not work. To update the certificate, go to http://www.topazsystems.com/sigweb.html and follow directions in the webpage to update your Topaz certificate by downloading the SigWeb Certificate Updater.

  10. After installing the Topaz signature pad: In FireLight, fill in the remaining information on the Agent On-Site Signature page, click the Sign with Topaz® signature tablet link, and sign on the Topaz signature tablet to sign your signature in FireLight.
