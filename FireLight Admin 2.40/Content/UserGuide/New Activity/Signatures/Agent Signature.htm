Agent Signature [User Instructions]

NOTE: In FireLight Admin, the Groups/App tab settings for "Disable Send Email Request for Agent" and "Disable Sign Now for Agent" settings determine whether Send Email Request or Sign Now is available for agent signers.

NOTE: The EXTERNAL role code governs the behavior of the signing ceremony for the signature request. The DEFAULT role code will be in effect for agents coming in a) without a role code, or b) with an invalid role code.

  1. In the List of Required Signers, click the Agent button to begin the agent signature process.

  2. Choose a method to obtain the Agent’s signature.

    • If the primary or additional Agent is present, click Sign Now.

    • If the primary or additional Agent is not present and would prefer to sign using a secure web site, click Send Email Request.

  3. Follow the instructions below for "Sign Now" or "Generate Link Without Email" or "Send Email Request."

Sign Now

  1. If the primary or additional Agent is present, click Sign Now on the Agent Signature Choice page.

  2. If initials are required, an "Initials are required in this document set" message appears on the lower left.

    Within the document, click in the Initials text box with the gray border.

    Enter the agent's initials in the dialog, and click the OK button.

    NOTE: Signer’s initials are entered in this dialog only once for all documents in the Initials document set for that signer type. Simply click the initials checkbox on all succeeding forms.

    Following is an example of how initials and capture date appear on the form.

  3. Near the top of the page, click a document button and review all pages for that document.

  4. When finished reviewing a document, select the checkbox for I have reviewed and agree with the terms expressed within this document. Repeat the review and consent process for each document.

  5. When you have reviewed and approved all documents, click Sign to begin the signature process.

  6. OPTIONAL for Agents only: If the organization allows Agents to sign without reviewing all of the documents, you can select the Confirm all documents without reviewing individually checkbox and proceed to signature capture.

    As an Agent, do the following to sign the document without first reviewing all of the document pages.

    1. Select the Confirm all documents without reviewing individually checkbox.

    2. Click OK on the Confirm Documents dialog.

    3. Click Sign.

  7. On the Agent On-Site Electronic Signature page, enter the City and ensure your agent information is correct. FireLight inserts Agent Full Name into the signature pad. If available, select a font for your signature in the Select Font list. To override the text script, click in the signature block and simply sign on the pad. To clear the signature and use the text script, click Clear Signature.

    Click one of the following buttons:

    • I Consent to authorize the signature and complete the signing process.
    • I Decline to decline the signature.
    • Cancel to cancel the signature process and return to the List of Required Signers page.
    • Clear Signature to clear the signature and use the text script.

  8. Go to Pre-Submission Review Step [User Instructions] or go to Submit the Activity [User Instructions] depending on whether the organization has a post-signature pre-submission review step.

Generate Link Without Email

The Generate Link Without Email feature allows the agent to generate a Passcode and link to a FireLight website that other agents who do not have personal access to email can use to log in to FireLight and sign their application.

  1. Click the Send Email Request button on the Agent Signature Choice page. Optionally, click Cancel to go to the List of Required Signers page.

  2. Click the Generate Link Without Email button to generate a Passcode and link to a FireLight website that agents who do not have personal access to email can use to log in to FireLight and sign their application.

    In the following example the Passcode is ix452xtx and the link is https://firelight.insurancetechnologies.com/WZr.

  3. Provide the Passcode and link to the Agent via phone or text. Click Back to Signers at the end of the page to return to the List of Required Signers page.

  4. The Agent logs in to the website using the provided URL link and the Passcode.

    If the organization uses Passcode Only Validation, enter the Passcode. The Last 4 of SSN/Government ID and Birth Date will not display as a validation option if the FireLight Admin setting is selected to use Passcode Only Validation.

    The Electronic Signatures page opens.

  5. On the Electronic Signatures page, the Agent clicks a button to do any of the following:

    • Sign Activity – opens the Agent Signature page to review the application pages and begin the signing process. Go to step 6.

    • Review Documents – opens document pages in a separate tab or window. Agent scrolls through the pages, uses the PDF controls to print or save the documents to the Agent’s PC, and closes the tab or window to exit the documents review.

    • Contact Agent – opens a message box to send a request for information to the originating Agent. On the message box, the Agent enters a message and clicks Send. The agent signer needs access to an email account entered in the From Address field to receive a reply to the message.

  6. After clicking the Sign Activity button, the Agent clicks a document button, reviews the document and selects I have reviewed and agree with the terms expressed within this document checkbox below the documents names.

    The Agent repeats the review and consent process for each document unless the organization has enabled the option for agents to select the "Confirm all documents without reviewing individually" checkbox and clicks OK on the Confirm Documents dialog to confirm all documents without individual document review.

  7. When the Agent signer has reviewed and approved all documents, the Capture Electronic Signature page opens.

    On the Capture Electronic Signature page, the Agent enters their information.

    Agent selects a font in Select Font (if the feature is enabled), and clicks the I Consent button to submit their signature. Agent can click Clear Signature to clear the system signature and sign in the signature block using a mouse, finger or stylus. Optionally, the Agent can click I Decline to decline signing.

    After the Agent clicks, I Consent, the following verification page appears.

    The Agent clicks one of the following:

    • Ok on the message or clicks Log Off on the navigation bar to log off and exit FireLight.

    • View Activity PDF on the navigation bar to view the activity documents.

    • Contact Agent on the navigation bar to send an email to the agent.

  8. The agent receives an email that the other agent has signed and approved the application.

  9. If additional agent signatures are required, the activity status is In Signatures. The originating agent opens the activity and completes the agent signing process.

  10. Go to Pre-Submission Review Step [User Instructions] or go to Submit the Activity [User Instructions] depending on whether the organization has a post-signature pre-submission review step.

Send Email Request

  1. Click the Send Email Request button on the Agent Signature Choice page.

  2. Enter the primary or additional agent's Signer Name and Signer Email as shown below. Click Send Email Request to send the email signature request to the agent via email.

    After sending the email request to the Agent, FireLight assigns an access Passcode and sends it to the Agent via email. Click Back to Signers to close the signature request.

    NOTE: You can send a reminder and a Passcode by clicking links within a message in My Messages [User Instructions].

  3. FireLight sends the agent request via email. The Agent is instructed to click a URL link and enter a Passcode to access and complete the e-signature process for the application. The URL link is valid for a set amount of time determined by the organization.

  4. The Agent clicks the URL on the email and enters the Passcode from the Passcode Email and clicks Enter to go to electronic Signatures in FireLight. A message appears if the Agent enters an invalid or expired Passcode.

  5. On the Electronic Signatures page, the Agent clicks a button to do any of the following:

    Sign Activity – opens the Agent Signature page to review the document set pages and begin the signing process. Go to step 6.

    Review Documents – opens document pages in a separate tab or window. Agent scrolls through the pages, uses the PDF controls to print or save the documents, and closes the tab or window.

    Contact Agent – opens a message box to send a request for information to the Agent. On the message box, the Agent enters a message and clicks Send.

  6. When the Agent reviews and approves all documents, the Capture Electronic Signature page opens. The Agent enters their information.

    If the organization enabled script fonts, the Agent can select a script font in the Select Font list. If the organization did not enable script fonts, FireLight uses the standard block font.

    To override the text script, the Agent can click in the signature box and sign on the pad using a mouse or stylus.

    The Agent clicks one of the following:

    • I Consent to authorize the signature.

    • I Decline to decline the signature.

    • Clear Signature to clear the signature and use the text script.

  7. After signing and consenting, the following verification screen appears.

    The Agent clicks one of the following:

    • Click OK on the message or click Log Off on the navigation bar to log off.

    • Click View Activity PDF on the navigation bar to view the activity documents.

    • Click Contact Agent on the navigation bar to send an email to the agent.

  8. A notification appears when the required signatures are completed. Click the CONTINUE button on the upper right. A message appears. This is a fully configurable message by organization.

  9. Go to Pre-Submission Review Step [User Instructions] or go to Submit the Activity [User Instructions] depending on whether the organization has a post-signature pre-submission review step.
