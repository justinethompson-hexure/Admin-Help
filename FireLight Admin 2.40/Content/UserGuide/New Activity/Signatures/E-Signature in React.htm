E-Signature in React for Embedded Users Only[User Instructions]

NOTE: Do not use the PreSubmit review queue for activities that use e-Sign in React. The PreSubmit review queue is incompatible and will cause errors.

The current release of e-Signature in React focuses on the Client API, specifically using the Sign Now and Send Email Request functionality. If your organization enabled this feature, it allows FireLight® embedded clients to enter into the e-Signature ceremony to complete the signing process for their parties. This feature impacts the Initialization page, Client Signature Choice Page, Send Email to Signer Page, Client Consent Capture, Presentation of Required Signers and Completed Signatures, and the Sign Now and Send Email Request functionality, which includes Document Sets, Initials, and capturing a signature. This enhancement includes LexisNexis and SMS Text functionality.

IMPORTANT! Within FireLight, most dates and times in the UI are displayed in the user's time zone (as configured in Preferences [User Instructions]). Signatures are different. Signature timestamps on forms are shown in UTC. Signature dates and times elsewhere in the UI, including dates obtained from the Signature Date control, are converted to the time zone of the signer’s state of residence.

After Data Entry, Signatures can be the next step in the application or activity process. This is where the user determines whether to use electronic signatures.

NOTE: The images will not look the same as the client organization's because the client organization has the control to change the look and feel of this process in an embedded experience.

  1. Decide whether or not to use electronic signature (E-Signature) and click one of the following options in Electronic Signatures. Upon making this choice, FireLight locks the application. You cannot make changes after signing without negating the captured e-signatures and requiring the re-capture of all e-signatures.

    On the Electronic Signatures view:

    • If you need to make changes to the application before signing, click Back to Application on the upper right to return to Data Entry.

      NOTE: Upon making this choice, FireLight locks the application. If you need to unlock the application for additional data entry, on the navigation bar click Other Actions and then click Unlock Application. If using the API process, the unlock will take you back to your original environment, (Example: If your organization is starting the signing process, and you unlock the application, you are redirected back to your organization's environment outside of FireLight to restart the signing process).

    • If Data Entry is complete and ready for signatures, choose which signature process to use:

      • Click Use E-Signature to start the E-Signature process.

      • Click Decline E-Signature to have the client sign paper copies of the application.

    (Optional) How to Stop the Signature Process

    Two buttons on the upper right of the List of Required Signers view allow you to stop the signature process. Click the Unlock Application button to begin the unlock process, which clears all captured signatures (see the note for "Unlock Application" above for details). Click the Back to Application button to return to the application without clearing any captured signatures.

    NOTE: If your organization requires the agent to sign first, FireLight disables client signer buttons until after the agent signs. If your organization requires the agent to sign last, FireLight disables the Agent button (as shown above) until after all clients sign.

  2. In the Request Signers list, click a signer type (Owner, Annuitant, Insured, Beneficiary, Agent, etc.) button. If enabled for your organization, select more than one signer type button for the signer. A single signer can sign for multiple signer types, e.g., when the signer is the Insured and the Owner.

    Next, click the Sign Now or Send Email Request button. Follow the applicable Sign Now or Send Email Request instructions below.

Sign Now

  1. Fill in information in the Agent Identification Verification and the Client Identification Verification sections and click the Verified button.

  2. If SMS Text Authentication is enabled through the signature control, the Mobile Number can be mapped to the Client Identification page or entered manually. SMS Text can be used in conjunction with Validate Identity Verification and LexisNexis Services.

  3. If the Mobile Number is not mapped, in the Client Identification Verification section, enter the Mobile Number in the field and click Send SMS Text. A passcode is sent to the client's mobile phone.

  4. On the Verify Client dialog, enter the Passcode and click Submit.

  5. On the Identification Verification page, click Verified as shown in step 3.

  6. Optionally, if LexisNexis Client Identification Verification Authentication is enabled and the correct information is mapped to the Party's first name, last name, street address, city, state, zip, date of birth, phone number and SSN, a Client Authentication page shows a series of three to five questions that verify the user's identity upon selecting answers to the questions and clicking Submit. LexisNexis may also present a bonus question for the user. The Signing process is not halted if the user fails the quiz, but the failure is noted on the Audit Report. (Additional contracting is required to use LexisNexis.)

    The user moves forward to the Signing ceremony. If the user clicks Cancel, the user is taken back to the Client Identification Verification page.

  7. If signature initials are required in a Document Set, an "Initials are Required. Click the Initials box to begin." message appears and the system scrolls to the Initials box. Click the Initials box.

    Enter the signer's initials and click the Apply Initials button.

    When initials are complete, an "Initials are complete. You can move forward to Signatures." pop-up appears. Close the toast.

    If more than one Document Sets are in the package, a "Document Set # or #" (e.g., Document Set 1 of 2) appears at the top of the view.

  8. Click a document name button to review the document. After review, select the I have reviewed and agree with the terms expressed within this document checkbox to proceed.

  9. On the Capture Electronic Signature view, fill in the information.

    The organization can limit the selections in the State list to only the approved states on the Sales Agreement. If so, the signer selects an approved state from the State list. The organization can also leave the State blank, so the signer is required to select the correct state for the signing ceremony.

    Select a font (if applicable) and do one of the following:

    • Click I Consent to accept the electronic signature.

    • Click I Decline to decline signing.

    • Click Cancel to cancel the e-signature.

    • Click Clear Signature, use a mouse to sign on the pad to override the text script, and click I Consent. mouse.

The signer clicks OK to close the message and selects another State in the list.

    The organization can set the number of state not matching attempts to be unlimited in Admin or the organization can set a limit on the number of attempts for state not matching. If the number of state mismatch attempts is limited, on the final attempt to sign with the state not matching, a message like the following Signing Error message appears.

    The agent receives a message and email with content similar to the following.

    An audit message is written out for the Signing Ceremony for the signer each time the warning message is displayed during the signature process. If the final configurable base message is presented to the signer, a final audit record is also written out.

  10. After a signer clicks the I Consent button, FireLight returns to the Request Signers page and displays the Completed Signatures. Select a checkbox for a remaining signer and click the Sign Now or Send Email Request button to continue the signature capture process.

    NOTE: A signer can re-sign the activity. In Manage Requests, the signer clicks the Re-Sign button and clicks OK on the Re-Sign Signature dialog. In the Request Signers list, select the signer and click the Sign Now or Send Email Request button to start the signing process.

  11. The Agent signs the activity. Select the Agent checkbox and click the Sign Now button.

    If the optional "Agent Not Required to View All Pages" setting is enabled in FireLight Admin, the agent can select the Confirm all documents without reviewing individually checkbox and click OK on the Confirm Documents dialog to skip the document review process before signing.

  12. When signatures are completed for each signer, click the OK button in Electronic Signatures Complete and then click the CONTINUE button to go to the next (Review) step or, if all steps are completed, to submit the activity to the back office for processing if the "Enable Auto Submit" setting is enabled in FireLight Admin.

Send Email Request or Generate Link Without Email

  1. On the Request Signers page, select the checkbox for a signer and click the Send Email Request button. Only one signer type can be selected at a time for e-Sign in React.

  2. Fill in the Signer Name, Signer Email, Your Name, Your Email, Signer Last 4 Digits of SSN/Government ID and Signer Birth Date on the Send Email to Client to Request Signatures page. NOTE: When sending the email request to an agent, the Signer Last 4 Digits of SSN/Government ID and Signer Birth Date fields do not appear on the Send Email to Signer to Request Signatures view.

  3. Next, preview the Subject and Message of the email near the end of the page. If the email is editable, add or change text within the email as applicable. You cannot change any of the variables within angle brackets, e.g., <CLIENT_NAME>. The organization determines whether emails are editable.

  4. Click the Send Email Request button.

  5. Agent signers receive a slightly different view.

    6. NOTE: If the signer does not have an email address, click the Generate Link Without Email button. At the bottom of the page, click the Copy Passcode and Copy Link buttons to copy the passcode and URL link to provide the FireLight URL and login information via phone or other means.

  6. Click the Back To Signers button to return to the Request Signers page.

  7. The signer receives the email that is shown in the Message field and clicks the URL link within the email (or enters the URL provided by the agent into a browser) to open the Welcome page in FireLight.

    The signer enters their Last 4 Digits of SSN/Government ID and Birth Date or enters a Passcode (if enabled in FireLight Admin) provided by FireLight in a separate email or provided to them by the agent via some other means and clicks Enter.

    If the organization requires Passcode Only Validation, the signer enters the Passcode on the Welcome dialog and clicks Enter.

    The Electronic Signatures page opens.

  8. On the Electronic Signatures page, the signer clicks a button to do any of the following:

    • Sign Activity – opens the Signature page to review the application pages and begin the signing process. Go to step 6.

    • Review Documents – opens document pages in a separate tab or window. The signer scrolls through the pages, uses the PDF controls to print or save the documents to the signer’s PC, and closes the tab or window to exit the documents review.

    • Contact Agent – opens a message box to send a request for information to the Agent. On the message box, the signer enters a message and clicks Send. The signer needs access to an email account entered in the From Address field to receive a reply to the message.

  9. After clicking the Sign Activity button, the signer clicks a document button, reviews the document and selects I have reviewed and agree with the terms expressed within this document checkbox below the documents names.

    The signer repeats the review and consent process for all remaining documents.

    OPTIONAL for Agents only: If the organization allows Agents to sign without reviewing all of the documents, the agent signer can select the Confirm all documents without reviewing individually checkbox and proceed to signature capture.

  10. When the signer has reviewed and approved all documents, the Capture Electronic Signature page opens. On the Capture Electronic Signature page, the signer enters their information.

    After the signer clicks I Consent, the following verification page appears.

    The signer clicks one of the following:

    • OK on the message or clicks Log Off on the navigation bar to log off and exit FireLight.

    • View Activity PDF on the navigation bar to view the activity documents.

    • Contact Agent on the navigation bar to send an email to the agent.

  11. The agent receives an email that the signer has signed and approved the application.

  12. If an additional agent signature is required, the activity status is In Signatures. The agent opens the activity and completes the agent signing process.

SMS Text Authentication

FireLight® supports SMS Text Authentication and validation for all signature workflows including Sign Now, Remote Sign (Send Email Request), and Client Fill and Sign. This provides an additional layer of validation because both the email and the phone number would have to be compromised to forge the signature. When the SMS Text option is selected in a Signature field, the Mobile Number field appears at the end of the Client Identification Verification section.

Sign Now

  1. The user enters the mobile phone number in the Mobile Number field and clicks the Send SMS Text button.

    NOTE: If your organization uses the Validate Identity Verification feature, the last 4 digits of the SSN/Government ID number, Birth Date, and Mobile Number are blank and need to be filled in during signer verification. The entered values must match values entered in the application.

    NOTE: If the organization uses Passcode Only Validation, SMS Text is the method of authentication in lieu of the passcode email. Enter the information including the Mobile Number and click Send SMS Text.

  2. The user enters the Passcode sent to the signer’s mobile number in the dialog and clicks the Submit button.

  3. FireLight verifies the mobile number and replaces the Send SMS Text button with a Verified button. The user clicks the Verified button and proceeds with the normal Sign Now process. NOTE: The SSN/Government ID and Birth Date are not included in the Client Identity Verification when the organization uses Passcode Only Validation along with SMS Text Authentication.

Send Email Request

  1. On the Send Email Request workflow, the user enters the signer identification data, including the Signer Mobile Number, and clicks the Send Email Request button.

    The user clicks the Back to Signers button to return to the Request Signers page.

    NOTE: If your organization uses Passcode Only Validation along with SMS Text Authentication, the Signer Last 4 Digits of SSN/Government ID and Signer Birth Date fields are not included.

  2. The signer receives an email request like the following and clicks the URL link.

  3. The signer clicks the link in the email request to open the login dialog in FireLight. The signer enters Last 4 Digits of SSN/Government ID and Birth Date and clicks the Send Passcode button. The agent signer does not need to enter the Last 4 Digits of SSN/Government ID and Birth Date.

    NOTE: If the organization uses Passcode Only Validation, the following dialog appears. The signer clicks the Send Passcode button.

  4. If not using Passcode Only Validation, the signer enters the Passcode for the recent request sent to their mobile number and clicks the Verify button to proceed with the normal Send Email Request process.
