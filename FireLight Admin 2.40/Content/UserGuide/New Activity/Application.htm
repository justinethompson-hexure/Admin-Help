Application [User Instructions]

Application activity is available if your organization has implemented the Application activity feature.

  1. On the Home page under Start New or on the New Activity menu, click Application.

    NOTE: If you enter FireLight® through your organization’s CRM system and FireLight launches directly into a new business application, skip steps 1 through 4.

  2. For a single carrier implementation, select a Jurisdiction and Product Type from the drop-down lists. In Product Type, select the All option to view a list of all available products.

    For a multi-carrier implementation, select a Jurisdiction, Product Type, and Carrier from the drop-down lists.

    NOTE: Click the information icon next to Jurisdiction to view a description of Jurisdiction. Click OK to close the dialog.

  3. If an information icon is next to a product name, click the icon to view product information in a More Info dialog. Click OK to close the dialog.

  4. Click a product name in the grid to begin an application for the product. Products sold by a subsidiary carrier may show the subsidiary carrier’s name depending on your organization’s settings.

    If your organization enabled the optional Use Resident State in Form Selection feature in FireLight Admin, select a Residence jurisdiction from the list. The Residence list determines available form selection. Click Back to return to the previous page and select a different product.

  5. FireLight displays a list of Required Forms for the application. Any Optional Forms appear below the list of Required Forms. You can select one or more optional forms to include within the application. FireLight dynamically adds conditional forms when the business case triggers them to be included.

    If you enter FireLight through your organization’s CRM system, the client data associated with the individual selected in the CRM will post over to FireLight. If the posted over data does not contain a product ID or jurisdiction ID, FireLight will open to the FireLight Home page. When you select a product and jurisdiction as shown below, you can choose whether to use the client posted over data to pre-fill the associated fields on the forms for the application. Ensure the Create the application for [client name] checkbox is selected to pre-fill applicable form fields with the posted over client information from the CRM system as shown below. Clear the checkbox to omit the posted over client data and not pre-fill the applicable form fields.

    NOTE: The posted over client data can continue to be used for any applications created during the FireLight session. However, if you enter FireLight through your organization’s CRM system and the posted over client data includes a product ID and jurisdiction ID, FireLight will open to an application for the existing product and jurisdiction included in the posted over client data.

  6. Click Create Application.

  7. Make any changes to the default application name and click Create. Click Cancel to return to Required Forms without creating the new application.

  8. Fill in the fields on the application form and click CONTINUE when Data Entry reaches 100%. See Data Entry Step [User Instructions] for detailed information.

    9. NOTE: If any changes to the Sales Agreement affect product availability, the following alert message will appear, "This product is not available since the sales agreement is invalid. Contact your admin for more info," and the activity is made read-only, disallowing completion. The following conditions cause an unlocked activity to be made read-only: 

    • End Date has passed current date,

    • An older end date was entered,

    • Start date is for a future date,

    • Jurisdiction selected for the activity is no more checked for the sales agreement.

  9. NOTE: Skip this step if your organization does not provide pre-signature review in the pre-submission review queue.

    1. On the action menu, click Request Review to request a pre-submission review before the signature process.
    2. Select the checkbox next to one or more names for Reviewer Name. Click Send Email Request to initiate the pre-submission review.

  10. Complete the signing process. See Signatures Step [User Instructions] for detailed information. Simplified instructions follow.

    1. Click Use E-Signature to electronically sign the documents.

    2. In the List of Required Signers, click a signer type button (e.g., click Owner).

    3. Click Sign Now to sign on this device.

    4. Fill in the Identification Verification fields and click Verified.

    5. The signer reviews the documents, selects the checkbox for I have reviewed and agree with the terms expressed within this document, and clicks Sign.

    6. The signer enters their signing information (Signer Full Name and City) and clicks I Consent to complete their signature capture.

    7. In the List of Required Signers, click Agent. You review the documents and click Sign.

    8. Enter your Agent On-Site Signature information and click I Consent.

    9. Click CONTINUE.

  11. NOTE: Skip this step if your organization does not provide a post-signature review within the pre-submission review queue. See Pre-Submission Review Step [User Instructions] for detailed information. Simplified instructions follow.

    1. Click an option on the action menu:

      • Submit to submit the application. Go to step 12.

      • Request Review to request a pre-submission review.

    2. After clicking Request Review, click a button to Send Request to Reviewer(s) or to Decline E-Review.

      If you decline e-review, click OK on the Electronic Review Declined dialog.

      NOTE: If your organization requires a pre-submission review, this page does not appear and you go directly to step c.

    3. Select the checkbox next to one or more names for Reviewer Name. Click Send Email Request. Click Back to App to return to the application.

    NOTE: The application could also go into a post-submission review if your organization requires it. Upon final reviewer approval, the application is automatically submitted. Note that this occurs without use of the EnableAutoSubmit Admin Groups option nor the FLI_AUTO_SUBMIT dataitem.

  12. If no pre-submission review, click CONTINUE to submit the new application.

Click the browser Back button to return to FireLight Admin Help.
