Paramed Exam Ordering

APPS

Pre Signature

Following is a sample wizard user experience for using APPS for scheduling a Paramed Examination prior to signing the application.

  1. Start an application.

  2. Select the Use APPS for Paramed Examination checkbox.

  3. Enter the information for the Insured that is not pre-filled from the application.

  4. Next to Would you like to check and see if a specific date is available for the paramed exam?, click the Yes button.

  5. Click in the Requested Exam Date field and click an exam date in the calendar.

  6. Click the Check Availability button. A message displays: "Back Office Action is being performed. A validation is being performed." Wait for the back office action to conclude.

    The Response field displays the number of available appointments for the requested exam date.

    The following fields display the Examiner information for the selected exam date: Examiner ID, Address, City, State, Zip, Phone, and Email.

  7. In the Available Times list, select a date and time for the paramed exam appointment.

  8. Click the Book Exam Appointment button. A message displays: "Back Office Action is being performed. A validation is being performed." Wait for the back office action to conclude. If a specified time is not available, a message displays "Specified time is not available for this examiner [time - appointment number] in the Response field.

  9. A confirmation number appears in the Confirmation ID field.

  10. When data entry is 100%, click the CONTINUE button on the upper right to proceed to the next step in the application process.

    NOTE: Application History includes the information for the web service calls to APPS for the paramed exam scheduling.

Post Signature

Following is a sample wizard user experience for using APPS for scheduling a Paramed Examination after signing the application.

  1. Start an application.

  2. Select the Use APPS for Paramed Examination checkbox.

  3. Enter the information for the Insured that is not pre-filled from the application.

  4. Complete the application. When Data Entry reaches 100%, click CONTINUE on the upper right to go to the next step of the application process.

  5. The Insured completes the application signing process.

  6. Enter a date to schedule the Paramed exam and click the Check Availability button to proceed to the scheduling process as shown below. Or click the Skip button to skip the Paramed exam and have the vendor contact the client directly.

    Select the date and time and click the Book Exam Appointment button.

    Click the CONTINUE button on the appointment success to return to the signature process if other signers remain, or go to the next step in the application process.

  7. When a "post-signature" paramed order submission process has been completed and then there is a request to either "Unlock the Application" (which would void all previously captured signatures) or there is a request to have the insured "Re-Sign" the application, a cancellation request is sent to the vendor canceling any previous reservation and/or paramed order that has been sent. (This supports a carrier's position that a paramed order is only to be submitted on a signed case.) This process then requires a repeat of the paramed scheduling / ordering workflow for the signer.

    NOTE: Application History includes the information for the web service calls to APPS for the paramed exam scheduling and cancellation (if applicable).

ExamOne

Pre Signature ExamOne Paramed Exam

Following is a sample wizard user experience for an ExamOne Paramed Examination scheduling prior to signing the application.

  1. Start an application.

  2. Select the Use ExamOne for Paramed Examination checkbox.

  3. Select the applicable location button for Mobile Appointment or In Center Examination.

  4. Enter any information for the Insured that was not pre-filled from the application data entries.

  5. In Start Date and End Date, select the date range for finding paramed exam appointments. The Response field displays the number of appointments available during the selected date range. In the following example, the Response field displays the number of Mobile appointments available.

  6. Schedule a Mobile or an In Center Location paramed exam. For Mobile exams, the examiner will go to the Insured's house, work, or other location to complete the exam.

    Mobile Times: 

    1. Click the Get Mobile Times button. A message displays: "Back Office Action is being performed. A validation is being performed." Wait for the back office action to conclude.

    2. In the Mobile Times list, select an appointment date and time for the paramed exam. A single Insured exam is for 45 minutes if a single insured. If it is for two Insureds, the user can book two back-to-back exams, totaling 90 minutes.

    3. Click the Confirm Mobile Times button. A message displays: "Back Office Action is being performed. A validation is being performed."

      The Response field displays the appoint confirmation information.

    4. When data entry is 100%, click the CONTINUE button on the upper right to proceed to the next step in the application process.

    In Center Locations:

    1. Click the Get In Center Location button. A message displays: "Back Office Action is being performed. A validation is being performed."

    2. The Response field displays the number of locations available.

    3. In the In Center Locations list, select a location for the paramed exam. The options in the list include the exam location's address and distance from the Insured's address.

    4. Click the Get In Center Times button. A message displays: "Back Office Action is being performed. A validation is being performed."

    5. The Response field displays the number of time slots available.

    6. In the In Center Times list, select an appointment date and time for the paramed exam.

    7. Click the Confirm In Center button. A message displays: "Back Office Action is being performed. A validation is being performed." An appointment confirmation appears in the Response field below the Confirm In Center button.

    8. When data entry is 100%, click the CONTINUE button on the upper right to proceed to the next step in the application process.

Post Signature ExamOne Paramed Exam

Following is a sample wizard user experience for an ExamOne Paramed Examination scheduling for an In Center location after signing the application.

  1. Start an application.

  2. Select the Use ExamOne for Paramed Examination checkbox.

  3. Select the applicable location button for Mobile Appointment or In Center Examination.

  4. Enter any information for the Insured that was not pre-filled from the application data entries.

  5. Skip the paramed exam scheduling section.

  6. When Data Entry is 100%, go to the next step in the application process.

  7. When the Insured completes the signing process:

    1. Enter the Insured's address information.

    2. Enter the Start Date and End Date for the paramed exam scheduling.

    3. Click the Check Mobile Availability button to check for Mobile exam availability or click the Check In Center Availability button to check for In Center exam availability.

    4. Click the Check Availability button.

    5. Select a location in the In Center Locations list and click the Get In Center Times button.

    6. Select a time in the In Center Times list and click the Book Exam Appointment button.

      NOTE: If the client clicks the Skip button, FireLight submits the paramed exam order information, but scheduling does not occur. ExamOne will reach out to the client to schedule that appointment.

    7. Click the Continue button when "Appointment confirmed at ..." text appears below the Book Exam Appointment button.

    8. If other signers need to sign the application, FireLight returns to the List of Required Signers for the application.

  8. The agent receives email messages during the following events:

    • Paramed exam is cancelled

    • Paramed exam is complete

    • Paramed exam status update

    • Paramed exam status and agent action is needed (e.g., when a client may have some questions or the paramed vendor is having difficulties contacting the client)
