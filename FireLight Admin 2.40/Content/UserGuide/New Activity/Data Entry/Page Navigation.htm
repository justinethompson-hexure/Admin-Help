Page Navigation [User Instructions]

Form controls are used for a forms only UI activity. Wizard controls are used for wizards and for forms that are in a combined wizards and forms UI activity.

Page List

FORMS: On the left below the numbered steps, click the button next to OPEN to open the list of pages for each form. Click a page number in the list to go to the page. Click a button to expand or collapse the list of pages within a form. Pages with validation errors (required fields, etc.) show in red text. Click the button next to CLOSE to close the page list.

WIZARDS: On the left below the numbered steps, click the button next to the current page name to open the list of pages for each wizard and for each form if in a combined wizard and form package as shown below. Click a wizard page name or a form page number to go to the page.

An icon indicates pages with validation errors (required fields, etc.). Click the button next to the page name to close the page list.

Locked Wizards and Forms

When an activity is locked (i.e., in Signatures, Complete, Submitted, etc.), the wizards and forms are available for viewing. The wizards and forms remain in read-only status to the user after locking an activity. The navigation page list reflects the wizard and form pages as it does while in unlocked status, but the user is unable to edit. When a user navigates back to the activity details after locking, they are navigated to the first page of the wizard. The user can then view the wizard and form entries in locked/read-only mode. No edits can be made unless the activity is unlocked (if status allows).

When an activity is locked, the pencil icon changes to a lock icon and the wizard and form features and functions at the time of locking are retained. If changes are made in FireLight Admin, Wizard Designer or Form Designer, (controls added, deleted, etc.) those changes will not apply to the locked wizard or form; the locked wizard or form will not be updated with the changes made in the Wizard Designer or Form Designer. If changes are made in the Wizard Designer or Form Designer, the changes will only be reflected if the Wizard or Form is unlocked.

Section Breaks (Wizards Only, Optional)

If a section break exists between wizard pages, data entry must be 100% complete in the section break before the Next button and the next section become available. In the following example, the user must complete data entry in all required fields and correct all errors on the Owner Information page before the next section becomes available.

Navigation links also rely on the required sections to be completed before the user can access the link. If the link is not accessible due to workflow sections incomplete, a message appears. When the information is complete in the section required, the link becomes accessible.

When data entry on the page is complete, the Page List shows a green check mark to indicate data entry is complete on the page and other pages or the next section are available.

After data entry is complete on the section break page: if an entry on a succeeding page causes an error to occur on a prior section break page, the section break will not block the user from navigating to other pages as it would in the initial data entry on the section break page. However, if the user closes the activity and re-opens it, the workflow would be blocked again until the user supplies the missing information and corrects any errors to continue.

In the following example, if data entry on the Insured_Information page causes an error to occur on the Owner Information page via rules, the Owner Information page would still show a green check mark indicating the page is complete in the Page List.

Page Error List

On the left in the DATA ENTRY numbered step, click the percent complete xx% to open an error list for the page. Click an error in the list to go to the field with the error.

Next and Previous Page Buttons

FORMS: On the left and right of a form page, click the next button to go to the next page in an activity or click the previous button to go to the previous page.

WIZARDS: At the bottom of a wizard page or a combined wizard+form UI page, click the Next button at the bottom of a page to move to the next page in an activity, or click the Previous button to go to the previous page.

Loading Multiple Copies of a Form

If multiple copies of a form exist, any of the following could exist for loading the copies.

NOTE: The multiple copies feature depends on the organization using the FORM_INDEX template moniker and rules in FireLight® Admin, so multiple copies of a form may not be available.

  • Click the checkbox to load the next copy of the form.

  • Each succeeding form copy has a number appended to the right of the form name. In the following example, 1035 EXCHANGEROLLOVERTRANSFER eFORM 2 is the second copy of the 1035 EXCHANGEROLLOVERTRANSFER eFORM form.

  • The copy number (“2” in this example) might also appear after the form name in the package.

  • The copy number of available copies might also appear on the form.
