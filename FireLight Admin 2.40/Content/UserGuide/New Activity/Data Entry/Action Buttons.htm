Action Buttons [User Instructions]

Action buttons can initiate an action, upload a document or photo, open a selection list, open a CRM, validate an address, and more. Following are some examples.

Example: Aura Next Underwriting Interview

  1. Create a new activity in FireLight.

    NOTE: All Aura Next interview data in a copied FireLight activity is cleared when it is copied.

  2. Fill in answers for the pre-underwriting fields in FireLight. The information from the pre-underwriting fields determine which questions to display in the Aura Next interview. Click the custom action button.

    Aura Next opens and the underwriting interview appears. The interview questions are specific to the organization and the activity details, such as jurisdiction, issue age, etc.

  3. Select or enter an answer to the questions on each page of the interview. You can open the interview multiple times to complete it.

    Click the Previous and Next buttons to move through the pages. When you reach the end of the interview, the Next button is disabled. Click the Close button to save and exit the interview.

  4. The applicable answered questions from the interview are filled into FireLight.

  5. At any time while the activity is unlocked before submittal, you can click the custom action button to open Aura Next to access and change the data in the interview.

    If any of the required pre-underwriting FireLight data has changed, the Aura Next interview is updated with the most current data. If applicable, new interview questions will appear and some may be removed. For example, a user changes the insured age from 59 to 60, which triggers different questions in the interview.

Example: Ceding Company Details for Replacement Processing

If enabled for your organization, do the following:

  1. Enter replacement information on the form or wizard, which triggers the addition of the ACORD 951 eForm to the application.

  2. Click the custom action button, search for and select the ceding company name. FireLight® pre-fills the ceding company information on the form.

    NOTE: If your organization implements an additional search feature, you can also search by product line of business.

  3. On the replacement instruction page, click the Print custom action button to print the forms that require ink signatures. When the forms with ink signatures are completed click the Upload custom action button to upload the ink signed documents.

  4. Continue with the next step of the application process in FireLight.

Example: LexisNexis Client Validation

If LexisNexis client validation during the data entry step is enabled for your organization, do the following:

  1. Begin an activity.

  2. On the form or wizard, complete the data entry for the client First Name, Last Name, Street Address, City, State, Zip, DOB, SSN, and Phone Number for the signer type (Owner, Insured, Annuitant, Joint Owner, Joint Annuitant, Payor).

    NOTE: If the form or wizard has a CRM (Redtail, Salesforce, etc.) action button, click the CRM action button and select the client to auto-fill the data from the CRM.

  3. Click the LexisNexis validation action button on the form or wizard for LexisNexis to validate the client. When the agent clicks the custom action button, FireLight automatically saves any data that was entered on the activity and the activity cannot be deleted in All Activities. This includes any data group and data group properties that are populated (i.e., Owner First Name, Owner Last Name, etc.).

Example: LexisNexis Risk Classifier

If LexisNexis Risk Classifier during the data entry step is enabled for your organization, do the following:

  1. On the form or wizard, fill in the required fields on the page that has the LexisNexis Risk Classifier action button.

  2. Click the LexisNexis Risk Classifier action button to initiate a call to the LexisNexis Risk web service. For example, click the Get LexisNexis Risk Score button as shown above. When the agent clicks the custom action button, FireLight automatically saves any data that was entered on the activity and the activity cannot be deleted in All Activities. This includes any data group and data group properties that are populated (i.e., Owner First Name, Owner Last Name, etc.).

  3. LexisNexis Risk Classifier service returns Risk Scores, Result Codes, up to 4 Reason Codes and 4 Descriptions, a Result Message, and an Order Number in the associated fields on the form or wizard.

  4. The carrier's underwriting department uses the risk scores, descriptions, and general information from the Motor Vehicle Department and Consumer Credit Bureaus on the signer to return back a faster approval/rejection rating.

  5. All of the actions taken with the LexisNexis Risk Classifier are included in the audit history and the report that is returned back to the client.

    For example, "FireLight made a web service call on behalf of Hexure to LexisNexis Risk Classifier for the `Owner` for activity `test riskclassifier - APM Test LN Service` and received score_result, national_credit_file, motor_vehicle_report pdf reports with Order Number HJ4654."

    If an error occurs, and no files are sent, a "no" is added to show no reports were received. For example, "FireLight made a web service call on behalf of Hexure to LexisNexis Risk Classifier for the `Owner` for activity `LN RC test - JB Forms Only` and received no pdf reports with Order Number KI4657."

    If an error occurs during the LexisNexis Call, no order number is added to the audit. For example, "FireLight made a web service call on behalf of Hexure to LexisNexis Risk Classifier for the `Owner` for activity `LN RC test - JB Forms Only` and received no pdf reports."

Example: Open a Selection List

In this example, click the button, click a letter at the top of the list, and select a Primary Occupation.

Example: Provider Lookup (Form View)

If enabled for your organization, do the following:

  1. Begin an application or other activity.
  2. Click the Provider Lookup button.
  3. On the upper left of the dialog in Search, type the first few letters of the provider's last name or business name.
  4. Select a State in the list.
  5. Click the search button to filter the list.
  6. In the list of names on the left, click a name to view the provider's information on the right.
  7. Click the Select button to populate the provider's information in the associated fields on the form or wizard.

Example: Redtail CRM Lookup (Form View)

If enabled for you or your organization, do the following:

  1. If you have a user-specific license, enter your Redtail login information in CRM Preferences on the Preferences page once and skip this step in future activities. If the firm has supplied a corporate license, skip this step.
  2. Begin an application or other activity.
  3. Click the Redtail Lookup CRM button.
  4. On the upper right of the My Contacts dialog, type the first few letters of the contact's last name or commercial name and click the search button to filter the list.
  5. In the list of names on the left, click a name to view the contact's information on the right.

  6. Click the Select button to populate the contact's information in the associated fields on the form or wizard.

Example: Salesforce CRM Lookup (Wizard View)

If enabled for you or your organization, do the following:

  1. If you have a user-specific license, enter your Salesforce login information in CRM Preferences on the Preferences page once and skip this step in future activities. If the firm has supplied a corporate license, skip this step.
  2. Begin an application or other activity.
  3. Click the Salesforce CRM button.
  4. Near the top of the dialog in My Contacts, type the first few letters of the contact's last name or commercial name and click the search button to filter the list.
  5. Click a name in the list to view the contact’s information below.
  6. Click the Select button to populate the contact's information in the associated fields on the wizard.

Example: Upload a Document or Photo

In this example, click the Upload Photo or Upload Doc button, then select the Document Type (or select Other). Drag and drop a single file from a system or network location to the drop area. Or click in the drop area to choose a file on the Open dialog and click Open to add the document. After adding a file, click the Upload button. FireLight accepts image files in .jpg, .tiff, .png, .gif, and .bmp formats and .xps files. After upload, FireLight converts the image files into .pdf files and stores them the same as other files. FireLight adds the document or photo to the Documents that you can view or remove (Documents) or print (Display/Print PDF) from the Other Actions menu.

NOTES:

  • You can upload image files that are not in .pdf format only from a custom button.
  • If the file size of the upload exceeds the maximum file size limit set by your organization, FireLight halts the upload and displays a notification message that the file is too large.

If the user attempts to add more than one file at a time, a message displays, "Please drop one file!" in the drop area.

Data Entry Review

Data Entry Review allows for an incomplete PDF package to be sent, along with metadata, prior to Client Fill and Sign, meaning the activity and documents will not be at 100% before going through the established Pre-Signature/Pre-Submit Review Queue. Any clients who choose to implement data entry review may require provider setup and should consult their project manager.

Submit Application to Data Entry Review on Custom Action Call

  • When the custom action Data Entry Review is clicked, the data entry review process begins.

  • The application PDF, documents and corresponding metadata are available for the provider to send.

  • Application is shown in READ ONLY mode.

  • The application enters the Pre-Submit Review Queue.

  • During Data Entry Review, the following are not available from the Other Actions menu:

    1. Request Client to Fill App

    2. Request Client to Fill & Sign

    3. Unlock

    4. Share

    5. Transfer

The Delete Button on the All Activities page is hidden while the application is in Data Entry Review.
