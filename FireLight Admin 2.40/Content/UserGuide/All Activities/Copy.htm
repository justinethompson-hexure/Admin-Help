Copy [User Instructions]

NOTE: If your organization enabled the Localization feature, you must update the copy of an activity in the same locale as the original activity.

The Copy feature is helpful when a new activity is nearly identical to an existing one.

Do one of the following to copy an activity or click the Cancel button on the Copy dialog to return to the All Activities page without making a copy.

When “Copy As Is” or "Copy With Changes" is initiated, FireLight® removes the original activity's unused data from the new activity at the time it is created. Additionally, when an activity is locked, any unused conditional data that may have been added and then subsequently made inactive is removed. This prevents the extra data from interfering with rules and from being sent when the activity is submitted.

When a User Is Associated with Multiple Firms

When a user is associated with multiple firms, the user has visibility and access to activities that were created under firms other than the firm for the current session. The copy feature for activities in FireLight behave in the following way and do so consistently.

Saved/Inflight activities are always associated with the firm under which they were originally created regardless of the firm ID associated with the current session. Different firms will likely have different sales agreements/product availability. For this reason, the following is the behavior for “Copy As Is” and “Copy with Changes” functionality.

  • “Copy As Is” copies the case and leverages the Firm ID information under which the case was originally created. This ensures that a case is not created under the firm for the current session as that firm may not have that product included in their sales agreement with the associated carrier.

  • “Copy with Changes” allows the user to reuse common demographic information in the newly created activity. The Firm ID and Role Codes associated with the current session are used in the “Copy with Changes” scenario. As part of the “Copy with Changes” workflow, the user goes through the selection process for the new jurisdiction, product type, product, etc., and those options are driven by the credentials for the current session.

LexisNexis Integration

When either the "LexisNexis" or "LexisNexisRisk" custom action call has occurred, the application is able to be copied, but the LexisNexis values will not be copied over to the next application.

  • For InstantID, those data items include LexID, NAS Score, NAP Score, CTI Score, Risk Indicators, and Pass/Fail Score.

  • For Risk Classifier, those data items include Risk Score, Result Code, Result Message, Reason Code 1-4, and Reason Messages 1-4.

Copy As Is

  1. In an activity, click the Copy button to open the Copy dialog.

  2. Click the Copy As Is button to copy the activity as-is (without changes to the Jurisdiction, Product, etc., or optional forms).

  3. Enter the new Name in the Copy Activity dialog and click OK. The new application opens in the Form Entry mode.

Copy with Changes

Copy with Changes will clear the original product codes, CUSIP number, carrier codes, provider data, and rule scope data. The user-entered data and pre-filled values are retained.

How Admin Group Settings Affect the Copy Feature

NOTE: If the "Disable Copy As Is" option or the "Disable Copy with Changes" option is selected in Admin, FireLight hides the Copy dialog and launches the correct copy option.

Disable Change Carrier Copy with Changes (Distributors Only)

In FireLight Admin on the Groups page, App tab, distributors can select the "Disable Change Carrier Copy with Changes" option to hide the Carrier selection list during the copy process. In All Activities, when a user clicks the Copy button, FireLight opens the Create New activity page instead of the Copy dialog and shows only the active carrier's products.

Disable Copy As Is

In FireLight Admin on the Groups page, App tab, organizations can select the "Disable Copy As Is" option to disable the ability to copy an activity "as is." In All Activities, when a user clicks the Copy button, FireLight opens the Create New activity page instead of the Copy dialog.

Disable Copy with Changes

In FireLight Admin on the Groups page, App tab, organizations can select the "Disable Copy with Changes" option to disable the ability to copy an activity with changes. In All Activities, when a user clicks the Copy button, FireLight opens the Copy Activity dialog instead of the Copy dialog.
