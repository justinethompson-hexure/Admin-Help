Filter Activities - User List [User Instructions]

Filter Activities

  1. In the area below the toolbar and above the list of activities, do any of the following:

    • If the group has user sharing enabled, select a user in the User list to filter the activities by user. See "User List" below for instructions. "My Activities" is the default.
    • If the organization has multiple locales enabled, select a Locale in the list to filter the activities by locale. "Any" is the default.
    • In Activity Type, select an activity in the list to filter the activities by activity type (PreSale, Application, Illustration, Quote, etc.). "All" is the default.

    • In Status, select a status from the list (Any, Data Entry, Locked, Signatures, PreSubmit Review, Review Queue, Submit Requested, Complete, Terminated, Purged, etc.) to filter activities by status. For example, select Data Entry to view activities that are in the Data Entry status. See Activity Display Statuses [User Instructions] for a list of available statuses.

      NOTE: Purged applications appear only when you set Status to Purged. Purged applications are not included in any other Status views.

    • In Sort, select a sort option from the list to filter the activities by that option (Date, Activity Name, Product Name).

  2. Click the numbered navigation buttons at the bottom of the page to move through multiple pages of activity listings.

Export Filtered Activities

Click the Export button on the toolbar to export the filtered data to a .csv file. See Export.

User List

When you have visibility into multiple users’ activities, through either a single activity share, global all activities user share, or group case access determined by the Group settings in FireLight® Admin, you can select a user from the User list to view their activities. The User list shows the five most recent users.

NOTE: The Active Agent User List setting in FireLight Admin restricts the list of agents in the User List on the All Activities page to only agents that have had activity since the last purge date. It can have an effect on SSO logins that use "On Behalf Of" in their 1228/103. If an assistant is listed as a user in the 1228 and a different agent user is listed in the 103, and the agent has not created any activities recently, the agent user is not available in the User List, which can cause a permissions error. 

  1. On the navigation bar, click All Activities.

  2. In the User list, click a user ID to view only the selected user’s activities. Click My Activities to view only your own activities. The default setting is My Activities.

    NOTE: The first time you share a case with a new user, the user may not appear in the User List until you search for the user name. Click the search button to the right of the User list to open the User List dialog. If the user does not appear in the list, enter their user name in the search text box and click the search button to add the user to the dialog. Then click the user name in the dialog to add the user name to the User list on the All Activities page.

    NOTE: The All Activities page displays a people icon next to activities that are shared.

    An activity that is linked to another activity, for example, an application that is linked to an illustration, displays a link icon next to the activity.

  3. To view more users’ activities, click the search button to the right of the User list to open the User List dialog and click a user to add to the User list and view their activities. The User list contains up to five additional users in addition to My Activities (your own cases) for a total of 6 users. The additional five users are based on the most recently accessed activities. For example, if you have access to more than five other users, only the five most recently accessed users appear in the list.

    The list of users that appear in the User List dialog include the following:

    • Users in groups with whom you have case access enabled in FireLight Admin
    • Users with whom you have a global share of all activities and it is enabled in FireLight Admin
    • Users with whom you have a single activity shared in FireLight and it is enabled in FireLight Admin

    FireLight removes a user name from the User list and User List dialog in the following scenarios:

    • If an Activity User Share is revoked, unless already there due to global sharing, case access, etc.
    • If a global All Activities User Share is revoked, unless already there due to activity sharing, case access, etc.
    • If the Activity is deleted or purged that has an Activity User Share, unless already there due to global sharing, case access, etc.
    • If Case Access is removed for a group role, unless already there due to activity sharing, global sharing, etc.

NOTE: FireLight creates audits in the activity History when an individual activity is shared with another user and also when it is revoked.
