Pre-Built Rule Templates

Age:

Age:

Calculates how many full years have passed between a past date and now. Useful for setting an age field from a date of birth field.

Template Parameters:

dataitemid_date The name of the control containing a string representation of a birth date in “mm/dd/yyyy” format

dataitemid_age The name of the control where the calculated age is stored

Use Case:

Calculate the attained age of a person from a date of birth

Age Nearest Birthday:

Age Nearest Birthday:

Calculates the age based on which birthday is closer. Calculates how many days have passed between the birthday and now and then divides by 365.25 days per year. If today is exactly halfway between the two birthdays then it will round up to the next birthday.

Template Parameters:

dataitemid_date The name of the control containing a string representation of a birth date in “mm/dd/yyyy” format

dataitemid_age The name of the control where the calculated age is stored

Use Case:

Calculate the insured age rather than the attained age from a date of birth

Alpha-Special Chars:

Alpha-Special Chars:

If the field is not blank then this rule uses a regex to check if the field contains only capital letters, lowercase letters, the single quote mark, commas, periods, and dashes. This rule is useful for validating name fields.

Dataitemid The name of the control that can contain only alpha or special characters

Use Case:

Validate a name field to ensure no strange characters are entered

Calculate Age Nearest

Calculate Age Nearest

Calculates the age of a person from a field containing the birth date of the individual. The rule uses the following logic:

  1. If the current date is within 6 months of the last birthday, the rule will report the user’s age on their previous birthday.

  2. If the current date is within 6 months of their next birthday, the rule will provide the user’s age at their upcoming birthday.

Template Parameters:

Dataitemid_dob The name of the control containing a string representation of a birth date in “mm/dd/yyyy” format

Dataitemid_age The name of the control where the calculated age is stored

Use Case:

Calculate the insured age rather than the attained age from a date of birth.
