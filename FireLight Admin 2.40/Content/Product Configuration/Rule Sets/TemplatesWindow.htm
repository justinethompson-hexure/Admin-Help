Templates Window

On the Rule Sets page, the Templates Window includes Pre-Built Rule Templates in ® for the most common rules that are used with input controls.

View Pre-Built Rule Templates

  1. On the navigation bar, click Product Config and click Rule Sets to go to the Rule Sets page.
  2. On the Rule Set Detail tab, click Templates Window to open the Templates Window and view the pre-built rule templates.

  3. On the left click a rule template to view the Xml tab.

  4. Click the Template Values tab to view the template values and the rule description.

  5. Click the X on the upper right to close the Templates Window.

Add a New Rule Template

When a rule is going to be used multiple times, you can add a new rule template using the Templates Window. This will add the template to the standard Templates Window for common usage.

will check the validity of the rule when you click the Save button. The rule can be saved with warnings, but any errors will prohibit the rule from being saved.

A template rule requires you to fill in required data to make the rule complete. What values must be provided and where they are placed within the resulting rule is determined by tagging areas of the template with “{}”. Each unique name with “{}” will define one required replacement value. For example, if you define a “{OwnerAge}” within the template rule, the user of the template rule is asked to supply an “OwnerAge” value. If the tag name within the “{}” contains the string “DataItemId”, the Rules Editor will require you to type in a valid control name for the replacement parameter.

Add a New Rule Template

  1. On the navigation bar, click Product Config and click Rule Sets to go to the Rule Sets page.

  2. On the Rule Set Detail tab, click Templates Window.

  3. On the Templates Window, click the New button.

  4. In Template Name, type a name for the new rule template.

    For example, type Age Nearest Birthday.

  5. On the Xml tab, type the XML for the new rule.

  6. On the Template Values tab, type a description of the template in Description.

  7. In Template Type, select an option in the list. The Template Type is used to tell the Rules engine which rule to load for the template you are defining. This leverages a different rule (e.g., diget, diset, compare, etc.) depending on where the rules engine is used.

    Rules is the normal rule template type you use to define rules to be used within forms. This is the default.

    Validation is a rule used to define validation rules you can apply to the parameters you define within a template. This functionality is inaccessible.

    BackOffice is a rule used to define rules to be used by back office. This functionality is inaccessible.

  8. Under Template Type, type parameter values as applicable.

  9. (Optional) Click the Edit Validation button and write validation logic in the Validation Script text box to ensure you type in correct values for the parameters within your script template. Click the Back button to return to the Template Values tab.

  10. On the upper right, click the Save button.

    The new rule template appears in the list on the left.

  11. Click the X on the upper right to close the Templates Window.

Delete a Rule Template

  1. On the navigation bar, click Product Config and click Rule Sets to go to the Rule Sets page.
  2. On the Rule Set Detail tab, click Templates Window link.
  3. On the left of the Templates Window, select the rule template to delete and click the red X to the right of the template name.

  4. On the Delete Confirmation dialog, click Yes to delete the rule template.

  5. Click the X on the upper right to close the Templates Window.
