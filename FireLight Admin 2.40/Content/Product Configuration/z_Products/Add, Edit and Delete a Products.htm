Add, Edit, and Delete Products

On the navigation bar, click Product Config and then click Products to go to the Products page. Adding a product creates a menu selection for a product in and associates the product to a Product Type (Variable Annuity, Universal Life, Non-Insurance, etc.).

Add a Product

  1. On the upper left of the Products page, click Add New Product.

  2. Enter the following information for the new product.

    Product Name Type the name of the new product. This is what is presented to your end users to select in .

    Product Type Select the type of the new product. This is the classification the end users select when filtering by product type (Variable Annuity, Universal Life, Preneed, Non-Insurance Product, etc.). Account Opening and E-Delivery are Non-Insurance Product type.

    Product Category Displays the product category based on the type of product/line of business. For a non-insurance product type, you can choose the product category. For all other product types, the category is read-only and driven by the ProductCategory and ProductType tables in the client Config databases.

    Rank Order Enter a number (integer value) for the rank order of the new product. Rank order is used to order the product list in the drop-down for product selection in the FireLight application. If the Rank Order is not entered, the product is listed in alphabetical order by product name. The rank order must be used on the other products as well. Empty values are placed first because they don't have a value to sort by. The rank ordering kicks in for any rank order values after that. For example, if the rank order does not matter for other products, you could set a pre-authorization product to 1 and set all of the other products to 2 to place the pre-authorization product first in your product selection list.

    CUSIP ID Type the CUSIP ID that integrates with an external illustration system, like ForeSight®. When integrating with an external illustration system, uses an ACORD 103 transaction to prepopulate the application with data collected within the external system. The CUSIP ID in is used to map the product illustrated within the external system to the product in . The CUSIP ID is also used in the Review Queue Script to add products to the review queue process. This is a required field.

    Billing Code Select a Billing Code for the product (Normal, Back Office, Service Forms, or Special). Billing code is used for reporting purposes.

    Product Code Type the Product Code to define a product identifier code that can be used for DOL purposes.

    Carrier Code Type the Carrier Code to define a product identifier code that gets exposed to the distributor for ID purposes.

    Product DTCC Member Id Type the Product DTCC Member ID for the product. See Fees, Expenses and Commissions Repository in Features.

    Product Organization Name Type the organization name of the selling carrier for the product.

    Full Settling DTCC Member Select the checkbox if your organization is a full settling DTCC member.

    Product Locale Select a locale for the product.

    Last Updated displays the date and time in UTC when the product configuration was last updated.

    State Approved grid In the grid, select the checkbox for each jurisdiction in which the product is approved. At least one state must be selected. The product jurisdictions approved will then appear in the State Approvals jurisdiction list in Sales Agreements. A Sales Agreement cannot be created for a state in which the product has not been approved.

    Check /Uncheck All selects or clears all checkboxes in the State Approved grid. This checkbox switches between selecting and clearing all of the State Approved checkboxes.

  3. Click the links to add Product Features, Product Documents, and Quote Detail if applicable.
  4. At the end of the Products page, click Save Product Config to save your entries.
  5. Next Steps: add Packages and Sales Agreements for the product and grant access to the new product in Groups. Then deploy the product to the applicable environments. See Deploy Data.

Edit a Product

  1. On the navigation bar, click Product Config/Products.
  2. In the product list grid, click a product name.
  3. Below the grid, update the product information, including state approvals. See "Add a Product" on this page.
  4. Click the Save Product Config button to save your updates.

Delete a Product

  1. On the navigation bar, click Product Config/Products.
  2. In the product list grid, click the red X to the right of the product name and product type.

  3. In the confirmation dialog, click Yes to delete the product from the system or click No to close the dialog without deleting the product.
