Rule Sets

On the navigation bar, click Product Config and click Rule Sets to go to the Rule Sets page.

Use the Rule Sets page to define and manage a collection of rule sets, which are added along with forms to packages. Rule sets can contain groups of rules by category (distributor only rule set, owner rule set, product specific rule set, etc.), which allow you to use the same rules across multiple forms, thus avoiding duplication of rules across multiple forms packages. For example, you could use the same Owner rule set across different forms in multiple packages that have owner-related data items.

Following are informative topics about rule sets and a description of the Rule Sets page.

Key Advantages of Rule Sets

Key Advantages of Rule Sets
  • You can group rules into rule sets by category, e.g., distributor rule set, owner rule set, product rule set, etc., then add the applicable rule sets to packages, which allows maximum reuse of the same rules across multiple packages. Because rules are tied to the data items within a package, only the applicable rules will fire.
  • If you would like to begin working on rules for a future release, you can set beginning and ending dates on the current rule set, make a copy of the current rule set for a future release, make updates to the future rule set, set beginning and ending dates for the future rule set, and then add the rule set to the applicable packages.
  • For testing new rules in a test environment, you can add a new rule set, copy rules from an existing rule set to the new rule set, update the new rule set, and then replace the old rule set with the new rule set in one or more packages for testing.
  • Rule sets are saved independently, which allows multiple users to work on rules at the same time.
  • Lock rule sets for editing to eliminate overwriting by other users.
  • Deploy rule sets across forms and wizards.
  • Deploy rule sets configurations independently on the Deployments page. See Deploy Data.

Rules Processing Overview

Rules Processing Overview

Conversion Process:

  • With the initial implementation of rule sets in FireLight® v2.11, FireLight converted all of your previous form level rules to rule sets with the rule set names matching the form names. If the forms have an Edition set in the Form Detail, the rule set names include the Edition at the end of the rule set name. For example, FireLight converts rules in a form named "Medical_History" with Edition 3/17 to a rule set named "Medical_History_3/17."
  • During the rule set conversion process, FireLight automatically assigned a Primary Rule Set to any form with rules containing [FORM_INDEX], [FORM_INDEX_PLUS_1] and/or [FORM_COUNT]. See Assign a Primary Rule Set for more information.
  • When you upload a new form, FireLight does not convert the new form to a rule set. You need to add or update existing rule sets for the data items on the form and then add the applicable rule sets to the package along with the form.

High Level Architecture for Rules Processing:

  • Controls are tied to data containers.
  • Rules work against the data containers. They have no knowledge of the associated form controls.
  • Rules engine is event based. Rules are fired when a value within a container changes.
  • The Rules Engine uses a stack-based or postfix type language. Think of it like working from the inside out. Arguments are passed in first, followed by the action.

Organization of Rules:

  • You do not have to duplicate common rules on other forms. You can use rule sets to maintain rules by category (distributor-specific, owner-specific, product-specific, etc.) and then add the applicable rule sets to the packages.
  • You can create new rule sets or edit existing rule sets to avoid duplicate rules.
  • When the application forms run, FireLight sorts the rules by rule content (not by name) and removes identical rules.
  • When working on a rule set, you can lock the rule set to prevent other Admin users from overwriting your work.
  • If you create a new rule on a form in the form Designer, you can tie a rule set to a particular data item.

Rule Sets Page Grid and Associated Fields

Rule Sets Page Grid and Associated Fields

The top section of the Rule Sets page displays a grid with a list of rule sets that have been successfully added. The following topics describe the rule sets grid and associated fields on the upper right of the Rule Sets page and instructions for using the top area of the page.

The rule set grid contains a row of information for each rule set.

Sort the rule sets alphabetically by clicking Rule Set Name in the header row of the grid.

Grid Columns

Rule Set Name displays the rule set name.

Description displays a description of the rule set.

Lock/Unlock column contains a lock button that is used to allow or prevent other admin users from editing the rule set.

Delete column contains a red X that is used to delete the rule set from the system.

Fields Below the Grid

Exclude Expired Select this checkbox to exclude expired rule sets from the rule set list grid. This filters the list to show only active rule sets in the grid.

Total displays the total number of rule sets that are currently in the grid.
