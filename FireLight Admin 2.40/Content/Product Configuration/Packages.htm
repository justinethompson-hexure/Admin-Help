Packages

On the navigation bar, click Product Config and click Packages to go to the Packages page.

Use the Packages page to define a collection of forms, wizards, and rule sets that make up a specific package. A package contains the forms and wizards that users see when they create an activity.

Add and Edit a Package

Add and Edit a Package

Create a package of forms, wizards, and rule sets for a new package or edit an existing package.

  1. On the navigation bar, click Product Config and then click Packages.

    The Package list grid displays saved packages. Click a package name in the grid and then click the Forms/Wizards in Package tab to display the forms and wizards that make up the package. On the tab, form names appear in black text, wizards appear in green text, and templates appear in red text.

  2. Do one of the following:

    • Duplicate an existing package.

      If an existing package applies to this product, you can duplicate the package and then make changes to the duplicate copy to create a new package.

      In the Package list grid, click the [Duplicate] link to the right of the source package name to create the duplicate package. The duplicate package is added to the grid with “Copy of” added to the package name. Update the Package Name field for the duplicate package as shown below.

    • Add a new package.

      Click the Add New Package link on the upper left. Update the Package Name field for the new package.

  3. Below the Package list grid on the Package Detail tab, add or edit information for the package.

  4. (Optional) Make a package available for only designated role codes.

  5. (Optional) Automatically add the package to an Activity based on Role Codes. This allows for these packages to be added to the Activity for users with the specified Role Code without it being included in a Sales Agreement.

    Guidelines:

    • Auto add is available for all ACORD types with the exception of: E-Delivery (121-12100), E-Delivery Deliver Only (121-12123), E-Delivery Sign Only (121-12122), Illustration (111), and Quote (212). Auto add is limited to the "Area Type" of "Activity" only; "Reviewer" and "Reviewer Optional" are not included in this scope.

    • Auto add is limited to the package Area Type of "Activity" only; "Reviewer" and "Reviewer Optional" are not included in this scope.

    • When an end user creates a new Activity, FireLight checks to determine if the user's Role Code is listed in the Auto Add to These Role Codes text box for the Activity type. If the session user has multiple role codes associated with them, the first role code from the session user's role code list is used. If the system finds a match, it adds that package automatically at the position indicated by the Package Position selection on the Packages page. If it does not find a match, nothing changes. This check occurs only at the time of the Activity creation. Should the values in the Auto Add to These Role Codes text box change after the Activity is created, it does not affect the Activity. Any forms in the package that is auto added are included in the Forms list in the Activity data entry. Signing Ceremony Only forms are not displayed.

    Auto Add Packages to Role Codes:

    1. A Firm Orgs link is above the Auto Add to These Role Codes text box on the right.

      1. Click the Firm Orgs link to open the Firm Orgs dialog.

        NOTE: If the Auto Add to These Role Codes text box is populated, and there is a wild card entry for a Firm Org, that Firm Org is selected when the Firm Orgs dialog opens. The wild card must be the exact wild card for the Firm Org for this to occur. For example, "ABC_*" would cause Firm Org ABC to be selected, but "AB*" would not.

      2. Select the firm organizations for which to include the firms' role codes that have access to the package for the Activity. Click the X on the upper right to close the Firm Orgs dialog. The entries in the Auto Add to These Role Codes text box are replaced with the wild card entries for the selected Firm Org(s). Example: Selecting Firm Org "ABC" would result in "ABC_*" being entered into the text box.

      3. Click Save Package on the Packages page.

        NOTE: If the user navigates between packages and does not click Save Package after each Firm Orgs list selections, the changes is lost.

      4. Deploy the package using the Package Configuration on the Deployments page. See Deploy Data for deployment instructions.

    2. Below the Auto Add to These Role Codes text box in the Package Position list, select a position for the package either before or after other packages in the Sales Agreement:

      • Blank (default)

      • Before

      • After

    3. If an Activity is locked, the packages are not automatically added (or removed).

    4. When the new fields for Package Position and Auto Add to These Role Codes are updated, the system creates an audit and displays it in the Package History tab. Old Value displays the previous value, New Value displays the value after the update, Action Date displays the date and time the update occurred, and Login displays the user that made the update.

For example, when the FireLight® Illustration feature is implemented using the ForeSight Illustration Services, FireLight has an API that returns calculation results based on the provided request values. This API forwards the request to the ForeSight Illustration Services web service to provide the Calculation Results. Part of what FireLight adds is a call to the Rules engine to validate the inputs before ForeSight Illustration Services is called. FireLight has a requirement to manipulate app data only when the app comes from an API request. This data manipulation can include mapping Data Items or Adding Data Items. The purpose is to be able to transform data, but do this only for External API calls. An API client sends a request that is similar to a "Guided Illustration." This request would have minimal data items, including the product requested, optional features, and minimal client info, etc. These values need to be expanded into the values ForeSight Illustration Services requires to run an Illustration calculations result. A Rule Set can be created to handle this data transformation. To limit this Rule Set to only API requests, it needs to be added to a package that is available only to a role code that is specific to API calls. During the API call when the Rules are run it provides this API specific role code to include any API-specific packages. The API logic also needs to use the updated set of data items to pass to ForeSight Illustration Services.

  7. Add a form to the Forms/Wizards in Package section.

    1. Below the upper grid, click the Forms/Wizards in Package tab.
    2. On the left under All Forms/Wizards, type the beginning characters of a form or wizard name in the search box and click the Search button to search for a specific form or wizard.

    3. Click a form name in the Forms/Wizards in Package section to select an insertion point for the new form or wizard. The new form or wizard is inserted above the selected form.

      If this is a new package, or an insertion point is not selected in an existing package, the new form or wizard is inserted at the top of the list.

    4. Select a form or wizard to add to the package: in the All Forms/Wizards list on the left, click a form or wizard name and then click the right arrow button to add the selected form or wizard to the package.

    5. Point to the Approved States link below the form or wizard name to view a list of state approvals that were selected on the Form Library or Wizard Library page. If needed, go back to the Form Library or Wizard Library page to edit the list.

    6. Point to the Properties link below the form or wizard name to view the following form or wizard properties:
      Status displays "Composing," which is the default form or wizard status.
      Edition displays the Edition that is defined on the Form Library page.
      Expires displays "This Form Expires On" date from the Form Library page or "This Wizard Expires On" date from the Wizard Library Page.
      Replace by displays the "Will Be Replaced By" form name that will (or did) replace the form on the expiration date.

  8. Set the inclusion option and order of forms and wizards in the package.

    On the right t, from the Forms/Wizards in Package section, select a form inclusion option in the list next to the form name.

    Form Inclusion Options
    Option Description
    Required

    The form or wizard is always required in the package.

    NOTE: A form or wizard that is included in a package for requiring a supplemental document (illustration, etc.) to be added to the signing ceremony needs to be required in the package.
    Optional The form or wizard is optional in the package based on a variable added to the form or wizard on the Designer page.
    Conditional The form or wizard is required in the package based on a variable added to the form or wizard on the Designer page. This is used for forms or wizards that are added to the package only upon selection of data on a given form or wizard; for instance, a 1035 exchange form is conditionally added to the package upon the election of the 1035 funding option.
    Print Only The form is included only within the Display/Print PDF document dialog. It is not included within the package submitted to the back office.
    Submit Only The form is included within the package submitted to the back office only. It is not included in the Display/Print PDF document dialog.
    Input Only: Required The input only form is always required in the package, but it is not included within the Display/Print PDF document dialog nor included within the submitted application. An example of an input only form would be a pre-application questionnaire that is included in the package but is not printable or viewable in the submitted application package.
    Input Only: Conditional The input only form is conditional in the package based on a variable added to the form on the Designer page, but it is not included within the Display/Print PDF document dialog nor within the submitted application.
    Signing Ceremony Only The form is used for acknowledgment purposes and for data presentation during the signing ceremony. It can be used in forms only packages or in forms and wizards combination packages. The Signing Ceremony Only form is not visible in the forms list during data entry in FireLight but can be viewed in the Display/Print PDF document dialog where it is selected by default. If the activity is not locked, the checkbox for the form can be cleared. If the form does not have signature controls on it, the form appears in the signing ceremony for each signer. If the form has signature controls, the form appears in the signing ceremony for only the signers set in the signature controls. The signatures, after being collected, are visible on the form in the edit view of a locked activity, the Display/Print PDF document dialog, and the form accomplishment during the signing ceremony for any other signers. (The Signing Ceremony Only form is included in the submission to the carrier's back office unless the provider explicitly excludes the form by use of the SigningCeremonyOnly document type.) NOTE: Add fields to the form with the same field names as a form or wizard field controls to pre-fill data onto the Signing Ceremony Only form in FireLight. All fields that are on the Signing Ceremony Only form must have a counterpart on a wizard or form included in the package. If a field is required and present only on the Signing Ceremony Only form, the activity will not reach 100%.
    Signing Ceremony Only: Conditional The form is a conditional version of the Signing Ceremony Only form above. A rule must be added to trigger the conditional form.
    Signing Ceremony Only: All When this option is selected, the form will always be displayed during the signing ceremony, even if a signature field is present and the user is not the user who will be signing.
    Signing Ceremony Only: All Conditional The form is a conditional version of the Signing Ceremony Only: All form above. A rule must be added to trigger the conditional form.

  9. (Optional) Remove a form or wizard from the package.

    Click a form or wizard name under Forms/Wizards in Package and then click the left arrow button to move the form or wizard back to the All Forms/Wizards list on the left.

  10. (Optional) Add multiple copies of a form.

    On the right under Forms/Wizards in Package in the Copies list, select the number of copies of the form to include in the package. This feature allows you to add multiple copies of a form to a package and then use the Form Designer page to add monikers to the form controls with data items that change.

  11. Re-order the forms and wizards in the Forms/Wizards in Package list.

    Click a form or wizard and then click the up arrow or down arrow button to the right of the list. Or, press and hold the Ctrl key, select multiple forms and wizards, and click the up arrow or down arrow button to move multiple forms and wizards at a time within the list. Default print order in follows the order of the forms and wizards in the Forms/Wizards in Package list. In , a Conditional form or wizard is included only when the condition has been met to include it. When an application activity is in locked status, included forms are labeled “Application PDF.”

  12. Add a rule set in the Rule Sets in Package list.

    1. Below the upper grid, click the Rule Sets In Package tab.
    2. On the left under All Rule Sets, begin typing the name of a rule set and click the search button to search for a specific rule set.

    3. Click the name of the rule set to add to the package and click the right arrow button to add the selected rule set to the package in the Rule Sets in Package list on the right.

  13. (Optional) Remove a Rule Set from a package.

    On the right under Rule Sets in Package, click a rule set name and click the left arrow button to remove the selected rule set from the Rule Sets in Package list and return it to the All Rule Sets list on the left.

  14. Click the Save Package button to save the package.

  15. Next Steps: Add Sales Agreements for the product and grant access to the new product in Groups.

Replace a Form, Wizard, or Rule Set in a Package

Replace a Form, Wizard, or Rule Set in a Package

The Replace Form/Wizard/Rule Set in Package dialog is a convenient way to replace a form, wizard, or rule set within an existing package. This can avoid breaking the linkage between Form Index fields and the rules written using Form Index fields by replacing the Primary Rule Set in the Form or Wizard at the Package level to reduce the chances of missing replacement of the appropriate Rule Set for the Form or Wizard.

Replace a form or wizard:

  1. On the navigation bar, click Product Config and then click Packages.
  2. Click the Replace Form/Wizard/Rule Set in Package link below the Package grid.
  3. In the Replace Form/Wizard/Rule Set in Package dialog on the Form/Wizard in Package tab, either type in the Form/Wizard to Replace text box the name of the form/wizard or select from the Form/Wizard to Replace list.
  4. Select the checkbox for the package in which to update the form or wizard in the Packages list. Click the Select checkbox to select all packages.
  5. Select the replacement form or wizard in the Replace with Form/Wizard list.

  6. Click the Replace Form/Wizard button and click Yes on the Replace dialog. The Form History/Wizard History updates include the replacement information.

Replace a rule set:

  1. Click the Replace Form/Wizard/Rule Set in Package link below the Package grid.

  2. In the Replace Form/Wizard/Rule Set in Package dialog on the Rule Set in Package tab, select the Rule Set to Replace in the list.

  3. In the Replace Form/Wizard/Rule Set in Package dialog on the Rule Set in Package tab, either type in the Rule Set to Replace text box the name of the form/wizard or select from the Rule Set to Replace list.

  4. Select the checkbox for the package in which to update the rule set in the Packages list. Click the Select checkbox to select all packages.

  5. Select the replacement rule set in the Replace with Rule Set list.

  6. Click the Replace Rule Set button.

  7. Click Yes on the Replace dialog to confirm the rule set replacement. The Package History updates include the rule set replacement information.

Delete a Package

Delete a Package
  1. On the navigation bar, click Product Config and then click Packages.
  2. In the package grid, click the red X to the right of a saved package name.

  3. Click Yes on the confirmation dialog to delete the package from the system.

View Package History

View Package History

Spotlight Video: See the History for Packages, Products, Sales Agreements, and Admin History Report Spotlight Feature Video for more information.

Go to the Package History tab to view the following package history details in a grid: Action, Old Value, New Value, Action Date, and Login. All changes happening in the Replace Form/Wizard/Rule Set in Package dialog are also included.

  1. On the navigation bar, click Product Config and then click Packages.
  2. In the package grid, click a package name.

  3. Click the Package History tab.

  4. In Utilities, Reports page, Admin Reports, run the Admin History Report for Package History Audit Report to view an audit history report for one or all packages during a selected time frame.

NOTE: The system purges all package deployment and audit history that is more than one year old.
