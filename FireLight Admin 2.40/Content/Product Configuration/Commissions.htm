Commissions

On the navigation bar, click Product Config and click Commissions to go to the Commission Netting page.

Use the Commission Netting page to import commission netting files (DPfAs) into FireLight®. DPfAs are in .xml format.

Add a Commission Netting File

Delete a Commission Netting File

In the upper grid, locate the commission netting DBfA file to delete and click the red X on the far right of the row.

Upload DPfA Files

  1. On the navigation bar, click Product Config and click Commissions to view the Commission Netting page.

  2. Click the Upload New 1204 button.
  3. Use Choose File to browse for one or several DPfA files.

  4. To begin the upload process, click the Upload button.
  5. By default, the Start Date is set to the current date. To stage the files before they go live, enter the needed Start Date manually.

    6. At the end of the upload, the result is displayed.

Replace an Existing DPfA File

1. On the navigation bar, click Product Config and click Commissions to view the Commission Netting page.

2. Click the Upload New 1204 button.

3. Use Choose File to browse for one or several DPfA files. To begin the upload process, click the Upload button.

4. When a DPfA file provides commission information for an existing carrier/distributor combination, the user will be prompted to confirm that they want to overwrite the existing file.

5. Click the Confirm button to overwrite the existing file.

6. After confirmation, the result is displayed.

Delete DPfA File

  1. On the navigation bar, click Product Config and click Commissions to view the Commission Netting page.

  2. Click the "x" icon to delete the appropriate DPfA file.

  3. On the Delete Confirmation dialog box, click OK.

Be aware that the Upload New 1204 button and the ability to delete a previously uploaded DPfA will be disabled for distributors.

Deploying DPfA Files

Once the DPfA is uploaded, tested, and ready for use in the Staging and/or Production environments, it can be deployed to those environments. In the Admin Deployments view, a new Synchronization Scope has been added for Commission Netting – Commission Netting Configuration. Select this scope and click Load Deployment Information to view and deploy DPfA files.

Calculating the Net Commission

The first step is finding the XTbML by table identity (obtained via data item Commission_XTBML_Identity) or by parameters:

  • CarrierCode

  • CarrierCommCode

  • ProductCode

  • Sub Distributor (obtained via data item FLI_SUB_FIRM_ID)

  • Jurisdiction (obtained via data item FLI_ISSUED_STATE_CODE)

  • CommOption/CommCode

  • One or several CommEvents

  • Commissions Netting is allowed (this check is disabled by default)

The next step is traversing the table to find the rate value.

Depending on the found XTbML, three use cases are possible:

  1. No banding uses the table that does not have any axis but provides the value itself.

  2. Keys banding is used when commissions banding is done with non-ordered items. These key values are obtained via the data item Commission_Key_Code.

  3. Axis banding is used when banding is done around typed values. The available types are Dates, Duration, Age, and Currency. The Axis values are used to look up the banding information. The values can be obtained via special data items Commission_Date, Commission_Duration, Commission_Age, and Commission_Amount or a rule if it is calculated.
